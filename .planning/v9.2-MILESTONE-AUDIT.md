### Gap Details

#### DSET-03: directory_path Never Stored on Initial Import

**Severity:** Load-bearing integration gap
**Symptom:** After importing via the discovery flow, `directory_path` in the `datasets` table is always NULL. When user clicks re-import (↺), the `onReimportDataset` handler finds `null` and falls back to re-opening the discovery picker instead of the seamless stored-path re-import flow.

**Root cause:** The per-directory `path` value IS in the `native:request-alto-import` payload (each directory entry has `path: s.path` in `main.ts` line 831), and Swift's `runAltoImport` receives it. But when Swift chunks cards back via `native:import-chunk`, the path is NOT included in the chunk payload. `handleNativeImportChunk` calls `bridge.importNative(sourceType, allCards)` which sends `{ sourceType, cards }` to the `etl:import-native` handler — with no `directoryPath`. The protocol already declares `directoryPath?: string` on `WorkerPayloads['etl:import-native']`, but nothing fills it.

**Fix options:**
- (a) Include `directoryPath` in the `native:import-chunk` payload from Swift, pass through NativeBridge → WorkerBridge.importNative → etl:import-native handler
- (b) Store path in a side channel (e.g., a separate bridge message after all chunks complete)
- (c) Pass `directoryPath` through `importNative()` signature and `etl:import-native` handler to `CatalogWriter.upsertDataset`

**Files affected:**
- `native/Isometry/Isometry/NativeImportCoordinator.swift` — `sendChunk` payload excludes path
- `src/native/NativeBridge.ts` — `handleNativeImportChunk` has no path parameter
- `src/worker/WorkerBridge.ts` — `importNative()` takes only sourceType and cards
- `src/worker/handlers/etl-import-native.handler.ts` — receives directoryPath but never gets a non-undefined value

#### DSET-04: Missing refreshDataExplorer() After Re-Import Commit

**Severity:** Cosmetic — DataExplorer stats panel shows stale card counts until next user action
**Symptom:** After committing a re-import, the CatalogSuperGrid refreshes correctly, but the DataExplorer stats panel (card count, dataset count) does not update.

**Fix:** Add `void refreshDataExplorer()` call at `src/main.ts` after `coordinator.scheduleUpdate()` in the `alto-reimport-result` commit handler (line ~909). The delete flow at line 737 already does this correctly.

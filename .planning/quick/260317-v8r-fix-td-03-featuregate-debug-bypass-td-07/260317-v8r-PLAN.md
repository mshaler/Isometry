  <name>Task 1: Fix TD-03 — FeatureGate debug-toggleable bypass</name>
  <files>native/Isometry/Isometry/FeatureGate.swift</files>
  <action>
Replace the `#if DEBUG return true` unconditional bypass with an environment-variable-based override mechanism:

1. In the `isAllowed` method's `#if DEBUG` block, check for an environment variable `ISOMETRY_ENFORCE_GATES`. If the env var is set to "1" or "true", fall through to the normal tier comparison instead of returning true.
2. Default behavior (no env var set) remains `return true` — zero friction for normal development.
3. Add a static computed property `static var isEnforcingGates: Bool` that checks `ProcessInfo.processInfo.environment["ISOMETRY_ENFORCE_GATES"]` so tests can introspect.

The implementation should look like:
```swift
static func isAllowed(_ feature: NativeFeature, for currentTier: Tier) -> Bool {
    #if DEBUG
    if ProcessInfo.processInfo.environment["ISOMETRY_ENFORCE_GATES"] != "1" {
        return true
    }
    #endif
    return currentTier >= requiredTier(for: feature)
}
```

This allows: (a) normal dev with all features unlocked, (b) `ISOMETRY_ENFORCE_GATES=1` in Xcode scheme env vars or XCTest launch arguments to test tier gating during development.

**HTML describe block:**
The HTML fixture (`html-snapshot.json`) is an array of HTML strings. But ImportOrchestrator's `html` case wraps a single string in `[data]` if it's a string, or maps array items. The fixture is a JSON array of HTML strings — pass as raw JSON string to importFileSource. Check: ImportOrchestrator line 196-202 handles the html case by converting to `string[]`. Since `importFileSource` passes data as-is to `orchestrator.import(source, data)`, and the html case does `typeof data === 'string' ? [data] : Array.isArray(data) ? data.map(...)`, passing a JSON string will result in `[jsonString]` — a single HTML card from the JSON text.

Actually, looking more carefully at the html case: it checks `typeof data === 'string'` first, wrapping in array. But the fixture is a JSON array of HTML strings. To import multiple HTML files, the data must be passed as an array (not a JSON string). But `importFileSource` passes string data. For HTML, we need to pass individual HTML strings. The simplest approach: load the fixture, parse it to get the array, and import each HTML string individually OR pass the first HTML string.

Better approach: Read the fixture, get the HTML strings array, and call importFileSource for each one, or pass the raw fixture string (which will be treated as a single HTML string). For the E2E test, pass the first HTML entry as a string:

```typescript
describe('HTML parser E2E', () => {
```

Add a 7th describe block to `tests/etl-validation/file-format-e2e.test.ts` for cross-format dedup collision detection (SC4).

The test imports the same card title from two different formats (JSON and CSV) and asserts 2 distinct rows exist with different source values. DedupEngine deduplicates by (source, source_id) — same title with different source = distinct rows.

```typescript
describe('Cross-format dedup collision detection', () => {
  let db: Database;
  beforeEach(async () => { db = await createTestDb(); });
  afterEach(() => db.close());

  it('same title from JSON and CSV produces two distinct rows', async () => {
    // Import a minimal JSON fixture with a known title
    const jsonData = JSON.stringify([
      { title: 'Shared Title Card', body: 'From JSON source', tags: [] }
    ]);
    const jsonResult = await importFileSource(db, 'json', jsonData);
    expect(jsonResult.inserted).toBe(1);

    // Import a CSV fixture with the same title but different source
    const csvData = JSON.stringify([
      { path: 'test.csv', content: 'title,content\nShared Title Card,From CSV source' }
    ]);
    const csvResult = await importFileSource(db, 'csv', csvData);
    expect(csvResult.inserted).toBe(1);

    // Query all cards with this title
    const stmt = db.prepare<{ name: string; source: string }>(
      "SELECT name, source FROM cards WHERE name = 'Shared Title Card' AND deleted_at IS NULL"
    );
    const rows = stmt.all();
    stmt.free();

    // Assert 2 distinct rows with different source values
    expect(rows.length).toBe(2);
    const sources = rows.map(r => r.source).sort();
    expect(sources).toEqual(['csv', 'json']);
  });
});
```

This test proves that DedupEngine's source-aware dedup creates separate rows for the same title imported from different file formats — the dedup key includes source, so (json, '0') and (csv, 'test.csv:0') are distinct entries.

  <action>
In `src/worker/handlers/graph-algorithms.handler.ts`, implement these 6 pure algorithm functions (each takes UndirectedGraph and optional params, returns a Map or partial metric data):

1. **computeShortestPath(g: UndirectedGraph, sourceCardId: string | undefined)**:
   - Import `dijkstra` from `graphology-shortest-path`
   - If sourceCardId is provided, use `dijkstra.bidirectional(g, source, target)` — but since we need source-to-all-on-path, use `dijkstra.singleSource(g, sourceCardId)` to get distances to all reachable nodes
   - Per CONTEXT.md: sp_depth written ONLY for cards on the actual source-to-target path — other cards get sp_depth = null
   - For the response: when shortest_path is requested with sourceCardId, find the shortest path using `dijkstra.bidirectional(g, sourceCardId, targetCardId)` — but since targetCardId is not in params yet, use source-only mode: write sp_depth = distance from source for ALL reachable cards from source
   - Actually re-reading CONTEXT.md: `shortest_path?.sourceCardId` is the only param. Write sp_depth for all reachable cards from sourceCardId. Return `pathCardIds: []` (path requires target which comes in Phase 117). Set `reachable` based on whether source exists in graph
   - If sourceCardId is undefined, auto-select the node with highest degree as source
   - Unreachable nodes get sp_depth = null. Never store Infinity

2. **computeBetweennessCentrality(g: UndirectedGraph)**:
   - Import `betweennessCentrality` from `graphology-metrics/centrality/betweenness`
   - If g.order > 2000, pass `{ nodeSampling: Math.ceil(Math.sqrt(g.order)) }` option for sqrt(n) pivot sampling
   - Returns `Record<string, number>` mapping nodeId to centrality score
   - Normalize scores to [0, 1] range

3. **computeLouvainCommunity(g: UndirectedGraph, resolution?: number)**:
   - Import `louvain` from `graphology-communities-louvain`
   - Call `louvain(g, { resolution: resolution ?? 1 })` — returns `Record<string, number>`
   - Isolated nodes (degree 0) get community_id = null (filter them out post-Louvain)

4. **computeClusteringCoefficient(g: UndirectedGraph)**:
   - Import from `graphology-metrics` — use `localClusteringCoefficient` or compute manually
   - Nodes with degree < 2 get clustering_coeff = 0 (mathematically defensible)
   - Returns `Record<string, number>`

5. **computeMinimumSpanningTree(g: UndirectedGraph)**:
   - Custom Kruskal's implementation (~50 LOC) with Union-Find
   - Sort edges, iterate with union-find to select non-cycle edges
   - Returns `{ edges: Array<[string, string]>, componentCount: number }`
   - MST produces spanning forest for disconnected graphs — edge count = n - componentCount

6. **computePageRank(g: UndirectedGraph, alpha?: number, iterations?: number)**:
   - Import `pagerank` from `graphology-metrics/centrality/pagerank`
   - Use alpha = 0.85 (default), iterations = 100 (default)
   - Returns `Record<string, number>` — scores should sum to ~1.0

**Then update handleGraphCompute to:**
- After building UndirectedGraph, compute connected components using BFS to get componentCount
- Iterate over `payload.algorithms` array and call the corresponding compute function
- Collect results into a single `GraphMetricsRow[]` array, one row per card
- For each row, merge results from all requested algorithms (partial — only overwrite columns for requested algorithms)
- Call `sanitizeAlgorithmResult()` on each row
- Call `writeGraphMetrics(db, rows)` once (transactional batch write)
- Return updated response with `algorithmsComputed`, `componentCount`, `pathCardIds`, `reachable`, `durationMs`

**Error handling:** If any algorithm throws, catch and skip that algorithm (no partial writes from failed algorithms). Log the error. Still write results from successful algorithms.

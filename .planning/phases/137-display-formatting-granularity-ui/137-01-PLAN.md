  <name>Time Bucket Display Formatter</name>
  <files>src/views/supergrid/formatTimeBucket.ts, tests/views/supergrid/formatTimeBucket.test.ts, src/views/pivot/PivotGrid.ts</files>
  <behavior>
    - formatTimeBucket('2026-03') returns 'Mar 2026' (month)
    - formatTimeBucket('2026-03-15') returns 'Mar 15, 2026' (day)
    - formatTimeBucket('2026-W14') returns 'Week 14, 2026' (week)
    - formatTimeBucket('2026-Q1') returns 'Q1 2026' (quarter)
    - formatTimeBucket('2026') returns '2026' (year — passthrough)
    - formatTimeBucket('__NO_DATE__') returns 'No Date' (sentinel)
    - formatTimeBucket('some_category') returns 'some_category' (non-time passthrough)
    - Format detection is regex-based on the bucket string shape, not on granularity parameter
  </behavior>
  <implementation>
    Create `src/views/supergrid/formatTimeBucket.ts` as a pure function:

    1. Export `formatTimeBucket(bucketLabel: string): string`
    2. Check for NO_DATE_SENTINEL first — return 'No Date'
    3. Pattern-match the bucket string to detect granularity:
       - `/^\d{4}-\d{2}-\d{2}$/` → day: parse with `new Date(str + 'T00:00:00Z')`, format with `d3.utcFormat('%b %d, %Y')`
       - `/^\d{4}-W\d{2}$/` → week: extract year and week number, return `'Week {ww}, {yyyy}'`
       - `/^\d{4}-\d{2}$/` → month: parse with `new Date(str + '-01T00:00:00Z')`, format with `d3.utcFormat('%b %Y')`
       - `/^\d{4}-Q[1-4]$/` → quarter: extract year and quarter, return `'Q{n} {yyyy}'`
       - `/^\d{4}$/` → year: return as-is
       - Else → passthrough (non-time field value)
    4. Use `import * as d3 from 'd3'` per project convention (no named sub-package imports)
    5. `d3.utcFormat` (not `d3.timeFormat`) per D3 formatting decision in STATE.md

    Then integrate into PivotGrid._renderOverlay():
    - Import `formatTimeBucket` from `../supergrid/formatTimeBucket`
    - Replace both `.text(spanInfo.label)` calls (line ~519 for col headers, line ~551 for row headers) with `.text(formatTimeBucket(spanInfo.label))`
    - This is a 2-line surgical change in PivotGrid.ts
  </implementation>
</feature>

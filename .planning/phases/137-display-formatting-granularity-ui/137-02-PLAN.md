<task type="auto" tdd="true">
  <name>Task 1: Conditional granularity selector visibility</name>
  <files>src/ui/ProjectionExplorer.ts, tests/ui/ProjectionExplorer.test.ts</files>
  <behavior>
    - When colAxes and rowAxes contain no time-classified fields, granularity label + select are hidden (display:none)
    - When a time field (e.g., created_at, modified_at, due_at) is added to colAxes or rowAxes, granularity label + select become visible
    - Time field detection uses SchemaProvider.getFieldsByFamily('Time') when schema is available
    - When schema is not available, falls back to hardcoded set: created_at, modified_at, due_at
    - Visibility updates on pafv.subscribe() callback (axis changes) and on initial render
  </behavior>
  <action>
    Add a private method `_hasTimeAxis(): boolean` to ProjectionExplorer:
    1. Get current axes: `const { colAxes, rowAxes } = this._config.pafv.getState()`
    2. Build time field set: if `this._config.schema` exists, use `this._config.schema.getFieldsByFamily('Time').map(c => c.name)` to build a Set; otherwise fallback to `new Set(['created_at', 'modified_at', 'due_at'])`
    3. Return true if any axis field in colAxes or rowAxes is in the time field set

    Add a private method `_syncGranularityVisibility(): void`:
    1. Find the granularity label and select elements (the `span` with textContent 'Granularity' and the second `.z-controls__density` select)
    2. Store references to the granularity label and select as instance fields `_granLabel` and `_granSelect` during `_buildZControls()` for efficient access
    3. Set `display = this._hasTimeAxis() ? '' : 'none'` on both elements

    Call `_syncGranularityVisibility()`:
    - At the end of `_buildZControls()` (initial state)
    - Inside the pafv.subscribe() callback (when axes change)
    - Inside `_syncZControls()` (when SuperDensityProvider state changes)

    This is a surgical change — the granularity `<select>` and its label already exist. We only add show/hide logic.
  </action>

  </verify>

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Chevron margin, chip glyph gap, resize handle visual border |
| sm | 8px | Plugin row padding gap, chip internal gap, button gap |
| md | 12px | Data section padding, config grid gap |
| lg | 16px | Sidebar section padding, grid wrapper padding, header padding |
| xl | 24px | Config panel horizontal padding, header horizontal padding |
| 2xl | 32px | Plugin row left-indent (32px = 16 base + 16 category indent) |
| 3xl | 48px | Not used in this phase |

**Source:** Observed in harness.css and pivot.css — existing tokens match this scale.

Exceptions:
- Column resize handle touch target: 4px visual width, pointer events captured on a 12px hotspot (existing `.pv-resize-handle--width` pattern)
- Row resize handle touch target: 4px visual height, 12px pointer hotspot (existing `.pv-resize-handle--height` pattern)
- Zoom slider: full sidebar width minus 32px horizontal padding (fluid, not fixed)
- SuperCalc footer row height: 32px (matches `--space-2xl` convention for data rows)
- Sort indicator chevron: 10px glyph, 4px margin-left of header label

| Role | Token | Value (dark default) | Usage |
|------|-------|----------------------|-------|

**Accent reserved for:**
1. Active sort column header indicator arrow (`↑` / `↓` glyph color)
2. Column and row resize handle on hover (`--pv-resize-hover`)

4. Zoom slider thumb and focus ring (browser native `accent-color: var(--pv-accent)`)
5. HarnessShell sidebar active badge background (`--hns-badge-active-bg`)
6. Aggregate cell left accent border (reuses `.pv-agg-cell` pattern from Phase 99)

**Sort indicator at rest:** `var(--pv-muted-fg)` — `#94a3b8` (dark) / `#64748b` (light). Dimmed to avoid visual noise when sorted by a non-prominent column.

**SuperCalc footer background:** `var(--pv-accent-light)` — `#1e3a5f` (dark) / `#dbeafe` (light). Matches `.pv-agg-cell` pattern established in Phase 99.

**Source:** pivot.css (complete --pv-* token system), harness.css (--hns-* tokens), CONTEXT.md (D-016 --pv-* namespace mandate).

## Interaction Contract

### SuperSize — Column Resize (`supersize.col-resize`)

- **Trigger:** Pointer down on `.pv-resize-handle--width` (right edge of column header, 4px wide, 12px pointer hotspot)
- **During drag:** `cursor: col-resize` on `<body>`, `pointer-events: none` on overlay to prevent hover interference
- **Visual feedback:** Resize handle turns `var(--pv-accent)` immediately on hover (no delay)
- **Width clamping:** minimum 48px (`MIN_COL_WIDTH` from SuperGridSizer), maximum unconstrained
- **Updates:** `--pv-col-width-{N}` CSS custom property updated on each pointermove (16ms throttle)
- **Commit:** On pointerup, final width written to `GridLayout.colWidths` Map
- **Double-click auto-fit:** Max content width + 24px padding (`AUTO_FIT_PADDING`), capped at 400px (`AUTO_FIT_MAX`)
- **Shift+drag normalize:** Dragging with Shift sets all columns to same width as dragged column

### SuperSize — Header Resize (`supersize.header-resize`)

- **Trigger:** Pointer down on `.pv-resize-handle--height` (bottom edge of row header area, 4px tall)
- **During drag:** `cursor: row-resize` on `<body>`
- **Updates:** `--pv-row-height` CSS custom property updated on each pointermove
- **Height clamping:** minimum 24px, maximum 120px
- **Commit:** On pointerup, final height written to `GridLayout`

### SuperSize — Uniform Resize (`supersize.uniform-resize`)

- **Trigger:** `.pv-resize-handle--cell` corner handle on grid corner cell
- **Behavior:** Simultaneously adjusts `--pv-col-width` (default) and `--pv-row-height` by same delta ratio
- **Cursor:** `nwse-resize` (existing class)

### SuperZoom — Wheel Zoom (`superzoom.wheel`)

- **Trigger:** Ctrl+wheel or trackpad pinch gesture over grid area
- **Scale formula:** Asymmetric — port `wheelDeltaToScaleFactor()` from `SuperZoom.ts` verbatim
- **Range:** [0.5, 3.0] clamped (ZOOM_MIN / ZOOM_MAX from SuperPositionProvider)
- **Reset:** Cmd+0 resets to 1.0
- **Application:** Updates `--pv-zoom` CSS custom property via `afterRender` hook; does NOT use CSS `transform: scale`
- **Event:** `{ passive: false }` — must call `e.preventDefault()` on Ctrl+wheel to block browser zoom

### SuperZoom — Slider (`superzoom.slider`)

- **Location:** HarnessShell sidebar under SuperZoom category when `superzoom.slider` is enabled
- **Element:** `<input type="range">` with `min="0.5" max="3" step="0.05"`
- **Label:** "Zoom" in 11px uppercase muted style (matches `.hns-data-label` pattern)
- **Value display:** Numeric readout "1.0x" at 11px, weight 500, right-aligned next to slider
- **Sync:** Slider position reflects current `--pv-zoom` value; wheel zoom updates slider in real-time

### SuperSort — Header Click (`supersort.header-click`)

- **Trigger:** Click on any leaf column header span (`.pv-col-span--leaf`)
- **Cycle:** none → ASC → DESC → none (three-state)
- **Sort indicator:** Unicode arrow appended inside header label — `↑` for ASC, `↓` for DESC, no glyph for unsorted
- **Arrow styling:** 10px, weight 400, `var(--pv-accent)` when active, `var(--pv-muted-fg)` on hover-preview (before click)
- **Active header background:** `var(--pv-cell-hover-bg)` persistent (not just on hover) when sorted
- **Only one column sorted at a time** in this phase — clicking a new column clears previous sort

### SuperSort — Chain Sort (`supersort.chain`)

- **Trigger:** Shift+click on a leaf column header
- **Behavior:** Adds column to sort priority chain (primary → secondary → tertiary)
- **Visual:** Sort indicators show numeric priority suffix — `↑1`, `↓2`, etc. at 9px
- **Chain limit:** Up to 3 columns; Shift+clicking a 4th replaces the oldest

### SuperScroll — Virtual Scrolling (`superscroll.virtual`)

- **Implementation:** Data windowing in `transformData` hook — filter `CellPlacement[]` array to visible row range only
- **NOT DOM virtualization** — preserves D3 data join ownership (D-005 pattern)
- **Scroll sentinel:** Two zero-height spacer `<tr>` elements (top and bottom) whose `height` style maintains total scroll area
- **Visible range:** `getVisibleRange(scrollTop, rowHeight, containerHeight, totalRows)` — port from `SuperGridVirtualizer.ts`
- **Buffer:** 2 rows above and below visible range rendered as buffer (prevents flash on fast scroll)
- **Trigger:** `onScroll` hook fires on grid scroll event, calls `registry.run('transformData')` only if new range differs

### SuperScroll — Sticky Headers (`superscroll.sticky-headers`)

- **Implementation:** CSS `position: sticky; top: 0` on overlay header row elements
- **z-index:** Headers at z-index 20, data cells at z-index 1 (existing Layer 2 overlay pattern)
- **No JS required** — pure CSS sticky positioning within overflow scroll container

### SuperCalc — Footer Row (`supercalc.footer`)

- **Position:** Sticky-positioned `<div>` below grid scroll area, managed by `afterRender` hook
- **Layout:** One cell per column, same widths as data columns (`--pv-col-width-{N}`)
- **Cell styling:** `var(--pv-accent-light)` background, `border-left: 3px solid var(--pv-accent)` on first cell only
- **Font:** 13px monospace (`var(--pv-number-font)`), weight 600, right-aligned
- **Default function:** SUM — displayed as "∑ {value}" with ∑ glyph at `var(--pv-muted-fg)`
- **Empty/no-data:** Shows "—" at `var(--pv-cell-empty-fg)` when column has no numeric data

### SuperCalc — Config (`supercalc.config`)

- **Location:** Per-column dropdown in HarnessShell sidebar (only plugin requiring sidebar UI beyond a toggle)
- **Element:** `<select>` with options: SUM, AVG, COUNT, MIN, MAX, NONE
- **Label:** Column name truncated to 16 chars, 11px uppercase muted
- **Layout:** Vertical list of column controls, one per column present in current data

| Element | Copy |
|---------|------|

## Layout Contract: 5-Slot CSS Grid

Phase 175 expands SuperWidget from a 4-row grid to a 5-slot grid that also includes the sidebar column. This is the central structural change of the phase.

### New Grid Template

```
[data-component="superwidget"] {
  display: grid;
  grid-template-columns: auto 1fr;
  grid-template-rows: auto auto 1fr auto;
  grid-template-areas:
    "sidebar header"
    "sidebar tabs"
    "sidebar canvas"
    "sidebar status";
}
```

| Slot | data-slot | grid-area | Background Token | Notes |
|------|-----------|-----------|-----------------|-------|
| Sidebar | sidebar | sidebar | --bg-surface | DockNav mounts here; bimodal: 48px / 280px; border-right: 1px solid var(--border-subtle) |
| Header | header | header | --sw-header-bg (--bg-surface) | CommandBar mounts here; border-bottom: 1px solid var(--sw-slot-border) |
| Tabs | tabs | tabs | --sw-tabs-bg (--bg-surface) | TabBar strip; unchanged from Phase 174; border-bottom: 1px solid var(--sw-slot-border) |
| Canvas | canvas | canvas | --sw-canvas-bg (--bg-secondary) | ViewCanvas / EditorCanvas / ExplorerCanvas; overflow: hidden; 1fr row |
| Status | status | status | --sw-status-bg (--bg-primary) | Status bar; min-height: 0; always in DOM |

### Sidebar Slot Interaction

- Sidebar width switches between `48px` (icon-only) and `280px` (icon-thumbnail) via a class toggle on `[data-slot="sidebar"]` — matching the existing WorkbenchShell `.workbench-sidebar--icon-only` / `.workbench-sidebar--icon-thumbnail` pattern.
- `flex: 1 1 auto` and `min-height: 0` remain on the SuperWidget root to preserve flex chain height propagation.
- The sidebar slot uses `grid-row: 1 / -1` (spans all 4 rows) to match current behavior where DockNav sits full-height beside all content.

### Temporary Explorer Passthrough Slots (D-02)

Two container divs are injected into the canvas slot to receive explorer content until Phase 176 replaces them with the real sidecar grid:

| Container | Class | max-height | border | padding |
|-----------|-------|-----------|--------|---------|
| Top passthrough | .sw-explorer-slot-top | 50vh | border-bottom: 1px solid var(--border-subtle) | var(--space-md) |
| Bottom passthrough | .sw-explorer-slot-bottom | 30vh | border-top: 1px solid var(--border-subtle) | var(--space-md) |

These classes mirror the existing `.workbench-slot-top` / `.workbench-slot-bottom` visual contract exactly so Phase 176 has a predictable target.

## Interaction Contract

### Tab interactions
Unchanged from Phase 174. No new tab interactions in Phase 175.

### Sidebar width toggle
- Triggered by the same mechanism that currently triggers `.workbench-sidebar--icon-only` / `.workbench-sidebar--icon-thumbnail` class toggle
- Target: `[data-slot="sidebar"]` instead of `.workbench-sidebar`
- Transition: none (instant width change matches existing WorkbenchShell behavior)

### View switching
- `commitProjection()` lifecycle unchanged — canvas slot receives mount/destroy calls
- No visual regression from current behavior: the canvas area renders identically
- Success criterion SHEL-01: app loads and renders identically to pre-migration

### StateCoordinator drain (SHEL-05)
- Big-bang swap — no mid-flight teardown of a live shell
- SuperWidget is fully wired before any providers fire notifications
- No UI interaction contract needed: ordering is a code-sequencing concern, not a user-facing interaction

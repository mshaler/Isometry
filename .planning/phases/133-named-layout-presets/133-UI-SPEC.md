## Component Inventory

### New CSS Classes Required (Phase 133)

This phase adds a new 'Presets' category to the existing command palette. No new UI surfaces. All new elements follow established BEM patterns from `command-palette.css`.

#### Preset Suggestion Toast

A non-blocking informational toast shown on dataset switch when a preset association exists. Follows the import-toast pattern (top-right, slide-in from top) but uses the action-toast color scheme (accent-border, bg-surface).

```
.preset-suggestion-toast          — container, fixed top-right (z-index: 9999)
.preset-suggestion-toast__text    — "{presetName}" preset available copy
.preset-suggestion-toast__apply   — inline button "Apply" affordance
.preset-suggestion-toast__dismiss — "×" dismiss button (optional, auto-dismiss at 5000ms)
```

Position: fixed top-right, mirroring `.import-toast` layout.
Entry animation: `translateY(-8px)` → `translateY(0)` with `opacity: 0` → `1`, matching import-toast.
Border: `1px solid var(--accent-border)` on `var(--bg-surface)`.
Auto-dismiss: 5000ms (D-09 from CONTEXT.md).

#### Preset Name Input State (Command Palette Reuse)

The 'Save Layout as Preset' command reuses the existing `.command-palette__input`. No new elements needed. The input placeholder text changes to "Name your preset…" while in preset-naming mode. Visual treatment is identical to the existing input — no style additions required.

#### Preset Category in Palette

New category label: "PRESETS" — uses existing `.command-palette__category` class verbatim. No new styles.

Preset commands use existing `.command-palette__option` and `.command-palette__label` classes. No new styles.

Delete Preset commands: use `var(--danger)` on `.command-palette__label` (inline style or modifier class `.command-palette__label--danger`).

| Preset Name | Panel Configuration Intent |

- Destructive palette options (Delete Preset) render in `var(--danger)` color — no modal confirmation, relies on Cmd+Z undo.

## Interaction Contract

### Command Palette Preset Flows

**Apply Built-in or Custom Preset:**
1. User opens palette (Cmd+K)
2. Types preset name or scrolls to "PRESETS" category
3. Selects "Apply Preset: {name}" — Enter or click
4. `WorkbenchShell.restoreSectionStates()` called with preset's `Record<storageKey, boolean>`
5. Mutation registered with MutationManager (undoable — Cmd+Z restores previous state)
6. Dataset-to-preset association saved: `preset:association:{datasetId}` = preset name
7. Action toast appears at bottom-center: "Applied preset "{name}"" for 2000ms

**Save Custom Preset:**
1. User opens palette (Cmd+K)
2. Selects "Save Layout as Preset"
3. Palette input placeholder changes to "Name your preset…" — input cleared
4. User types name, presses Enter
5. `WorkbenchShell.getSectionStates()` captures current state
6. Saved to ui_state: `preset:name:{name}` = JSON serialized `Record<storageKey, boolean>`
7. Action toast: "Preset "{name}" saved" for 2000ms
8. New "Apply Preset: {name}" option appears in palette immediately

**Delete Custom Preset:**
1. User opens palette (Cmd+K)
2. Types "Delete" or preset name to filter
3. Selects "Delete Preset: {name}" (red label)
4. Preset removed from ui_state immediately
5. Action toast: "Preset "{name}" deleted" — no undo (deletion is not undoable)
6. Built-in presets never appear under Delete category

**Dataset-Preset Suggestion:**
1. User switches dataset (via DataExplorer)
2. If `preset:association:{datasetId}` exists, `preset-suggestion-toast` appears top-right
3. Toast shows: 'Layout preset "{name}" was last used here. [Apply]'
4. [Apply] button: applies preset immediately (same as step 3-7 above)
5. Toast auto-dismisses after 5000ms if not interacted with
6. Toast does NOT stack with import-success toast — shown 500ms after import toast clears

### Keyboard Navigation

All new interactions follow existing keyboard patterns:
- Preset options in palette: arrow key navigation, Enter to confirm, Escape to cancel — identical to all other palette options
- Preset naming: typing in existing input, Enter to save, Escape to cancel and close palette
- Toast [Apply] button: focusable, Enter/Space activates

### Accessibility

- Preset category "PRESETS" in palette: `role="group"` with `aria-label="Presets"` — follows existing category pattern
- Dataset-preset suggestion toast: `role="status"` live region so screen readers announce it
- [Apply] button in toast: `type="button"`, accessible label "Apply preset {name}"
- Delete Preset option: no additional ARIA beyond existing palette option pattern

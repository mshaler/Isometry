Create `src/presets/presetCommands.ts`:

Export a `createPresetCommands` factory function:

```typescript
export function createPresetCommands(deps: {
  presetManager: LayoutPresetManager;
  registry: CommandRegistry;
  palette: CommandPalette;
  actionToast: ActionToast;
  getActiveDatasetId?: () => string | null;
}): void
```

This function:

1. **Generates commands** from `presetManager.listAll()`:
   - For each preset: `{ id: 'preset:apply:{name}', label: 'Apply Preset: {name}', category: 'Presets', execute: () => { ... } }`
   - Apply execute handler: calls `presetManager.applyPreset(name)`, then `actionToast.show('Applied preset \u201C{name}\u201D')` (curly quotes per UI-SPEC). If `getActiveDatasetId` returns a value, also calls `presetManager.setAssociation(datasetId, name)`.
   - For each custom (non-built-in) preset: `{ id: 'preset:delete:{name}', label: 'Delete Preset: {name}', category: 'Presets', execute: () => { ... } }`
   - Delete execute handler: calls `presetManager.deleteCustom(name)`, then `actionToast.show('Preset \u201C{name}\u201D deleted')`, then calls `refreshCommands()` to update registry.

2. **Save command**: `{ id: 'preset:save', label: 'Save Layout as Preset', category: 'Presets', execute: () => { ... } }`
   - Save execute handler: This is the tricky part. It needs to prompt for a name. Since the command palette closes after execute (see `_executeSelected` which calls `close()` then `execute()`), the save command must re-open the palette in a "naming mode".
   - Implementation approach: Dispatch a custom event `isometry:preset-name-prompt` that CommandPalette listens for. When received, palette opens with placeholder "Name your preset\u2026" (per UI-SPEC), and on Enter, calls `presetManager.saveCustom(inputValue)` then `actionToast.show('Preset \u201C{name}\u201D saved')` then `refreshCommands()`.
   - Alternative simpler approach: Use `window.prompt('Name your preset:')` for MVP, then enhance later. But this conflicts with the UI-SPEC which says "reuses existing palette text input infrastructure."
   - Best approach: Add a `promptForInput(placeholder: string, onConfirm: (value: string) => void)` method to CommandPalette. When called, it opens the palette with the custom placeholder, hides command results, and on Enter calls onConfirm with the input value then closes. The save command's execute handler calls `palette.promptForInput('Name your preset\u2026', async (name) => { await presetManager.saveCustom(name); actionToast.show(...); refreshCommands(); })`.

3. **`refreshCommands()` internal function**: Unregisters all `preset:*` commands from registry, regenerates from `presetManager.listAll()`, re-registers. This handles dynamic command list updates after save/delete.
   - To support unregister, add an `unregisterByPrefix(prefix: string)` method to CommandRegistry that removes commands whose id starts with the prefix. Add this method in CommandRegistry.ts.

4. **Initial registration**: calls `refreshCommands()` once to populate.

Add `promptForInput(placeholder: string, onConfirm: (value: string) => void)` to `CommandPalette`:
- Opens palette, clears input, sets placeholder to provided string
- Hides command results (listbox hidden or empty)
- Stores onConfirm callback
- On Enter in this mode: calls `onConfirm(inputValue.trim())` if non-empty, then closes palette and resets placeholder to default
- On Escape: closes palette, resets placeholder, clears callback
- Flag: `_promptMode = true` to distinguish from normal search mode (skip search-as-you-type when in prompt mode)

Add `unregisterByPrefix(prefix: string)` to `CommandRegistry`:
- Removes all commands from `this.commands` array and `this.byId` map where `cmd.id.startsWith(prefix)`

Wire in `src/main.ts`:

- Call `createPresetCommands({ presetManager, registry, palette, actionToast, getActiveDatasetId: () => stateManager.getActiveDatasetId() })`

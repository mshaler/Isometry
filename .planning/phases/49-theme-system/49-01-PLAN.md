<objective>
Restructure the CSS design token system for multi-theme support and eliminate all hardcoded colors from TypeScript rendering code.

Purpose: This is the foundation that all theme switching depends on. Without scoped token palettes and clean CSS var() references everywhere, toggling `data-theme` would leave hardcoded colors unchanged, creating a broken visual experience.

Output: design-tokens.css with dark/light/system token blocks, all view renderers using CSS var() references, theme transition CSS, FOWT-prevention body styling.
</objective>

Create `src/providers/FilterProvider.ts`:

- Internal state: `{ filters: Filter[], searchQuery: string | null }`
- `addFilter(filter)`: validates field and operator via allowlist assertions, pushes to array, calls scheduleNotify()
- `removeFilter(index)`: removes by index, calls scheduleNotify()
- `clearFilters()`: empties array, clears searchQuery, calls scheduleNotify()
- `setSearchQuery(query: string | null)`: sets search, calls scheduleNotify()
- `getFilters()`: returns readonly copy
- `compile(): CompiledFilter`:
  - Start with `clauses = ['deleted_at IS NULL']`, `params = []`
  - For each filter, validate field/operator, compile to clause+params based on operator
  - If searchQuery set, add FTS clause with tokenized query
  - Return `{ where: clauses.join(' AND '), params }`
- `subscribe(cb): () => void`: adds to Set, returns removal function (PROV-11)
- `scheduleNotify()`: uses `queueMicrotask` with pendingNotify guard per CONTEXT.md
- `toJSON()`: serializes state to JSON string
- `fromJSON(json)`: static factory that parses and validates (corrupt JSON → throw)
- Implements `PersistableProvider` interface (toJSON, setState, resetToDefaults)

Create `src/providers/index.ts` with re-exports of types.ts, allowlist.ts, and FilterProvider.

IMPORTANT: The `compile()` method validates field/operator at compile time even if addFilter already validated. This handles the case where state is restored from JSON (CONTEXT.md: corrupt JSON resets that provider to defaults).

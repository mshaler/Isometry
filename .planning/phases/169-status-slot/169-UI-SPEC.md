## Layout and DOM Contract

The status slot is rendered inside `[data-slot="status"]` on the SuperWidget.
This slot is always in the DOM (never `display: none`), has `min-height: 0`,
and `overflow: hidden`.

**Inner structure — three `<span>` elements, one wrapper `<div>`:**

```
div.sw-status-bar
  span.sw-status-bar__item[data-stat="cards"]    "42 cards"
  span.sw-status-bar__sep                         "·"
  span.sw-status-bar__item[data-stat="connections"] "18 connections"
  span.sw-status-bar__sep                         "·"
  span.sw-status-bar__item[data-stat="last-import"]  "Imported 2 min ago"
```

**CSS class conventions — follow existing `--sw-*` namespace:**
- Wrapper: `.sw-status-bar` — `display: flex; align-items: center; gap: var(--space-xs); padding: var(--space-xs) var(--space-sm); font-size: var(--text-sm); color: var(--text-muted);`
- Items: `.sw-status-bar__item` — no additional style beyond inheritance
- Separators: `.sw-status-bar__sep` — `color: var(--text-muted); user-select: none;`

**Slot update rule (STAT-04):** Only `.sw-status-bar` inner spans are mutated
via `textContent` assignment. No canvas slot, no tab containers, no DataExplorerPanel
DOM is touched during a status update. The `data-render-count` attribute on
`[data-slot="status"]` must NOT be incremented on status updates (slot-scoped
update pattern from CANV-06 constraint).

## States

All states are always visible — no conditional hide/show of the slot itself:

| State | Display |
|-------|---------|
| Zero state (no data) | `0 cards · 0 connections · No imports yet` |
| Populated | `42 cards · 18 connections · Imported 2 min ago` |
| Singular card | `1 card · 18 connections · Imported 5 min ago` |
| Singular connection | `42 cards · 1 connection · Imported 5 min ago` |

No loading spinner, no skeleton, no animation. Count changes happen
instantaneously (`textContent` swap, no CSS transition needed).

## Copywriting Contract

| Element | Copy |
|---------|------|
| Cards span (populated) | `{N} cards` — singular: `1 card` |
| Connections span (populated) | `{N} connections` — singular: `1 connection` |
| Timestamp span (populated) | `Imported {relative}` — e.g. `Imported 2 min ago`, `Imported 1 hour ago`, `Imported yesterday` |
| Timestamp span (no imports) | `No imports yet` |
| Zero-count card span | `0 cards` |
| Zero-count connection span | `0 connections` |
| Separator between spans | `·` (U+00B7 MIDDLE DOT) — no spaces in markup, gap from CSS flexbox |

Relative time format (CONTEXT.md D-02):
- Under 1 minute: `just now`
- 1–59 minutes: `{N} min ago`
- 1–23 hours: `{N} hour ago` / `{N} hours ago`
- 24+ hours same calendar day: `yesterday`
- Older: `{M} {D}` short date (e.g. `Apr 19`)

No external dependency for relative time. Implement as a minimal inline formatter
(< 20 lines). No Intl.RelativeTimeFormat polyfill needed — target is macOS 14+ /
iOS 17+ where Intl is fully supported, but the custom formatter keeps it trivial.

Primary CTA: none — the status bar is read-only, no user actions.
Destructive actions: none in this phase.
Error state: none displayed in slot — if Worker query fails, preserve last-known
counts (silent failure, no error copy in the slot).

Source: CONTEXT.md D-01, D-02, D-03 — pre-populated.

## Interaction Contract

The status slot is **read-only and non-interactive**. No click targets, no hover
states, no focus rings. `pointer-events: none` is acceptable if needed to prevent
accidental text selection, but the default behavior (selectable muted text) is fine.

No destructive confirmations. No dialogs. No modals.

Update trigger: piggybacks on `refreshDataExplorer()` in `main.ts`. Each call to
`refreshDataExplorer()` also calls a status slot update function. The status slot
update queries card count, connection count, and last import timestamp via the
Worker bridge in a single compound query (single roundtrip). The relative timestamp
is formatted client-side from the raw `imported_at` value returned.

# Phase 146 — UI Design Contract

> Visual and interaction contract for Phase 146: DockNav Shell + SidebarNav Swap.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — manual CSS custom properties |
| Preset | not applicable |
| Component library | none (pure TypeScript + D3, no framework) |
| Icon library | Lucide SVGs via `iconSvg()` helper in `src/ui/icons.ts` |
| Font | -apple-system / BlinkMacSystemFont / system-ui (San Francisco on Apple) — `var(--font-sans)` |

No shadcn gate applies. This project uses a hand-authored CSS token system in `src/styles/design-tokens.css` covering 5 themes. All new DockNav CSS must use existing custom properties exclusively — no new raw hex values or hardcoded sizes.

## Spacing Scale

Declared values from `src/styles/design-tokens.css` (theme-independent tokens):

| Token | Value | Usage |
|-------|-------|-------|
| `--space-xs` | 4px | Icon-to-label gap within a dock item |
| `--space-sm` | 8px | Padding around section header divider text |
| `--space-md` | 12px | Internal dock item padding (vertical) |
| `--space-lg` | 16px | Dock item touch target side padding |
| `--space-xl` | 24px | Vertical gap between dock sections |

Exceptions:
- Dock container width: exactly 48px (D-01, D-06 — not a spacing token, set as a fixed width on `.workbench-sidebar`)
- Dock item touch target: 48×48px per item (DOCK-01 — icon square is 48×48)
- Section header divider text area: min-height 20px (compact, unobtrusive — macOS Finder style per D-03)

## Typography

All sizes from `src/styles/design-tokens.css` — no new font-size values.

| Role | Size | Token | Weight | Line Height | Usage |
|------|------|-------|--------|-------------|-------|
| Dock item label | 10px | `--text-xs` | 400 (regular) | 1.2 | Item label below icon (stacked layout) |
| Section header divider | 10px | `--text-xs` | 600 (semibold) | 1.2 | Muted divider text (Integrate, Visualize, etc.) |

Weights: regular (400) for item labels; semibold (600) for section header dividers only.

Note: DockNav is icon-dominant. Labels are secondary — 10px at weight 400, centered below icon. Section headers are not interactive — small caps dividers at 10px weight 600.

## Color

All values resolved via CSS custom properties. No raw hex in dock CSS.

| Role | Token | Value (dark default) | Usage |
|------|-------|---------------------|-------|
| Dominant (60%) — dock background | `var(--bg-surface)` | #252540 | Dock container background (matches workbench sidebar) |
| Secondary (30%) — dock items inactive | `var(--text-secondary)` | #a0a0b0 | Inactive dock item icon fill + label color |
| Accent (10%) — active item | `var(--accent)` | #4a9eff | Active dock item icon color ONLY (D-04) |
| Section divider text | `var(--text-muted)` | #858596 | Section header label (Integrate, Visualize…) |
| Dock right border | `var(--border-subtle)` | rgba(128,128,128,0.2) | Border between dock and main content |
| Item hover background | `var(--cell-hover)` | rgba(128,128,128,0.08) | Hover state on dock items |

Accent reserved for: active dock item icon color only. No background fill, no edge bar, no border on active state (D-04 is explicit: icon color change only).

Destructive: no destructive actions exist in this phase. `var(--danger)` is not used.

All 5 themes (dark, light, system, nextstep, material) are covered by the existing token definitions. DockNav uses only tokens already present in all 5 theme blocks — A11Y-03 is satisfied by token coverage, not new theme-specific rules.

### New: `DockNav`

**File:** `src/ui/DockNav.ts` (new)
**CSS:** `src/styles/dock-nav.css` (new — replaces `sidebar-nav.css`)

#### DOM structure

```
nav.dock-nav[role="navigation"][aria-label="Main navigation"]
  └── ul.dock-nav__list
        ├── li.dock-nav__section
        │     ├── span.dock-nav__section-header  (text divider — not interactive)
        │     └── ul.dock-nav__section-items
        │           └── button.dock-nav__item[data-section-key][data-item-key][data-view-key]
        │                 ├── span.dock-nav__item-icon  (Lucide SVG via iconSvg())
        │                 └── span.dock-nav__item-label
        └── ... (repeat per section)
```

#### States

| State | CSS Selector | Visual Treatment |
|-------|-------------|-----------------|
| Default | `.dock-nav__item` | Icon color `var(--text-secondary)`, label color `var(--text-secondary)` |
| Hover | `.dock-nav__item:hover` | Background `var(--cell-hover)`, icon/label color `var(--text-primary)` |
| Active | `.dock-nav__item--active` | Icon color `var(--accent)`, label color `var(--accent)` — NO background fill |
| Focus-visible | `.dock-nav__item:focus-visible` | `outline: 2px solid var(--accent); outline-offset: -2px` |

#### Section header divider

- `span.dock-nav__section-header`: display block, text centered, `var(--text-muted)`, `var(--text-xs)`, weight 600
- Rendered as non-interactive visual divider (not a button)
- Separated from adjacent items by `var(--space-sm)` top padding
- Finder sidebar style: compact, unobtrusive (D-03)

#### Icon sizing

- Icon SVG container: 24×24px within the 48×48px item touch target
- Icon centered horizontally and vertically in item
- `iconSvg()` called with size 20 (20px SVG viewBox within 24px container)

#### Lifecycle API

```typescript

  onActivateSection: (sectionKey: string) => void;
}

class DockNav {
  mount(container: HTMLElement): void;
  destroy(): void;
  setActiveItem(sectionKey: string, itemKey: string): void;
}
```

Mirrors `SidebarNav` API surface exactly to minimize WorkbenchShell and main.ts diff (Claude's discretion — lean toward reuse).

## Interaction Contracts

### Click to activate

- User clicks any `.dock-nav__item` button
- `DockNav` fires `onActivateItem(sectionKey, itemKey)` callback
- `DockNav.setActiveItem()` updates `.dock-nav__item--active` class via event delegation (D3 v6.0 performance pattern — single click handler on the `ul`, not per-button)
- `ViewManager.switchTo()` is invoked by main.ts on receiving the callback (unchanged from SidebarNav wiring)

### Keyboard: Cmd+1-9

- Shortcut bindings in `CommandRegistry` reference `viewOrder` from `section-defs.ts` — unchanged
- After swap, `DockNav.setActiveItem()` is called to sync visual state when keyboard activates a view
- DOCK-06 regression tests in `tests/shortcuts/` are the gate — must pass without modification

### Event delegation pattern

Single `click` listener on `.dock-nav__list` (or `.dock-nav` root). Read `data-section-key` and `data-item-key` attributes off `event.target.closest('.dock-nav__item')`. Matches the event delegation pattern established in v6.0 performance phase.

## CSS File Contract

**New file:** `src/styles/dock-nav.css`

Selector namespace: all selectors must begin with `.dock-nav` — no bare element selectors, no global resets (matches SHEL-06 scoping rule from workbench.css).

Key rules:

```css
/* Container */
.dock-nav { display: flex; flex-direction: column; height: 100%; padding: 0; margin: 0; }

/* Items list */
.dock-nav__list { list-style: none; padding: 0; margin: 0; }

/* Each dock item button */
.dock-nav__item {
  display: flex;
  flex-direction: column;
  align-items: center;
  justify-content: center;
  width: 48px;
  height: 48px;
  padding: 0;
  background: transparent;
  border: none;
  color: var(--text-secondary);
  cursor: pointer;
  transition: background-color var(--transition-fast) ease, color var(--transition-fast) ease;
}

/* Section header divider */
.dock-nav__section-header {
  display: block;
  text-align: center;
  font-size: var(--text-xs);
  font-weight: 600;
  color: var(--text-muted);
  padding: var(--space-sm) 0 var(--space-xs);
  overflow: hidden;
  /* At 48px width, show first 3-4 chars. Text-overflow is intentional. */
  white-space: nowrap;
  text-overflow: clip;
}
```

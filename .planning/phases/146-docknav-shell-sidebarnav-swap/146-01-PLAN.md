   ```typescript
   export interface DockNavConfig {
     onActivateItem: (sectionKey: string, itemKey: string) => void;
     onActivateSection: (sectionKey: string) => void;
     announcer?: { announce: (message: string) => void };
   }
   ```

2. Export a DockNav class with:
   - Constructor takes DockNavConfig
   - mount(container: HTMLElement): void — builds DOM from DOCK_DEFS and appends to container
   - destroy(): void — removes DOM, clears event listener, nulls references
   - setActiveItem(sectionKey: string, itemKey: string): void — updates active state visually
   - updateRecommendations(sourceType: string | null): void — no-op stub (dock has no recommendation badges, but main.ts calls this)

3. DOM structure (per UI-SPEC):
   ```
   nav.dock-nav[role="navigation"][aria-label="Main navigation"]
     ul.dock-nav__list
       li.dock-nav__section  (one per DockSectionDef)
         span.dock-nav__section-header  (section label text)
         ul.dock-nav__section-items
           li (one per item)
             button.dock-nav__item[data-section-key][data-item-key]
               span.dock-nav__item-icon (iconSvg(icon, 20))
               span.dock-nav__item-label (item.label)
   ```

4. Event delegation (per D-04 and v6.0 performance pattern):
   - Single click listener on the nav element (not per-button)
   - On click, find closest `.dock-nav__item` via event.target closest
   - Read data-section-key and data-item-key attributes
   - Call _activateItem(sectionKey, itemKey) which:
     a. Removes `.dock-nav__item--active` and aria-current from previous active
     b. Adds `.dock-nav__item--active` and aria-current="page" to clicked item
     c. Stores active composite key as `${sectionKey}:${itemKey}`
     d. Calls config.onActivateItem(sectionKey, itemKey)

5. setActiveItem(sectionKey, itemKey): same as _activateItem but does NOT call config.onActivateItem (external sync only — prevents loops). Uses a _setActive(sectionKey, itemKey) helper that both paths share for the DOM update.

6. Import `'../styles/dock-nav.css'` at top (side-effect import for Vite CSS bundling).
   Import DOCK_DEFS from './section-defs'.
   Import iconSvg from './icons'.

7. Store item button references in a Map<string, HTMLButtonElement> keyed by composite key for fast setActiveItem lookup.

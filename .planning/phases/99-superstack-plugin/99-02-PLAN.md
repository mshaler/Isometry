    The `state` parameter is the shared `SuperStackState` object (containing `collapsedSet: Set<string>`) created in HarnessShell and passed to both spanning and collapse plugins. The `onCollapseToggle` callback triggers a full PivotTable re-render when collapse state changes.

       - Query all `.pv-col-span:not(.pv-col-span--leaf)` and `.pv-row-span:not(.pv-row-span--leaf)` in `root`
       - For each non-leaf header element:
         - Read `data-level` attribute (already set by PivotGrid or SuperStackSpans)
         - Read the header's value text content
         - Compute collapse key from level + parentPath + value
         - Add class `pv-col-span--collapsible` (or `pv-row-span--collapsible`)
         - Set `data-collapse-key` attribute on the element for pointer event lookup
         - Prepend a `<span class="pv-span-chevron">` with `▼` (expanded) or `▶` (collapsed)
         - If collapsed: add class `pv-col-span--collapsed` (or `pv-row-span--collapsed`), append `<span class="pv-span-count">(n)</span>` where n is the count of hidden children
         - The hidden child count can be computed from the original span vs collapsed span (original colSpan stored as data-original-span by spanning plugin, or computed from the collapsedSet)
       - Store `data-parent-path` on each non-leaf header for key reconstruction

    2. `onPointerEvent(type: string, e: PointerEvent, ctx: RenderContext): boolean`:
       - Only handle `type === 'pointerdown'`
       - Find the clicked element — check if it has class `pv-col-span--collapsible` or `pv-row-span--collapsible` (use `e.target` and walk up to find the header div)
       - If not a collapsible header, return `false` (don't consume)
       - Extract `data-collapse-key` from the element
       - Toggle: if key is in `state.collapsedSet`, delete it; otherwise, add it
       - Call `onCollapseToggle()` to trigger re-render
       - Return `true` (event consumed)

    **A. Create SuperStackAggregate.ts** (`src/views/pivot/plugins/SuperStackAggregate.ts`):

    Factory signature:
    ```typescript
    export function createSuperStackAggregatePlugin(state: SuperStackState): PluginHook;
    ```

    Plugin hooks:

    1. `transformData(cells: CellPlacement[], ctx: RenderContext): CellPlacement[]`:
       - If `state.collapsedSet.size === 0`, return cells unchanged (fast path)
       - For each collapsed group (from collapsedSet):
         - Identify which column indices are hidden children of that collapsed group
         - For each row, sum the values of all hidden child cells in that column group
         - Replace the hidden cells with a single cell at the collapsed parent column index
         - Mark the replacement cell with `meta: { aggregate: true, originalCount: n }`
       - Similarly for collapsed row groups: for each column, sum the values of all hidden child rows
       - Return the modified cells array
       - Note: The spanning plugin already handles hiding columns/rows via buildHeaderCells. The aggregate plugin needs to match: when spanning produces a collapsed column at index X, aggregate should produce SUM values at that index.
       - Implementation detail: The aggregate transformData runs AFTER spanning's afterRender has determined which columns are visible. The aggregate needs to know the mapping from "collapsed column index" to "original child column indices". This mapping comes from the buildHeaderCells output.
       - Simpler approach: The aggregate plugin's `afterRender` hook (not transformData) finds cells in the collapsed column and overwrites their text with the SUM. Steps:
         a. Read collapsedSet from shared state
         b. For each collapsed group, find the data cells in the collapsed column
         c. For each such cell's row, compute SUM from the original data map (ctx.data) using the hidden child column paths
         d. Set cell text to the SUM value
         e. Add class `pv-agg-cell` to the cell element

    2. `afterRender(root: HTMLElement, ctx: RenderContext)`:
       - For each collapsed column group in state.collapsedSet:
         - Determine the original child column paths that are now hidden
         - Find the data cell DOM elements in the collapsed column (by data attribute or position)
         - For each row:
           - Look up all child column values from `ctx.data` map: `getCellKey(rowPath, childColPath)` for each hidden child
           - Compute SUM (null values = 0)
           - Update cell text content to the SUM
           - Add class `pv-agg-cell` to the cell element
       - Same for collapsed row groups (sum across hidden child rows for each column)
       - The DOM cells to modify are the `.pv-data-cell` elements in the table (Layer 1), not the overlay

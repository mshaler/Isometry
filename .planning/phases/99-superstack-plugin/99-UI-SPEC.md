## Spacing Scale

Declared values — project uses named tokens from design-tokens.css, mapped to 4pt multiples:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline chevron padding |
| sm | 8px | Cell internal padding, chip gaps |
| md | 12px | Config panel internal gaps |
| lg | 16px | Panel padding, header padding |
| xl | 24px | Config panel horizontal padding |

Source: `--space-xs` through `--space-xl` in `/src/styles/design-tokens.css`. The project spacing scale runs xs=4, sm=8, md=12, lg=16, xl=24 (not a strict 8pt scale — md is 12px not 16px).

Exceptions: Collapsed header cell minimum touch target is 44px height (to ensure pointer events register reliably on narrow collapsed column stubs).

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / data cell | 13px (var(--text-base)) | 400 | 1.4 |
| Label / leaf header | 12px (var(--text-sm) = 11px, mapped to 0.75rem = 12px in pivot.css) | 500 | 1.3 |
| Parent header / group label | 13px (0.8125rem, consistent with pv-col-span) | 600 | 1.3 |
| Muted / zone label | 11px (var(--text-sm)) | 600 | 1.0 |

Source: Existing `--text-*` tokens plus pivot.css font-size declarations (0.8125rem = 13px, 0.75rem = 12px, 0.6875rem = 11px). All new SuperStack CSS must use the same rem-based sizes as the pivot module — do not introduce new px sizes.

**Collapse indicator text:** Inline Unicode within the header label span. Format: `▶ {value} ({n})` collapsed, `▼ {value}` expanded. Same font-size as the parent header (600 weight, 13px).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--pv-bg)` (#f8fafc light / #0f172a dark) | Grid background, data cell base |
| Secondary (30%) | `var(--pv-header-bg)` (#f1f5f9 light / #1e293b dark) | All header cells — row and column |
| Accent (10%) | `var(--pv-accent)` (#3b82f6 light / #60a5fa dark) | Specific elements only (see below) |
| Destructive | `var(--danger)` | Not applicable in this phase — no destructive actions |

Accent reserved for:
- Collapse indicator chevron glyph color when hovering a collapsible header (`color: var(--pv-accent)`)
- The `pv-agg-cell` tint border-left stripe (3px left-border accent mark on aggregated cells)
- Insertion line and resize handle hover (already established in Phase 97, unchanged)

**Aggregate tint (`pv-agg-cell`):** Background is `var(--pv-accent-light)` — `#dbeafe` in light mode, `#1e3a5f` in dark mode. This is the existing `--pv-accent-light` token introduced in Phase 97. Left border stripe: `3px solid var(--pv-accent)`. Do not introduce a new color token — use what already exists.

**Parent-level collapsed header background:** `var(--pv-header-parent-bg)` (#e2e8f0 light / #334155 dark) — same as non-leaf headers, unchanged. The collapsed state is indicated by chevron direction and the `(n)` count suffix, not by background color change.

**Collapsed header hover state:** `var(--pv-cell-hover-bg)` (#eff6ff light / #1e3a5f dark) — consistent with existing hover pattern, confirming the cell is interactive.

Source: All color tokens pre-exist in `/src/styles/pivot.css`. No new tokens needed for this phase.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Not applicable — this phase has no user-facing buttons or primary actions |
| Collapsed header indicator | `▶ {value} ({n})` where n = count of hidden child columns/rows |
| Expanded header indicator | `▼ {value}` (no count suffix when expanded) |
| Cardinality guard bucket | `Other ({n})` — when leaf columns exceed MAX_LEAF_COLUMNS=50, excess values collapse into this label |
| Empty state (no data) | Existing `.pv-empty` class: "No data to display" (inherited, no change) |
| Error state | Not applicable — plugin errors are silent fallbacks to base PivotSpans behavior; no user-visible error copy needed in this phase |
| Aggregate cell tooltip | Not required in this phase — tint background communicates computed state visually |

**Collapse toggle:** No separate button label. The entire non-leaf header cell is the click target. The chevron + count suffix is the only affordance — no "click to collapse" tooltip needed because the pointer changes to `cursor: pointer` on collapsible cells.

Source: Collapse indicator format (`▶ 2024 (12)` / `▼ 2024`) locked in CONTEXT.md decisions section.

## Interaction Contract

These interactions are mandated by CONTEXT.md decisions and must be implemented exactly as specified:

### Collapse toggle (superstack.collapse)

- **Trigger:** Single pointer click (`pointerdown` or `click` via `onPointerEvent` hook) anywhere on a non-leaf header cell
- **Visual state: collapsed:** Chevron `▶`, value label, `(n)` count in muted text (`var(--pv-muted-fg)`), `cursor: pointer`
- **Visual state: expanded:** Chevron `▼`, value label only, `cursor: pointer`
- **Scope:** Both row headers and column headers are collapsible (any non-leaf span)
- **Re-render trigger:** Click toggles the collapsedSet in the plugin instance; calls PivotGrid re-render with same data (full clean-slate re-render per CONTEXT.md)
- **Persistence:** Session-only — survives re-renders, resets on plugin disable/re-enable or page refresh. collapsedSet lives on the plugin instance, not localStorage
- **Key format:** `"level\x1fparentPath\x1fvalue"` — load-bearing from Phase 7 SuperStackHeader, must be preserved exactly
- **Leaf cells:** Not collapsible — no chevron, no pointer cursor change, no click handler

### Aggregate display (superstack.aggregate)

- **Trigger:** Automatic when a group is collapsed — no user interaction beyond the collapse toggle
- **Collapsed column:** Each data cell in the collapsed column column shows the SUM of all hidden child column values for that row
- **Collapsed row:** Each data cell in the collapsed row shows the SUM of all hidden child row values for that column
- **Visual indicator:** `.pv-agg-cell` CSS class — `background: var(--pv-accent-light)`, `border-left: 3px solid var(--pv-accent)`
- **Aggregate function:** SUM only for this phase — global, not per-group. No UI for changing function (deferred per CONTEXT.md)

### Cardinality guard (superstack.spanning)

- **Threshold:** MAX_LEAF_COLUMNS = 50 (carried from SuperStackHeader Phase 7)
- **Behavior:** Excess leaf values beyond 50 collapse into a single "Other (n)" bucket cell
- **Visual:** "Other (n)" rendered as a regular collapsed header cell with the same styling as any collapsed group, but no clickable toggle (it is a terminal bucket, not a collapsible group)

## New CSS Classes Required

All new classes use the `pv-` prefix per D-016. Add to `/src/styles/pivot.css`.

| Class | Purpose |
|-------|---------|
| `.pv-col-span--collapsible` | Applied to any non-leaf column header. Sets `cursor: pointer`. |
| `.pv-col-span--collapsed` | Applied to a collapsed column header cell. Chevron points right (▶). |
| `.pv-row-span--collapsible` | Applied to any non-leaf row header. Sets `cursor: pointer`. |
| `.pv-row-span--collapsed` | Applied to a collapsed row header cell. Chevron points right (▶). |
| `.pv-agg-cell` | Applied to data cells showing computed aggregate values. Accent-light background + 3px accent left border. |
| `.pv-span-chevron` | `<span>` inside the header label for the ▶/▼ glyph. Color: `var(--pv-muted-fg)` default, `var(--pv-accent)` on parent `:hover`. |
| `.pv-span-count` | `<span>` inside collapsed header for the `(n)` count suffix. Font-size 11px, color `var(--pv-muted-fg)`. |

No new CSS tokens needed. All values reference existing `--pv-*` tokens from Phase 97.

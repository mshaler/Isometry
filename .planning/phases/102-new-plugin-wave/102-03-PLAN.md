    **1. Create SelectionState type and constructor (`SuperSelectClick.ts`):**

    ```typescript
    export interface SelectionState {
      selectedKeys: Set<string>;
      anchor: { rowIdx: number; colIdx: number } | null;
      listeners: Set<() => void>;
    }
    export function createSelectionState(): SelectionState {
      return { selectedKeys: new Set(), anchor: null, listeners: new Set() };
    }
    ```

    **2. Create `createSuperSelectClickPlugin(selectionState: SelectionState, onSelectionChange: () => void)` factory:**
    - `onPointerEvent(type, e, ctx)`:
      - Only handle `'pointerdown'`
      - Walk up from `e.target` to find `.pv-data-cell` ancestor (bounded by root)
      - If no data cell found: clear all selection, call `onSelectionChange()`, return false
      - Read cell key from `data-key` attribute
      - If `e.metaKey` (Cmd+click): toggle key in `selectedKeys` (add if missing, remove if present)
      - If NOT metaKey: clear `selectedKeys`, add this key
      - Set `anchor` to `{ rowIdx, colIdx }` from cell's `data-row` and `data-col` attributes
      - Call `onSelectionChange()` to trigger re-render
      - Return true (consumed)
    - `afterRender(root, ctx)`: iterate all `.pv-data-cell` elements in root. For each:
      - If cell's `data-key` is in `selectedKeys`: add `.selected` class and set `data-selected="true"`
      - Otherwise: remove `.selected` class and remove `data-selected` attribute
    - `destroy`: clear `selectedKeys`, set `anchor` to null

    **3. Create `createSuperSelectLassoPlugin(selectionState: SelectionState, onSelectionChange: () => void)` factory:**
    - Internal state: `_dragging: boolean`, `_startX/Y: number`, `_overlay: HTMLDivElement | null`
    - `onPointerEvent(type, e, ctx)`:
      - `'pointerdown'`: if target is a `.pv-data-cell` or inside grid area, start lasso drag. Set `_dragging = true`, record `_startX = e.clientX`, `_startY = e.clientY`. Do NOT consume yet (let click plugin handle single clicks). Return false.
      - `'pointermove'`: if `_dragging` and distance from start > 5px threshold, create `.pv-lasso-overlay` div with `position: absolute`, sized from start to current pointer position. Append to grid container. Return true (consuming moves during drag).
      - `'pointerup'`: if `_dragging` and overlay exists, compute bounding rect of overlay. Find all `.pv-data-cell` elements whose bounding rects intersect the lasso rect. If NOT `e.metaKey`, clear existing selection. Add intersecting cell keys to `selectedKeys`. Remove overlay from DOM. Call `onSelectionChange()`. Return true.
    - Grid container needs `position: relative` (set in afterRender if not already)
    - `afterRender`: ensure grid container has `position: relative`
    - `destroy`: remove overlay if exists, reset drag state

    **4. Create `createSuperSelectKeyboardPlugin(selectionState: SelectionState, onSelectionChange: () => void)` factory:**
    - `onPointerEvent(type, e, ctx)`:
      - Only handle `'keydown'` type (NOTE: PluginRegistry dispatches keydown events through onPointerEvent with type 'keydown')
      - Check: event must have `shiftKey = true` and be an arrow key (ArrowUp/Down/Left/Right)
      - If no `anchor` in selectionState, return false
      - Compute new cell position: anchor + direction delta (ArrowRight: colIdx+1, ArrowLeft: colIdx-1, ArrowDown: rowIdx+1, ArrowUp: rowIdx-1)
      - Build key for new cell position using pattern `${rowIdx}:${colIdx}`
      - Add new key to `selectedKeys`
      - Update anchor to new position
      - Call `onSelectionChange()`
      - `e.preventDefault()` to prevent scroll
      - Return true
    - `destroy`: no-op (state managed by click plugin)

    **1. Create AuditPluginState type and constructor (`SuperAuditOverlay.ts`):**

    ```typescript
    export interface AuditPluginState {
      inserted: Set<string>;   // cell keys of newly inserted cards
      updated: Set<string>;    // cell keys of modified cards
      deleted: Set<string>;    // cell keys of deleted cards
      sources: Map<string, string>; // cell key -> source type (e.g., "csv", "apple_notes")
    }
    export function createAuditPluginState(): AuditPluginState {
      return {
        inserted: new Set(),
        updated: new Set(),
        deleted: new Set(),
        sources: new Map(),
      };
    }
    ```

    **2. Create `createSuperAuditOverlayPlugin(auditState: AuditPluginState)` factory:**
    - `afterRender(root, ctx)`: iterate all `.pv-data-cell` elements in root. For each cell:
      - Read cell key from `data-key` attribute
      - If key is in `auditState.inserted`: add `.audit-new` class
      - If key is in `auditState.updated`: add `.audit-modified` class
      - If key is in `auditState.deleted`: add `.audit-deleted` class
      - If key is NOT in any set: remove all 3 audit classes
    - `destroy`: remove `.audit-new`, `.audit-modified`, `.audit-deleted` from all `.pv-data-cell` elements

    **3. Create `createSuperAuditSourcePlugin(auditState: AuditPluginState)` factory:**
    - `afterRender(root, ctx)`: iterate all `.pv-data-cell` elements in root. For each cell:
      - Read cell key from `data-key` attribute
      - If key is in `auditState.sources`: set `data-source` attribute to the source type string, add `.audit-source` class
      - If key is NOT in sources: remove `data-source` attribute and `.audit-source` class
    - CSS handles the coloring via `data-source` attribute selectors (see Task 2 CSS)
    - `destroy`: remove `.audit-source` class and `data-source` attribute from all cells

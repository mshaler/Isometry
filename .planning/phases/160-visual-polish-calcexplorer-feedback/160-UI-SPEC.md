## Spacing Scale

The project spacing scale uses named tokens, not numeric steps. Active tokens for this phase:

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| `--space-xs` | 4px | Inline glyph gap, dismiss-bar icon padding |
| `--space-sm` | 8px | `.calc-row` horizontal padding (replaces hardcoded `8px`) |
| `--space-md` | 12px | `.calc-row` vertical rhythm; slot-level padding candidate |
| `--space-lg` | 16px | Explorer panel header internal padding |
| `--space-xl` | 24px | Not used in this phase |

Slot-level padding decision (CONTEXT.md D-03, Claude's Discretion): use `--space-md` (12px) on `.workbench-slot-top` and `.workbench-slot-bottom` as the slot container padding. This is slightly more generous than `--space-sm` (8px) and keeps explorers from feeling cramped against slot edges without wasting vertical space.

`.calc-row` hardcoded `padding: 4px 8px` migrates to `padding: var(--space-xs) var(--space-sm)` in this phase. Phase 155 D-04 mandates zero hardcoded values; this is in scope.

Exceptions: none.

## Typography

Three-tier scale declared in CONTEXT.md D-04 through D-07. Applied consistently across all 8 explorers.

| Role | Token | Size | Weight | Line Height | Usage |
|------|-------|------|--------|-------------|-------|
| Explorer panel header | `--text-base` | 13px | 600 (semibold) | 1.4 | Section titles in all 8 explorer panels |
| Explorer label | `--text-sm` | 11px | 400 (normal) | 1.4 | Column names, field labels, chip text |
| Explorer meta / hint | `--text-xs` | 10px | 400 (normal) | 1.3 | Secondary info, counts, type glyphs |

Source: CONTEXT.md D-04, D-05, D-06, D-07.

Note: `--text-sm` in design-tokens.css is 11px (not 13px). `--text-base` is 13px. The existing `.calc-row label` uses `--text-sm, 13px` — the fallback value is incorrect and must be removed per Phase 155 D-03. After this phase, label font-size is `var(--text-sm)` (11px) with no fallback.

## Color

All colors are CSS custom properties that adapt across the 5 themes (dark, light, system, nextstep, material). No hex values hardcoded in phase 160 CSS.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `--bg-primary` | Explorer slot backgrounds, calc-row backgrounds |
| Secondary (30%) | `--bg-surface` | Explorer panel header backgrounds, hover states |
| Accent (10%) | `--accent` | Reserved — see below |
| Destructive | `--danger` | Not used in this phase |

**Accent reserved for:**
- Active aggregation label text: `color: var(--text-primary)` with `font-weight: 600` (CONTEXT.md D-08). Note: active state uses `--text-primary`, NOT `--accent` — this aligns with the existing `.collapsible-section__header--active` pattern which uses `border-left-color: var(--accent)` only for structural emphasis, not text. CalcExplorer active labels follow the same distinction.

**Inactive state:** `color: var(--text-secondary)`, `font-weight: 400` (CONTEXT.md D-08).

**Type glyph color:** `color: var(--text-muted)` — visible but secondary (CONTEXT.md D-11). Glyph color does not change with active/inactive state.

**Border treatment (VCSS-05):** Zone boundaries use `1px solid var(--border-subtle)`. The existing `.workbench-slot-top` already has `border-bottom: 1px solid var(--border-subtle)` and `.workbench-slot-bottom` has `border-top: 1px solid var(--border-subtle)`. No additional background color shifts (CONTEXT.md D-01).

## Interaction States

### CalcExplorer row — inactive (aggregation = 'off')
- Label: `font-size: var(--text-sm)`, `font-weight: 400`, `color: var(--text-secondary)`
- Type glyph: `color: var(--text-muted)`
- Select: visually neutral, no state change

### CalcExplorer row — active (aggregation != 'off')
- Label: `font-size: var(--text-sm)`, `font-weight: 600`, `color: var(--text-primary)`
- Type glyph: `color: var(--text-muted)` (unchanged)
- Select: visually neutral (CONTEXT.md D-09 — select stays neutral)
- CSS modifier class: `.calc-row--active` applied by `_render()` when aggregation != 'off'

### Explorer panel header (all 8 explorers)
- `font-size: var(--text-base)` (13px), `font-weight: 600`, `color: var(--text-primary)`
- Consistent across AlgorithmExplorer, VisualExplorer, ProjectionExplorer, LatchExplorers, PropertiesExplorer, DataExplorer, NotebookExplorer, catalog-actions

### Explorer label (all 8 explorers)
- `font-size: var(--text-sm)` (11px), `font-weight: 400`, `color: var(--text-secondary)`

### Explorer meta/hint (all 8 explorers)
- `font-size: var(--text-xs)` (10px), `font-weight: 400`, `color: var(--text-muted)`

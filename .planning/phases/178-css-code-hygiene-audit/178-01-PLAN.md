Phase 1 — Identify all unique hardcoded px values across the 8 target CSS files. Group them into semantic categories. Exclude design-tokens.css definitions themselves (those ARE the tokens) and values that are genuinely one-off (e.g., `0px`, `1px` borders — use judgment per D-01 but `0` and `1px` borders are fine as literals).

Phase 2 — Add new tokens to design-tokens.css in the theme-independent `:root` section. Follow existing naming:
- Layout sizes: `--size-*` (e.g., `--size-sidebar: 240px`, `--size-toolbar: 36px`, `--size-tab-height: 32px`)
- Widths/heights that repeat: `--size-*` with semantic name
- Icon sizes: `--icon-sm: 14px`, `--icon-md: 16px`, `--icon-lg: 20px` (if repeated)
- Line heights or gaps: extend `--space-*` scale if values fit, otherwise `--size-*`
- Do NOT duplicate tokens that already exist (check existing --space-xs through --space-xl, --radius-*, --text-*)
- Do NOT add theme-variant tokens — layout sizes are theme-independent

Phase 3 — In each of the 8 CSS files, replace hardcoded px values with `var(--token-name)`:
- Match repeated values to existing or newly created tokens
- For truly one-off values that have a clear semantic meaning, create a token (per D-01 aggressive policy)
- For `0px`, just use `0` (CSS spec: zero needs no unit)
- For `1px` in border shorthand, keep as literal (universally 1px, no semantic name adds value)
- For `border-radius`, replace with `--radius-*` tokens
- For `font-size`, replace with `--text-*` tokens
- For `padding`/`margin`/`gap`, replace with `--space-*` tokens

Phase 4 — Audit overflow:hidden in these 8 files (~33 occurrences):
- For each `overflow: hidden` / `overflow-x: hidden` / `overflow-y: hidden`:
  - Determine if it is a band-aid (hiding content that should be handled by flex/grid layout, text-overflow, or proper sizing)
  - If band-aid: replace with proper CSS fix (e.g., `min-width: 0` + `text-overflow: ellipsis` for flex children, `overflow: auto` for scrollable containers)
  - If intentional: add `/* intentional: [reason] */` comment on the same line (per D-02)

Phase 5 — Also address superwidget.css line 1 TODO: `/* TODO: :focus-visible ring deferred to KBNV-01 */`
- Add a `:focus-visible` outline rule for the superwidget tab bar buttons: `outline: 2px solid var(--accent); outline-offset: -2px;` (standard focus ring pattern used elsewhere in the codebase)
- Remove the TODO comment since it's now implemented

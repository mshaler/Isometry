  <behavior>
    - cycle('name') on empty state -> sorts = [{field:'name', direction:'asc'}]
    - cycle('name') when asc -> sorts = [{field:'name', direction:'desc'}]
    - cycle('name') when desc -> sorts = [] (removed)
    - cycle('folder') when 'name' is sorted -> sorts = [{field:'folder', direction:'asc'}] (replaces, not appends)
    - addOrCycle('name') on empty state -> sorts = [{field:'name', direction:'asc'}]
    - addOrCycle('folder') when 'name' sorted -> sorts = [{field:'name',...}, {field:'folder', direction:'asc'}]
    - addOrCycle('name') when 'name' asc in chain -> sorts have 'name' cycled to desc
    - addOrCycle('name') when 'name' desc in chain -> 'name' removed from chain
    - addOrCycle when at maxSorts (3) and new field -> no change (silently ignored)
    - getPriority('name') returns 1-indexed position; 0 if not sorted
    - getDirection('name') returns 'asc'|'desc'|null
    - clear() empties all sorts
    - hasActiveSorts() returns true when sorts exist, false when empty
    - getSorts() returns defensive copy (mutating returned array does not affect internal state)
    - Constructor accepts optional initial SortEntry[] for session restore
    - Constructor accepts optional maxSorts (default 3)
  </behavior>

       private _calcReorderTargetIndex(e: DragEvent, dimension: 'col' | 'row', headerCells: HTMLElement[]): number

<task type="auto" tdd="true">
  <name>Task 2: FLIP animation for headers and data cells after reorder/transpose</name>
  <files>src/views/SuperGrid.ts, tests/views/SuperGrid.test.ts</files>
  <behavior>
    - Before provider mutation in drop handler, FLIP snapshot captures getBoundingClientRect() for all .col-header, .row-header, and .data-cell elements
    - After _renderCells rebuilds DOM, FLIP animation plays: each element animates from old position to new position over 200ms
    - Elements with no old position (newly appearing cells) are not animated (shown immediately)
    - Elements that disappeared are not animated (already removed from DOM)
    - FLIP snapshot is consumed once (set to null after playing)
    - Hidden cells (hide-mode collapsed) do not participate in FLIP
    - Both same-dimension reorder AND cross-dimension transpose trigger FLIP
  </behavior>
  <action>
    1. **Add FLIP instance variable:**
       - `private _flipSnapshot: Map<string, DOMRect> | null = null;`

    2. **Add _getElementFlipKey helper:**
       ```
       private _getElementFlipKey(el: HTMLElement): string | null
       ```
       - For .data-cell: use dataset['cellKey'] (already set by _renderCells)
       - For .col-header: construct from dataset['level'] + dataset['value'] prefixed with 'ch-'
       - For .row-header: construct from dataset['level'] + dataset['value'] prefixed with 'rh-'
       - Return null if no identifying data available

    3. **Add _captureFlipSnapshot method:**
       ```
       private _captureFlipSnapshot(): void
       ```
       - If no _gridEl, return
       - Create new Map
       - querySelectorAll('.data-cell, .col-header, .row-header') on _gridEl
       - For each element, get FLIP key, store getBoundingClientRect()
       - Assign to this._flipSnapshot

    4. **Add _playFlipAnimation method:**
       ```
       private _playFlipAnimation(): void
       ```
       - If no _flipSnapshot or no _gridEl, return
       - Consume snapshot (read and null out)
       - querySelectorAll('.data-cell, .col-header, .row-header') on _gridEl
       - For each element:
         - Get FLIP key
         - Look up old rect from snapshot
         - If no old rect, skip (new element — no animation)
         - Get new rect via getBoundingClientRect()
         - Compute deltaX = oldRect.left - newRect.left, deltaY = oldRect.top - newRect.top
         - If both deltas are 0, skip (no movement)
         - Call el.animate() with Web Animations API:
           `[{ transform: translate(dx, dy) }, { transform: 'none' }]`
           `{ duration: 200, easing: 'ease-out' }`

    5. **Wire FLIP into the drop handler:**
       - In _wireDropZone drop handler, BEFORE the provider mutation (reorderColAxes call or setColAxes/setRowAxes call), call `this._captureFlipSnapshot()`
       - This must be synchronous, before any async work. The provider mutation triggers StateCoordinator → _fetchAndRender which is async.

    6. **Wire _playFlipAnimation into _renderCells:**
       - At the END of _renderCells (after all DOM is built), call `this._playFlipAnimation()`
       - This ensures the FLIP fires after the new DOM positions are established
       - The snapshot is consumed once, so subsequent _renderCells calls (not from a drop) will find null and skip

    7. **Ensure data-cell key is set:**
       - Verify that _renderCells already sets dataset['cellKey'] on data cells. If not, add it using buildCellKey(). (Check existing code — the cell key is likely already used for D3 data join keying.)

    8. **Write TDD tests:**
       - Test _captureFlipSnapshot: mount grid, capture snapshot, verify snapshot has entries for headers and data cells
       - Test _playFlipAnimation: populate _flipSnapshot with mock positions, call _renderCells, verify animate() was called on elements that moved (mock element.animate)
       - Test FLIP snapshot is consumed (null after _playFlipAnimation)
       - Test that elements with no old position are not animated
       - Integration: fire same-dim drop → verify _captureFlipSnapshot was called before provider mutation

    Note: FLIP with cells that are added/removed during re-query — new cells just appear without animation, disappeared cells are already gone. This is the correct behavior per RESEARCH.md recommendation. Use requestAnimationFrame if needed to ensure layout is computed before reading new positions — but since _renderCells synchronously builds DOM and _playFlipAnimation runs at the end, layout should be computed by the time getBoundingClientRect is called.
  </action>

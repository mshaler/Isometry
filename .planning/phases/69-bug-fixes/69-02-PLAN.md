<objective>
Fix deleted_at null-safety bugs in NetworkView.ts and TreeView.ts where SQL queries reference `deleted_at IS NULL` on the connections table, which has no `deleted_at` column. This causes SQLite "no such column" errors that silently break edge rendering in Network and Tree views.

Purpose: The connections table uses CASCADE deletion (ON DELETE CASCADE from cards), not soft-delete. Querying `deleted_at IS NULL` on connections is a copy-paste bug from cards query patterns. Removing the invalid clause restores correct connection data retrieval.

Output: Fixed SQL queries in NetworkView.ts and TreeView.ts, unit test verifying connection queries work correctly.
</objective>

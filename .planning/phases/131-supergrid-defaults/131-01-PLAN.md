    **Step 1: Add applyDefaults helper to PAFVProvider** (new public method):

    Add a public `applySourceDefaults(sourceType: string, schema: SchemaProvider)` method to PAFVProvider that:
    1. Imports `resolveDefaults` from `./ViewDefaultsRegistry`
    2. Calls `resolveDefaults(sourceType, schema)`
    3. If result.colAxes.length > 0: calls `this.setColAxes(result.colAxes)`
    4. If result.rowAxes.length > 0: calls `this.setRowAxes(result.rowAxes)`
    5. Does nothing if both are empty (no-op — existing generic defaults remain)

    This method is the single entry point for applying source-type defaults. It uses existing setColAxes/setRowAxes which already notify subscribers.

    **Step 2: Wire into main.ts bridge.importFile wrapper** (around line 1424-1428):

    After `const result = await originalImportFile(source, data, options);` and before `auditState.addImportResult(result, source);`, add:
    ```typescript
    // SGDF-01/02/03: Apply source-type defaults on first import
    pafv.applySourceDefaults(source, schemaProvider);
    ```

    Do the same in the `bridge.importNative` wrapper (around line 1446), using `sourceType` parameter:
    ```typescript
    pafv.applySourceDefaults(sourceType, schemaProvider);
    ```

    NOTE: This applies defaults on EVERY import, not just first import. The first-import gate (SGDF-06) is handled in Plan 02. For now, calling applySourceDefaults after each import is correct because it's idempotent — if the user hasn't changed axes, re-applying the same defaults is a no-op. Plan 02 adds the flag gate to prevent overwriting user customizations.

    **Step 3: Remove redundant _getSupergridDefaults from PAFVProvider** (optional cleanup):

    The existing `_getSupergridDefaults()` method (lines 132-171) does schema-aware fallback for the generic VIEW_DEFAULTS.supergrid. With ViewDefaultsRegistry now handling source-type-specific defaults with its own fallback logic, `_getSupergridDefaults()` becomes dead code for the source-type path. However, it's still used by `setViewType()` when switching TO supergrid view without a source context. Leave it in place — do NOT delete it. The two mechanisms are complementary: _getSupergridDefaults handles view-type switching, ViewDefaultsRegistry handles source-type first-import.

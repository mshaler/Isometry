## Spacing Scale

Declared values — sourced from `src/styles/design-tokens.css` (theme-independent `:root` block):

| Token | Value | Usage |
|-------|-------|-------|
| --space-xs | 4px | Icon gaps, inline padding |
| --space-sm | 8px | Compact element spacing, badge internal padding |
| --space-md | 12px | Default element spacing (note: project uses 12px, not 16px) |
| --space-lg | 16px | Section padding |
| --space-xl | 24px | Layout gaps, legend panel inset |

Exceptions:
- Badge circle diameter: 16px (not on the spacing scale — fixed size for readability at small node sizes)
- Edge stroke thickness range: 1px minimum, 6px maximum (D3-controlled, not token-driven)
- Weight picker select: 4px 8px internal padding (matches existing `.nv-pick-dropdowns select` pattern)

## Color

All tokens sourced from `src/styles/design-tokens.css`. Values shown are dark theme defaults (`:root`).

| Role | Token | Dark Value | Usage |
|------|-------|-----------|-------|
| Dominant (60%) | --bg-primary | #1a1a2e | App background, NetworkView canvas background |
| Secondary (30%) | --bg-surface | #252540 | Legend panel, AlgorithmExplorer sidebar, tooltip surface |
| Accent (10%) | --accent | #4a9eff | Hop count badge background, weight picker focus ring |
| Destructive | --danger | #ff4a4a | Not applicable to this phase |

Accent (`--accent`) reserved for:
- Hop count badge background fill (GALG-01)
- Focus ring on weight attribute picker `<select>` (GALG-04)
- Single-source shortest path: NOT used — distance coloring uses `d3.interpolateWarm` scale instead

Distance coloring scale (GALG-02):
- Uses `d3.interpolateWarm` continuous scale: yellow (hop 0 / source) through orange to red (max hop distance)
- Domain: linear mapping from 0 (source node) to maxHop (farthest reachable node)
- Unreachable nodes: retain existing node fill color (no coloring applied)

Edge betweenness thickness (GALG-03):
- Stroke width range: 1px (minimum centrality) to 6px (maximum centrality)
- Color: existing edge stroke color unchanged — only stroke-width changes
- Threshold: edges with zero betweenness render at 1px (floor, not hidden)

## Component Inventory

### New: Hop Count Badge (GALG-01)

Class: `.nv-hop-badge`

Render as an SVG `<g>` child of the target node group in NetworkView's D3 force layout.

| Property | Value |
|----------|-------|
| Shape | Circle, 16px diameter |
| Position | Top-right of target node circle, translated (+node_r * 0.6, -node_r * 0.6) |
| Background fill | var(--accent) — #4a9eff |
| Text | Hop count integer, --text-xs (10px), weight 600, fill #ffffff |
| Visibility | display:none when no shortest path active; shown when path result present |
| Border | 1.5px solid var(--bg-primary) — separation ring from node beneath |

### New: Distance Color Legend Entry (GALG-02)

Extends the existing `.nv-legend` panel. Adds one new section block (`.nv-legend__section`) containing:
- Heading: "Distance from source" at --text-sm, weight 600
- A gradient scale bar: `<div class="nv-legend__scale-bar">` with `background: linear-gradient(to right, {warm_yellow}, {warm_red})`
- Two endpoint labels: "Near" (left) and "Far" (right) at --text-xs, color var(--text-secondary)

Scale bar dimensions: full width of legend panel (max 176px), height 8px, border-radius var(--radius-sm).

### New: Weight Attribute Picker (GALG-04)

Rendered inside the AlgorithmExplorer sidebar panel (`.algorithm-explorer__params`), below the existing Dijkstra algorithm section.

| Property | Value |
|----------|-------|
| Element | `<label>` wrapping a `<select>` |
| Label text | "Edge weight" |
| Placeholder option | "Uniform (weight = 1)" |
| Font size | var(--text-sm) — 13px |
| Width | 100% of params column |
| Styling | Inherits existing `.nv-pick-dropdowns select` pattern |

Shown only when Shortest Path or Weighted Dijkstra algorithm is selected. Hidden for other algorithms.

### Modified: Edge Stroke Rendering (GALG-03)

No new CSS class. D3 `.attr("stroke-width", ...)` binding on existing edge `<line>` elements.

Edge betweenness range scale: `d3.scaleLinear().domain([0, maxBetweenness]).range([1, 6])`.

Legend addition: Extends `.nv-legend` with a stroke preview row showing "Min" (1px line) and "Max" (6px line) labeled swatches using existing `.nv-legend__stroke-preview` and `.nv-legend__stroke-line` classes.

### Modified: NotebookExplorer New Card Flow (BUGF-02)

No new visual components. The fix is behavioral: after `MutationManager.execute(createCardMutation(...))` completes, the explorer must:
1. Call `_selection.select(newCardId)` to auto-select the new card
2. Trigger a Recent Cards list refresh (re-query and re-render the list)
3. Transition the editor from idle/buffering state to active editing state

Visual result: the new card appears at top of Recent Cards list with a selected state indicator, and the title input field is focused and empty, ready for typing.

## Interaction States

### Hop Count Badge

| State | Visual |
|-------|--------|
| No active path | Badge hidden (display:none on SVG group) |
| Path active, target node | Badge visible with hop integer |
| Node is not target | No badge rendered |
| Path recomputing | Badge retains last value until new result arrives |

### Distance Coloring

| State | Visual |
|-------|--------|
| No algorithm active | All nodes retain default fill |
| Single-source SSSP active | Reachable nodes colored by d3.interpolateWarm(hop/maxHop) |
| Source node | Colored at domain position 0 (warmest yellow) |
| Unreachable node | Retains default fill — no coloring applied |
| Algorithm reset | All nodes revert to default fill in one transition (--transition-normal: 300ms) |

### Edge Betweenness Thickness

| State | Visual |
|-------|--------|
| No betweenness computed | All edges at default 1px stroke-width |
| Betweenness computed | Edges scaled 1px–6px by centrality score |
| Max betweenness edge | 6px stroke-width |
| Zero betweenness edge | 1px stroke-width (floor) |

### Weight Picker

| State | Visual |
|-------|--------|
| Dijkstra not selected | Picker hidden |
| Dijkstra selected, no numeric columns | Picker shows only "Uniform (weight = 1)" |
| Dijkstra selected, numeric columns present | Picker shows column names + "Uniform" placeholder |
| Column selected | Run button enabled; selection persists until algorithm changes |

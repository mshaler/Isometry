**BUGF-01 — SubscriptionManager.tierForProductID():**

The current implementation at line 219-223 already returns `.free` for unknown product IDs. However, there is a subtle bug: a product ID like `"com.example.production.v2"` contains the substring `"pro"` (inside "production") and would incorrectly return `.pro`. Fix by matching on the dotted segment pattern instead of bare substring:

Change `tierForProductID` (line 219-223) to:
```swift
nonisolated static func tierForProductID(_ productID: String) -> Tier {
    let components = productID.lowercased().split(separator: ".")
    if components.contains("workbench") { return .workbench }
    if components.contains("pro") { return .pro }
    return .free
}
```

Add tests to `SubscriptionManagerTests.swift`:
- `productIDContainingProSubstringReturnsFree`: `#expect(SubscriptionManager.tierForProductID("com.example.production.v2") == .free)` — verifies the substring bug is fixed
- `caseInsensitiveProReturnsPro`: `#expect(SubscriptionManager.tierForProductID("works.isometry.PRO.monthly") == .pro)` — verifies case-insensitive matching

**BUGF-02 — Two bugs in NotebookExplorer card creation:**

**Bug 1: Recent Cards list doesn't refresh after card creation.**

Root cause: `_evaluateBufferingCommit()` creates a card via `this._mutations.execute(mutation)` then calls `this._selection.select(newCardId)`. The MutationManager subscription in `src/main.ts` calls `coordinator.scheduleUpdate()` which re-renders views, but `refreshDataExplorer()` is NOT called from the MutationManager subscriber. Recent Cards only refreshes on dataset switch, vacuum, or file drop.

Fix in `src/main.ts`: Find the MutationManager subscription block (search for `mutations.subscribe`). In that callback, add a call to `void refreshDataExplorer()` so that any mutation (including card creation) triggers a Recent Cards list update.

The MutationManager subscription is approximately around the area where `coordinator.scheduleUpdate()` is called from the mutation subscriber. Find it by searching for the pattern where `mutations.subscribe(() => { ... coordinator.scheduleUpdate() ... })`. Add `void refreshDataExplorer();` inside that callback.

If the subscription looks like:
```typescript
mutations.subscribe(() => {
    coordinator.scheduleUpdate();
});
```

Change to:
```typescript
mutations.subscribe(() => {
    coordinator.scheduleUpdate();
    void refreshDataExplorer();
});
```

**Bug 2: Selection doesn't trigger editor load — stays in idle/buffering state.**

Root cause: In `_evaluateBufferingCommit()` at line 551, `this._selection.select(newCardId)` fires the `_onSelectionChange` subscriber. However, the issue is that `_creationState` is still `'buffering'` when `_onSelectionChange` fires, and `_showEditor()` (called at line 688) needs to properly transition from buffering to editing.

Inspect `_showEditor()` to verify it sets `this._creationState = 'editing'` and shows the right elements. If it doesn't, add:

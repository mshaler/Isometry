## States and Interactions

### TimelineView (TMLN-01, TMLN-02, TMLN-03)

**Normal render state:**
- Horizontal SVG time axis at top using `d3.scaleUtc()`
- Fixed 120px label column on left showing swimlane group values
- Cards rendered as SVG `<g class="card">` at temporal x-position derived from `due_at`
- Sub-row stacking via `computeSubRows()` for overlap — each sub-row adds `CARD_HEIGHT + SWIMLANE_PADDING` to swimlane height

**Empty state — no `due_at` data (TMLN-02):**
This is a CONTEXTUAL empty state, not a zero-row DB result. Triggered when all cards in the result set have `due_at = null`. TimelineView must detect this internally and render the empty panel BEFORE calling the D3 SVG render path.

- Container: `.view-empty` centering wrapper
- Panel: `.view-empty-panel` (existing class)
- Icon: Unicode clock glyph `🕐` at 32px via `.view-empty-icon`
- Heading (`.view-empty-heading`): "No scheduled cards"
- Description (`.view-empty-description`): "Add a due date to any card to see it on the timeline."
- No CTA button (adding due dates is done in card detail — not a filter action)

**Loading state:** `.view-loading` + `.spinner` + `.spinner-label "Loading..."` — shared with all views, managed by ViewManager 200ms timeout. No change needed.

**Error state:** `.view-error-banner` with `.error-category` heading + `.error-detail` body + `.retry-btn` (shared pattern from `views.css`). Timeline-specific error: "Timeline failed to load — database query error. Try reloading."

### NetworkView (NETW-01, NETW-02, NETW-03, NETW-04)

**Normal render state:**
- Force-directed SVG graph: nodes as `<circle>` sized by degree (8px–28px radius), edges as `<line>`
- Node labels as SVG `<text>` at 10px below each node
- Zoom/pan via `d3.zoom()` on SVG container
- Drag-to-pin: `d3.drag()` sets `fx/fy`; pinned nodes show distinct cursor (`cursor: grab` → `cursor: grabbing`)
- Hover: hovered node + its direct edges stay at full opacity; all other nodes/edges dim to `DIM_*_OPACITY`

**Empty state — no nodes (NETW-01 regression):**
If the primary card query returns zero rows, NetworkView must show empty panel before graph render.
- Icon: Unicode node glyph `◉` at 32px via `.view-empty-icon`
- Heading: "No cards to display"
- Description: "Import data or adjust your filters to see connections in the network."
- CTA: `.clear-filters-btn` "Clear Filters" — only shown when filters are active (state check required)

**Secondary query empty state — no connections:**
If cards exist but zero connection edges are returned from the `db:exec` connections query:
- Render nodes only (no edges) — this is a valid state, NOT an empty state
- No empty panel; graph renders with nodes and no edges
- AlgorithmExplorer Run button remains enabled (community detection works on nodes alone)

**Algorithm overlay states (NETW-02):**
- Community colors: D3 schemeTableau10 applied to node fills, legend panel shown
- Centrality sizing: node radius scaled to [MIN_RADIUS, MAX_RADIUS] by centrality score, gradient scale bar in legend
- Path/MST highlighting: path edges use `var(--accent)` stroke at full opacity; non-path edges dim
- Stale state: `.algorithm-explorer__stale-dot` shown when graph data changes after last Run

**AlgorithmExplorer controls (NETW-03):**
- Radio group for algorithm selection: existing `.algorithm-explorer__radios` pattern
- Run button: `.algorithm-explorer__run` — enabled when valid algorithm selected, disabled during compute
- Parameter sliders: `.algorithm-explorer__params input[type=range]` with value readout
- Status text: `.algorithm-explorer__status` at `--text-xs` — shows "Running…" / "Done" / stale warning

**Source/target picker (NETW-04):**
- `.nv-pick-dropdowns` with two `<select>` elements for source and target card selection
- `.nv-pick-instruction` text at `--text-sm` in `var(--text-secondary)`
- Legend panel `.nv-legend` — `.nv-legend--visible` class toggled by JS when algorithm active

## SVG Render Contracts

### TimelineView SVG structure

```
svg.timeline-svg
  g.timeline-axis         // d3.axisTop(scaleUtc) — top of SVG
  g.timeline-lanes        // one <g.swimlane> per group value
    g.swimlane
      rect.swimlane-bg    // var(--bg-secondary) fill, spans full width
      text.swimlane-label // 120px fixed column, var(--text-secondary)
      g.card[data-id]     // D3 key: d.id — renderSvgCard() per card
  line.timeline-today     // accent-colored today marker, dashed
```

Key invariant: `g.card` CSS class must be preserved — it matches ListView/GridView for morphTransition compatibility (established in Phase 6 comments).

D3 key function `d => d.id` is MANDATORY on every `.data()` call (D3 key function rule, locked decision).

### NetworkView SVG structure

```
svg.network-svg
  g.network-zoom-container  // d3.zoom() applied here
    g.edges                 // <line.edge[data-id]> — D3 key: d.id
    g.nodes                 // <g.node[data-id]>  — D3 key: d.id
      circle
      text                  // node label, 10px, below circle
  div.nv-legend             // absolute positioned, outside SVG in container
  div.nv-tooltip            // absolute positioned, pointer-events: none
```

Secondary query sequence (load order):
1. Primary `db:exec` → card rows → initial node render
2. Secondary `db:exec` → connection rows → edge render
3. `graph:simulate` Worker call → position update → final node/edge position render

All three steps must complete before hiding `.view-loading`. If step 1 returns zero rows, show empty panel immediately without executing steps 2 and 3.

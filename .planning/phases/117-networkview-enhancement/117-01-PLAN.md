**graph-algorithms.handler.ts changes:**

3. In `computeShortestPath`, add `targetCardId` parameter and reconstruct the actual path when both source and target are provided:
   - After computing `singleSourceLength(g, source)` which gives distances, use BFS predecessor map to reconstruct the path from source to target
   - Implementation: after the existing BFS via `singleSourceLength`, do a second BFS from source tracking predecessors (`pred: Map<string, string>`). Then walk backward from target through predecessors to reconstruct `pathCardIds` as ordered array `[source, ..., target]`
   - If target is unreachable (not in lengths), return `reachable: false, pathCardIds: []`
   - If no targetCardId provided, keep existing behavior (pathCardIds: [])
   - Function signature becomes: `computeShortestPath(g, sourceCardId, targetCardId?)`

4. In `computeMinimumSpanningTree`, also return the MST edge pairs:
   - Change return type to include `mstEdges: Array<[string, string]>`
   - In the Kruskal loop where `union(source, target)` succeeds, push `[source, target]` to mstEdges array
   - Return `{ mstNodes, mstEdges, componentCount }`

5. In `handleGraphCompute`:
   - Pass `payload.params?.shortest_path?.targetCardId` to `computeShortestPath`
   - Capture `mstEdges` from `computeMinimumSpanningTree` result
   - Add `mstEdges` to response when spanning_tree algorithm ran and mstEdges.length > 0:
     ```typescript
     if (mstEdges.length > 0) { response.mstEdges = mstEdges; }
     ```

**NetworkView.ts changes:**

1. Add algorithm encoding state fields to the class:
```typescript
// Algorithm encoding state
private _algorithmActive = false;
private _metricsMap: Map<string, { centrality: number | null; pagerank: number | null; community_id: number | null; clustering_coeff: number | null; sp_depth: number | null; in_spanning_tree: number | null }> = new Map();
private _activeAlgorithm: string | null = null;
private _pathCardIds: string[] = [];
private _mstEdges: Array<[string, string]> = [];
private _sourceCardId: string | null = null;
private _targetCardId: string | null = null;
```

2. Add `applyAlgorithmEncoding(result)` public method that receives the graph:compute response + algorithm ID:
```typescript
async applyAlgorithmEncoding(params: {
    algorithm: string;
    pathCardIds?: string[];
    mstEdges?: Array<[string, string]>;
    reachable?: boolean;
}): Promise<void>
```
   - Sets `_algorithmActive = true`, `_activeAlgorithm = params.algorithm`
   - Stores pathCardIds and mstEdges
   - Fetches graph_metrics via `this.bridge.send('graph:metrics-read', {})` and builds `_metricsMap`
   - Calls `_reapplyEncoding()` to update visuals

3. Add `_reapplyEncoding()` private method that updates existing D3 elements:
   - **Node fill**: If community data exists in metricsMap, use `d3.scaleOrdinal(d3.schemeCategory10)` keyed by `community_id % 10`. Else keep existing colorScale.
   - **Node radius**: Determine active size metric based on `_activeAlgorithm`:
     - `centrality` -> use centrality values
     - `pagerank` -> use pagerank values
     - `clustering` -> use clustering_coeff values
     - `community`, `spanning_tree`, `shortest_path` -> use centrality if available, else degree
   - Build `d3.scaleSqrt().domain([minMetric, maxMetric]).range([MIN_RADIUS, MAX_RADIUS])` for the active metric
   - Apply with D3 transition(300ms): `nodesGroup.selectAll('g.node circle').transition().duration(300).attr('r', ...).attr('fill', ...)`
   - Update text label y-positions to match new radii

   - **Edge highlighting for shortest path** (NETV-02):
     - Build `pathEdgeSet: Set<string>` from adjacent pairs in `_pathCardIds`: for each consecutive pair `[pathCardIds[i], pathCardIds[i+1]]`, add both `"${a}-${b}"` and `"${b}-${a}"` to the set
     - Path edges: `stroke: var(--accent)`, `stroke-width: 3.5`, `stroke-opacity: 1.0`
     - Non-path edges: `stroke-opacity: 0.1`
     - Source/target node rings: select nodes by id, add `stroke: var(--accent)` / `stroke: var(--danger)`, `stroke-width: 2.5`
     - Dim non-path nodes to `opacity: 0.2`

   - **Edge highlighting for MST** (NETV-03):
     - Build `mstEdgeSet: Set<string>` from `_mstEdges` pairs (both directions)
     - MST edges: `stroke: var(--latch-time)`, `stroke-width: 2.5`, `stroke-opacity: 1.0`
     - Non-MST edges: `stroke-opacity: 0.1`

   - **Composition** (path + MST both present): path edges get path style; MST-only edges get MST style; neither get dimmed

4. Add `resetEncoding()` public method:
   - Sets `_algorithmActive = false`, clears metricsMap, pathCardIds, mstEdges, sourceCardId, targetCardId
   - Restores default degree->size via existing `degreeScale` and source->color via existing `colorScale`
   - Restores default edge opacity/stroke
   - Removes source/target node rings

5. Add `setPickedNodes(sourceId: string | null, targetId: string | null)` public method:
   - Stores source/target IDs for ring rendering
   - Applies ring highlight on matching nodes (stroke: accent/danger, stroke-width: 2.5)
   - Adds S/T badge SVG groups (circle r=8 + text) at top-right of node

6. Store `_lastDegreeScale` and `_lastColorScale` during render() for reset use.

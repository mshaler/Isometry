|-------|-------|-------|

| sm | 8px (`--space-sm`) | Compact element spacing, picker instruction margin |
| md | 12px (`--space-md`) | Legend panel internal padding |
| lg | 16px (`--space-lg`) | Default element spacing, legend row height |
| xl | 24px (`--space-xl`) | Legend panel bottom/right inset from SVG edge |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Source/target node badge (S/T) is a 20px diameter circle overlaid on the node. Not from the spacing scale — sized relative to node MIN_RADIUS (8px base).
- Legend color swatch: 12px × 12px square (fits at --space-md). Acceptable deviation.

## Color

All colors are resolved via CSS custom properties. No hardcoded hex. Values below show the token name + dark-theme resolved value for reference.

| Role | Token | Dark-theme value | Usage |
|------|-------|-----------------|-------|
| Dominant (60%) | `var(--bg-primary)` | `#1a1a2e` | SVG canvas background, overlay behind legend |
| Secondary (30%) | `var(--bg-surface)` / `var(--bg-card)` | `#252540` / `#1e1e2e` | Legend panel background, picker dropdown |
| Accent (10%) | `var(--accent)` | `#4a9eff` | Path edges, source/target ring highlight, S/T badge fill |
| MST accent | `var(--latch-time)` | `#10b981` | MST edges only (teal/green distinct from path blue) |
| Danger | `var(--danger)` | `#ff4a4a` | Target node ring + T badge fill |
| Muted edge | `var(--text-muted)` | `#858596` | Non-path, non-MST default edge stroke (unchanged from existing `EDGE_STROKE`) |

Accent reserved for:
1. Shortest path highlighted edges (stroke color)
2. Source node ring highlight + S badge background
3. Active state border on `algorithm-explorer__run` button

MST accent (`var(--latch-time)`) reserved for MST edge stroke only.

Danger (`var(--danger)`) reserved for target node ring highlight + T badge background only.

Community node fill palette: `d3.scaleOrdinal` with D3 `schemeCategory10` (source: CONTEXT.md locked decision). The 10 colors from `d3.schemeCategory10` are fixed hex values in D3 — do NOT substitute with CSS tokens. Communities beyond 10 wrap via modulo. The existing source-provenance `colorScale` is replaced entirely when algorithms are active.

Dimming contract:
- Non-path edges when shortest path active: `stroke-opacity: 0.1`
- Non-MST edges when spanning tree active: `stroke-opacity: 0.1`
- Non-path nodes when shortest path active: `opacity: 0.2` (matches existing `DIM_NODE_OPACITY = 0.2`)

## Legend Panel Specification

Position: absolute, bottom-right of the SVG container. `position: absolute; bottom: 24px; right: 24px;`

Visual:
- Background: `var(--bg-surface)` at 90% opacity via `background: rgba(37, 37, 64, 0.9)` (dark) — use `backdrop-filter: blur(4px)` as progressive enhancement
- Border: `1px solid var(--border-muted)`
- Border-radius: `var(--radius-md)` (8px)
- Padding: 12px (`--space-md`)
- Box-shadow: `var(--overlay-shadow-heavy)`
- Max-width: 200px
- z-index: 10

Visibility: Hidden (`display: none`) when no algorithm active. Shown (`display: flex; flex-direction: column; gap: 8px`) when graph:compute has succeeded and encoding is active.

Content sections (each conditionally rendered):
1. Algorithm name heading — always present when visible
2. Community color swatches — present when community algorithm ran (one row per community_id, up to 10; if > 10 show "10+ communities" text only)
3. Size scale row — present when centrality, PageRank, or clustering_coeff is the active size metric. Shows "Small → Large" label with a linear gradient bar from `var(--bg-card)` to `var(--accent)`
4. Path stroke preview — present when shortest path is active. Shows a short horizontal line in path stroke style + "Shortest Path" label
5. MST stroke preview — present when spanning tree is active. Shows a short horizontal line in MST stroke style + "Min Spanning Tree" label

## Interaction Contract

### Node Encoding Switch (NETV-01)

Trigger: `graph:compute` response received with non-error result.

Transition: D3 `transition().duration(300)` (matches `--transition-normal: 300ms`).

Before transition: existing degree/source encoding
After transition: community fill via `d3.schemeCategory10[community_id % 10]`, radius from `d3.scaleSqrt` over centrality/PageRank/clustering_coeff range.

Reset: CTRL-04 Reset button (deferred to Phase 118, but the encoding restore path must be designed for it now — Reset calls a `resetEncoding()` method that restores `MIN_RADIUS`..`MAX_RADIUS` degree scale and source-provenance `colorScale`).

### Edge Highlighting (NETV-02 / NETV-03)

Shortest path active state:
- Path edges: `stroke: var(--accent)`, `stroke-width: 3.5px`, `stroke-opacity: 1.0`
- Non-path edges: `stroke-opacity: 0.1` (D3 transition 300ms)
- Source node: ring (`stroke: var(--accent)`, `stroke-width: 2.5px`) + S badge
- Target node: ring (`stroke: var(--danger)`, `stroke-width: 2.5px`) + T badge

MST active state (spanning_tree only, no path computed):
- MST edges: `stroke: var(--latch-time)`, `stroke-width: 2.5px`, `stroke-opacity: 1.0`
- Non-MST edges: `stroke: var(--text-muted)`, `stroke-width: 1px`, `stroke-opacity: 0.1`

Composition (both path + MST computed):
- Path edges: path style (blue, 3.5px) takes priority
- MST-only edges: MST style (teal, 2.5px)
- Neither: full dim (0.1 opacity)

### Shortest Path Picker (NETV-05)

Entry: User selects "Shortest Path" radio in AlgorithmExplorer.

Pick mode visual indicators:
- Instruction text (`.nv-pick-instruction`) appears above the Run button: "Step 1: Click source node on graph"
- After first click: text updates to "Step 2: Click target node on graph"
- After second click: text updates to "Ready — press Run"
- Instruction disappears after Reset

Source node visual:
- Ring: `stroke: var(--accent)` (#4a9eff), `stroke-width: 2.5px`, `fill: none`
- S badge: filled circle, radius 8px, `fill: var(--accent)`, white text "S" at `font-size: 9px`, positioned at top-right of node circle (offset +radius, -radius)

Target node visual:
- Ring: `stroke: var(--danger)` (#ff4a4a), `stroke-width: 2.5px`, `fill: none`
- T badge: filled circle, radius 8px, `fill: var(--danger)`, white text "T" at `font-size: 9px`, positioned at top-right of node circle

Keyboard fallback dropdowns (`.nv-pick-dropdowns`):
- Two `<select>` elements labeled "Source" and "Target" with `aria-label`
- Populated with all card names from current render set, sorted alphabetically
- `font-size: var(--text-sm)` (11px), `width: 100%`, matches existing AlgorithmExplorer param style
- Selecting from dropdown highlights the corresponding node on graph with a ring (same ring style as click-select, minus the badge until both are selected)

Sync: click-select updates dropdown selection; dropdown change triggers ring highlight on the graph node.

### Hover Tooltip (CTRL-03 — Phase 118, but CSS contract defined here)

Not implemented in Phase 117. Reserve `.nv-tooltip` class name for Phase 118 use. Do not implement.

## Accessibility Contract

All interactions extend existing A11Y patterns from Phase 50 (WCAG 2.1 AA).

- S/T badge SVG groups: `aria-label="Source node"` / `aria-label="Target node"` on the `<g>` element
- Legend panel: `role="region"`, `aria-label="Graph encoding legend"`
- Pick instruction element: `role="status"`, `aria-live="polite"` — updates are announced automatically
- Source/target `<select>` elements: `<label>` element association via `for`/`id` pair
- Node ring highlights: supplementary to fill/size changes — color is not the sole encoding (size + position also differentiate)
- D3 `schemeCategory10` community palette at 0.2 opacity DIM state: dimmed nodes remain distinguishable by position/size — acceptable degraded state for color-blind users

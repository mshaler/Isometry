Add the Worker Bridge notification protocol, progress emission in the ETL pipeline, per-request timeout fix, and standalone FTS optimize — the complete backend infrastructure for import progress reporting.

Purpose: This is the plumbing that enables live import progress from Worker to main thread. Without it, the user sees nothing during large imports. The per-request timeout fix also addresses a correctness issue (shared config mutation) identified in research.

Output: WorkerNotification type system, onnotification callback on WorkerBridge, progress emission at SQLiteWriter batch boundaries, per-request timeout override, standalone FTS optimize for incremental imports. All TDD with comprehensive tests.

3. Inside the batch loop, after the `setTimeout(0)` yield, emit progress. Use `performance.now()` for rate calculation with a 3-batch exponential moving average to smooth the rate display (per CONTEXT.md "rate display should update smoothly"):
   ```typescript
   let smoothedRate = 0;
   for (let i = 0; i < cards.length; i += BATCH_SIZE) {
     const batch = cards.slice(i, i + BATCH_SIZE);
     const batchStart = performance.now();
     this.insertBatch(batch);
     const batchElapsed = performance.now() - batchStart;

     const processed = Math.min(i + BATCH_SIZE, cards.length);
     const batchRate = batchElapsed > 0 ? Math.round((batch.length / batchElapsed) * 1000) : 0;
     smoothedRate = smoothedRate === 0 ? batchRate : Math.round(0.7 * smoothedRate + 0.3 * batchRate);

     if (i + BATCH_SIZE < cards.length) {
       await new Promise(resolve => setTimeout(resolve, 0));
     }

     onProgress?.(processed, cards.length, smoothedRate);
   }
   ```

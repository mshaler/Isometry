## Spacing Scale

Declared values (project-defined tokens from design-tokens.css — NOT standard 8pt multiples):

| Token | Value | Usage |
|-------|-------|-------|
| --space-xs | 4px | Icon gaps, tight inline padding |
| --space-sm | 8px | Compact element spacing, icon-to-label gap in sidebar items |
| --space-md | 12px | Default element padding (section headers, settings items) |
| --space-lg | 16px | Panel padding, sidebar item height padding |
| --space-xl | 24px | Stub placeholder vertical padding |

> Source: design-tokens.css. Note: project uses xs=4, sm=8, md=12, lg=16, xl=24 — not the standard 8pt "md=16" convention. All new CSS MUST use these var() tokens, never hardcoded pixel values.

Exceptions:
- Settings icon touch target: enlarge from 28×28px to 36×36px (MENU-02 — ~1.5x increase as specified in UAT handoff B1). This is the only permitted exception to the existing 28px button sizing.
- Sidebar leaf item min-height: 32px (adequate touch target within compact nav context).
- Sidebar top-level section header min-height: 36px (same as enlarged settings icon, consistent affordance weight).

## Color

All values reference existing design-tokens.css custom properties. No new color values introduced in this phase.

| Role | Token | Value (dark) | Usage |
|------|-------|-------------|-------|
| Dominant (60%) | --bg-primary | #1a1a2e | App background, view content area |
| Secondary (30%) | --bg-surface / --bg-secondary | #252540 / #16213e | Menubar, sidebar rail, section headers |
| Accent (10%) | --accent | #4a9eff (dark) / #2563eb (light) | Sidebar active item highlight only |
| Destructive | --danger | #ff4a4a (dark) / #dc2626 (light) | Not applicable in this phase |

Accent reserved for:
1. Active sidebar leaf item background (--accent-bg, not full --accent fill)
2. Active sidebar leaf item icon + label color (--accent)
3. Keyboard focus ring on all interactive elements (2px solid var(--accent), outline-offset: -2px)

Active sidebar item pattern: background uses var(--accent-bg) at ~12% opacity tint — NOT full --accent fill. Icon and label text color switch to var(--accent). This mirrors the existing .view-tab--active pattern but uses the subtler opacity variant to avoid over-emphasis in a persistent sidebar.

Inactive sidebar items: color: var(--text-secondary). Hover: background: var(--cell-hover).

> Source: design-tokens.css, view-tab-bar.css (.view-tab--active uses full --accent background with white text — sidebar active state intentionally differs to avoid heavy visual weight across 8 persistent sections).

## Component Inventory

### Menubar (CommandBar modifications)

The existing `.workbench-command-bar` div has three zones: [app-icon] [input-center] [settings-trigger].

Changes required per MENU-01..04:

**MENU-01 — Center wordmark:**
- Remove the current app icon button (◆ diamond) from the left zone
- Replace center zone (currently the command-palette input button) with a static `<span class="workbench-command-bar__wordmark">` containing the text "Isometry"
- The wordmark is non-interactive (no click handler, no hover state)
- Centered using `text-align: center` + `flex: 1` on the span container
- Typography: var(--text-md), weight 600, color: var(--text-primary), letter-spacing: 0.02em

**MENU-02 — Command Palette trigger (left zone):**
- The command palette trigger moves to the left zone, replacing the removed app-icon
- Retains the ◆ diamond icon OR the existing input pill — decision: retain the ◆ icon button (smaller footprint, left-aligned, consistent with existing ARIA label)
- Keep existing `.workbench-command-bar__app-icon` class + 28×28px sizing for the palette trigger

**MENU-03 — Settings icon enlargement:**
- `.workbench-command-bar__settings-trigger` width/height: 36px (up from 28px)
- Font-size for the icon character: var(--text-xl) = 18px (up from var(--text-md) = 14px)
- Touch target is now 36×36px — meets 44px minimum only on desktop; acceptable given this is a macOS WKWebView target

**MENU-04 — Remove ViewSwitcher from menubar:**
- The `.workbench-tab-bar-slot` and `.view-tab-bar` components are removed from CommandBar/WorkbenchShell
- No replacement in menubar. Visual gap: none — the freed horizontal space is absorbed by the flex layout
- ViewSwitcher moves to Visualization Explorer panel (Phase 87 — not in scope here)

New menubar DOM structure:
```
.workbench-command-bar
  └─ .workbench-command-bar__app-icon  (Command Palette ◆, 28×28, LEFT)
  └─ .workbench-command-bar__wordmark  (static "Isometry" text, flex:1, CENTER)
  └─ .workbench-command-bar__settings-trigger  (⚙ gear, 36×36, RIGHT)
```

### Sidebar Navigation (new component: SidebarNav)

Phase 86 introduces a sidebar navigation rail as a distinct layout column left of the workbench panel rail. This replaces the current single-column vertical stack with a two-column layout: [sidebar-nav | workbench-content].

**Layout change:**
- `.workbench-shell` changes from `flex-direction: column` to a two-column layout
- Left column: `.workbench-sidebar` — fixed width 200px, full height, `overflow-y: auto`
- Right column: `.workbench-main` — `flex: 1`, `flex-direction: column`, contains existing panel rail + view content
- Sidebar and main column share the same height as the shell (both `height: 100%`)

**Sidebar structure:**

```
.workbench-sidebar
  └─ .workbench-sidebar__nav  (role="navigation", aria-label="Workbench navigation")
       └─ .sidebar-section (×8, one per top-level item)
            ├─ .sidebar-section__header (button, role="button", aria-expanded)
            │    ├─ .sidebar-section__icon  (Unicode glyph)
            │    ├─ .sidebar-section__label  (text)
            │    └─ .sidebar-section__chevron  (▸ collapsed / ▾ expanded)
            └─ .sidebar-section__items (ul, role="list")
                 └─ .sidebar-item (li > button, role="menuitem")
                      ├─ .sidebar-item__icon
                      └─ .sidebar-item__label
```

**8 top-level sections with icons and sub-items:**

| # | Section | Icon | Sub-items |
|---|---------|------|-----------|
| 1 | Data Explorer | 🗄 (U+1F5C4) | Catalog / CAS, Extensions |
| 2 | Properties Explorer | 🔧 (U+1F527) | (leaf — no sub-items) |
| 3 | Projection Explorer | 📐 (U+1F4D0) | (leaf — no sub-items) |
| 4 | Visualization Explorer | 📊 (U+1F4CA) | List, Gallery, Kanban, Grid, SuperGrid, Map, Timeline, Charts, Graphs |
| 5 | LATCH Explorers | 🏷️ (U+1F3F7) | Location Explorer, Alphanumeric Explorer, Time Explorer, Category Explorer, Hierarchy Explorer |
| 6 | GRAPH Explorers | 🕸 (U+1F578) | Path, Centrality, Community, Similarity, Link, Embed |
| 7 | Formula Explorer | ƒ (U+0192) | DSL Formulas, SQL Queries, Graph Queries, Audit View |
| 8 | Interface Builder | 🧩 (U+1F9E9) | Formats, Templates, Apps |

> Icon choices follow existing project convention: Unicode glyphs/emoji used throughout WorkbenchShell.ts and CollapsibleSection stubs. No icon library dependency.

**Three-state toggle model (SIDE-01..05):**

Each top-level section has three states:
1. `hidden` — section header and all sub-items invisible (data-state="hidden")
2. `collapsed` — section header visible, sub-items hidden (data-state="collapsed", aria-expanded="false")
3. `visible` — section header visible, sub-items visible (data-state="visible", aria-expanded="true")

Toggle behavior: clicking the section header cycles collapsed ↔ visible. The `hidden` state is set programmatically (not user-toggled in this phase). Default state at mount: all sections `collapsed` (items hidden, headers visible).

Sections 6 (GRAPH Explorers), 7 (Formula Explorer), and 8 (Interface Builder) render placeholder panels with stub copy when expanded (navigation stubs — no functional content).

**Leaf item active state:**

A leaf item becomes `active` when:
- Its panel is currently visible in the workbench content area, OR
- Its view type is currently selected (for Visualization Explorer sub-items)

Active class: `.sidebar-item--active`
Active styles:
- background: var(--accent-bg) — ~12% tint
- color: var(--accent) on both icon and label
- font-weight: 600 (label only — distinguishes from inactive 400)
- border-left: 2px solid var(--accent) — positional indicator
- No box-shadow, no border-radius fill (keeps sidebar visually lightweight)

Inactive hover: background: var(--cell-hover); color: var(--text-primary)

## Interaction Contracts

### Sidebar Section Toggle

| Trigger | Action |
|---------|--------|
| Click section header | Toggle between collapsed ↔ visible |
| Keyboard Enter / Space on header | Same as click |
| aria-expanded | "true" when visible, "false" when collapsed |
| Collapse animation | max-height 0 → auto, 200ms ease-out (reuses existing .collapsible-section transition) |

### Sidebar Leaf Item Activation

| Trigger | Action |
|---------|--------|
| Click leaf item | Set item to active state, activate corresponding panel/view |
| Keyboard Enter on item | Same as click |
| Only one leaf active at a time | Previous active item clears --active class on new activation |
| aria-current="page" | Set on currently active leaf item |

### Menubar Wordmark

| Property | Value |
|----------|-------|
| Interactive | No — static text, no pointer events |
| Cursor | default |
| Hover | No visual change |

## Accessibility Contracts

All patterns must be WCAG 2.1 AA compliant (established in v4.4, Phase 50).

| Contract | Implementation |
|----------|---------------|
| Keyboard navigation | Tab through sidebar section headers; Enter/Space to expand; Tab into leaf items when section is visible |
| Roving tabindex | Within an expanded section's sub-items, use roving tabindex (established pattern from Phase 84) |
| Focus trap | None required — sidebar is persistent nav, not a modal |
| Screen reader section role | `<nav role="navigation">` wrapper with `aria-label="Workbench navigation"` |
| Leaf items | `role="menuitem"` or native `<button>` — either is acceptable; `<button>` preferred for keyboard compat |
| Chevron ARIA | `aria-hidden="true"` on chevron icon (decorative); `aria-expanded` on the header button |
| Active leaf | `aria-current="page"` on active `.sidebar-item` |
| Touch targets | Section headers 36px min-height, leaf items 32px min-height |
| Focus ring | 2px solid var(--accent), outline-offset: -2px — matches existing app-wide focus pattern |

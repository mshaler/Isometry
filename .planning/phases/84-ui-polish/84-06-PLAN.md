# Plan 84-06: Replace WorkbenchShell stub strings with explicit section state

## Objective

SECTION_CONFIGS embeds hardcoded 'coming soon' stub strings for three sections. These appear if mount ordering drifts or mount silently fails. Replace with an explicit loading/ready/empty state model. Syncs with Plan 02's collapsible-section__body--has-explorer class.

<tasks>

<task id="1">
**Add setState method to CollapsibleSection**

Add `setState(state: 'loading' | 'ready' | 'empty')` to the CollapsibleSection component (or equivalent section body manager):

- **loading** — neutral: blank body or minimal spinner, no stub text
- **ready** — explorer mounted: hide loading UI, section body visible
- **empty** — intentionally unpopulated: show emptyContent if configured

Method must be idempotent (safe to call multiple times with same state).
</task>

<task id="2">
**Remove stubContent from explorer-backed sections**

Update SECTION_CONFIGS:
```typescript
// Before:
{ title: 'Properties',  icon: '🔧', storageKey: 'properties',  stubContent: 'Properties explorer coming soon' },
{ title: 'Projection',  icon: '📐', storageKey: 'projection',  stubContent: 'Projection explorer coming soon' },
{ title: 'LATCH',       icon: '🏷️', storageKey: 'latch',       stubContent: 'LATCH explorer coming soon'      },

// After:
{ title: 'Properties',  icon: '🔧', storageKey: 'properties' },
{ title: 'Projection',  icon: '📐', storageKey: 'projection' },
{ title: 'LATCH',       icon: '🏷️', storageKey: 'latch'      },
```

Set `setState('loading')` at section creation time for these three sections.
</task>

<task id="3">
**Call setState('ready') after explorer mount**

After each explorer is appended to its section body:
```typescript
propertiesSection.setState('ready');
propertiesSection.body.classList.add('collapsible-section__body--has-explorer');
```

Same pattern for Projection and LATCH sections. The class addition syncs with Plan 02's CSS fallback.
</task>

<task id="4">
**Add three behavioral tests**

```typescript
it('Properties section shows loading state (no stub text) before explorer mounts')
it('Properties section transitions to ready state after explorer mount')
it('ready state is stable after repeated provider notifications')
```
</task>

</tasks>

# Plan 84-02: Fix :has() behavioral dependency in LatchExplorers

## Objective

Two uses of `:has()` in the codebase: one behavioral (JS DOM query in LatchExplorers.ts) and one layout (CSS in workbench.css). WKWebView on macOS 12 / iOS 15 has incomplete `:has()` support. Replace behavioral usage with data-attribute selector; add class-based CSS fallback for layout usage.

<tasks>

<task id="1">
**Add data-time-field attribute at mount time**

In `src/ui/LatchExplorers.ts`, when rendering each time-field presets container, add a `data-time-field` attribute:

```typescript
presetsContainer.dataset['timeField'] = field;
```
</task>

<task id="2">
**Replace :has() DOM query with attribute selector**

At line ~695 in LatchExplorers.ts, replace:
```typescript
// Before:
const presetsContainer = this._rootEl.querySelector(
  `.latch-time-presets:has(button[data-field="${field}"])`
) as HTMLElement | null;

// After:
const presetsContainer = this._rootEl.querySelector(
  `.latch-time-presets[data-time-field="${field}"]`
) as HTMLElement | null;
```
</task>

<task id="3">
**Add class-based CSS primary rule**

In `src/styles/workbench.css`, add the class-based rule BEFORE the existing `:has()` rules:

```css
/* Primary rule — class-based, always works: */
.collapsible-section__body--has-explorer {
  max-height: 2000px;
}
```

Keep existing `:has()` rules as progressive enhancement (no deletion).
</task>

<task id="4">
**Set class in WorkbenchShell after explorer mount**

In `src/ui/WorkbenchShell.ts`, after appending each explorer's root element to the section body:

```typescript
sectionBody.classList.add('collapsible-section__body--has-explorer');
```
</task>

<task id="5">
**Add behavioral test**

```typescript
it('finds time presets container by data-time-field attribute without :has()')
```

Mount a time field section, assert presetsContainer is found via [data-time-field] selector, assert UI updates correctly when preset is activated.
</task>

</tasks>

## Component Inventory

### ViewZipper Strip

The ViewZipper is a horizontal strip mounted at the top of the Visualization Explorer panel's content area. It contains two zones:

**Zone 1 — Tab Row (9 tabs):**

```
[ List ] [ Gallery ] [ Kanban ] [ Grid ] [ SuperGrid ] [ Map ] [ Timeline ] [ Charts ] [ Graphs ]
```

Each tab:
- Element: `<button role="tab" aria-selected="true|false">`
- Container: `<nav role="tablist" aria-label="View type">`
- CSS class: `.vzip-tab`, `.vzip-tab--active` for active state
- Inactive: background `transparent`, color `--text-secondary`
- Hover: background `--bg-card`, color `--text-primary`
- Active: background `--accent`, color `white`, font-weight 600
- Border radius: `--radius-sm` (4px)
- Padding: `--space-xs` `--space-sm` (4px 8px)
- Font size: `--text-sm` (11px)
- Transition: `background-color var(--transition-fast)`, `color var(--transition-fast)`

Keyboard: roving tabindex, ArrowLeft/ArrowRight cycles tabs, Home/End for first/last. Matches existing `ViewTabBar` keyboard pattern.

**Zone 2 — Play/Stop Control:**

```
[ ▶ Play ]   (cycling inactive)
[ ■ Stop ]   (cycling active — accent ring)
```

- Element: `<button class="vzip-play-btn">` / `<button class="vzip-stop-btn">`
- Same strip as tabs, right-aligned via `margin-left: auto`
- Play state: background `transparent`, color `--text-secondary`, border `1px solid --border-subtle`
- Stop state (cycling active): background `--accent-bg`, color `--accent`, border `1px solid --accent-border`
- Height: 28px, padding: `--space-xs` `--space-sm` (4px 8px)
- Border radius: `--radius-sm` (4px)
- Transition: `background-color var(--transition-fast)`, `color var(--transition-fast)`, `border-color var(--transition-fast)`
- ARIA: `aria-label="Play auto-cycle"` / `aria-label="Stop auto-cycle"`, `aria-pressed="false|true"`

### ViewZipper Strip Container

```
.vzip-strip {
  display: flex;
  align-items: center;
  gap: var(--space-xs); /* 4px — smallest on-scale gap; tabs are visually separated by padding alone */
  padding: var(--space-xs) var(--space-sm);
  background: var(--bg-surface);
  border-bottom: 1px solid var(--border-subtle);
  flex-shrink: 0;
  overflow-x: auto;
}
```

**Source:** Directly mirrors `.view-tab-bar` pattern from `view-tab-bar.css`. New class name to avoid conflict since ViewTabBar will be deprecated from the shell.

### Crossfade Transition

When auto-cycling or when user clicks a tab:
- View canvas fade-out: `opacity 0 → 1` over `--transition-normal` (300ms)
- No position animation in this phase (position-morph deferred to follow-on phase per ROADMAP)
- Implementation: CSS transition on `.workbench-view-content` or a wrapper div with class `.vzip-transition-frame`
- Auto-cycle hold time: 2000ms per view (hard constant, not configurable in this phase)
- Cycle sequence: List → Gallery → Kanban → Grid → SuperGrid → Map → Timeline → Charts → Graphs → (loop)

### Sidebar Active State Sync

When user clicks a tab in ViewZipper, the corresponding leaf item in the Visualization Explorer sidebar section must also receive `.sidebar-item--active`. This is a state sync requirement, not a new visual pattern — it reuses existing `sidebar-item--active` CSS.

## States and Interactions

### Tab States (9 tabs)

| State | Visual |
|-------|--------|
| Inactive | Transparent bg, `--text-secondary`, weight 400 |
| Hover | `--bg-card` bg, `--text-primary`, weight 400 |
| Active | `--accent` bg, white text, weight 600 |
| Focus-visible | `outline: 2px solid var(--accent); outline-offset: -2px` |

### Play/Stop Button States

| State | Visual |
|-------|--------|
| Play (idle) | Transparent bg, `--text-secondary`, `--border-subtle` border |
| Play hover | `--cell-hover` bg, `--text-primary` |
| Stop (cycling) | `--accent-bg` bg, `--accent` text, `--accent-border` border |
| Stop hover | `--accent-bg` bg, `--accent-hover` text |
| Focus-visible | `outline: 2px solid var(--accent); outline-offset: -2px` |

### Cycling Behavior

1. User clicks Play: `aria-pressed="true"` on Stop button, cycling begins, current tab advances every 2000ms
2. Each cycle step: crossfade transition (300ms), active tab updates, sidebar item updates
3. User clicks Stop (or clicks any tab manually): cycling halts, `aria-pressed="false"`, current view held
4. Manual tab click during cycling: halts cycling first, then switches view (same as Stop + tab click)

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Play" (compact single-verb label — deliberate choice for strip context; full intent expressed via `aria-label="Play auto-cycle"`) |
| Stop action | "Stop" (compact single-verb label — deliberate choice for strip context; full intent expressed via `aria-label="Stop auto-cycle"`) |
| Tab labels | "List", "Gallery", "Kanban", "Grid", "SuperGrid", "Map", "Timeline", "Charts", "Graphs" |
| Play button aria-label | "Play auto-cycle" |
| Stop button aria-label | "Stop auto-cycle" |
| Tablist aria-label | "View type" |
| Empty state heading | Not applicable — ViewZipper renders regardless of data state |
| Error state | Not applicable — ViewZipper has no failure mode; view switch errors surface through existing `ErrorBanner` |
| Destructive confirmation | None — no destructive actions in this phase |

**Note on button labels:** Visible labels ("Play", "Stop") use single verbs to keep the compact strip uncluttered. The `aria-label` attributes ("Play auto-cycle", "Stop auto-cycle") provide the full verb + noun form for screen readers. This pattern is intentional and consistent — do not expand the visible label without design review.

**Tab label source:** `ViewTabBar.ts` `VIEW_LABELS` array. Labels are unchanged; UAT spec (B3) uses: List, Gallery, Kanban, Grid, SuperGrid, Map, Timeline, Charts, Graphs. Note: existing ViewTabBar uses "Calendar" for the `calendar` view type; UAT spec renames it "Map". Use UAT spec labels.

**Source:** `docs/UAT-HANDOFF-DesignWorkbench.md` section B3.

## Accessibility Requirements

- ViewZipper strip: `role="tablist"`, each tab `role="tab"` with `aria-selected`
- Roving tabindex pattern on tab row: active tab has `tabindex="0"`, others `tabindex="-1"`
- ArrowLeft / ArrowRight cycles focus through tabs (wraps at ends)
- Home / End jumps to first / last tab
- Play/Stop: `aria-pressed` reflects cycling state
- All focus-visible states: `outline: 2px solid var(--accent); outline-offset: -2px`
- Screen reader announcement on view switch: reuse existing `Announcer` utility with message "Switched to {ViewLabel} view"
- Screen reader announcement on cycle start: "Auto-cycle started"
- Screen reader announcement on cycle stop: "Auto-cycle stopped on {ViewLabel} view"

**Source:** `src/accessibility/` — existing `Announcer` utility from v4.4 (WCAG 2.1 AA mandate, A11Y-01..11 reqs).

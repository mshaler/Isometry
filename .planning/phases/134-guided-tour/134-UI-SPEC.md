| Property | Value |

## Spacing Scale

All spacing via existing CSS custom properties. No new tokens introduced.

| Token | Value | Usage |
|-------|-------|-------|
| `--space-xs` | 4px | Icon gaps, inline padding, close button padding |
| `--space-sm` | 8px | Toast action button padding, gap between toast elements |
| `--space-md` | 12px | Toast container padding, popover internal padding |
| `--space-lg` | 16px | Toast bottom/top/right offset from viewport edge, section padding |
| `--space-xl` | 24px | Popover max-width inner padding |

Exceptions:
- Tour popover footer button row: `gap: var(--space-sm)` (8px between Prev/Next/Done buttons)
- Tour opt-in toast auto-dismiss: 8 seconds (timing, not spacing — from CONTEXT.md D-08)
- Touch/click targets on tour navigation buttons: minimum 32px height via `padding: var(--space-xs) var(--space-md)` (4px + 12px = 28px height + line-height ≥ 32px)

## Color

All colors reference existing design tokens. No new values.

| Role | Token | Value (dark) | Usage |
|------|-------|-------------|-------|
| Dominant (60%) | `--bg-surface` | `#252540` | Tour popover background |
| Secondary (30%) | `--bg-card` | `#1e1e2e` | Tour popover card wrapper, overlay backdrop |
| Accent (10%) | `--accent` / `--accent-border` | `#4a9eff` / rgba | Next button, active step dot, tour prompt "Start Tour" action |
| Destructive | n/a | — | No destructive actions in this phase |

Accent reserved for:
- "Start Tour" action button in opt-in toast
- "Next" / "Done" tour navigation buttons
- Active step indicator dot in popover footer
- Focus-visible outline on all tour interactive elements (`outline: 2px solid var(--accent)`)

Tour overlay backdrop: `var(--overlay-bg)` (`rgba(0,0,0,0.5)` dark / `rgba(0,0,0,0.3)` light) — driver.js overlay color configured to this token value.

Step highlight border: `var(--accent-border)` — the highlighted element gets a ring matching the project's existing selection pattern.

"Prev" and "Dismiss" buttons: `var(--text-secondary)` color, no accent fill — visually subordinate to Next/Done.

### Tour Opt-In Toast

Reuses existing `.action-toast` / `.preset-suggestion-toast` CSS pattern (fixed position, `var(--bg-surface)` background, `var(--accent-border)` border). New class: `.tour-prompt-toast`.

Position: bottom-center (matching `.action-toast` — `bottom: var(--space-lg)`, `left: 50%`, `transform: translateX(-50%)`).

```
.tour-prompt-toast {
  /* Inherits action-toast layout pattern */
  position: fixed;
  bottom: var(--space-lg);
  left: 50%;
  transform: translateX(-50%) translateY(8px);
  z-index: 9998;              /* Below tour overlay (driver.js z-index ~10000) */
  display: flex;
  align-items: center;
  gap: var(--space-sm);
  padding: var(--space-md);
  background: var(--bg-surface);
  border: 1px solid var(--accent-border);
  border-radius: var(--radius-md);
  font-size: var(--text-md);
  color: var(--text-primary);
  opacity: 0;
  pointer-events: none;
  transition: opacity var(--transition-normal), transform var(--transition-normal);
  min-width: 240px;
  max-width: 420px;
}

.tour-prompt-toast.is-visible {
  opacity: 1;
  transform: translateX(-50%) translateY(0);
  pointer-events: auto;
}
```

Auto-dismisses after 8 seconds (D-08 from CONTEXT.md).

## Tour Step Inventory

5-7 steps total (D-01). Exact 7 steps defined here — executor implements all 7.

| Step | `data-tour-target` | Title | Body template | Notes |
|------|-------------------|-------|---------------|-------|
| 1 | `[data-tour-target="sidebar-nav"]` | "Your views" | "Isometry offers 9 ways to explore your data. The ✦ badge marks the best view for your current dataset — click it to switch." | Must-have (D-02) |
| 2 | `[data-tour-target="supergrid"]` | "SuperGrid: PAFV projection" | "Each cell shows cards where the row axis matches {rowAxis} and the column axis matches {columnAxis}. The cell count is your data density." | Must-have (D-02). Template substitutes live axis names via PAFVProvider. Falls back to "the row axis" / "the column axis" if unavailable (D-05). |
| 3 | `[data-tour-target="supergrid-density"]` | "Density controls" | "Switch between Spreadsheet, Matrix, and Summary layouts to change how much data each cell shows." | Explorer/grid surface |
| 4 | `[data-tour-target="latch-explorers"]` | "LATCH panel" | "Drag any column to the row or column axis to instantly reshape the grid. Location, Alphabet, Time, Category, and Hierarchy — every axis tells a different story." | Explorer panel (D-02 "explorer panel") |
| 5 | `[data-tour-target="notebook-explorer"]` | "Notebook" | "Click any cell to open a card. The Notebook panel lets you add rich notes directly to that record." | Explorer panel |
| 6 | `[data-tour-target="command-palette-trigger"]` | "Command palette" | "Press Cmd+K to open the command palette. Every action — switching views, applying presets, restarting this tour — lives here." | Must-have (D-02) |
| 7 | `[data-tour-target="supergrid"]` | "You're ready" | "That's Isometry. Press Cmd+K anytime, or explore on your own." | Completion step, no "Next" — only "Done" |

Step 7 is the completion step: only a "Done" button is shown (no "Next"). On Done, persist `tour:completed:v1` to ui_state.

**Skipping behavior (D-06/D-07):** After a view switch, re-query the current step's `data-tour-target` via `requestAnimationFrame` callback after ViewManager.switchTo() resolves. If selector not found, advance to the next step whose target exists in the DOM. No pause/resume prompt shown to user.

## `data-tour-target` Attribute Convention

Attributes are added to existing DOM elements — no wrapper divs introduced.

| Value | Element | Location |
|-------|---------|----------|
| `"sidebar-nav"` | Root `.sidebar-nav` element | `src/ui/SidebarNav.ts` render output |
| `"supergrid"` | Root `.supergrid` container | SuperGrid root element |
| `"supergrid-density"` | Density toggle control | SuperGrid toolbar density control |
| `"latch-explorers"` | LATCH explorer panel container | WorkbenchShell LATCH section |
| `"notebook-explorer"` | Notebook explorer panel | WorkbenchShell notebook section |
| `"command-palette-trigger"` | Cmd+K trigger button or sentinel element | SidebarNav or WorkbenchShell header |

All values are kebab-case strings. Attribute set directly on existing DOM elements via `element.setAttribute('data-tour-target', '{value}')` in the respective TypeScript render methods.

- Fallback: if field value is empty/undefined, replace with the generic fallback string defined per step in the step inventory table above

## Accessibility Contract

- All tour popover buttons are focusable (`tabindex` managed by driver.js)
- On tour open: focus moves to first focusable element in popover
- On tour close: focus returns to the element that triggered the tour (the "Start Tour" button or the command palette)
- Escape key: closes tour (driver.js default behavior — do not override)
- `aria-live="polite"` region updates when step advances (announce step title)
- Spotlight/highlighted element: `aria-describedby` pointing to popover description (if driver.js supports; otherwise omit rather than hack)
- Toast: `role="status"` on `.tour-prompt-toast` for screen reader announcement
- Reduced motion: `@media (prefers-reduced-motion: reduce)` disables all slide/fade transitions on popover and toast

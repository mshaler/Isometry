3. Create `src/tour/TourEngine.ts` exporting a `TourEngine` class:
   - Constructor accepts `{ getAxisNames: () => { rowAxis: string | null; columnAxis: string | null } }` config object
   - `start()` method: Creates a driver.js instance via `driver({ ... })` with configuration:
     - `showProgress: true`
     - `progressText: 'Step {{current}} of {{total}}'`
     - `overlayOpacity: 0.5`
     - `animate: true` (driver.js handles reduced-motion itself, but also add CSS override)
     - `allowClose: true`
     - `showButtons: ['next', 'previous', 'close']`
     - `nextBtnText: 'Next'`
     - `prevBtnText: 'Back'`
     - `doneBtnText: 'Done'`
     - `onDestroyed` callback: calls `this._onComplete?.()` (hook for Plan 02 to persist completion)
     - Build steps array from TOUR_STEPS, resolving `{rowAxis}` / `{columnAxis}` placeholders by calling `this._config.getAxisNames()`. If axis name is null, use the fallback body.
     - For step 7 (isLastStep), set `showButtons: ['previous', 'close']` and override `doneBtnText` to 'Done' (driver.js uses close as done on last step).
     - Before calling `driverInstance.drive()`, filter out steps whose `document.querySelector(step.target)` returns null (per D-07 — silently skip missing targets).
   - `destroy()` method: calls `driverInstance.destroy()` if active
   - `isActive()` method: returns whether tour is currently running
   - `onComplete` setter: stores callback for `onDestroyed`
   - `handleViewSwitch()` method: called after ViewManager.switchTo() resolves. Uses `requestAnimationFrame` to re-query the current step's selector. If the element exists, calls `driverInstance.refresh()` (or equivalent driver.js API to reposition). If the element does NOT exist, calls `driverInstance.moveNext()` to advance to next step. If no more steps have valid targets, calls `driverInstance.destroy()`.

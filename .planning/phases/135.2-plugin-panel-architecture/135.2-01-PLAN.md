```typescript
// PanelMeta — describes a registerable panel
export interface PanelMeta {
  id: string;           // e.g., 'properties', 'projection', 'latch', 'calc', 'algorithm', 'notebook'
  name: string;         // Human-readable, e.g., 'Properties'
  icon: string;         // Icon name from icons.ts, e.g., 'sliders'
  description: string;  // Short description for tooltip/aria-label
  dependencies: string[]; // Panel IDs this depends on (auto-enabled per D-02)
  defaultEnabled: boolean;
}

// PanelHook — lifecycle hooks a panel plugin implements
export interface PanelHook {
  mount(container: HTMLElement): void;   // Mount panel content into the provided container
  unmount?(): void;                      // Remove panel content (optional — destroy covers this)
  resize?(): void;                       // Called when drawer width changes
  update?(): void;                       // Called by registry broadcast (D-03)
  destroy(): void;                       // Full cleanup
}

// PanelFactory — creates a fresh PanelHook instance
export type PanelFactory = () => PanelHook;
```

Create `src/ui/panels/PanelRegistry.ts` mirroring `PluginRegistry` pattern but adapted for panels:

- Private `_panels: Map<string, PanelEntry>` (insertion-ordered) where `PanelEntry = { meta: PanelMeta; factory: PanelFactory; instance: PanelHook | null }`
- Private `_enabled: Set<string>` tracking active panels
- Private `_listeners: Array<() => void>` for change notification
- `register(meta: PanelMeta, factory: PanelFactory): void` — stores entry, auto-enables if `defaultEnabled`
- `enable(id: string): void` — enables with dependency traversal (D-02), creates instance via factory, calls `_notify()`
- `disable(id: string): void` — disables with dependent traversal, calls `instance.destroy()`, sets instance to null, calls `_notify()`
- `isEnabled(id: string): boolean`
- `getAll(): PanelMeta[]` — all registered panel metadata in registration order
- `getEnabled(): PanelMeta[]` — only enabled panel metadata
- `getInstance(id: string): PanelHook | null` — returns active instance or null
- `getOrder(): string[]` — returns panel IDs in current order (supports reorder)
- `setOrder(ids: string[]): void` — reorders panels (for drag-to-reorder persistence)
- `onChange(listener: () => void): () => void` — subscribe, returns unsubscribe
- `broadcastUpdate(): void` — calls `update()` on all enabled panel instances that implement it (D-03)
- `destroyAll(): void` — destroy all enabled instances, clear enabled set
- Private helpers: `_enableSingle`, `_enableWithDeps`, `_disableSingle`, `_disableWithDependents`, `_findDependents`, `_notify` (same pattern as PluginRegistry)

Create `src/ui/panels/PanelDrawer.ts`:

```typescript
export interface PanelDrawerConfig {
  registry: PanelRegistry;
  bridge: { send(cmd: string, payload: unknown): Promise<unknown> };  // for ui_state persistence
}
```

Constructor receives config + mounts into a provided container element. Creates three DOM elements that get inserted into the container:
1. `.panel-icon-strip` — vertical toolbar with `role="toolbar"` and `aria-label="Panel controls"`
2. `.panel-drawer` — collapsible drawer surface
3. `.panel-drawer__resize-handle` — resize drag target with `role="separator"`, `aria-orientation="vertical"`, `aria-valuenow`, `aria-valuemin="200"`, `aria-valuemax="480"`

**Icon strip rendering:**
- For each panel from `registry.getAll()` (in `getOrder()` order), create a `<button>` with:
  - `class="panel-icon-strip__btn"`, `aria-label="{panel.description}"`, `title="{panel.description}"`, `aria-pressed="{isEnabled}"`
  - `innerHTML = iconSvg(panel.icon, 16)`
  - `data-panel-id="{panel.id}"`
- Use roving tabindex pattern: first button `tabindex="0"`, rest `tabindex="-1"`, ArrowUp/ArrowDown navigate
- Click handler: toggle panel (enable if disabled, disable if enabled). If enabling and drawer is closed, open drawer. If disabling and no panels remain enabled, close drawer.

**Drag-to-reorder on icon strip:**
- On pointerdown: start 150ms timer
- If 150ms elapses without pointerup: enter drag mode — set `.panel-icon-strip__btn--dragging` class, track pointermove to reorder icons visually (swap DOM positions when pointer crosses button midpoint)
- On pointerup in drag mode: commit new order to `registry.setOrder()`, persist to ui_state key `panel-drawer:order` via `bridge.send('ui:set', { key: 'panel-drawer:order', value: JSON.stringify(ids) })`
- If pointerup before 150ms: treat as toggle click (no drag)

**Drawer management:**
- When a panel is enabled: create `CollapsibleSection` with `storageKey: 'panel-section:{panelId}'`, mount it inside `.panel-drawer__scroll`, call `panel.instance.mount(section.getBodyEl())`, set section state to 'ready'
- When a panel is disabled: call `panel.instance.destroy()` (registry handles this), remove CollapsibleSection from drawer, call section.destroy()
- Open/close: toggle `.panel-drawer--closed` class. Drawer transitions via CSS width.

**Resize handle:**
- `pointerdown` → `setPointerCapture`, track `pointermove`
- On `pointermove`: `newWidth = clamp(e.clientX - drawerLeft, 200, 480)`, set `--panel-drawer-width` on drawer element, update `aria-valuenow`
- On `pointerup`: `releasePointerCapture`, persist width to ui_state key `panel-drawer:width` via `bridge.send('ui:set', { key: 'panel-drawer:width', value: String(newWidth) })`
- Keyboard on resize handle: ArrowLeft/ArrowRight adjust by 16px, Home → 200px, End → 480px

**Initialization:**
- `async init()`: read persisted order from `bridge.send('ui:get', { key: 'panel-drawer:order' })` and apply to registry via `setOrder()`; read persisted width from `bridge.send('ui:get', { key: 'panel-drawer:width' })` and set `--panel-drawer-width`
- Subscribe to `registry.onChange()` to re-render icon strip active states

**Public API:**
- `mount(container: HTMLElement): void` — inserts icon strip, drawer, resize handle into container (between sidebar and view)
- `async init(): Promise<void>` — loads persisted state
- `destroy(): void` — removes DOM, cleans up event listeners, unsubscribes from registry

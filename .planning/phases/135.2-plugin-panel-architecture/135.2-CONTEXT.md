# Phase 135.2: Plugin-Based Panel Architecture - Context

**Gathered:** 2026-04-02
**Status:** Ready for planning

<domain>
## Phase Boundary

Replace the panel-rail + sidebar dual-surface with a plugin-registry-based panel system. Each explorer panel (Properties, Projection, LATCH, Calc, Algorithm, Notebook) becomes a registered plugin with standardized mount/unmount lifecycle, rendered in a collapsible drawer with icon strip toggle. Also fix ProjectionExplorer chip well overflow (D-08 from 135.1, still unfixed after revert).

</domain>

<decisions>
## Implementation Decisions

### Panel Plugin Lifecycle
- **D-01:** New `PanelRegistry` class — purpose-built for explorer panels. Mirrors the architectural pattern of SuperGrid's `PluginRegistry` (register + factory + lifecycle) but with panel-specific interfaces (`mount`, `unmount`, `resize`, `update`, `destroy`). Separate from grid plugins.
- **D-02:** Panel dependencies allowed — panels can declare dependencies on other panels (auto-enabled when a dependent panel is activated). Models real relationships (e.g., Calc may depend on Projection).
- **D-03:** Registry-broadcast `update()` hook — PanelRegistry broadcasts a generic `update()` to all mounted panels when key providers change (SchemaProvider reload, PAFVProvider axis change, etc.). Centralizes the notification pattern instead of each panel subscribing independently.

### Panel Drawer Layout
- **D-04:** Collapsible panel drawer replaces the current panel-rail. Drawer slides open/closed as a surface between the sidebar and the visualization view. When closed, a thin icon strip (~40px) with panel toggle icons remains visible.
- **D-05:** Thin icon strip always visible — ~40px vertical strip with icons for each registered panel. Click an icon to toggle that panel open/closed in the drawer. Active panels highlighted.
- **D-06:** Drawer is resizable via drag handle. Width persisted in `ui_state` (key: `panel-drawer:width`). Min/max constraints apply.

### Panel Visibility & Toggling
- **D-07:** Icon strip is the primary toggle surface. SidebarNav loses its explorer section toggles — sidebar stays focused on navigation (views, data sources, settings). Clean separation: sidebar = navigation, icon strip = panels.
- **D-08:** Multiple panels can be open simultaneously in the drawer. Panels stack vertically with independent CollapsibleSections. Icon strip shows active/inactive state per panel.
- **D-09:** Drag-to-reorder on icon strip — users can reorder panel icons by dragging. Order persisted in `ui_state` (key: `panel-drawer:order`).

### Chip Well Overflow (D-08 from 135.1)
- **D-10:** CSS `max-height: 120px` + `overflow-y: auto` on each ProjectionExplorer drop zone well. Action buttons below wells stay visible. Same approach as the reverted 135.1 fix, now landing in the new drawer context.

### Claude's Discretion
- Default drawer width before user resizes (reasonable starting point, e.g., 280-320px)
- Specific icon choices for each panel in the icon strip (Lucide or similar monoline style)
- Internal PanelMeta/PanelHook interface design — follow PluginTypes patterns but adapt to panel needs
- Animation/transition timing for drawer open/close
- Drag-to-reorder implementation approach (HTML5 DnD vs pointer events)

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `PluginRegistry` — Battle-tested register/enable/disable/destroy lifecycle with 27 plugins. Architectural model for PanelRegistry (different interfaces, same pattern).
- `CollapsibleSection` — Already handles expand/collapse/loading states with `storageKey` persistence. Reuse directly inside the drawer for each panel.
- `ui_state` table — Existing persistence mechanism for UI preferences. Will store drawer width and panel order.

### Established Patterns
- Factory pattern (`PluginFactory`) — Plugins created fresh on enable, `destroy()` on disable. Same pattern for panels.
- CSS custom property theming (`data-theme` + token palettes) — Drawer and icon strip must adopt.
- `storageKey` convention — Each CollapsibleSection already persists open/collapsed state via this key.

### Integration Points
- `WorkbenchShell` — Panel-rail creation logic (lines 98-106) will be replaced with PanelRegistry + drawer mount. `getPanelRailEl()` deprecated/removed, new `getDrawerEl()` or equivalent.
- `main.ts` — Explorer mounting currently targets panel-rail section bodies via `shell.getSectionBody()`. Must retarget to PanelRegistry-managed mounts.
- `SidebarNav` — Explorer toggle sections removed. Section definitions in `SidebarNav.ts` simplified.
- `workbench.css` — `.workbench-panel-rail` rules replaced with drawer + icon strip layout rules.

</code_context>

<specifics>
## Specific Ideas

- The SuperGrid plugin architecture (v8.0, 27 plugins) proved the registry pattern at scale. This phase applies the same philosophy to the explorer panel surface.
- Icon strip acts like a vertical toolbar — similar to VS Code's Activity Bar or Figma's left tool strip.
- Drawer open/close should use a CSS transition for smooth animation (not instant show/hide).
- Panel order persistence means the icon strip order survives page refresh — drag-to-reorder is a power-user feature.

</specifics>

| Role | Token | Dark value | Light value | Usage |

| Dominant (60%) | --bg-primary | #1a1a2e | #ffffff | Data Explorer panel body background |

1. Active dataset row left-border indicator in Catalog SuperGrid
2. Drag-over drop zone border color (via `--drag-over-bg` fill)
3. Active sidebar leaf item background (via `--accent-bg`) and text color
4. Focus-visible outlines on all interactive elements (2px solid --accent)

**Active dataset row highlight:** Use `--accent-bg` (rgba(74, 158, 255, 0.12)) as background fill + `--accent-border` (rgba(74, 158, 255, 0.3)) as left border (3px solid). Do NOT use bold text as the sole differentiator — the background fill is the primary signal. Text may be `var(--text-primary)` at weight 600.

**Source:** CONTEXT.md "Catalog as SuperGrid" — active dataset row highlight method left to discretion; resolved above. `src/styles/design-tokens.css` — all token values.

## Component Inventory

### DataExplorerPanel

New component. Pattern: `mount(container) / update() / destroy()` — identical lifecycle to existing explorer panels (PropertiesExplorer, ProjectionExplorer, etc.).

```
.data-explorer
  .collapsible-section  (Import/Export)
  .collapsible-section  (Catalog)
  .collapsible-section  (Apps)
  .collapsible-section  (DB Utilities)
```

All four sections use the existing `.collapsible-section` / `.collapsible-section__header` / `.collapsible-section__body` / `.collapsible-section__chevron` structure from `workbench.css`. No new section CSS classes needed.

The panel mounts in `.workbench-panel-rail` (replacing existing workbench explorer panels while Data Explorer is active). Panel-switch via `data-state` attribute on `.workbench-panel-rail` — matches Phase 86 pattern.

### Import/Export Section

```
.data-explorer-import
  button.dexp-import-btn          (primary CTA — accent background)
  .dexp-drop-zone                 (drag-and-drop target)
  .dexp-export-row
    button.dexp-export-btn        (CSV)
    button.dexp-export-btn        (JSON)
    button.dexp-export-btn        (Markdown)
```

**Drop zone:** dashed border (2px dashed `--border-muted`), rounded corners (`--radius-md`), `--bg-primary` background. On `dragover`: border becomes 2px dashed `--accent`, background becomes `--drag-over-bg`. Min-height: 64px. Centered text label.

**Import button:** `background: var(--accent); color: #fff; border-radius: var(--radius-md); padding: var(--space-sm) var(--space-lg); font-size: var(--text-sm); font-weight: 600;` — matches `.app-dialog__btn--confirm` styling pattern.

**Export buttons:** Ghost style — `background: transparent; border: 1px solid var(--border-muted); color: var(--text-primary); border-radius: var(--radius-md); padding: var(--space-xs) var(--space-sm); font-size: var(--text-sm);` — matches `.app-dialog__btn--cancel` styling pattern.

### Catalog Section (SuperGrid instance)

Full SuperGrid instance bound to `datasets` table. Inherits all `supergrid.css` and `--sg-*` tokens.

**Default visible columns:** name, source_type, card_count, connection_count, last_imported_at.

**Active row treatment:** Row where `is_active = 1` receives:
- `background: var(--accent-bg)` on the row
- `border-left: 3px solid var(--accent)` on the first cell

**Row click interaction:** Click on any non-active dataset row opens AppDialog confirmation. Active row click is a no-op (no dialog, no state change).

**SuperGrid body max-height:** Catalog section body uses `.collapsible-section__body--has-explorer` class (max-height: 2000px) — same pattern as PropertiesExplorer. Add `.data-explorer__catalog` to the `:not(.collapsible-section--collapsed)` :has() guard in `workbench.css`.

Reuses `.collapsible-section__stub`, `.collapsible-section__stub-icon`, `.collapsible-section__stub-text` from `workbench.css`. No new CSS. Icon: "🔌" or system grid emoji. Label: "Coming soon".

## Interaction Contracts

### SidebarNav Wiring

- `data-explorer` section header click: toggles section `data-state` between `visible` / `collapsed`
- `data-explorer` > `catalog` item click: activates Data Explorer panel AND programmatically expands the Catalog collapsible section (collapses Import/Export, Apps, DB Utilities)
- When Data Explorer is active, workbench explorer panels (Properties, Projection, LATCH, Visual, Notebook, Calc) hide via `display: none` on their containers
- Re-activating any workbench section from SidebarNav: Data Explorer hides, target workbench panel shows

**State tracking:** `.workbench-panel-rail[data-active-panel="data-explorer"]` selector — matches Phase 86 `data-state` attribute pattern.

### Drag-and-Drop (Import)

1. `dragenter` / `dragover`: add class `.dexp-drop-zone--active` → border color changes to `--accent`, background to `--drag-over-bg`
2. `dragleave` / `drop`: remove `.dexp-drop-zone--active`
3. On `drop`: read `dataTransfer.files[0]`, pass to existing ImportOrchestrator flow
4. HTML5 native drag events only — no library dependency (matches Phase 88 CONTEXT.md "Claude's Discretion")

### Dataset Row Click (Catalog)

1. User clicks non-active dataset row in Catalog SuperGrid
2. Immediately open AppDialog confirmation (no loading state before confirmation)
3. User cancels: dialog closes, no state change
4. User confirms "Switch Dataset": dialog closes → trigger ViewManager eviction path (Phase 85 path: `evictAll()` → DELETE → re-import → SchemaProvider refresh → provider resets)
5. During eviction: show loading spinner in `.workbench-view-content` (existing loading overlay pattern)

### Vacuum / Optimize

1. User clicks Vacuum button
2. No confirmation dialog — button text changes to "Optimizing…" immediately
3. Run `VACUUM; REINDEX;` via Worker bridge
4. On completion: button text reverts to "Vacuum / Optimize"
5. If >500ms: show `import-toast` style notification "Database optimized"

### Export Database File

- Web: trigger `Blob` download of the sql.js `db.export()` ArrayBuffer as `isometry-backup-{date}.sqlite`
- Native: invoke share sheet via existing NativeBridge postMessage
- No confirmation dialog required

## Accessibility

All interactive elements follow pre-existing patterns from `accessibility.css`:

- Every `<button>` has an ARIA label if icon-only
- Drop zone: `role="region" aria-label="Drop file to import"` + `aria-live="polite"` status update on drop
- Catalog SuperGrid: active row has `aria-selected="true"` on the active row element
- AppDialog: uses native `<dialog>` element with `showModal()` — browser-native focus trap
- Keyboard: Tab navigates section headers → items; Enter/Space activates buttons and section toggles
- Focus-visible: `outline: 2px solid var(--accent); outline-offset: -2px` on all interactive elements (pre-existing rule in `accessibility.css`)

<task type="auto" tdd="true">
  <name>Task 2: Add unrecognized structure warning to JSONParser</name>
  <files>src/etl/parsers/JSONParser.ts, tests/etl/parsers/JSONParser.test.ts</files>
  <behavior>
    - parse('{"config": {}, "settings": {}}') returns 1 card AND errors array containing a warning with message listing "config, settings" as found keys
    - parse('{"items": [{"title": "A"}]}') returns 1 card with NO warning (known wrapper key)
    - parse('[{"title": "A"}]') returns 1 card with NO warning (already an array)
    - parse('{"title": "Test", "content": "Body"}') returns 1 card with NO warning (object has recognizable card fields)
  </behavior>
  <action>
    1. Modify `extractNestedArray` in `src/etl/parsers/JSONParser.ts`:
       - The method currently returns `data` unchanged when no known wrapper key is found
       - Change signature to `extractNestedArray(data: unknown, errors: ParseError[]): unknown` so it can push warnings
       - Before the final `return data` fallthrough, check if the object has ANY keys that match HEADER_SYNONYMS values (name, title, content, body, etc.)
       - If the object has NO recognizable card field keys: push a ParseError to the errors array with message: `"Unrecognized JSON structure. Found keys: [key1, key2, ...]. Expected an array or object with 'items', 'data', 'records', or 'cards' key."`
       - If the object DOES have recognizable card fields (title, name, content, body, etc.): no warning — it's a valid single-card object
       - Still return `data` unchanged in both cases (backward compatible — single object still becomes 1 card)

    2. Update the `parse()` method call site to pass `errors` array to `extractNestedArray`:
       - Change `data = this.extractNestedArray(data)` to `data = this.extractNestedArray(data, errors)`

    3. Add tests to `tests/etl/parsers/JSONParser.test.ts`:
       - Test: unrecognized structure `{"config": {}, "settings": {}}` produces 1 card + 1 warning error listing keys
       - Test: known wrapper `{"items": [...]}` produces NO warning
       - Test: recognizable card fields `{"title": "Test", "content": "Body"}` produces NO warning
       - Test: warning message contains the actual keys found
  </action>

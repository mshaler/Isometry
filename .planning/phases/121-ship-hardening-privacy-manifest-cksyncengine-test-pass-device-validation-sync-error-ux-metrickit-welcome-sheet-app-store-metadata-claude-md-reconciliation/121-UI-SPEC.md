# Phase 121 — UI Design Contract

> Visual and interaction contract for Phase 121: Ship Hardening.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — native SwiftUI only |
| Preset | not applicable |
| Component library | SwiftUI system components (List, Section, Button, Sheet, Alert) |
| Icon library | SF Symbols (system only — no third-party icon library) |
| Font | -apple-system / San Francisco (SwiftUI `.font()` semantic styles) |

**Notes:** This phase has no web-layer (TypeScript/CSS) UI work. All new surfaces are SwiftUI native. The existing CSS design token system (`design-tokens.css`) is unchanged and not extended in this phase. shadcn gate: not applicable (no React/Next.js/Vite target in scope).

## Spacing Scale

All SwiftUI spacing must use multiples of 4pt. Map to the established CSS scale for conceptual consistency.

| Token | Value | SwiftUI Usage |
|-------|-------|---------------|
| xs | 4pt | Icon-to-label gap, inline padding |
| sm | 8pt | Compact row padding, List row vertical spacing |
| md | 16pt | Default Section padding, sheet content insets |
| lg | 24pt | Between major content groups within a sheet |
| xl | 32pt | Welcome sheet vertical rhythm between icon/title/buttons |
| 2xl | 48pt | Welcome sheet top/bottom insets |

Exceptions:
- Welcome sheet app icon: 80pt × 80pt (standard App Store icon display size — not a spacing token)
- Touch targets on interactive elements: minimum 44pt height (iOS HIG requirement, applies to all buttons in sync error banner and welcome sheet)
- Recovery overlay: `.padding(40)` existing value retained (matches existing `recoveryOverlay` pattern)

## Typography

Uses SwiftUI semantic font styles. Maps to these size equivalents from `design-tokens.css` for reference.

| Role | SwiftUI Style | Approx. Size | Weight | Usage |
|------|---------------|-------------|--------|-------|
| Body | `.body` | 17pt | Regular (400) | Sync error message text, settings section body rows |
| Label | `.subheadline` | 15pt | Regular (400) | Section headers, detail disclosure text, countdown copy |
| Heading | `.headline` | 17pt | Semibold (600) | Welcome sheet tagline, banner error title |
| Display | `.title2` | 22pt | Bold (700) | Welcome sheet app name |

Rules:
- Maximum 2 weights per surface: Regular (400) and Semibold/Bold (600/700).
- `.caption` (12pt, Regular) is permitted for the CKError domain/code disclosure detail only — it is intentionally de-emphasized power-user information.
- Line height: SwiftUI default (system-managed, approximately 1.3–1.5 for body).
- No custom font sizes. Use semantic styles exclusively — they respect Dynamic Type automatically.

## Color

All colors use SwiftUI semantic color roles — no hardcoded hex values in SwiftUI code. Maps to existing `design-tokens.css` concepts for conceptual alignment.

| Role | SwiftUI Semantic | Usage |
|------|-----------------|-------|
| Dominant (60%) | `.background` / `Color(UIColor.systemBackground)` | Welcome sheet background, Settings sheet background |
| Secondary (30%) | `.secondarySystemBackground` / `Color(UIColor.secondarySystemBackground)` | List row backgrounds, grouped List sections |
| Accent (10%) | `.accentColor` (system blue) | Primary CTA buttons only: "Load Sample Data", "Retry" on sync error banner |
| Destructive | `.red` / `Color.red` | "Re-sync All Data" button label and confirmation destructive role |
| Warning | `.orange` | Sync error banner icon (`exclamationmark.triangle.fill`) |

Accent reserved for:
1. "Load Sample Data" primary button in WelcomeSheet
2. "Retry" button in sync error banner
3. "Export Diagnostics" button in Settings Diagnostics section (secondary bordered style)

Destructive color reserved for:
1. "Re-sync All Data" button in Settings Cloud Sync section (`.foregroundStyle(.red)` or `Button(role: .destructive)`)
2. Alert confirmation action in Re-sync confirmation dialog

## New UI Surfaces — Interaction Contracts

### 1. Sync Error Banner (SUXR-01..03)

**Placement:** Persistent banner at top of content area (above `WebViewContainer`, inside the `ZStack` in `ContentView.body`). Displayed when `SyncStatusPublisher.status == .error(let message)`.

**Layout:**
```
┌─────────────────────────────────────────────────────────┐
│ ⚠ [Error title — .headline]          [Retry]  [×]       │
│   [Human-readable message — .subheadline]                │
│   ▼ Details  [CKError domain/code — .caption, collapsed] │
│   Retrying in 15s...  [countdown — .caption, secondary]  │
└─────────────────────────────────────────────────────────┘
```

**States:**
- `error(message)`: banner visible, countdown running
- `syncing`: banner hidden (sync in progress, no error)
- `idle`: banner hidden

**Behavior rules:**
- Banner stays visible until sync succeeds (status returns to `.idle`) or user dismisses with [×]
- Dismiss [×] button: hides banner for current error only; if a new error occurs, banner reappears
- "Details" disclosure: `DisclosureGroup` showing raw CKError domain + code string in `.caption` weight; collapsed by default
- Retry button: `.bordered` style with `.accentColor`; triggers immediate retry, resets countdown to 5s
- Countdown label: `"Retrying in {N}s..."` using `.caption` foreground `.secondary`; hides during active retry
- Exponential backoff sequence: 5s → 15s → 60s → 300s (5 min); resets after successful sync
- SF Symbol for banner icon: `exclamationmark.triangle.fill` in `.orange`

**CKError human-readable message mapping:**
| CKError Code | Human Message |
|---|---|
| `.networkUnavailable` / `.networkFailure` | "iCloud sync paused — check your internet connection." |
| `.notAuthenticated` | "Sign in to iCloud to sync your data." |
| `.quotaExceeded` | "Your iCloud storage is full. Free up space to continue syncing." |
| `.serverResponseLost` / `.serviceUnavailable` | "iCloud is temporarily unavailable. Retrying automatically." |
| `.zoneBusy` | "iCloud is busy. Retrying automatically." |
| `.changeTokenExpired` | "Sync state expired. Tap Re-sync All Data in Settings to restore." |
| `.userDeletedZone` / `.zoneNotFound` | "iCloud sync zone was reset. Tap Re-sync All Data in Settings." |
| All other errors | "Sync error ({domain} {code}). Retrying automatically." |

### 2. Welcome Sheet (WLCM-01)

**Trigger:** `ContentView.onAppear` checks `UserDefaults.standard.bool(forKey: "hasSeenWelcome")`. If false, presents sheet.

**Presentation:** `.sheet(isPresented:)` — not fullScreenCover. No swipe-to-dismiss on first launch (`interactiveDismissDisabled(true)`).

**Layout (single-page, no carousel):**
```
┌──────────────────────────────────────┐
│                                      │
│  [48pt top inset]                    │
│                                      │
│       [App Icon — 80×80pt]           │
│                                      │
│       Isometry              [.title2.bold]
│       See everything,                │
│       connected.            [.headline, secondary]
│                                      │
│  [24pt gap]                          │
│                                      │
│  ┌──────────────────────────────┐    │
│  │   Load Sample Data           │    │  ← .buttonStyle(.borderedProminent)
│  └──────────────────────────────┘    │    accent color, full width
│                                      │
│  ┌──────────────────────────────┐    │
│  │   Start Empty                │    │  ← .buttonStyle(.bordered), secondary
│  └──────────────────────────────┘    │
│                                      │
│  [48pt bottom inset]                 │
│                                      │
└──────────────────────────────────────┘
```

**Behavior rules:**
- "Load Sample Data": calls existing `SampleDataManager` via JS bridge (`window.__isometry?.sampleDataManager?.generate()`), sets `hasSeenWelcome = true`, dismisses sheet
- "Start Empty": sets `hasSeenWelcome = true`, dismisses sheet — no data loaded
- After dismiss: user lands on current view (default `"list"`) with data visible if loaded
- `hasSeenWelcome` reset: available in Settings (Debug section, DEBUG builds only — not visible in Release)
- VoiceOver: app icon `Image` requires `.accessibilityLabel("Isometry app icon")`. Both buttons have automatic VoiceOver labels from their text. Standard SwiftUI focus order (top-to-bottom).

**Tagline:** "See everything, connected." (Claude's discretion per CONTEXT.md)

### 3. Settings — Diagnostics Section (MKIT-01..02)

**Placement:** New `Section` in `SettingsView`, inserted above the existing "About" section.

**Layout:**
```
Section("Diagnostics") {
    HStack {
        Text("Crashes")
        Spacer()
        Text("\(crashCount)")      // .foregroundStyle(.secondary)
    }
    HStack {
        Text("Hangs")
        Spacer()
        Text("\(hangCount)")       // .foregroundStyle(.secondary)
    }
    Button("Export Diagnostics") {
        // presents share sheet or Files save
    }
    .buttonStyle(.bordered)        // accent color, NOT borderedProminent
}
```

**Behavior rules:**
- Crash/hang counts sourced from `MXCrashDiagnostic` + `MXHangDiagnostic` payloads received since last export
- Counts display as plain integers; "0" when no diagnostics received yet
- "Export Diagnostics" presents `UIActivityViewController` (iOS) or `NSSavePanel` (macOS) with raw `MXDiagnosticPayload` JSON
- No destructive confirmation required for export

### 4. Settings — Cloud Sync Section (SUXR-03)

**Placement:** New `Section` in `SettingsView`, inserted above "Diagnostics" section (below "Subscription").

**Layout:**
```
Section("Cloud Sync") {
    Button("Re-sync All Data") {
        // presents confirmation alert
    }
    .foregroundStyle(.red)        // destructive visual treatment
}
```

**Confirmation alert:**
- Title: "Re-sync All Data?"
- Message: "This will re-download all data from iCloud. Your local data will be replaced."
- Actions: "Cancel" (`.cancel` role) + "Re-sync" (`.destructive` role)
- On confirm: calls `SyncManager` re-upload/re-download trigger

## Accessibility Contract

All new SwiftUI surfaces must meet these requirements (extends existing WCAG 2.1 AA posture from v4.4):

| Surface | Requirement |
|---------|-------------|
| Welcome sheet app icon | `.accessibilityLabel("Isometry app icon")` — image is decorative but labeled for context |
| Welcome sheet buttons | Automatic VoiceOver from button text. No additional label needed. |
| Welcome sheet tagline | Standard `Text` — auto-announced. `.accessibilityAddTraits(.isHeader)` on app name. |
| Sync error banner | `.accessibilityLabel("iCloud sync error. \(humanMessage)")` on banner container. Retry and dismiss buttons have standard labels. |
| Sync error countdown | `.accessibilityHidden(true)` — countdown is secondary; VoiceOver should not interrupt with timer updates |
| Sync error details | `DisclosureGroup` is natively accessible — VoiceOver announces expand/collapse state |
| Settings Diagnostics section | Standard `List` rows — automatic accessibility. Export button standard. |
| Settings Re-sync button | Standard `Button` with `.foregroundStyle(.red)` — no additional ARIA equivalent needed in SwiftUI |
| Re-sync confirmation alert | Standard SwiftUI `Alert` — fully accessible by default |

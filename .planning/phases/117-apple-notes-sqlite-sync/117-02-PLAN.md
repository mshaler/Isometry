Create `src/services/sync/AppleNotesSyncService.ts` with:

```typescript
interface SyncProgress {
  phase: 'extracting' | 'writing' | 'cleanup' | 'complete';
  current: number;
  total: number;
  message: string;
}

interface SyncOptions {
  onProgress?: (progress: SyncProgress) => void;
  batchSize?: number; // Default 100
}

interface SyncResult {
  success: boolean;
  nodesWritten: number;
  edgesWritten: number;
  duration: number;
  error?: string;
}

class AppleNotesSyncService {
  constructor(db: Database, adapter: AppleNotesAdapter);

  async fullSync(options?: SyncOptions): Promise<SyncResult>;
  async incrementalSync(options?: SyncOptions): Promise<SyncResult>;
  getSyncState(): SyncState | null;
}
```

Implementation details:
1. Accept `AppleNotesAdapter` as an injected dependency (NOT instantiated internally)
   - **Runtime boundary note:** AppleNotesAdapter uses better-sqlite3 (Node.js native), while AppleNotesSyncService targets sql.js (WASM/browser). These cannot run in the same process. Production wiring (Tauri IPC or Node backend) is deferred to 117-04.
   - For tests: mock the adapter interface
2. Instantiate `NodeWriter` with the provided sql.js db
3. Use `createSettingsService(db)` for sync state persistence
4. Sync state key: `apple_notes_sync_state` (JSON-serialized SyncState)
5. Use the injected adapter instance for all extraction operations
6. `fullSync()`:
   - Call `adapter.fullSync()` to get SyncResult with nodes/edges
   - Report progress: extracting → writing → cleanup → complete
   - Call `nodeWriter.upsertNodes()` in batches (default 100)
   - Call `nodeWriter.upsertEdges()` in batches
   - Save sync state to settings
7. `incrementalSync()`:
   - Load previous sync state from settings
   - Call `adapter.incrementalSync(state)`
   - Process nodes/edges as above
   - Call `nodeWriter.softDeleteBySource()` for deletedIds
   - Save updated sync state

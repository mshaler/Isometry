## Zone Theme Label Contract

The header slot text content is the ONLY user-visible copy introduced in the v13.0 substrate.
These values are locked in `ZONE_LABELS` in `SuperWidget.ts`.

| `zoneRole` value | Header text content | Assertion |
|------------------|---------------------|-----------|
| `'primary'` | `"Primary"` | `widget.headerEl.textContent === 'Primary'` |
| `'secondary'` | `"Secondary"` | `widget.headerEl.textContent === 'Secondary'` |
| `'tertiary'` | `"Tertiary"` | `widget.headerEl.textContent === 'Tertiary'` |

Header label updates when `zoneRole` changes between consecutive `commitProjection` calls.
Header label does NOT update when `zoneRole` is unchanged (header `data-render-count` stays stable).

## Console Warning Contract

`console.warn` calls are asserted by INTG-04 and INTG-06 cross-seam tests.

| Trigger | Message pattern |
|---------|-----------------|
| Invalid projection at `commitProjection` | `"[SuperWidget] commitProjection rejected: {reason}"` |
| Unknown `canvasId` returned undefined from factory | `"[SuperWidget] canvasFactory returned undefined for canvasId: {canvasId}"` |

Test assertion pattern: spy on `console.warn` before the call, assert `.toHaveBeenCalledWith(expect.stringContaining('[SuperWidget]'))`.

## CanvasFactory Signature Change Contract

Phase 166 introduces a minimal, explicit type change (CONTEXT.md D-01 through D-04).

| Item | Before (Phase 164/165) | After (Phase 166) |
|------|------------------------|-------------------|
| `CanvasFactory` type | `(canvasId: string) => CanvasComponent \| undefined` | `(canvasId: string, binding: CanvasBinding) => CanvasComponent \| undefined` |
| `commitProjection` call | `this._canvasFactory(proj.canvasId)` | `this._canvasFactory(proj.canvasId, proj.canvasBinding)` |
| `getCanvasFactory()` closure | `entry.create()` | `entry.create(binding)` |
| Canvas lifecycle trigger | `prev.canvasType !== proj.canvasType \|\| prev.canvasId !== proj.canvasId` | also includes `prev.canvasBinding !== proj.canvasBinding` |

## Playwright WebKit Smoke Test Contract

The WebKit smoke test (INTG-07) is a CI hard gate. It runs sequentially after chromium specs.
Source: CONTEXT.md D-07, D-08, D-09.

### playwright.config.ts Change

Add `webkit` project alongside existing `chromium`. The smoke spec uses project-scoped
filtering (`--project=webkit`) to run exclusively in WebKit.

```ts
// Add to projects array:
{
  name: 'webkit',
  use: { ...devices['Desktop Safari'] },
}
```

### Smoke Test Scope (INTG-07)

The WebKit smoke test exercises transition matrix cases 1–3 only (fast, focused):

| Step | Transition | DOM Assertion |
|------|------------|---------------|
| 1 | Mount widget, commit Explorer projection | `[data-canvas-type="Explorer"]` present |
| 2 | Commit View/Bound projection | `[data-canvas-type="View"]` present, `[data-sidecar]` present, `[data-canvas-type="Explorer"]` absent |
| 3 | Commit Editor projection | `[data-canvas-type="Editor"]` present, `[data-canvas-type="View"]` absent, header text is `"Tertiary"` or current zoneRole label |

### Smoke Spec Location

`e2e/superwidget-webkit-smoke.spec.ts` — placed in existing `e2e/` directory.

### Page Harness

The smoke spec uses the existing Vite dev server (`http://localhost:5173`) with a
`page.evaluate()` inline bootstrap — same pattern as existing e2e specs that call
`window` bridge methods. A dedicated HTML harness page is NOT required.

### CI Pipeline Change

The existing `e2e` CI job gains a webkit step running after the chromium step.
No new parallel job. Failure blocks merge (hard gate).

<objective>
Implement TreeView: a collapsible hierarchy view that renders cards in a tree layout derived from connections of a selected label, using d3-hierarchy for layout computation and d3-stratify for flat-to-tree conversion.

Purpose: TreeView lets users explore containment/parent-child relationships spatially. The tree axis is defined by a user-selectable connection label, and orphan cards are shown separately. Collapse state persists across re-renders without re-stratifying.

Output: Working TreeView with expand/collapse, orphan list, selection, registered in views/index.ts
</objective>

Create `src/views/TreeView.ts` implementing IView for collapsible tree hierarchy.

**Constructor:**
- Accepts ViewConfig, optional `treeLabel: string` (default: 'contains'), optional SelectionProvider
- Internal state:
  - `root: d3.HierarchyNode<TreeNodeData> | null` — the hierarchy root (persisted for collapse state)
  - `treeLabel: string` — which connection label defines the hierarchy
  - `orphans: CardDatum[]` — cards not in the tree
  - `svg`, `treeLayer` — SVG elements
  - `orphanContainer` — HTML div for orphan list

**TreeNodeData interface (internal):**
```typescript
interface TreeNodeData {
  id: string;
  parentId: string | null;
  name: string;
  card_type: string;
}
```

**mount(container):**
1. Create SVG element (class: 'tree-view')
2. Create inner `g.tree-layer` with d3-zoom attached (same pattern as NetworkView)
3. Create sub-groups: `g.tree-links`, `g.tree-nodes`
4. Create `div.orphan-list` below SVG for orphan cards
5. Attach d3-zoom to SVG for pan/zoom

**render(cards):**
1. If cards is empty, clear SVG and orphan list, return
2. Fetch connections for visible cards via bridge (same approach as NetworkView — db:exec with parameterized IN clause)
3. Filter connections by `this.treeLabel` — only connections with matching label define the hierarchy
4. Build parent mapping: for each connection with this label, `source_id` is parent, `target_id` is child
5. Classify cards:
   - Cards that appear as a child in any connection → tree nodes with parentId
   - Cards that appear as source but never as target → root nodes (parentId = null)
   - Cards that appear in neither source nor target for this label → orphans
6. If multiple roots exist, inject synthetic `__forest_root__` node with all roots as children
7. Check if `this.root` exists (re-render vs first render):
   - First render: call `d3.stratify()` to build hierarchy
   - Re-render with same data shape: DO NOT re-stratify — preserve collapse state on existing root. Only re-stratify if the set of card IDs has changed.
8. Run `d3.tree().nodeSize([NODE_WIDTH, NODE_HEIGHT])` on the root
9. Render nodes with D3 data join (`root.descendants()` excluding the synthetic root)
10. Render links with D3 data join (`root.links()`)
11. Render orphans in the HTML orphan container

**Layout (Claude's discretion per CONTEXT.md — choosing top-down vertical):**
- Tree orientation: top-down vertical (root at top, children below)
- nodeSize: [120, 80] (120px horizontal spacing, 80px vertical)
- separation: (a, b) => a.parent === b.parent ? 1 : 1.5

**Node rendering:**
- Circle at computed (x, y) position
- Text label (card name) below circle
- Color by card_type (same scheme as NetworkView)

**Link rendering:**
- SVG `<path>` using d3.linkVertical() — smooth curves from parent to child
- Stroke: semi-transparent gray

**Orphan list rendering:**
- Simple HTML divs inside the orphan container
- Each orphan shows name and card_type icon
- Visually separated from tree with a horizontal rule and "Unconnected cards" label

**destroy():**
- Remove SVG and orphan container from parent
- Set root = null
- Clear event listeners

Write `tests/views/TreeView.test.ts` with jsdom environment:

- d3 data join key function `d => d.data.id` present

1. Add expand/collapse to TreeView.ts:

**Toggle function:**
```typescript
private _toggleNode(node: d3.HierarchyNode<TreeNodeData>): void {
  if (node.children) {
    // Collapse: stash children
    (node as any)._children = node.children;
    node.children = undefined;
  } else if ((node as any)._children) {
    // Expand: restore children
    node.children = (node as any)._children;
    (node as any)._children = undefined;
  }
  // Re-run layout on existing root — do NOT re-stratify
  this._treeLayout(this.root!);
  this._updateTreeRender();
}
```

**Click handler on nodes:**
- Primary click (no shift): toggle expand/collapse, THEN select via SelectionProvider.toggle(id)
- Shift-click: only select (addToSelection), do NOT toggle collapse — this prevents accidental collapse during multi-select

**_updateTreeRender() — partial re-render:**
- Uses the SAME D3 data join pattern as initial render
- Updates node positions (cx, cy from recomputed layout)
- Updates link paths (from recomputed parent-child coordinates)
- Enters new nodes (from expanded children)
- Exits removed nodes (from collapsed children) — animate fade out
- Does NOT re-stratify — the root hierarchy object is preserved with its _children stash

**Visual indicator for collapsed nodes:**
- Nodes with `_children` (collapsed): filled circle
- Nodes without children AND without _children (leaf nodes): small filled circle
- Nodes with children (expanded): hollow circle with border

**Selection visuals:**
- Subscribe to SelectionProvider changes
- Selected nodes get a 3px highlight stroke
- Store SelectionProvider unsubscribe function for destroy()

## Goal

Add user override and disabled-field layers to SchemaProvider so that `getFieldsByFamily()`, `getAxisColumns()`, `getFilterableColumns()`, and `getNumericColumns()` reflect user LATCH family reassignments and field disable state. User overrides always win over heuristic classification. The override/disabled maps are independent of `initialize()` and survive re-initialization. All existing consumers (getLatchFamily via latch.ts, LatchExplorers, PropertiesExplorer, ProjectionExplorer) automatically inherit the override behavior because they read from SchemaProvider.

## must_haves

1. SchemaProvider has `_latchOverrides: Map<string, LatchFamily>` and `_disabledFields: Set<string>` private fields, initialized empty in class body
2. `setOverrides(overrides)` replaces `_latchOverrides` and calls `_scheduleNotify()`
3. `setDisabled(disabled)` replaces `_disabledFields` and calls `_scheduleNotify()`
4. `getFieldsByFamily(family)` applies override map before filtering by family, excludes disabled fields
5. `getAxisColumns()` excludes disabled fields and applies override to returned `latchFamily` values
6. `getFilterableColumns()` excludes disabled fields
7. `getNumericColumns()` excludes disabled fields
8. `getColumns('cards')` does NOT filter disabled fields (used for validation via `isValidColumn`)
9. `getLatchFamilies()` applies override map and excludes disabled fields
10. `getHeuristicFamily(field)` returns the original PRAGMA-derived latchFamily (never overridden)
11. `getLatchOverride(field)` returns the user override or undefined
12. `hasAnyOverride()` and `hasAnyDisabled()` boolean accessors
13. `getDisabledFields()` returns `ReadonlySet<string>`
14. `getOverrides()` returns `ReadonlyMap<string, LatchFamily>` (needed for persistence serialization)
15. `getAllAxisColumns()` returns ALL columns (including disabled) with override-applied latchFamily — needed by PropertiesExplorer to show disabled fields greyed-out in place
16. `initialize()` does NOT reset `_latchOverrides` or `_disabledFields` -- they survive re-init
17. Tests cover: override merge, disabled exclusion, heuristic accessor, re-init survival, subscriber notification on setOverrides/setDisabled, getColumns validation unaffected by disabled, getAllAxisColumns includes disabled with overrides

<details>
Add the following to SchemaProvider class:

1. Import `LatchFamily` type (already imported from `../worker/protocol`).
2. Add private fields:
   ```typescript
   private _latchOverrides: Map<string, LatchFamily> = new Map();
   private _disabledFields: Set<string> = new Set();
   ```
3. Add public setter methods:
   ```typescript
   setOverrides(overrides: Map<string, LatchFamily>): void {
     this._latchOverrides = new Map(overrides);
     this._scheduleNotify();
   }
   setDisabled(disabled: Set<string>): void {
     this._disabledFields = new Set(disabled);
     this._scheduleNotify();
   }
   ```
4. Add read accessors:
   ```typescript
   getHeuristicFamily(field: string): LatchFamily | undefined
   // Returns this._cards.find(c => c.name === field)?.latchFamily

   getLatchOverride(field: string): LatchFamily | undefined
   // Returns this._latchOverrides.get(field)

   hasAnyOverride(): boolean
   // Returns this._latchOverrides.size > 0

   hasAnyDisabled(): boolean
   // Returns this._disabledFields.size > 0

   getDisabledFields(): ReadonlySet<string>
   // Returns this._disabledFields

   getOverrides(): ReadonlyMap<string, LatchFamily>
   // Returns this._latchOverrides
   ```
5. Modify `getFieldsByFamily(family)`:
   - Filter: exclude disabled fields, use effective family (override ?? original)
   ```typescript
   return this._cards.filter(c => {
     if (this._disabledFields.has(c.name)) return false;
     const effective = this._latchOverrides.get(c.name) ?? c.latchFamily;
     return effective === family;
   });
   ```
6. Modify `getAxisColumns()`:
   - Exclude disabled, apply override to latchFamily in returned objects
   ```typescript
   return this._cards
     .filter(c => !this._disabledFields.has(c.name))
     .map(c => ({
       ...c,
       latchFamily: this._latchOverrides.get(c.name) ?? c.latchFamily,
     }));
   ```
7. Modify `getFilterableColumns()`:
   - Exclude disabled: `return this._cards.filter(c => !this._disabledFields.has(c.name));`
8. Modify `getNumericColumns()`:
   - Exclude disabled: add `!this._disabledFields.has(c.name)` to filter
9. Modify `getLatchFamilies()`:
   - Apply override to family key, exclude disabled fields
10. Add `getAllAxisColumns()` — returns ALL columns (including disabled) with override-applied latchFamily:
   ```typescript
   getAllAxisColumns(): readonly ColumnInfo[] {
     return this._cards.map(c => ({
       ...c,
       latchFamily: this._latchOverrides.get(c.name) ?? c.latchFamily,
     }));
   }
   ```
   This is needed by PropertiesExplorer to show disabled fields greyed-out in place within their LATCH column (per user decision). `getAxisColumns()` (which excludes disabled) remains correct for ProjectionExplorer and LatchExplorers.
11. Leave `getColumns('cards')` UNCHANGED -- it returns raw PRAGMA data for validation
12. Leave `isValidColumn()` UNCHANGED -- disabled fields are still valid columns
13. Confirm `initialize()` does NOT touch `_latchOverrides` or `_disabledFields`

<objective>
Build the CommandBar component with app icon trigger, command input placeholder, and settings dropdown menu.

Purpose: CommandBar is the always-visible top bar of the workbench shell. It provides three triggers: (1) app icon that opens CommandPalette, (2) command input placeholder that opens CommandPalette, and (3) settings gear that opens a lightweight dropdown for theme/density/help/about quick actions. All triggers delegate to existing provider methods — CommandBar introduces zero new business logic.

Output: CommandBar.ts (mount/destroy lifecycle, settings dropdown with ARIA), CommandBar.test.ts (unit tests).

Use callback-based config for loose coupling (INTG-02 — no direct provider imports):

```typescript
export interface CommandBarConfig {
  onOpenPalette: () => void;        // App icon + command input click
  onCycleTheme: () => void;         // Settings > Theme toggle
  onCycleDensity: () => void;       // Settings > Density toggle
  onToggleHelp: () => void;         // Settings > Help
  getThemeLabel: () => string;      // Current theme display text
  getDensityLabel: () => string;    // Current density display text
}

export class CommandBar {
  constructor(config: CommandBarConfig);
  mount(container: HTMLElement): void;
  destroy(): void;
}
```

Settings dropdown behavior:
- Toggle via settings trigger button click
- When open: aria-expanded="true" on trigger, dropdown display:block
- When closed: aria-expanded="false" on trigger, dropdown display:none
- Close on Escape (document keydown listener, only active when dropdown open)
- Close on outside click (document click listener added when open, removed when closed)
- Close after any item click (VS Code pattern)
- Theme item text updates after cycle: "Theme: Dark" → "Theme: Light" → "Theme: System"
- Density item text updates after cycle: "Density: Compact" → "Density: Comfortable" → "Density: Spacious"

## Component Inventory

Phase 132 introduces two new UI elements and reuses one existing pattern.

### New: ✦ recommendation badge in SidebarNav view items

- **Placement:** Inline after the view label text within `.sidebar-item__label`, rendered as a sibling `<span>` element
- **Class:** `sidebar-item__badge` (new class)
- **Content:** Static Unicode glyph `✦` (U+2736 SIX POINTED BLACK STAR)
- **Appearance:**
  - `font-size: var(--text-xs)` (10px)
  - `color: var(--accent)`
  - `margin-left: var(--space-xs)` (4px gap from label text)
  - `vertical-align: middle`
  - `line-height: 1`
  - `flex-shrink: 0`
- **Tooltip:** `title` attribute on the `.sidebar-item` element set to e.g. `"Recommended for calendar data"` — per project pattern (`title` attribute tooltips throughout codebase)
- **Visibility:** Present only when a dataset is loaded and that dataset's source type has a registered `ViewRecommendation`. Removed (element hidden or not rendered) when dataset changes to a type with no recommendation. No animation on appear/disappear.
- **data-testid:** `sidebar-badge-recommended`
- **Active state:** Badge inherits `color: var(--accent)` from active item — no special override needed since accent item is already accent-colored

### New: auto-switch toast

- **Class:** Reuse existing `.import-toast` element — append a new message via the existing `ImportToast.show()` / `.setStatus()` API, or fire a separate `ImportToast` with the switch message. Does not introduce a new toast component.
- **Message format:** `"Switched to [ViewName] — best view for [source description]."` Examples:
  - `"Switched to Timeline — best view for calendar data."`
  - `"Switched to Timeline — best view for reminders."`
  - `"Switched to Tree — best view for Apple Notes."`
  - `"Switched to Tree — best view for Notes."`
  - `"Switched to Network — best view for Alto Index."`
- **Duration:** 3000ms auto-dismiss (matches existing import toast dismiss timing)
- **Position:** Fixed top-right, same slot as `.import-toast` — fires after the import-complete toast has cleared (sequential, not stacked)
- **Appearance:** Identical to existing `.import-toast` — `var(--bg-surface)` background, `1px solid var(--accent-border)` border, `var(--radius-md)` corners, `var(--text-base)` text

### CSS file: sidebar-item badge

Add `.sidebar-item__badge` to `src/styles/sidebar-nav.css`. No new CSS file needed.

## Interaction States

### ✦ recommendation badge

| State | Appearance |
|-------|-----------|
| Dataset has recommendation | `sidebar-item__badge` span present, `color: var(--accent)`, `title` tooltip set |
| Dataset has no recommendation | `sidebar-item__badge` span not rendered (or `display: none`) |
| Dataset changes | Badge updated synchronously via SidebarNav update mechanism; no transition animation |
| View item active (current view) | Badge retains `var(--accent)` color — same as active item color, no conflict |
| View item hover | Badge inherits `color: var(--text-primary)` from `.sidebar-item:hover` unless item is active |

### Auto-switch on first import

| State | Behavior |
|-------|---------|
| First import, recommendation exists | `ViewManager.switchTo(recommendedView)` called, auto-switch toast fires with view name |
| First import, no recommendation | No auto-switch, no toast; `view:defaults:applied:{datasetId}` flag still set |
| Subsequent imports (flag already set) | No auto-switch, no toast |
| User manually switches away after auto-switch | No re-trigger — flag gate prevents re-switch |
| Auto-switch fires alongside SuperGrid axis defaults (Phase 131) | Both fire in same first-import window under same `view:defaults:applied:{datasetId}` flag — no double-gate |

### View-specific default configuration (applied silently)

No new user-visible interaction. ViewRecommendation config (sort field, groupBy, filter) applied at the provider level, same as SuperGrid axis defaults in Phase 131. The view renders with the correct defaults — no toast, no animation, no notification beyond the auto-switch toast.

Notes on copy decisions:
- Badge tooltip uses "Recommended for [source description]" — not "Best view" (that phrasing is in the toast). Keeps tooltip minimal.
- Toast format `"Switched to [View] — best view for [data]."` mirrors the decision in CONTEXT.md D-07 verbatim.
- Source descriptions use natural-language labels ("calendar data", "reminders") not source-type identifiers ("native_calendar", "apple_notes") — user-facing copy should never expose internal enum values.
- "Apple Notes" capitalized as a proper noun; "Notes" for native_notes (generic).
- Period at end of toast message for grammatical completeness consistent with import-toast error copy style.

## Implementation Notes for Executor

1. **CSS placement:** Add `.sidebar-item__badge` rules to `src/styles/sidebar-nav.css`. No new CSS file needed. Approximately 6 lines of CSS.

2. **Badge DOM structure:** Within `.sidebar-item`, after `.sidebar-item__label`, insert `<span class="sidebar-item__badge" aria-hidden="true">✦</span>`. Use `aria-hidden="true"` — the tooltip on the parent item conveys the meaning for assistive technology. Do not put the tooltip on the badge span itself.

3. **Badge visibility toggle:** When dataset changes, update each view item's badge: check `ViewRecommendation` for the new dataset's source type. If recommendation exists, render badge and set `title` on `.sidebar-item`. If not, remove badge span and clear `title`. SidebarNav update mechanism TBD by planner (callback / event / direct DOM mutation per CONTEXT.md Claude's Discretion).

4. **Toast sequencing:** Auto-switch toast fires after the existing import-complete toast has cleared. Do not stack two toasts simultaneously. If `ImportToast` supports queueing, use that. Otherwise fire the auto-switch toast with a 500ms delay after import-toast dismiss. Use the same `.import-toast` element and `ImportToast` class instance — do not create a second toast element.

5. **No new design tokens needed.** All tokens exist in `design-tokens.css`. Do not add tokens for the badge.

6. **No new CSS files needed.** Extend `src/styles/sidebar-nav.css` only.

7. **ViewRecommendation interface:** Defined in Phase 132 planning. Separate from `DefaultMapping`. Badge and auto-switch logic reads from the new `ViewRecommendation` registry (part of extended `ViewDefaultsRegistry`).

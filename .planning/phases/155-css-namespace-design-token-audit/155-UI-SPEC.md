## Spacing Scale

The project uses a non-standard spacing scale (not a pure 8-point grid). The tokens in `design-tokens.css` are the canonical values. Use these exactly — do not add new spacing tokens unless a needed value is completely absent from this table.

| Token | Value | Usage |
|-------|-------|-------|
| --space-xs | 4px | Icon gaps, inline padding, tight chip spacing |
| --space-sm | 8px | Compact element spacing, label-to-control gaps |
| --space-md | 12px | Default element spacing (note: 12px, not 16px) |
| --space-lg | 16px | Section padding, button horizontal padding |
| --space-xl | 24px | Layout gaps, major section separation |

Note: The project spacing scale tops out at `--space-xl: 24px`. There is no 32px, 48px, or 64px token. Structural dimensions that are not theme-sensitive (e.g., `width: 80px` for number inputs, `border-radius: 50%` for dot indicators, flex ratios) remain as plain values per Decision D-04.

Source: `src/styles/design-tokens.css` lines 109–116.

Exceptions: None for this phase.

## Typography

The project uses a compact UI scale suited to dense panel layouts. All font sizes must use the token — never the raw px value.

| Role | Token | Value | Weight | Line Height |
|------|-------|-------|--------|-------------|
| Muted label / badge | --text-xs | 10px | 400 | 1.4 |
| Explorer label / control | --text-sm | 11px | 400 or 600 | 1.4 |
| Body / default | --text-base | 13px | 400 | 1.5 |
| Emphasized label | --text-md | 14px | 600 | 1.2 |

Font weights in use: regular (400) and semibold (600) only. No other weights are permitted.

Migration rule: Any hardcoded `font-size` value must be replaced with the nearest token above. Where the original code used `var(--text-sm, 13px)`, the fallback `13px` is stripped per Decision D-03 — write `var(--text-sm)` only.

Source: `src/styles/design-tokens.css` lines 141–147; `visual-explorer.css` as gold standard.

## Color

This phase uses only the existing token vocabulary. Do not introduce new color values. The 5-theme system is implemented via CSS custom properties on `[data-theme]` — replacing hardcoded hex values with tokens automatically enables correct theming.

| Role | Token | Dark Default | Usage |
|------|-------|-------------|-------|
| Dominant surface (60%) | --bg-primary | #1a1a2e | App background |
| Secondary surface (30%) | --bg-secondary / --bg-surface | #16213e / #252540 | Explorer panel backgrounds, sidebar |
| Accent (10%) | --accent | #4a9eff | Active states, focus rings, slider thumbs, run button border — explicit list below |
| Destructive | --danger | #ff4a4a | Stale/error indicator only (`--audit-modified` for stale state) |

Accent reserved for:
- Focus ring on interactive controls (`outline: 2px solid var(--accent)`)
- Slider thumb fill (`.visual-explorer__zoom-slider` thumb)
- Active chip / selected state highlight
- Hover color on zoom-label text

Text color tokens:
- `--text-primary` — primary content text
- `--text-secondary` — labels, secondary content; replaces hardcoded `#666`
- `--text-muted` — metadata, hints, inactive; replaces hardcoded `#666` in low-emphasis contexts

Border color tokens:
- `--border-subtle` — default dividers, panel borders; replaces hardcoded `#ccc`
- `--border-muted` — slightly more prominent borders

Background tokens for interactive elements:
- `--bg-card` — card backgrounds; replaces hardcoded `#fff`
- `--bg-surface` — panel/header backgrounds; replaces hardcoded `#f5f5f5` hover states

Migration map for algorithm-explorer.css (primary migration target):
| Hardcoded value | Replace with token |
|----------------|--------------------|
| `#666` | `var(--text-secondary)` |
| `#ccc` | `var(--border-subtle)` |
| `#fff` | `var(--bg-card)` |
| `#f5f5f5` | `var(--bg-surface)` |
| `#fb923c` (fallback only) | strip fallback, keep `var(--audit-modified)` |

## Token Replacement Rules

1. **No fallback values.** Write `var(--text-sm)` not `var(--text-sm, 13px)`. Breakage from missing token load is intentional surfacing per Decision D-03.
2. **Spacing/color categories only.** Tokens replace: `padding`, `margin`, `gap`, `font-size`, `color`, `background`, `background-color`, `border-color`. Structural dimensions (`width`, `min-width`, `border-radius` values in non-theme-sensitive contexts, `opacity`, flex ratios) stay as plain values per Decision D-04.
3. **Nearest token wins.** Map to the nearest existing token value. If no token matches (e.g., a `6px` gap has no `--space-2xs` token), Claude may add `--space-2xs: 6px` to `design-tokens.css` under `:root` spacing section, or remap to the nearest existing token. Document the choice in a CSS comment.
4. **TS class references must be updated.** When a CSS selector is renamed, grep all `src/explorers/*.ts` (and related TS files) for the old string literal and update it. D3 `.classed()` calls and `document.querySelector()` calls are the primary risk points.

## Component Specs

### Theme Picker (settings area)

Location: Settings trigger zone in CommandBar (onCycleTheme callback replaced by picker open).

Layout: A non-modal inline panel OR a dropdown from the settings icon — executor choice. The picker shows 5 named options in a single column list:

```
[ ] Modern Dark
[ ] Modern Light
[ ] Modern System
[ ] NeXTSTEP
[ ] Material 3
```

Interaction:
- Click any option: instant theme switch via ThemeProvider.setTheme(). No confirmation.
- Active option shows accent-colored left border (3px solid --accent) + bold label.
- Keyboard: arrow up/down to cycle options, Enter/Space to select, Escape to close.
- Persists via existing ThemeProvider.toJSON() / ui_state pipeline — no new storage key needed.

### Theme Lag Fix

Root cause: `--theme-transition` applies 200ms transitions on `background-color, color, border-color, fill, stroke` to a hardcoded list of element classes in `design-tokens.css`. On theme switch, all matching elements repaint simultaneously.

Fix approach: class-swap suppression. When ThemeProvider._applyTheme() fires, add `.no-theme-transition` to `<html>` before setting `data-theme`, then remove it on the next `requestAnimationFrame`. The `.no-theme-transition` escape hatch already exists in `design-tokens.css` (line 383-386) — it sets `transition: none !important` on itself and all descendants.

```
// in ThemeProvider._applyTheme():
document.documentElement.classList.add('no-theme-transition');
document.documentElement.setAttribute('data-theme', this._theme);
requestAnimationFrame(() => {
  document.documentElement.classList.remove('no-theme-transition');
});
```

This replaces the current pattern where `.no-theme-transition` is only removed (never re-added on subsequent switches).

### Recent-Cards Viewer (DB Utilities section)

Location: Inside DB Utilities CollapsibleSection in DataExplorerPanel, appended below existing stat rows and action buttons.

SQL query:
```sql
SELECT id, name, source, created_at
FROM cards
WHERE deleted_at IS NULL
ORDER BY created_at DESC
LIMIT 8
```

8 rows chosen (midpoint of 5-10 range from CONTEXT.md decisions).

Layout: A `<ul class="dexp-recent-cards">` list appended to the DB Utilities body after the action buttons. Each `<li class="dexp-recent-card-row">` contains:
- Card title (13px, weight 400, --text-primary, truncated with text-overflow: ellipsis)
- Metadata line: source type + created date (11px, --text-sm, --text-muted)

Row height: 36px min-height. Clickable: entire row is the click target.

Click interaction: calls SelectionProvider.select(cardId). NotebookExplorer auto-updates via its existing SelectionProvider subscription. SuperGrid highlights the row.

Empty state: "No cards yet" in 11px --text-muted, centered, 16px vertical padding.

Section heading above the list: "Recent Cards" in 11px --text-sm weight 600 --text-secondary, with 8px top margin from action buttons.

## Interaction States

### Recent-cards row
- Default: background transparent
- Hover: background --cell-hover (rgba(128,128,128,0.08) in dark, rgba(0,0,0,0.04) in light)
- Active/pressed: background --accent-bg
- Selected (when this card is the current SelectionProvider selection): left border 3px solid --accent, background --accent-bg

### Theme picker option
- Default: label at --text-primary, no background
- Hover: background --cell-hover
- Active selection: label at --text-primary weight 600, left border 3px solid --accent

### NeXTSTEP-specific interactive states
- Buttons: raised 3D bevel (inset box-shadow) at rest, reversed bevel (inset -1px -1px 0 #d4d4d4, inset 1px 1px 0 #555555) on :active
- No hover state color change — NeXTSTEP buttons do not change on hover

### Material 3-specific interactive states
- Buttons: filled with --accent (#6750a4), border-radius: 20px (pill shape for primary), 12px for secondary
- Hover: add 8% white overlay (state layer pattern)
- Active/pressed: add 12% white overlay

## Accessibility

All requirements inherited from v4.4 WCAG 2.1 AA baseline. Phase 90 additions:

- Theme picker: `role="radiogroup"` with `aria-label="Appearance"`, each option is `role="radio"` with `aria-checked`
- Recent-cards list: `<ul>` with `aria-label="Recent cards"`, each item is a `<li>` with `role="button"` and `aria-label="{card title}"`
- Theme switch: announce change via existing Announcer utility: "Theme changed to {name}"
- NeXTSTEP and Material 3 color palettes must maintain WCAG AA 4.5:1 contrast ratio for all text/background pairs (verified by values declared above)

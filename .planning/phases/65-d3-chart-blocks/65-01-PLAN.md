    **GREEN:** Create `src/ui/charts/ChartParser.ts` with:

    ```typescript
    export interface ChartConfig {
      type: 'bar' | 'pie' | 'line' | 'scatter';
      x?: string;
      y?: string;
      value?: string;
      color?: string;
      title?: string;
      legend?: boolean;
      limit?: number;
    }

    export interface ChartParseError {
      error: string;
    }

    export function parseChartConfig(text: string): ChartConfig | ChartParseError
    ```

    Implementation: Simple line-by-line `key: value` parser (NO yaml library). Steps:
    1. Split by newlines, trim each line
    2. Skip empty lines and lines starting with `#` (comments)
    3. Find first `:` in each line, split into key/value, trim both
    4. Build a Record<string, string> of raw key-value pairs
    5. Validate `type` is present and one of: bar, pie, line, scatter
    6. Validate required keys per chart type:
       - bar/line: `x` required, `y` defaults to 'count' if missing
       - pie: `value` required (alias for `x` internally)
       - scatter: both `x` and `y` required (y cannot be 'count')
    7. Parse `legend` as boolean ('true'/'false'), `limit` as parseInt
    8. Return ChartConfig on success, ChartParseError on any validation failure

    Run tests -- all MUST pass.

    **REFACTOR:** Clean up if needed, ensure exports are clean.

    Create `src/worker/handlers/chart.handler.ts` following the supergrid.handler.ts pattern:

    ```typescript
    import type { Database } from '../../database/Database';
    import { validateAxisField } from '../../providers/allowlist';
    import type { WorkerPayloads, WorkerResponses } from '../protocol';

    export function handleChartQuery(
      db: Database,
      payload: WorkerPayloads['chart:query'],
    ): WorkerResponses['chart:query']
    ```

    Logic:
    1. Validate xField via validateAxisField() (throws on invalid -- caller catches)
    2. If yField is not null, validate yField via validateAxisField()
    3. Determine base WHERE: use payload.where if non-empty, otherwise 'deleted_at IS NULL'
    4. Build SQL per chart type:
       - **scatter**: `SELECT {xField} AS x, {yField} AS y FROM cards WHERE {where} AND {xField} IS NOT NULL AND {yField} IS NOT NULL`
       - **bar/pie with yField=null** (count): `SELECT {xField} AS label, COUNT(*) AS value FROM cards WHERE {where} AND {xField} IS NOT NULL GROUP BY {xField} ORDER BY value DESC`
       - **bar with yField** (aggregate): `SELECT {xField} AS label, SUM({yField}) AS value FROM cards WHERE {where} AND {xField} IS NOT NULL GROUP BY {xField} ORDER BY {xField} ASC`
       - **line** (always count by x for trends): `SELECT {xField} AS label, COUNT(*) AS value FROM cards WHERE {where} AND {xField} IS NOT NULL GROUP BY {xField} ORDER BY {xField} ASC`
       - **pie** (always count): Same as bar count
    5. Append `LIMIT ?` if limit is defined, push limit to params
    6. Execute via db.prepare(sql) + bind + step/getAsObject loop (same as supergrid handler)
    7. Return discriminated union: `{ type: 'labeled', rows }` for bar/pie/line, `{ type: 'xy', rows }` for scatter

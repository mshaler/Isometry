**makePluginHarness.ts:**
- Import PluginRegistry, registerCatalog, FEATURE_CATALOG, RenderContext, CellPlacement, GridLayout from src/views/pivot/plugins/
- Define HarnessOptions interface: { rows?: number, cols?: string[], containerHeight?: number, data?: Map<string, number|null> }
- Define PluginHarness interface: { registry: PluginRegistry, ctx: RenderContext, enable(id: string): void, disable(id: string): void, runPipeline(): { cells: CellPlacement[], layout: GridLayout } }
- Export function makePluginHarness(opts?: HarnessOptions): PluginHarness
- Inside: create new PluginRegistry, call registerCatalog(registry), build RenderContext from opts (defaults: rows=10, cols=['Col1','Col2'], containerHeight=400, empty data map)
- Generate visibleRows as Array.from({length: opts.rows}, (_, i) => [`Row${i}`])
- Generate allRows = visibleRows (same reference by default)
- Generate visibleCols from opts.cols
- Generate rowDimensions/colDimensions as HeaderDimension[] with label/values from generated row/col arrays
- rootEl = document.createElement('div'), apply containerHeight if given via Object.defineProperty
- isPluginEnabled delegates to registry.isEnabled
- enable/disable delegate to registry.enable/disable
- runPipeline: builds default cells from data, calls registry.runTransformData, runTransformLayout, runAfterRender, returns { cells, layout }
- Default GridLayout: { headerWidth: 120, headerHeight: 30, cellWidth: 80, cellHeight: 24, colWidths: new Map(), zoom: 1 }

**usePlugin.ts:**
- Import afterEach from vitest
- Import type PluginHarness from makePluginHarness
- Import type PluginHook from PluginTypes
- Export function usePlugin(harness: PluginHarness, pluginId: string): PluginHook
- Calls harness.enable(pluginId)
- Gets the enabled plugin's hook instance from harness.registry (use getAll + isEnabled to find it, or access internal via any cast -- but cleaner: just enable and let tests call harness.registry methods)
- Actually: the PluginRegistry doesn't expose instances directly. So usePlugin should: enable the plugin, create a reference object, and register afterEach(() => harness.disable(pluginId)) which triggers destroy() through the registry's disable path
- Return type should be the plugin ID for chaining, OR return a proxy. Simplest: return { pluginId, destroy: () => harness.disable(pluginId) } and register afterEach. But CONTEXT.md says "Returns the PluginHook instance so tests can call hook.transformData(ctx) directly or spy on methods"
- Problem: PluginRegistry stores instances privately. Solution: add a getHook(id: string): PluginHook | null public method to PluginRegistry (minimal API surface addition). This is a small production code change but necessary for test tooling.
- Alternative: usePlugin accesses harness.registry['_plugins'] via bracket notation to get the instance. This avoids production code change. Use this approach since it's test-only code.
- usePlugin: harness.enable(pluginId), then const entry = (harness.registry as any)._plugins.get(pluginId), return entry.instance as PluginHook. Register afterEach(() => { harness.disable(pluginId); }).

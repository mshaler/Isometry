<what>Create a vitest test file with it()-based timing assertions for SQL query and ETL import budgets at 20K card scale. Some assertions must FAIL (defining Phase 76/77 work); others must PASS (proving the harness works).</what>
<how>

4. Import `ImportOrchestrator` from `src/etl/ImportOrchestrator.ts` and `clearTraces` from `src/profiling/PerfTrace.ts`.

**SQL Query Budget Tests (single describe block, shared 20K DB):**
- `beforeAll`: create Database, initialize, seedDatabase with cardCount: 20_000. Timeout: 120_000.
- `afterAll`: db.close().
- Helper: `p99(samples: number[]): number` — same as render-timing.test.ts (sort ascending, index at ceil(length * 0.99) - 1).
- Helper: `measureQuery(db, sql, iterations = 50): number` — runs query `iterations` times, returns p99 ms via performance.now().
- it('GROUP BY folder, card_type p99 < BUDGET_QUERY_GROUP_BY_20K_MS'): Run the query 50 times, compute p99, `expect(p99val).toBeLessThan(BUDGET_QUERY_GROUP_BY_20K_MS)`. This MUST FAIL today (24.93 > 12).
- it('GROUP BY strftime month p99 < BUDGET_QUERY_STRFTIME_20K_MS'): Same pattern. MUST FAIL (20.64 > 10).
- it('GROUP BY status p99 < BUDGET_QUERY_STATUS_20K_MS'): Same pattern. Should PASS (1.87 < 5).
- it('FTS 3-word search p99 < BUDGET_QUERY_FTS_20K_MS'): Same pattern. Should PASS (1.70 < 5).

**ETL Import Budget Tests (separate describe block, single shared DB):**
- Copy data generators from `tests/profiling/etl-smoke.bench.ts` (genAN, genCSV, genJSON, genMD, rw, rs, CARD_TYPES, FOLDERS, WORDS).
- `beforeAll`: create Database, initialize. Timeout: 120_000.
- `afterAll`: db.close().
- Helper: `timeImport(db, type, data): Promise<number>` — same as etl-smoke.bench.ts pattern.
- it('apple_notes 20K < BUDGET_ETL_20K_MS'): Generate 20K, measure, expect < 1000. Should PASS (182 < 1000).
- it('csv 20K < BUDGET_ETL_20K_MS'): Same. Should PASS (767 < 1000).
- it('json 20K < BUDGET_ETL_20K_MS'): Same. MUST FAIL (1771 > 1000).
- it('markdown 20K < BUDGET_ETL_20K_MS'): Same. MUST FAIL (1059 > 1000).
- Each test calls clearTraces() after timing (match etl-smoke.bench.ts pattern).

5. Set timeouts: 120_000 for each it() block (20K card operations are slow).
6. Run `biome check tests/profiling/budget.test.ts` — must pass.
7. Run `npx vitest run tests/profiling/budget.test.ts` — confirm mix of PASS and FAIL results. DO NOT fix failures — they are intentional "red" tests.

**Done-when:** File exists, runs, GROUP BY folder+card_type and strftime FAIL, status and FTS PASS, json and markdown ETL FAIL, apple_notes and csv ETL PASS. Biome clean.

## Color

All values reference existing design tokens. Resolves to dark-theme defaults; light/system/NeXTSTEP themes apply automatically via CSS custom properties.

| Role | Token | Dark Value | Usage |
|------|-------|------------|-------|
| Dominant (60%) | `--bg-secondary` | `#16213e` | Canvas slot background behind tab containers |
| Secondary (30%) | `--bg-surface` | `#252540` | Tab bar strip background (`--sw-tabs-bg`) |
| Accent (10%) | `--accent` | `#4a9eff` | Active tab button background only |
| Destructive | `--danger` | `#ff4a4a` | Not directly used — exists in re-used DataExplorerPanel sections (Vacuum button); no new destructive actions introduced in this phase |

Accent reserved for: active tab button (`[data-tab-active="true"]` background fill) only. No other elements in this phase receive accent treatment.

Inactive tab buttons: `color: var(--text-secondary)`, `background: transparent`.
Tab bar border: `1px solid var(--sw-slot-border)` → `var(--border-subtle)`.
Tab button hover: `background: var(--bg-card)`, `color: var(--text-primary)`.

Source: `src/styles/superwidget.css` (tab button rules), `src/styles/design-tokens.css`.

## Component Inventory

New elements introduced by this phase. All use existing tokens — no new CSS variables.

### Explorer Tab Bar (`[data-component="explorer-canvas"] [data-slot="tab-bar"]`)

- DOM: `<div data-slot="tab-bar">` inside `.explorer-canvas` wrapper, above tab containers
- Layout: flex row, `gap: 4px`, `padding: var(--space-xs) var(--space-sm)`, `border-bottom: 1px solid var(--border-subtle)`
- Background: `var(--bg-surface)` (matches SuperWidget tabs slot)
- Overflow: `overflow-x: auto; scrollbar-width: none` (matches SuperWidget tabs slot pattern)
- Mask fade: none (tab bar has only 3 fixed items; no overflow expected)

### Explorer Tab Button (`[data-tab-id]`)

- DOM: `<button data-tab-id="{id}">` — one per tab
- Default: `background: transparent; color: var(--text-secondary); font-size: var(--sw-tab-font-size); padding: var(--sw-tab-padding-y) var(--sw-tab-padding-x); border-radius: var(--sw-tab-radius); border: none; cursor: pointer; white-space: nowrap`
- Hover: `background: var(--bg-card); color: var(--text-primary)`
- Active (`[data-tab-active="true"]`): `background: var(--accent); color: white`
- Transition: `background-color var(--transition-fast), color var(--transition-fast)`

Pattern mirrors existing `[data-tab-role="tab"]` rules in `superwidget.css`. Use same token references.

### Tab Container (`[data-tab-container="{id}"]`)

- DOM: `<div data-tab-container="{id}">` — one per tab, all pre-mounted at `mount()` time
- Visible tab: `display: block` (or flex if needed for content layout)
- Hidden tabs: `display: none`
- Switch mechanism: CSS `display` toggle only — no DOM removal. Preserves scroll position and form state (CONTEXT.md D-03).
- Layout: `display: flex; flex-direction: column; width: 100%` (inherits `.explorer-canvas` flex column)

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Tab click | Call `switchTab(proj, tabId)`, then `commitProjection(newProj)`. SuperWidget calls `onProjectionChange(proj)` back on ExplorerCanvas. ExplorerCanvas sets `display: none` on all containers, then `display: block` on the active container. |
| Tab keyboard | Not in scope — deferred to KBNV-01 (REQUIREMENTS.md future). |
| Scroll within tab | Each tab container scrolls independently. CSS `display: none` preserves scroll offset. No scroll reset on switch. |
| Initial mount | All 3 tab containers rendered and appended to wrapper. Active tab set to `import-export`. |
| Destroy | All containers removed via wrapper `.remove()`. No individual container cleanup needed. |

## CSS Selector Conventions

Follow `--sw-*` namespace pattern for any new scoped tokens. New selectors should be scoped under `[data-component="superwidget"] .explorer-canvas` or a new `[data-component="explorer-canvas"]` wrapper attribute.

Preferred: `data-*` attributes for state (`data-tab-active`, `data-tab-id`, `data-tab-container`) over class modifiers. This matches the existing SuperWidget pattern (`data-tab-role`, `data-tab-active`).

New CSS goes into `src/styles/superwidget.css` (extends existing explorer-canvas block at line 153).

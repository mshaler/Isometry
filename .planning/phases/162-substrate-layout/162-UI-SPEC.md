## Spacing Scale

Project spacing tokens (from design-tokens.css, theme-independent):

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| --space-xs | 4px | Tab button icon gaps, inline padding |
| --space-sm | 8px | Tab button padding (horizontal), tab bar padding |
| --space-md | 12px | Slot internal padding baseline |
| --space-lg | 16px | Header slot vertical padding |
| --space-xl | 24px | Not used in Phase 162 |

Exceptions:
- Tab edge fade gradient begins at 32px from each edge (SLAT-04, CONTEXT.md D-04). This is a mask-image distance, not a spacing token.
- Status slot: min-height: 0 (zero-height empty state, SLAT-03). No padding applied when empty.
- SuperWidget root: flex: 1 1 auto; min-height: 0 (SLAT-07). No padding on root.

Note: The existing project spacing scale uses --space-xs=4, --space-sm=8, --space-md=12, --space-lg=16, --space-xl=24. This is a 4/8/12/16/24 scale rather than a strict 8-point doubling scale. All SuperWidget tokens must align to this existing scale.

## DOM Structure Contract

Grid template rows (CONTEXT.md D-07): header → tabs → canvas → status

```
[data-component="superwidget"]     — root; flex: 1 1 auto; min-height: 0
  [data-slot="header"]             — zone theme label placeholder
  [data-slot="tabs"]               — scrolling flex row; mask-image fade
    [data-tab-role="tab"] × N      — tab buttons (placeholder count: 3 in test)
    [data-tab-role="config"]       — config gear; margin-left: auto; right-aligned; aria-label="Configure"
  [data-slot="canvas"]             — 1fr; receives canvas content in Phase 164
  [data-slot="status"]             — min-height: 0; always present in DOM
```

CSS Grid declaration on root:
```css
display: grid;
grid-template-rows: auto auto 1fr auto;
grid-template-columns: 1fr;
```

Row mapping: header=auto, tabs=auto, canvas=1fr, status=auto (zero-height via min-height: 0 when empty).

Tab bar overflow (SLAT-04, CONTEXT.md D-04):
```css
[data-slot="tabs"] {
  display: flex;
  flex-direction: row;
  overflow-x: auto;
  scrollbar-width: none;         /* hide scrollbar visually */
  mask-image: linear-gradient(
    to right,
    transparent,
    black var(--sw-fade-width),
    black calc(100% - var(--sw-fade-width)),
    transparent
  );
}
[data-slot="tabs"]::-webkit-scrollbar { display: none; }
```

Config gear right-alignment (SLAT-02, CONTEXT.md D-08):
```css
[data-tab-role="config"] {
  margin-left: auto;
  flex-shrink: 0;
}
```

Status slot zero-height (SLAT-03):
```css
[data-slot="status"] {
  min-height: 0;
  overflow: hidden;
}
/* Status slot grows naturally when children are added — no display: none */
```

## Interaction Contract

Phase 162 is a pure DOM skeleton. No events are wired (CONTEXT.md D-06).

| Interaction | Behavior |
|-------------|----------|
| Tab button click | No handler in Phase 162 — arrives in Phase 163/164 |
| Config gear click | No handler in Phase 162 |
| Tab bar scroll | Native browser horizontal scroll via overflow-x: auto |
| mount(container) | Appends root element to container; idempotent |
| destroy() | Removes root element from DOM; nulls all slot references |

Tab button visual states (CSS only, no JS):
- Default: background transparent, color var(--text-secondary)
- Hover: background var(--bg-card), color var(--text-primary)
- Active (data-tab-active="true"): background var(--accent), color white

No `:focus-visible` ring is specified for Phase 162. Keyboard navigation is deferred (REQUIREMENTS.md Out of Scope: KBNV-01). The planner should note this as a future TODO comment in the CSS but not implement it.

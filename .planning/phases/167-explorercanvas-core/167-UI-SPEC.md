## Interaction Contract

### Canvas Slot Mounting

- `ExplorerCanvas.mount(container)` appends a `.explorer-canvas` wrapper div to `container` ([data-slot="canvas"]).
- Inside `.explorer-canvas`, `DataExplorerPanel.mount()` is called immediately — no async, no loading state.
- The canvas slot background (`--sw-canvas-bg = --bg-secondary`) shows behind the panel until mount completes (imperceptible — synchronous DOM write).
- No skeleton or spinner is shown — the panel DOM renders in a single synchronous pass.

### Canvas Slot Destruction

- `ExplorerCanvas.destroy()` calls `DataExplorerPanel.destroy()` then removes the `.explorer-canvas` wrapper.
- After destroy, the canvas slot is empty (no child nodes). Status slot is unaffected.
- No animation on destroy — immediate DOM removal.

### Sidebar DataExplorerPanel Removal (D-03)

- The Integrate dock section no longer renders a toggle for inline DataExplorerPanel.
- The `dataExplorerChildEl` element is removed from the sidebar DOM.
- No visible affordance is shown in the sidebar for DataExplorerPanel — the SuperWidget canvas slot is the only access point.
- If SuperWidget is not mounted/visible, DataExplorerPanel is inaccessible. No fallback UI in this phase.

### refreshDataExplorer() Continuity

- `refreshDataExplorer()` in main.ts calls `updateStats()` and `updateRecentCards()` on the DataExplorerPanel instance.
- The ExplorerCanvas must expose or allow access to the DataExplorerPanel instance reference so main.ts can still call these methods.
- No visual change in this phase — the same stat rows and recent card list update as before.

### Focus and Keyboard

- No new keyboard interactions are introduced in this phase.
- Existing CollapsibleSection expand/collapse keyboard behavior is preserved (managed by CollapsibleSection.ts).
- No roving tabindex changes — DataExplorerPanel manages its own focus tree.

## CSS Authoring Rules

1. New styles for ExplorerCanvas wrapper go in `src/styles/superwidget.css` — append to existing file, scoped to `[data-component="superwidget"] .explorer-canvas`.
2. Do NOT create a separate `explorer-canvas.css` file — keep SuperWidget styles co-located.
3. The `.explorer-canvas` wrapper must be `display: flex; flex-direction: column; width: 100%; height: 100%; overflow: auto;` to allow DataExplorerPanel sections to scroll within the canvas slot.
4. Do not add padding to `.explorer-canvas` — DataExplorerPanel's own sections use internal padding.
5. All token references must use `var(--token-name)` — no raw hex values in new CSS.

    **GREEN phase — src/superwidget/projection.ts:**

    Per D-01: Three string literal unions:
    ```typescript
    export type CanvasType = 'Explorer' | 'View' | 'Editor';
    export type CanvasBinding = 'Bound' | 'Unbound';
    export type ZoneRole = 'primary' | 'secondary' | 'tertiary';
    ```

    Per D-02: Projection interface with all readonly fields, enabledTabIds as ReadonlyArray<string>.

    Per D-04: Four standalone exported functions (no class):

    `switchTab(proj, tabId)`:
    - Guard: `!proj.enabledTabIds.includes(tabId)` -> return proj
    - Guard: `proj.activeTabId === tabId` -> return proj
    - Return `{ ...proj, activeTabId: tabId }`

    `setCanvas(proj, canvasId, canvasType)`:
    - Guard: `proj.canvasId === canvasId && proj.canvasType === canvasType` -> return proj
    - Return `{ ...proj, canvasId, canvasType }`

    `setBinding(proj, binding)`:
    - Guard: `binding === 'Bound' && proj.canvasType !== 'View'` -> return proj (per D-06)
    - Guard: `proj.canvasBinding === binding` -> return proj
    - Return `{ ...proj, canvasBinding: binding }`

    `toggleTabEnabled(proj, tabId)`:
    - If tabId IS in enabledTabIds: guard against removing activeTabId (return proj); otherwise filter it out
    - If tabId is NOT in enabledTabIds: return `{ ...proj, enabledTabIds: [...proj.enabledTabIds, tabId] }`
    - If resulting enabledTabIds would be identical: return proj

    CRITICAL anti-pattern: NEVER `return { ...proj }` in a guard path. Always `return proj` (the input reference directly). This is load-bearing per D-06 and PROJ-02/04/05.

    Also export `ValidationResult` type (needed by Plan 02) but do NOT implement `validateProjection` yet.

    ```typescript
    export type ValidationResult = { valid: true } | { valid: false; reason: string };
    ```

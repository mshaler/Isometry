## DOM Structure Contract

EditorCanvas creates the following DOM inside the SuperWidget canvas slot:

```
[data-slot="canvas"]
  └─ div.editor-canvas          ← wrapper div (EditorCanvas owns, isolation boundary)
       └─ [NotebookExplorer DOM]  ← mounted by NotebookExplorer internally

[data-slot="status"]            ← sibling slot, found via DOM traversal
  └─ div.sw-editor-status-bar   ← idempotent, setup on mount
       └─ span[data-stat="card-title"]  ← text content set reactively
```

### CSS class naming

| Element | Class | Source |
|---------|-------|--------|
| Wrapper div | `editor-canvas` | Mirrors ViewCanvas's `view-canvas` convention |
| Status bar container | `sw-editor-status-bar` | Mirrors `sw-view-status-bar` from ViewCanvas |
| Title span | `sw-status-bar__item` + `data-stat="card-title"` | Reuses shared `sw-status-bar__item` modifier; data attribute scopes lookup |

The `editor-canvas` wrapper must be `display: flex; flex-direction: column; width: 100%; height: 100%; overflow: hidden` to fill the canvas slot and give NotebookExplorer its scroll context.

Source: ViewCanvas.ts lines 82-83, 156-176 (idempotent DOM setup pattern).

## State Transitions and Copy

### Status Slot States

| State | Trigger | DOM Content |
|-------|---------|-------------|
| Idle (no card selected) | Mount with no active selection, or selection cleared | `span[data-stat="card-title"]` text = "No card selected" |
| Card selected | SelectionProvider emits a card ID; EditorCanvas queries title | `span[data-stat="card-title"]` text = card title string (e.g. "Project Brief") |

Status updates are reactive: SelectionProvider subscription in EditorCanvas calls `_updateStatus()` whenever selection changes. No loading intermediate state — update is synchronous DOM text assignment after the title query resolves.

### Canvas Wrapper States

| State | Visual | Notes |
|-------|--------|-------|
| Mounting | `editor-canvas` div appended, NotebookExplorer mounts inside | No loading spinner — NE renders immediately if a card is selected |
| Active | NE renders selected card content | Full editor UI managed by NotebookExplorer |
| No card selected | NE shows its own idle state; status slot shows "No card selected" | NE handles its own empty state rendering internally |
| Destroying | wrapper removed from DOM, status el reference nulled | No visual transition required |

## Interaction Contract

### SelectionProvider Subscription

EditorCanvas subscribes to SelectionProvider on mount. On each emission:
1. Call `selection.getSelectedIds()` to get current selection array.
2. If array is empty: set status to "No card selected".
3. If array has items: query card title for `ids[0]` and set status to that title string.

Unsubscribe handle stored as `_selectionUnsub`; called in `destroy()` before NE teardown.

### Lifecycle Ordering (D-14)

```
destroy-before-mount: always call destroy() on previous canvas before mount() on next
  EditorCanvas.destroy():
    1. selectionUnsub()           → stop reactive status updates
    2. notebookExplorer.destroy() → NE flushes shadow-buffer, unsubscribes mutation/selection/filter/chart
    3. wrapperEl.remove()         → detach from canvas slot
    4. statusEl = null            → release reference
```

### Idempotent Status DOM Setup

Follows ViewCanvas pattern: before writing text, check if `[data-stat="card-title"]` exists inside the status element. If not, create `div.sw-editor-status-bar` containing `span.sw-status-bar__item[data-stat="card-title"]` and append to `statusEl`. Then set `span.textContent`. This prevents duplicate DOM if `_updateStatus` is called multiple times before DOM teardown.

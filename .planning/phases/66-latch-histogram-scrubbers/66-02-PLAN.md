Create src/ui/HistogramScrubber.ts as a self-contained D3 mini bar chart with d3.brushX overlay:

**HistogramScrubber class design:**

```typescript
export interface HistogramScrubberConfig {
  field: string;               // Column name (e.g., 'priority', 'created_at')
  fieldType: 'numeric' | 'date';
  filter: FilterProvider;
  bridge: WorkerBridgeLike;    // Reuse LatchExplorers interface
  bins?: number;               // Default 10
}
```

Constructor takes HistogramScrubberConfig. Class has mount/update/destroy lifecycle matching CollapsibleSection pattern.

**mount(container: HTMLElement):**
1. Create wrapper div with class `latch-histogram`
2. Create SVG using d3.select().append('svg') with viewBox for responsive sizing
   - Dimensions: width=200 (full panel width), height=48 (compact)
   - Margins: { top: 2, right: 4, bottom: 14, left: 4 } (bottom for x-axis tick labels)
3. Create root `<g>` with margin transform
4. Create x-axis `<g>` at bottom
5. Append container to parent
6. Call _fetchAndRender() for initial data

**_fetchAndRender():**
1. Get current FilterProvider state: `const { where, params } = this._config.filter.compile()`
2. Send histogram:query to Worker: `bridge.send('histogram:query', { field, fieldType, bins, where, params })`
3. On response, call _render(bins)

**_render(bins: Array<{binStart, binEnd, count}>):**
1. If bins is empty, show a small "No data" text and return
2. Build x scale: d3.scaleBand with domain = bin labels, range = [0, innerW], padding 0.1
   - Numeric bins: label = binStart formatted to 1 decimal (or integer if whole)
   - Date bins: label = binStart (already 'YYYY-MM' string)
3. Build y scale: d3.scaleLinear with domain = [0, max(count)], range = [innerH, 0]
4. Render x-axis with d3.axisBottom(x).tickSize(0) — tiny font, rotate if many bins
5. D3 data join (D-003 mandatory key function: d => `${d.binStart}-${d.binEnd}`):
   - enter: append rect, grow from y=innerH with 200ms transition
   - update: transition to new y/height in 200ms
   - exit: shrink to height 0, remove
6. Bar fill: var(--accent) via CSS class, not inline color
7. Bar hover: add title element (native tooltip) showing "binStart-binEnd: count"

**d3.brushX overlay:**
1. After rendering bars, create d3.brushX() with extent [[0, 0], [innerW, innerH]]
2. Append brush group to root `<g>` AFTER bars (so brush overlay is on top)
3. On 'brush' event (live dragging): no-op (or optional: light highlight)
4. On 'end' event:
   - If selection is null (user clicked away to clear), call `filter.clearRangeFilter(field)`
   - Otherwise: map pixel selection [x0, x1] back to data domain:
     a. For numeric: use x.domain() bands, find which bins are covered by selection range
     b. Get binStart of first covered bin as min, binEnd of last covered bin as max
     c. Call `filter.setRangeFilter(field, min, max)`
5. Store brush instance for programmatic clear

**update():**
- Call _fetchAndRender() to refresh with latest filter state
- This is called when LatchExplorers._valuesDirty triggers

**destroy():**
- Remove brush
- Remove SVG
- Remove wrapper div
- Null out references

**clearBrush() (public):**
- Programmatically clear the brush selection (for Clear All button)
- Call brush.move(brushGroup, null) to visually clear
- Do NOT call filter.clearRangeFilter here (the caller handles that)

---

**LatchExplorers integration:**

1. Import HistogramScrubber at top of LatchExplorers.ts
2. Add private _histograms = new Map<string, HistogramScrubber>() field
3. In _populateTime(body): after each TIME_FIELDS field group (after the presets container), create and mount a HistogramScrubber:
   ```typescript
   const histogram = new HistogramScrubber({
     field, fieldType: 'date', filter: this._config.filter,
     bridge: this._config.bridge, bins: 12  // 12 months for date fields
   });
   histogram.mount(group);
   this._histograms.set(field, histogram);
   ```
4. In _populateHierarchy(body): after each HIERARCHY_FIELDS checkbox group, create and mount a HistogramScrubber:
   ```typescript
   const histogram = new HistogramScrubber({
     field, fieldType: 'numeric', filter: this._config.filter,
     bridge: this._config.bridge, bins: 10
   });
   histogram.mount(group);
   this._histograms.set(field, histogram);
   ```
5. In update(): after _fetchAllDistinctValues(), call _histograms.forEach(h => h.update())
6. In destroy(): call _histograms.forEach(h => h.destroy()), then _histograms.clear()
7. In _handleClearAll(): call _histograms.forEach(h => h.clearBrush()), and also call filter.clearRangeFilter() for all TIME_FIELDS and HIERARCHY_FIELDS to remove range filters
8. In _updateClearAllVisibility(): add check for range filters — `const hasRangeFilters = [...TIME_FIELDS, ...HIERARCHY_FIELDS].some(f => filter.hasRangeFilter(f))` — include in anyActive boolean
9. In _computeFamilyFilterCount(): for 'T' family, also count fields with active range filters; for 'H' family, also count fields with active range filters

IMPORTANT:
- D3 data join MUST use a key function (D-003 is non-negotiable)
- SVG must use viewBox for responsive sizing — no fixed pixel widths
- Use CSS classes for colors/styling, not inline styles (except positioning)
- d3.brushX from d3-brush is included in d3 v7.9 (already in project deps)

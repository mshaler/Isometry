## Layout Contract

This phase is primarily a layout surgery. The visual contract for the DOM restructuring is:

### Before (Phase 149 state)

```
.workbench-body (flex-row)
  .workbench-sidebar           ŌåÉ DockNav (width: 48px or 280px)
  .workbench-main (flex-col)
    .workbench-data-explorer   ŌåÉ toggled, max-height 35vh, display:none by default
    .workbench-main__content (flex-ROW)
      [PanelDrawer elements]   ŌåÉ icon strip + drawer + resize handle ŌĆö DELETED this phase
      .workbench-view-content  ŌåÉ flex:1 view host
```

### After (Phase 151 target)

```
.workbench-body (flex-row)
  .workbench-sidebar           ŌåÉ DockNav (unchanged)
  .workbench-main (flex-col)
    .workbench-main__content (flex-COL ŌåÉ direction changes from row to column)
      .workbench-slot-top      ŌåÉ new empty scaffold, display:none, no content yet
      .workbench-view-content  ŌåÉ flex:1 view host (unchanged)
      .workbench-slot-bottom   ŌåÉ new empty scaffold, display:none, no content yet
```

### CSS Rules for New Elements

`.workbench-main__content` direction change:
- `flex-direction: column` (was `row`)
- `min-height: 0`, `min-width: 0`, `overflow: hidden` (preserved from current rule)

`.workbench-slot-top`:
- `width: 100%`
- `flex-shrink: 0`
- `overflow-y: auto`
- `display: none` (hidden until Phase 152 wires toggle)
- `background: var(--bg-primary)`
- `border-bottom: 1px solid var(--border-subtle)` (matches former `.workbench-data-explorer` pattern)

`.workbench-slot-bottom`:
- `width: 100%`
- `flex-shrink: 0`
- `overflow-y: auto`
- `display: none` (hidden until Phase 153 wires toggle)
- `background: var(--bg-primary)`
- `border-top: 1px solid var(--border-subtle)`

`.workbench-view-content` (unchanged): `flex: 1 1 auto`, `min-height: 0`, `overflow: hidden`, `position: relative`, `display: flex`, `flex-direction: column`.

The former `.workbench-data-explorer` CSS rule is removed. Its content (DataExplorer mount point) migrates to `.workbench-slot-top` in Phase 152.

Source: CONTEXT.md D-03, D-04, D-05; existing `workbench.css`.

## Interaction Contract

Phase 151 introduces no new user-facing interactions. The layout is restructured but all visible behavior is identical to the Phase 149 baseline:

- DockNav remains in the sidebar column ŌĆö no change.
- View content renders in `.workbench-view-content` ŌĆö no change.
- Both new slot containers start as `display:none` ŌĆö user sees no new chrome.
- No toggle controls, no animations, no resize handles are introduced.

The only observable change is the removal of the side drawer column (icon strip + drawer + resize handle). If the PanelDrawer was closed (width 0) in Phase 149, the visual result after Phase 151 is identical ŌĆö no layout shift.

## Spacing Scale

Declared values (project token names; spacing scale uses 4/8/12/16/24 progression, not strict 8-point):

| Token | Value | Usage |
|-------|-------|-------|
| --space-xs | 4px | Icon gaps, badge padding, inline separators |
| --space-sm | 8px | Button padding (vertical), row gaps, compact stacking |
| --space-md | 12px | Cell padding, stat row padding, list item horizontal padding |
| --space-lg | 16px | Button padding (horizontal), import button, section padding |
| --space-xl | 24px | Dialog padding (all four modal surfaces) |

Source: `src/styles/design-tokens.css` — pre-populated from existing tokens.

Exceptions:

| Token / Value | Reason |
|---------------|--------|
| `--space-md = 12px` | Project convention — intermediate density step for cell/stat padding; pre-existing token from design-tokens.css, not a new value introduced by this phase. Falls between the strict 8-point stops of 8px and 16px to match the visual density of existing SuperGrid and catalog rows. |
| `44px` min touch target (action buttons) | Accessibility — matches existing `.disc-row` min-height pattern extended to 44px for icon buttons on touch targets, per WCAG 2.5.5 Target Size (AAA guideline, adopted as project standard). |
| `72px` catalog actions column width | Fixed-width column — accommodates two 28px icon buttons + 8px gap + 4px horizontal margin each side; not a spacing token, a column width constraint. |
| `0px / 180px` collapsible section heights | Animation bounds — collapsed (0px) and expanded (180px scroll region) are layout heights, not spacing values. |

## Component Inventory

### 1. Catalog Actions Column (DSET-01)

Added to existing `CatalogSuperGrid` — new `actions` column appended to the 5 existing columns (name, source_type, card_count, connection_count, last_imported_at).

**Column header:** empty (no label) — 72px fixed width.

**Per-row contents:** two icon buttons in a flex row with 8px gap, horizontally centered.

| Button | Icon | Class | Idle state | Hover state |
|--------|------|-------|-----------|-------------|
| Re-import | ↺ (U+21BA, 14px) | `.dset-action-btn .dset-action-btn--reimport` | opacity 0 (hidden) | opacity 1, color var(--accent), background var(--accent-bg) |
| Delete | ✕ (U+2715, 13px) or trash SVG path | `.dset-action-btn .dset-action-btn--delete` | opacity 0 (hidden) | opacity 1, color var(--danger), background var(--danger-bg) |

**Row hover reveals buttons:** parent `.sg-row:hover .dset-action-btn` → `opacity: 1`. Buttons always visible (opacity: 1 always) when `@media (hover: none)` (touch/mobile).

**Touch target:** min-width 28px, min-height 28px per button. Row min-height 36px inherited from SuperGrid row height.

**Disabled state:** Re-import button `opacity: 0.4; cursor: not-allowed; pointer-events: none` during an active import for that dataset (when ImportToast is visible for that source).

### 2. Delete Confirmation Dialog (DSET-02)

Reuses `.app-dialog` pattern exactly. Max-width: 420px (matches existing app-dialog contract).

**Structure:**
```
.app-dialog
  .app-dialog__title    "Delete '[dataset name]'?"
  .app-dialog__message  "[N] cards and all associated connections will be permanently removed. This cannot be undone."
  .app-dialog__actions
    .app-dialog__btn--cancel     "Cancel"
    .app-dialog__btn--delete     "Delete Dataset"   ← danger variant
```

**Delete button variant:** `.app-dialog__btn--delete` — background var(--danger), color #fff, border-color var(--danger). Hover: background `color-mix(in srgb, var(--danger) 85%, black)` or opacity 0.85.

**Keyboard:** Escape closes (Cancel), Enter does NOT confirm (safety — user must click the danger button explicitly).

### 3. Re-import Progress (DSET-03)

Reuses existing `ImportToast` component unchanged. No new UI required for progress phase.

**Fallback picker:** If stored path is inaccessible, silently fall back to native file picker (same UX as initial import). No extra dialog — the OS picker IS the UI.

### 4. Diff Preview Modal (DSET-04)

New component: `.dset-diff-modal` — extends `.app-dialog` base pattern. Max-width: 560px (wider than standard 420px to accommodate card name lists).

**Structure:**
```
.dset-diff-modal  (role="dialog", aria-labelledby="diff-modal-title")
  .dset-diff-modal__title  id="diff-modal-title"  "Review Changes — [dataset name]"
  .dset-diff-summary         (3 badge chips in a flex row, gap 8px)
    .dset-diff-badge[data-kind="new"]       "N new"
    .dset-diff-badge[data-kind="modified"]  "N modified"
    .dset-diff-badge[data-kind="deleted"]   "N deleted"
  .dset-diff-sections        (collapsible sections, gap 4px)
    .dset-diff-section[data-kind="new"]       (collapsed if count = 0)
    .dset-diff-section[data-kind="modified"]  (collapsed if count = 0)
    .dset-diff-section[data-kind="deleted"]   (collapsed if count = 0)
  .app-dialog__actions
    .app-dialog__btn--cancel   "Cancel"
    .app-dialog__btn--confirm  "Commit Changes"
```

**Summary badge (`.dset-diff-badge`):**
- `font-size: var(--text-sm)` (11px), `font-weight: 600`
- `padding: 4px 8px`, `border-radius: var(--radius-sm)` (4px)
- Colors per kind: new = audit-new palette, modified = audit-modified palette, deleted = audit-deleted palette (see Color section above)
- Zero-count badges: `opacity: 0.4` (visible but de-emphasized)

**Collapsible section (`.dset-diff-section`):**
- Header: `<button>` with section kind label + count, `font-size: var(--text-sm)` (11px), `font-weight: 600`, `color: var(--text-secondary)`, left border 2px solid matching badge color for that kind
- Chevron indicator: ▶ collapsed / ▼ expanded, 10px, `color: var(--text-muted)`
- Body: `<ul>` of card names, `font-size: var(--text-base)` (13px), `color: var(--text-primary)`, max-height 180px, overflow-y auto
- Animation: `max-height` transition 200ms ease from 0 to 180px (respects `prefers-reduced-motion`)

**Zero-changes fast path:** If DedupResult has zero toInsert + zero toUpdate + zero deletedIds, modal is skipped entirely. Show ImportToast briefly with text "No changes — dataset is already up to date." for 3 seconds. No commit/cancel decision required.

## Accessibility Contract

- All icon buttons have `aria-label` with dataset name (see Copywriting Contract)
- Delete confirmation dialog: `role="alertdialog"`, `aria-describedby` pointing to body message
- Diff preview modal: `role="dialog"`, `aria-labelledby="diff-modal-title"`
- Focus management: when dialog opens, first interactive element (Cancel button) receives focus
- Focus trap: Tab cycles within dialog, does not escape to page behind backdrop
- Keyboard: Escape closes both dialogs as cancel action
- Collapsible sections: `aria-expanded` attribute on trigger button, `aria-controls` pointing to section body id
- Reduced motion: all `max-height` and `transform` transitions replaced with `transition-duration: 0.01ms` when `prefers-reduced-motion: reduce`
- WCAG 2.1 AA: all text on surface colors meets 4.5:1 contrast (inherited from existing token adjustments in design-tokens.css, requirement A11Y-01)

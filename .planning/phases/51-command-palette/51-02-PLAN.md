    ```typescript
      private _cardSearchGeneration = 0;  // Race condition guard
      private _previousFocus: HTMLElement | null = null;  // Focus restoration

      // Handlers (stored for cleanup)
      private _keydownHandler: ((e: KeyboardEvent) => void) | null = null;
      private _inputHandler: ((e: Event) => void) | null = null;
      private _backdropClickHandler: ((e: MouseEvent) => void) | null = null;
    }
    ```

    **Constructor:** Takes `registry: CommandRegistry` and `searchCards: (query: string, limit: number) => Promise<SearchResult[]>` (WorkerBridge.searchCards bound reference -- avoids importing WorkerBridge type directly).

    Optionally accept an `announcer?: { announce: (msg: string) => void }` for screen reader announcements on open/close.

    **mount(container: HTMLElement): void:**
    1. Create overlay DOM structure (see RESEARCH.md DOM structure):
       - Outer div `.command-palette`
       - Inner div `.command-palette__card`
       - Input with COMBOBOX_ATTRS.input attributes applied (loop over entries, setAttribute)
       - Input: type="text", placeholder="Type a command or search...", class="command-palette__input"
       - Results div with COMBOBOX_ATTRS.listbox attributes applied, class="command-palette__results"
    2. Append to container
    3. Store element references
    4. Attach backdrop click handler: click on overlay (but not card) closes palette
    5. Do NOT register Cmd+K here -- main.ts will register it via ShortcutRegistry and call palette.open()

    **open(): void:**
    1. Capture `document.activeElement` as `_previousFocus` for focus restoration
    2. Add `is-visible` class to overlay
    3. Set `_visible = true`
    4. Set `aria-expanded` to `'true'` on input
    5. Clear input value
    6. Populate results with recents + all visible commands (empty-query state)
    7. Set `_selectedIndex = 0`
    8. Render results
    9. Focus input (use `requestAnimationFrame` before `focus()` per project pattern)
    10. Announce "Command palette opened" via announcer (if provided)

    **close(): void:**
    1. Remove `is-visible` class
    2. Set `_visible = false`
    3. Set `aria-expanded` to `'false'` on input
    4. Clear pending card search timer
    5. Clear `_pendingCards`
    6. Restore focus: `requestAnimationFrame(() => _previousFocus?.focus())`

    **destroy(): void:**
    1. Remove overlay element from DOM
    2. Remove keydown listener from document
    3. Null out all references

    **Private: _onInput(query: string): void (dual-path search):**
    1. Synchronous path: `this._registry.search(query)` for instant command results
    2. If query is empty: show recents + getVisible()
    3. Render command results immediately
    4. Async path: if query.length >= 2, debounce ~200ms, then:
       - Increment `_cardSearchGeneration`
       - Call `this._searchCards(query, 5)`
       - On resolve: check generation matches current, if so map SearchResult[] to PaletteCommand[] with category='Cards', then re-render with both command results + card results
    5. If query.length < 2: clear `_pendingCards`

    **Private: _onKeydown(e: KeyboardEvent): void:**
    - Escape: e.preventDefault(), close()
    - ArrowDown: e.preventDefault(), move selection down (wrap to 0 if at end)
    - ArrowUp: e.preventDefault(), move selection up (wrap to end if at 0)
    - Enter: e.preventDefault(), execute selected command
    - Update `aria-activedescendant` on input to `palette-option-{_selectedIndex}` after arrow key
    - Update `aria-selected` on option elements (true on selected, false on others)

    **Private: _executeSelected(): void:**
    1. Get command at `_selectedIndex` from `_currentResults`
    2. If not found, return
    3. Call `pushRecent(command.id)` (skip for card results -- only persist non-card commands)
    4. Call `close()`
    5. Call `command.execute()`

    **Private: _renderResults(commands: PaletteCommand[], cards: PaletteCommand[]): void:**
    1. Clear listbox innerHTML
    2. Combine results: commands first, then cards (cards always last per category order)
    3. Store combined in `_currentResults`
    4. If no results: render `.command-palette__empty` with "No results found"
    5. Group by category in order: Recents > Views > Actions > Cards > Settings
       - For empty query: prepend recent commands as "Recents" category
    6. For each category group:
       - Render `.command-palette__category` div with `role="presentation"` (not selectable)
       - For each command in group:
         - Render `.command-palette__option` div with COMBOBOX_ATTRS.option.role, id=`palette-option-{flatIndex}`
         - Icon span: use category icons (e.g., Views: magnifying glass/layout icon emoji or simple text -- Claude's discretion on icon characters)
         - Label span: command.label
         - If command.shortcut: render `<kbd class="command-palette__kbd">{shortcut}</kbd>`
         - aria-selected: 'true' if flatIndex === _selectedIndex, 'false' otherwise
    7. Reset `_selectedIndex` to 0 if out of bounds
    8. Update `aria-activedescendant` on input

    **Palette visibility check:** Add `isVisible(): boolean` method for external checks.

    **Update `src/palette/index.ts`:** Add `export { CommandPalette } from './CommandPalette'`.

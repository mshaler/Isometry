## Spacing Scale

All spacing uses existing `--space-*` tokens. No new tokens introduced.

| Token | Value | Usage |
|-------|-------|-------|
| --space-xs | 4px | Tab gap (--sw-tab-gap), inline icon padding, insertion line offset |
| --space-sm | 8px | Tab padding-x (--sw-tab-padding-x), tab bar horizontal padding, chevron button padding |
| --space-xs | 4px | Tab padding-y (--sw-tab-padding-y), tab bar vertical padding |
| --space-md | 12px | (unused in tab bar — reserved for canvas area) |
| --space-lg | 16px | (section-level, not used in tab bar) |

Exceptions:
- Tab close button (×): 16px × 16px minimum hit target, centered at 0px offset within tab padding area — no extra margin
- Chevron buttons (‹ ›): 28px × 28px minimum touch target, `flex-shrink: 0` so they never collapse
- Insertion line: 2px height, `position: fixed`, `z-index: 10000` — matches existing `.pv-insertion-line` pattern

Source: existing --sw-tab-padding-x / --sw-tab-padding-y already set in superwidget.css; spacing scale from design-tokens.css

## Typography

All tab bar text uses existing `--sw-tab-font-size` = `var(--text-sm)` = 11px.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Tab label | 11px (--text-sm) | 400 (regular) | 1.4 |
| Tab badge | 10px (--text-xs) | 400 (regular) | 1.0 (single line) |
| Chevron glyph | 11px (--text-sm) | 400 (regular) | 1.0 |
| Canvas type label | 11px (--text-sm) | 600 (semibold) | 1.4 |

Notes:
- Tab labels are `white-space: nowrap` — no wrapping, no truncation ellipsis for tabs under 120px max-width
- Tabs beyond 120px wide truncate with `text-overflow: ellipsis`
- Badge (TABS-10) renders as inline `<span>` after the label text, separated by a single space, at --text-xs
- Canvas type label in badge uses weight 600 to distinguish it from the user-assigned tab name

Source: existing --sw-tab-font-size token in superwidget.css; --text-xs / --text-sm from design-tokens.css

## Color

All colors use existing design-token custom properties. No new color values introduced.

| Role | Token | Value (dark) | Usage |
|------|-------|--------------|-------|
| Dominant (60%) | --sw-tabs-bg → --bg-surface | #252540 | Tab bar background |
| Secondary (30%) | --bg-card | #1e1e2e | Inactive tab hover state |
| Accent (10%) | --accent | #4a9eff (dark) / #2563eb (light) | Active tab indicator only |
| Destructive | --danger | #ff4a4a (dark) / #dc2626 (light) | Close button (×) hover state only |

Accent (`--accent`) is reserved for exactly these elements in this phase:
1. Active tab button background (`[data-tab-active="true"]`)
2. Insertion line color during drag reorder (matches `.pv-insertion-line` pattern: `background: var(--accent)` + `box-shadow: 0 0 4px var(--accent)`)
3. The + button gets `--accent` on hover only (not resting state — resting state is `--text-muted`)

Inactive tab at rest: background transparent, color `--text-secondary`
Inactive tab on hover: background `--bg-card`, color `--text-primary`
Active tab: background `--accent`, color white (hard-coded white, not `--text-primary`, matching existing `[data-tab-active="true"]` rule in superwidget.css)
Close button (×) at rest: color `--text-muted`, no background
Close button (×) on hover: color `--danger`, background `--danger-bg`
Chevron buttons at rest: color `--text-muted`, background transparent
Chevron buttons on hover: color `--text-primary`, background `--bg-card`
Dragged source tab: opacity 0.45 (matches existing ghost-chip dimming pattern in supergrid.css)

Source: superwidget.css existing selectors; design-tokens.css color values; pivot.css insertion line precedent

## Interaction Contract

### TABS-01 — Create tab (+ button)

- Button: `.sw-tab-strip__add`, text "+" (Unicode U+002B), `aria-label="New tab"`
- Position: after all tabs in the DOM, `margin-left: auto` alignment
- On click: create new TabSlot with `canvasType: 'view'`, `canvasBinding: 'bound'` (D-01); new tab appended to right; new tab becomes active immediately
- No animation on create (tab appears at end; scroll position shifts to reveal new tab via `scrollIntoView({ behavior: 'smooth', block: 'nearest', inline: 'nearest' })`)

### TABS-02 — Close tab (× button)

- Button: `.sw-tab__close`, text "✕" (Unicode U+2715), `aria-label="Close tab"`
- Visibility: `display: none` when only one tab remains (D-06); visible otherwise
- On click: removes tab; adjacent tab activates per browser convention (right → left fallback) (D-04)
- No confirmation dialog (D-05)
- Close button is a child element of the tab button itself; stop `click` propagation on `.sw-tab__close` to prevent triggering tab switch

### TABS-03 — Switch tab (click)

- Clicking anywhere on tab button body (excluding `.sw-tab__close`) calls `switchTab(projection, tabId)`
- No delay, no animation on canvas switch (canvas lifecycle handles it)

### TABS-04 — Active indicator

- `[data-tab-active="true"]` applies `background: var(--accent); color: white`
- Only one tab has `data-tab-active="true"` at any time
- The active tab button also has `aria-selected="true"` and `tabindex="0"`; inactive tabs have `aria-selected="false"` and `tabindex="-1"`

### TABS-05 — Drag reorder

- Drag affordance: Pointer Events API (`pointerdown` / `pointermove` / `pointerup`) on tab button — same API as SuperDynamic axis reorder (D-08)
- Drag threshold: 4px movement before drag mode activates (prevents accidental drag on click)
- Source tab: `data-tab-dragging="true"` applied, opacity drops to `var(--sw-tab-drag-opacity)` = 0.45
- Insertion line: `.sw-tab-insertion-line`, `position: fixed`, `width: 2px`, height spans full tab bar height, `background: var(--accent)`, `box-shadow: 0 0 4px var(--accent)`, `z-index: 10000`, `pointer-events: none` — positioned between tabs at drop target
- Drop: tab order updates in state; DOM re-renders tab list from new order array; insertion line hidden; dragged tab opacity restored
- No FLIP animation (tab count is small; simple re-render is sufficient per D-07 discretion note)
- `setPointerCapture()` called on `pointerdown` target to preserve move events outside element

### TABS-06 — Overflow chevrons

- Container: `[data-slot="tabs"]` becomes a flex row containing left chevron + `.sw-tab-strip` (scroll container) + right chevron + add button
- `.sw-tab-strip`: `overflow-x: scroll; scrollbar-width: none; display: flex; flex: 1 1 auto; min-width: 0`
- Left chevron hidden when `scrollLeft === 0`; right chevron hidden when `scrollLeft + clientWidth >= scrollWidth`
- Chevron click: `scrollBy({ left: ±(tab_width + gap), behavior: 'smooth' })` where tab_width is read from first tab's `offsetWidth` (D-09 discretion: native `scrollBy` with smooth behavior)
- Scroll position listener: `'scroll'` event on `.sw-tab-strip` updates chevron visibility after each scroll
- ResizeObserver on `.sw-tab-strip` recomputes chevron visibility when container width changes
- Chevron buttons: `data-direction="left"` / `data-direction="right"`, `aria-label="Scroll tabs left"` / `"Scroll tabs right"`, `flex-shrink: 0`

### TABS-07 — Keyboard navigation

- Roving tabindex pattern matching ViewTabBar.ts exactly:
  - Active tab: `tabindex="0"`; all other tabs: `tabindex="-1"`
  - `ArrowRight` → next tab (wraps); `ArrowLeft` → previous tab (wraps); `Home` → first tab; `End` → last tab
  - Key handler on `[data-slot="tabs"]` via event delegation (not per-tab listeners)
  - After key-triggered switch: `button.focus()` on the newly activated tab
- `role="tablist"` on `.sw-tab-strip` (the scrolling container, not the outer wrapper)
- Each tab: `role="tab"`, `aria-selected`, `tabindex`
- Canvas area: `role="tabpanel"`, `aria-labelledby="{tab-button-id}"` on `[data-slot="canvas"]`

### TABS-08 — Cmd+W shortcut

- Registers via existing `ShortcutRegistry` using `'Meta+KeyW'` key (discretion: ShortcutRegistry path per CONTEXT.md)
- Fires on active tab only; no-op when only one tab remains
- Same adjacency logic as × button close (D-04)

### TABS-09 — TabSlot type

- `TabSlot` wraps `Projection` and adds: `{ tabId: string; label: string; badge?: string }`
- `label` is the display string in the tab bar (defaults to canvas type label, editable later)
- `badge` is the optional secondary label from `onTabMetadataChange` (TABS-10)
- `TabSlot` does NOT duplicate any fields already on `Projection`; shell reads from the Projection fields via the wrapped reference

### TABS-10 — Metadata callback

- Canvas calls `onTabMetadataChange({ label?: string; badge?: string })` to update the tab bar without triggering canvas re-render
- SuperWidget receives this via the `CanvasComponent` interface extension (CANV-06 preserved — SuperWidget never imports canvas directly)
- Badge renders as `.sw-tab__badge` inline after label, visible only when badge string is non-empty

## Copywriting Contract

| Element | Copy |
|---------|------|
| + button aria-label | "New tab" |
| × button aria-label | "Close tab" |
| Left chevron aria-label | "Scroll tabs left" |
| Right chevron aria-label | "Scroll tabs right" |
| Default new tab label | Canvas type name: "View", "Editor", or "Explorer" — derived from canvasType at creation time |
| Tab badge (TABS-10) | Provided by canvas via onTabMetadataChange — no static default |
| Empty state | Not applicable — last tab cannot be closed; at least one tab always present |
| Error state | Not applicable — tab operations are synchronous pure-state mutations; no async failure path |
| Destructive confirmation | None — close is immediate, no confirmation (D-05) |

Notes:
- No tooltip on tab labels (label is visible text; tooltip adds no value)
- No truncated label tooltip either — overflow is clipped, which is acceptable at 120px max-width given labels are short canvas type names

Source: CONTEXT.md D-05, D-06; REQUIREMENTS.md TABS-01, TABS-02, TABS-10

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Tab list ARIA | `role="tablist"` on `.sw-tab-strip`, `aria-label="Workspace tabs"` |
| Tab item ARIA | `role="tab"`, `aria-selected="true/false"` on each `[data-tab-role="tab"]` |
| Tab panel ARIA | `role="tabpanel"` on `[data-slot="canvas"]`, `aria-labelledby` pointing to active tab id |
| Keyboard nav | Roving tabindex: `ArrowLeft` / `ArrowRight` / `Home` / `End` (matching ViewTabBar.ts) |
| Touch targets | × close button: 16×16px min; chevrons: 28×28px min; tab buttons: 28px min height |
| Focus indicator | `:focus-visible` ring deferred to KBNV-01 (noted in existing superwidget.css TODO comment) |
| Contrast | Active tab: white on `--accent` (#4a9eff) — WCAG AA at 3.3:1; acceptable for large text at 11px bold. Close button `--danger` hover on `--danger-bg` meets 4.5:1 |

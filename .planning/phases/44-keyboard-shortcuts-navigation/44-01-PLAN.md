  <behavior>

    - register('Cmd+Shift+Z', handler, { category: 'Editing', description: 'Redo' }) adds shortcut
    - register('Cmd+1', handler, { category: 'Navigation', description: 'List view' }) adds shortcut
    - Pressing Cmd+Z calls the registered handler (Mac)
    - Pressing Ctrl+Z calls the registered handler (non-Mac)
    - Pressing Cmd+1 calls the registered handler
    - Handler does NOT fire when target is INPUT element
    - Handler does NOT fire when target is TEXTAREA element
    - Handler does NOT fire when target is contentEditable element
    - getAll() returns array of { shortcut, category, description } for all registered shortcuts
    - unregister('Cmd+Z') removes the shortcut
    - destroy() removes the keydown listener and clears all registrations
    - '?' key (no modifier) fires handler for plain key shortcuts
    - Duplicate register for same shortcut replaces the previous handler

  <action>
    Create `src/shortcuts/ShortcutRegistry.ts`:

    ShortcutRegistry class with:
    - Private Map<string, ShortcutEntry> keyed by normalized shortcut string (e.g., 'cmd+1', 'cmd+z', 'cmd+shift+z', '?')
    - Single `document.addEventListener('keydown', handler)` in constructor
    - Built-in input field guard in the single handler (INPUT, TEXTAREA, isContentEditable check)
    - Platform detection: `navigator.platform.includes('Mac')` — metaKey for Mac, ctrlKey for non-Mac. Cmd+X in the shortcut string maps to metaKey on Mac, ctrlKey on non-Mac.
    - `register(shortcut: string, handler: () => void, meta: { category: string; description: string }): void`
    - `unregister(shortcut: string): void`
    - `getAll(): Array<{ shortcut: string; category: string; description: string }>`
    - `destroy(): void` — removes listener, clears map

    Shortcut string parsing:
    - Split by '+', last segment is the key, preceding segments are modifiers
    - Modifiers: 'Cmd' (maps to metaKey/ctrlKey per platform), 'Shift' (shiftKey), 'Alt' (altKey)
    - Plain keys like '?' have no modifier requirement
    - Normalization: lowercase for comparison

    The single keydown handler:
    1. Check input field guard — return early if in INPUT/TEXTAREA/contentEditable
    2. Build the shortcut string from the event (which modifiers are pressed + event.key)
    3. Look up in the Map
    4. If found: preventDefault() and call handler

    Create `src/shortcuts/index.ts` barrel export:
    ```typescript
    export { ShortcutRegistry } from './ShortcutRegistry';
    export type { ShortcutEntry } from './ShortcutRegistry';
    ```

      const displayName = viewType.charAt(0).toUpperCase() + viewType.slice(1);

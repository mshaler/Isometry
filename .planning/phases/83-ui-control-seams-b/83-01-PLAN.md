1. Write 3 cards via `new SQLiteWriter(db).writeCards(cards, false)`, then call `searchCards(db, 'Alpha')` — expect 1 result with matching name.

3. Write a card with content='quantum physics notes'. Search for 'quantum' — expect 1 result.
4. Search for 'zzz_nonexistent_zzz' — expect empty array.

**EFTS-02 describe block (4 tests):**
1. Write 1 card with name='OldProjectName'. Call `writer.updateCards([{...card, name: 'NewProjectName'}])`. Assert `searchCards(db, 'OldProjectName')` returns [] and `searchCards(db, 'NewProjectName')` returns 1 result. NOTE: The FTS update trigger fires on UPDATE OF name — this is the production path.
2. Write 501 cards via `writeCards(cards, true)` (triggers bulk FTS path: disable triggers, rebuild). Then search for one specific card by name — expect 1 result.
3. After the 501-card bulk import, assert cards_fts count matches cards count.
4. Write 2 cards, then soft-delete one (`UPDATE cards SET deleted_at = '...' WHERE id = ?`). searchCards() should return only 1 result (the non-deleted card). Note: The FTS index still has the row, but searchCards() JOIN includes `AND c.deleted_at IS NULL`.

- LATCH morph: d3-transition, 400ms, easeCubicOut, per-card stagger (15ms per card)
- Crossfade: opacity out (300ms) old container → opacity in (300ms) new container
- SVG↔HTML boundary MUST use crossfade (cannot morph SVG g to HTML div)
- List↔Grid: true morph (both SVG)
- List↔Kanban, Grid↔Kanban: crossfade (SVG vs HTML)
- LATCH↔GRAPH: always crossfade (different rendering paradigm)

       - Import type { ViewType, ViewFamily } from '../providers/types'
       - Define SVG_VIEWS set: new Set<ViewType>(['list', 'grid']) — views that render SVG cards
       - Define HTML_VIEWS set: new Set<ViewType>(['kanban']) — views that render HTML cards
       - Define VIEW_FAMILY map or import getViewFamily from PAFVProvider
       - `shouldUseMorph(fromType: ViewType, toType: ViewType): boolean`:
         a. Both must be in SVG_VIEWS (SVG→SVG only for morph)
         b. Both must be in same ViewFamily (LATCH→LATCH)
         c. Return true only if both conditions met
       - `morphTransition(svg: SVGSVGElement, cards: CardDatum[], computePosition: (d: CardDatum, i: number) => string, options?: { duration?: number, stagger?: number }): void`:
         a. Default duration 400, stagger 15
         b. D3 data join: d3.select(svg).selectAll<SVGGElement, CardDatum>('g.card').data(cards, d => d.id)
         c. .join(
              enter => enter.append('g').attr('class','card').attr('opacity',0).attr('transform', computePosition),
              update => update,
              exit => exit.transition().duration(200).attr('opacity',0).remove()
            )
         d. .transition().duration(duration).ease(d3.easeCubicOut).delay((_, i) => i * stagger)
            .attr('transform', computePosition).attr('opacity', 1)
       - `async crossfadeTransition(container: HTMLElement, mountNewView: () => void, duration?: number): Promise<void>`:
         a. Default duration 300
         b. Find existing .view-root child
         c. If exists: d3.select(outgoing).transition().duration(duration).style('opacity','0').on('end', () => outgoing.remove())
         d. Wait for transition to complete (via Promise wrapper)
         e. Create new div.view-root with opacity 0, append to container
         f. Call mountNewView() to populate the new container
         g. d3.select(incoming).transition().duration(duration).style('opacity','1')
    3. Update `src/views/ViewManager.ts` (GREEN phase):
       - Import shouldUseMorph, crossfadeTransition from './transitions'
       - In switchTo(), before destroying current view:
         a. Store previousViewType
         b. If currentView and shouldUseMorph(previousViewType, newViewType):
            - Keep the SVG container alive (don't clear innerHTML)
            - After mounting new view and fetching data, the new view's render() calls morphTransition internally via its own layout computation
            - The morph happens automatically because both views share the same SVG and use d3 data join with key function
         c. Else (crossfade):
            - Wrap old view in opacity-out transition
            - After fade out: destroy old view, clear container
            - Mount new view inside a new .view-root with opacity 0
            - Fade in
       - Store currentViewType to enable transition detection on next switchTo()
       - NOTE: The morph between List↔Grid works naturally because BOTH share the same SVG container and use the same g.card elements with the same key function. When switching, the new view computes new positions and the existing g.card elements animate to those positions. The transition is inherent in the D3 data join + transition pattern, not a separate step.

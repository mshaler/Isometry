## Component Inventory

### Stub Ribbon Row (new component — presentational only)

**DOM structure** — follows `dock-nav.css` class conventions exactly, rendered directly in SuperWidget.ts without a separate DockNav instance:

```
[data-slot="stories-ribbon"]
  div.stub-ribbon                       ← flex row container, 56px height
    ul.stub-ribbon__list                ← horizontal list
      li.stub-ribbon__section           ← section column
        span.stub-ribbon__section-header  ← "STORIES" uppercase label
        ul.stub-ribbon__section-items   ← horizontal items row
          li > button.stub-ribbon__item[disabled] ← each icon+label
            span.stub-ribbon__item-icon ← iconSvg() output
            span.stub-ribbon__item-label ← text label
```

**Whether to reuse `dock-nav.css` classes or add a `stub-ribbon.css`:**
Use a separate `src/styles/stub-ribbon.css`. Reasoning: stub rows must override `.dock-nav__item` hover and active states with permanent disabled treatment. Adding overrides to `dock-nav.css` would pollute the live nav ribbon. A separate file avoids specificity conflicts and keeps the nav ribbon styles untouched. The file should import `dock-nav.css`-matching token references rather than duplicate values.

**Whether to use a data-definition array:**
Yes — define a `STUB_RIBBON_DEFS` array in `section-defs.ts` following the `DockSectionDef` interface. This keeps icon/label choices in one place and mirrors the `DOCK_DEFS` pattern. SuperWidget.ts renders from the array, not hardcoded HTML strings.

Source: CONTEXT.md "Claude's Discretion" items 2, 3

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Mouse hover over stub item | No visual change — `pointer-events: none` blocks hover entirely |
| Click on stub item | No event fired — `pointer-events: none` and no event delegation on stub rows |
| Keyboard focus | `button[disabled]` is not focusable — no keyboard nav path into stub rows |
| Screen reader | `button` elements carry `disabled` attribute; AT announces as "dimmed/unavailable" |

No `aria-disabled` pattern needed — use native `disabled` attribute on `<button>` elements. Native `disabled` removes from tab order and announces correctly across NVDA/VoiceOver/TalkBack.

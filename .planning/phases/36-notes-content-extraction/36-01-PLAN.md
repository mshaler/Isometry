**Step 3 — Create `GzipDecompressor.swift`.**

Utility enum with a single static method:

```swift
import Foundation
import zlib

enum GzipDecompressor {
    enum GzipError: Error {
        case decompressFailed(Int32)
        case emptyInput
    }

    static func decompress(_ data: Data) throws -> Data
}
```

Implementation details:
- Guard against empty input (throw `.emptyInput`)
- Use `inflateInit2_(&stream, MAX_WBITS + 32, ZLIB_VERSION, Int32(MemoryLayout<z_stream>.size))` — the `+32` enables automatic gzip header detection. This is critical because ZDATA uses gzip format (magic bytes 0x1F 0x8B), NOT raw zlib/deflate. Using Foundation's `.zlib` decompression will fail.
- Loop with 16KB output chunks, appending to a Data buffer
- Call `inflateEnd` in all paths (success and error)
- Guard final status is `Z_STREAM_END`
- See 36-RESEARCH.md "Gzip Decompression" code example for verified implementation

Create `ProtobufToMarkdown.swift` — the AttributeRun walker that converts a protobuf Note to Markdown.

**Public interface:**

```swift
import Foundation

enum ProtobufToMarkdown {
    struct ConversionResult {
        let body: String              // Full Markdown body text
        let summary: String           // ~200-char auto-generated summary
        let tags: [String]            // Hashtags extracted from full body text
        let attachments: [AttachmentRef]   // Attachment metadata for ## Attachments section
        let noteLinks: [NoteLink]     // Note-to-note link targets for connection creation
        let isSnippetFallback: Bool   // true if fell back to ZSNIPPET
    }

    struct AttachmentRef {
        let identifier: String        // ZIDENTIFIER for DB lookup
        let typeUti: String?          // UTI from protobuf AttachmentInfo
        let humanType: String         // "Image", "Drawing", "PDF", etc.
        let filename: String?         // Resolved from DB (may be nil until Plan 02 wires lookup)
    }

    struct NoteLink {
        let targetIdentifier: String  // ZIDENTIFIER of linked note
        let displayName: String       // Text of the link in the note
    }

    /// Full three-tier extraction (per CONTEXT.md locked decision):
    /// Tier 1: Full Markdown reconstruction from protobuf
    /// Tier 2: Plain text + hashtags if Markdown reconstruction fails but decompression succeeded
    /// Tier 3: ZSNIPPET fallback if decompression itself fails
    static func extract(
        zdata: Data?,
        snippet: String?,
        attachmentLookup: [String: (filename: String?, typeUti: String?)]
    ) -> ConversionResult
}
```

**Markdown Reconstruction (Tier 1) — per locked decisions:**

Walk `note.attributeRun` entries against `note.noteText` using Unicode scalar indexing (Pitfall 3 from research: `attributeRun.length` counts Unicode scalars, not grapheme clusters).

For each run:
1. Extract the text slice from `noteText` using `String.UnicodeScalarView` for correct indexing
2. Check for U+FFFC (Object Replacement Character) — if found and `run.hasAttachmentInfo`, generate inline placeholder `[Type: filename]` using the attachment lookup map. Map UTI to human-friendly type using `humanFriendlyType()`.
3. Check for note-to-note links via `run.link`:
   - If link starts with `applenotes:` or `notes://` or `x-coredata://`, extract the identifier
   - Parse URL patterns: `applenotes:note/{identifier}`, `notes://showNote?identifier={UUID}`, `x-coredata://.../ICNote/{identifier}`
   - Record in `noteLinks` array and render as `[Linked Note: {display text}]` per locked decision
   - External URLs (https://, http://) render as `[text](url)` Markdown links
4. Apply paragraph-level formatting based on `run.paragraphStyle.styleType`:
   - `-1` (default/body): no prefix
   - `0` (Title): `# ` prefix per locked decision
   - `1` (Heading): `## ` prefix
   - `2` (Subheading): `### ` prefix
   - `4` (Monospaced): wrap in ``` fenced code block
   - `100` (dotted list): `- ` with indent based on `indentAmount`
   - `101` (dashed list): `- ` with indent
   - `102` (numbered list): `1. ` with indent
   - `103` (checkbox): `- [x] ` or `- [ ] ` based on `checklist.done`
   - `blockQuote` non-zero: `> ` prefix per locked decision
5. Apply inline formatting:
   - `fontWeight == 1` or `fontWeight == 3`: wrap in `**bold**`
   - `fontWeight == 2` or `fontWeight == 3`: wrap in `*italic*`
   - `strikethrough != 0`: wrap in `~~strikethrough~~`
6. Handle paragraph boundaries: when run text ends with `\n`, close current paragraph and start new line. Apply paragraph prefix to the NEXT line's content.

**Post-processing:**
- Append `## Attachments` section at end with `- [Type] filename` entries (per locked decision). Cross-reference: each inline `[Image: sunset.jpg]` matches `- [Image] sunset.jpg` in the Attachments section.
- Extract hashtags from the FULL body text using the same regex pattern as NotesAdapter.extractHashtags() — `#(\w+)`. Per locked decision: re-extract from full body, not just ZSNIPPET.
- Add bare type name tags for filterability (per locked decision): if note has image attachments, add "image" tag; if PDF, add "pdf" tag; etc.
- Generate ~200-char summary: take first 200 characters of plain text (strip Markdown syntax), trimmed at word boundary. Same pattern as AppleNotesParser.ts generateSummary.

public.image / public.jpeg / public.png / public.heic -> "Image"
com.apple.drawing.2 / com.apple.drawing -> "Drawing"

```

**Three-tier fallback (BODY-02):**
- Tier 1: Full Markdown reconstruction (as above)
- Tier 2: If Markdown reconstruction throws but decompression succeeded, extract `noteText` as plain text + extract hashtags. Set `isSnippetFallback = false` (still has real content, just no formatting).
- Tier 3: If decompression itself fails or ZDATA is nil, use ZSNIPPET. Set `isSnippetFallback = true`.
- Per locked decision: maximize extraction on partial failure.

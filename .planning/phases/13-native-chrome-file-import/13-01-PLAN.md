private func switchView(to viewName: String) {

<task type="auto">
  <name>Task 2: Add macOS Commands struct for File and Edit menus</name>
  <files>native/Isometry/Isometry/IsometryApp.swift</files>
  <action>
Add a `.commands` modifier to the WindowGroup scene in IsometryApp, guarded by `#if os(macOS)`.

**Create IsometryCommands struct** (inside IsometryApp.swift, after the IsometryAppDelegate class):

```swift
#if os(macOS)
struct IsometryCommands: Commands {
    var body: some Commands {
        // File menu: Import File (Cmd+I)
        CommandGroup(after: .newItem) {
            Button("Import File...") {
                NotificationCenter.default.post(name: .importFile, object: nil)
            }
            .keyboardShortcut("I", modifiers: .command)
        }

        // Edit menu: Undo/Redo (Cmd+Z / Cmd+Shift+Z)
        CommandGroup(replacing: .undoRedo) {
            Button("Undo") {
                NotificationCenter.default.post(name: .undoAction, object: nil)
            }
            .keyboardShortcut("Z", modifiers: .command)

            Button("Redo") {
                NotificationCenter.default.post(name: .redoAction, object: nil)
            }
            .keyboardShortcut("Z", modifiers: [.command, .shift])
        }
    }
}
#endif
```

**Modify IsometryApp.body** to include the commands modifier:

Change:
```swift
WindowGroup {
    ContentView(bridgeManager: bridgeManager)
        .onAppear { ... }
}
.onChange(of: scenePhase) { ... }
```

To:
```swift
WindowGroup {
    ContentView(bridgeManager: bridgeManager)
        .onAppear { ... }
}
#if os(macOS)
.commands {
    IsometryCommands()
}
#endif
.onChange(of: scenePhase) { ... }
```

**Note:** The Notification.Name extensions (.importFile, .undoAction, .redoAction) are defined in ContentView.swift (Task 1). The Commands struct uses them — they are visible because both files are in the same module.

**Undo/Redo approach:** The web runtime already has a full MutationManager with undo/redo stacks and keyboard shortcut handlers (Cmd+Z, Cmd+Shift+Z) registered via `registerShortcuts()` in shortcuts.ts. However, macOS Commands struct captures Cmd+Z before it reaches WKWebView. By replacing the `.undoRedo` command group and routing to `mutationManager.undo()` via evaluateJavaScript, we ensure the shortcut works correctly. The NotificationCenter approach from Commands -> ContentView -> evaluateJavaScript handles this.

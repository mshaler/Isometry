  <files>native/Isometry/Isometry/DatabaseManager.swift, native/Isometry/Isometry/Isometry.entitlements</files>
  <action>
Modify DatabaseManager.swift with these specific changes:

1. **Remove `useCoordinator` property** entirely -- Application Support never needs file coordination.

2. **Remove `resolveStorageDirectory()` static method** entirely -- no more iCloud ubiquity container preference.

3. **Replace `autoMigrateIfNeeded(to:)` with `migrateFromUbiquityIfNeeded(to:)`** -- reverse direction. New implementation:
   - Takes `appSupportDir: URL` parameter (the Application Support/Isometry/ directory)
   - Check if `appSupportDir/isometry.db` already exists -- if yes, skip (already migrated or never used iCloud)
   - Call `FileManager.default.url(forUbiquityContainerIdentifier: nil)` to find iCloud container
   - If iCloud container exists and `containerURL/Isometry/isometry.db` exists, copy it to `appSupportDir/isometry.db`
   - Non-destructive: do NOT delete the iCloud copy
   - Log success/failure via os.Logger
   - This method MUST run on a background thread (existing `Task.detached` pattern from current `makeForProduction`)

4. **Simplify `makeForProduction()` async factory**:
   - Always resolve to Application Support/Isometry/ (no more `resolveStorageDirectory()` call)
   - Run `migrateFromUbiquityIfNeeded(to:)` via `Task.detached` on background thread (Pitfall 3: `url(forUbiquityContainerIdentifier:)` blocks on IPC)
   - Return `try DatabaseManager()` (the production init already uses Application Support)

5. **Remove `saveCheckpointCoordinated(_:)` method** entirely -- NSFileCoordinator is no longer needed.

6. **Simplify `saveCheckpoint(_:)`** -- remove the `if useCoordinator` branch, always call `saveCheckpointDirect(_:)` directly. Inline the direct save logic into `saveCheckpoint()` and remove the now-unnecessary `saveCheckpointDirect` private method (or keep it as the sole implementation -- executor's choice for clarity).

7. **Remove `init(baseDirectory:)` iCloud detection logic** -- the `useCoordinator` detection (`path.contains("Mobile Documents")`) is no longer needed. Keep the `init(baseDirectory:)` for tests but set it to never use coordinator (since the property is removed, this simplifies to just setting paths).

8. **Remove `import Foundation` NSFileCoordinator usage** -- ensure no NSFileCoordinator references remain.

Then modify Isometry.entitlements:
- Replace `CloudDocuments` with `CloudKit` in `com.apple.developer.icloud-services`
- Replace `com.apple.developer.ubiquity-container-identifiers` key with `com.apple.developer.icloud-container-identifiers` key, keeping value `iCloud.works.isometry.Isometry`
- Add `aps-environment` key with value `development` (required for CKSyncEngine push notification handling)

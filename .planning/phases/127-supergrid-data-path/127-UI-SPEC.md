## Spacing Scale

Declared values from `src/styles/design-tokens.css` (theme-independent `:root` block):

| Token | Value | Usage |
|-------|-------|-------|
| --space-xs | 4px | Icon gaps, inline padding, cell internal spacing |
| --space-sm | 8px | Compact element spacing, empty state gap |
| --space-md | 12px | Section padding, error banner padding |
| --space-lg | 16px | Default element spacing, toolbar gaps |
| --space-xl | 24px | Empty state panel padding, major gaps |

**Source:** `design-tokens.css` `:root` spacing block (pre-populated, not re-specified).

Exceptions:
- SuperGrid cell padding spreadsheet mode: `--sg-cell-padding-spreadsheet: 4px` = `--space-xs` (sub-grid exception, multiples-of-4 compliant)
- SuperGrid cell padding matrix mode: `--sg-cell-padding-matrix: 8px` = `--space-sm` (sub-grid exception, multiples-of-4 compliant)
- SuperGrid row height: `--sg-row-height: 40px` default (grid structural token)
- Touch targets: not applicable (macOS desktop app inside WKWebView)

## Typography

All sizes from `src/styles/design-tokens.css` theme-independent `:root` typography scale:

| Role | Token | Size | Weight | Line Height |
|------|-------|------|--------|-------------|
| Body | --text-base | 13px | 400 (regular) | 1.5 |
| Label / Cell | --text-sm | 11px | 400 (regular) | 1.4 |
| UI element / description | --text-md | 14px | 400 (regular) | 1.5 |
| Heading / empty state / column + row headers | --text-xl | 18px | 700 (bold) | 1.2 |

**Weights used in this phase:** regular (400) for cell data and labels; bold (700) for column/row headers (`.sg-header`) and empty state heading.

SuperGrid cell font: `var(--sg-cell-font-size)` = `var(--text-sm)` (11px), scaled by `var(--sg-zoom, 1)`.

SuperGrid header font: `font-weight: bold` per `.sg-header` (weight 700 — existing rule, not a new addition).

**Source:** `design-tokens.css` typography scale + `supergrid.css` `.sg-header` rule (pre-populated).

## Color

All tokens from `src/styles/design-tokens.css`. Dark theme (default) values shown; all tokens resolve per `[data-theme]` at runtime.

| Role | Token | Dark Value | Light Value | Usage |
|------|-------|-----------|-------------|-------|
| Dominant (60%) | --bg-primary | #1a1a2e | #ffffff | Grid background surface, main view area |
| Secondary (30%) | --bg-surface / --sg-header-bg | #252540 | #eeeeef | Column headers, row headers, corner cell |
| Accent (10%) | --accent | #4a9eff | #2563eb | Active cell ring, selection border, retry button border |
| Destructive | --danger | #ff4a4a | #dc2626 | Error banner background tint only |

**Primary focal point:** the pivot grid cell area — the central data surface where row/column intersections render card counts and chips. All other UI elements (headers, footer rows, empty state, error banner) are secondary to this area.

**Accent reserved for:**
- Active cell focus ring (`--sg-active-ring-shadow`)
- Selected cell/row outline (`--sg-selection-border`)
- Retry button border and text color (`.retry-btn`)
- CalcExplorer aggregate type selector — focus state only

**Derived tokens used in this phase:**
- `--sg-cell-alt-bg` — alternating row tint (every other row in spreadsheet mode)
- `--sg-gridline` = `--border-subtle` — all cell borders
- `--sg-selection-bg` — selected cell fill
- `--danger-bg` / `--danger-border` — error banner background and border
- `--cell-hover` — cell hover state fill
- `--cell-empty-bg` — card chip background inside occupied cells

**Source:** `design-tokens.css` full token definitions (pre-populated).

## Interaction States

Each cell in the PivotGrid must cycle through exactly these states:

| State | Visual Indicator | Token |
|-------|-----------------|-------|
| Default | No highlight | --bg-primary background |
| Alternate row | Subtle tint | --sg-cell-alt-bg |
| Hover | Light fill | --cell-hover |
| Active / focused | 1px inset ring | --sg-active-ring-shadow + --sg-active-crosshair-bg |
| Selected | Outline + tinted fill | --sg-selection-border (2px solid) + --sg-selection-bg |
| Empty cell (no cards) | Dimmed italic chip | .supergrid-card with --cell-empty-bg fill |
| Occupied cell | Count + card chip(s) | .supergrid-card with --border-muted dashed border |

Footer aggregate rows (SGRD-03):
- Rendered below last data row, separated by a 2px solid `--border-muted` top border
- Font: `--font-mono` (`--sg-number-font`), right-aligned, weight 400
- Label column: aggregate function name in `--text-muted` color at 11px

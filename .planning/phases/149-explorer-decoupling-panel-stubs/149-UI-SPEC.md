All values from `design-tokens.css` — use tokens, not raw px.

| Role | Token | Size | Weight | Line Height |
|------|-------|------|--------|-------------|
| Body | --text-base | 13px | 400 | 1.5 |
| Label | --text-sm | 11px | 600 | 1.4 |
| Section header | --text-xs | 10px | 600 | 1.0 (single line, uppercase) |

Visual distinction for section headers comes from uppercase transform and 10px size — not from weight. Two weights: 400 (body) and 600 (all non-body roles).

## Color

All values are CSS custom property references — they adapt automatically across all 5 themes (dark, light, system, nextstep, material). Never use raw hex in new code.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | --bg-primary | Workbench background, view-content area |
| Secondary (30%) | --bg-surface | DockNav column, PanelDrawer surface, panel icon strip |
| Muted surface | --bg-secondary | DataExplorer area background |
| Accent (10%) | --accent / --accent-bg | Active dock item highlight, open panel icon state |
| Destructive | --danger | Not applicable in this phase |

Accent reserved for:
- Active/selected dock item (`.dock-nav__item--active` — accent background fill)

Text:
- `--text-primary` — CollapsibleSection headers, placeholder heading
- `--text-secondary` — Dock item labels (inactive), panel icon buttons (inactive)
- `--text-muted` — Dock section headers (uppercase caps), "Coming soon" body copy

## Layout Contract

### PanelDrawer Relocation (DCPL-01, DCPL-02)

The existing `.workbench-panel-tray` container (currently `display: none`) is replaced with a visible side drawer inside `.workbench-main`.

New `.workbench-main` flex row order (left to right):
1. `.panel-drawer` — collapsible side panel (default 300px, 0px when closed)
2. `.panel-drawer__resize-handle` — 8px drag target (hidden when closed)
3. `.workbench-view-content` — flex: 1 1 auto, takes remaining width

`.workbench-main` direction stays `flex-row` (change from `flex-column`).

The `.workbench-data-explorer` special case remains a `flex-column` item above `.workbench-view-content` — it is NOT folded into PanelDrawer this phase.

### PanelDrawer Icon Strip

The `.panel-icon-strip` is hidden via `display: none` (not deleted). DockNav is now the sole activation surface. Preserve the element in DOM for iteration flexibility.

### Coming Soon Placeholder Panels (STUB-01, STUB-02, STUB-03)

Each stub (Maps, Formulas, Stories) renders a centered placeholder `div` inside a `PanelHook`-mounted container. Placeholder layout:

```
[Lucide icon, 32×32, color: --text-muted]
[Heading: "Maps" / "Formulas" / "Stories",  14px, 600, --text-primary]
[Body: "Coming soon",  13px, 400, --text-muted]
```

Centered with flexbox: `align-items: center; justify-content: center; flex-direction: column; gap: 8px; height: 100%; padding: 24px`.

## Interaction Contract

### Dock Item Click → Panel Toggle (D-02 from CONTEXT.md)

| Interaction | Behavior |
|-------------|----------|
| Click dock item mapped to an explorer (e.g., Properties) | Calls `PanelRegistry.enable()` / `disable()` for that panel; PanelDrawer opens if any panel enabled, closes when all disabled |
| Click same dock item again when panel is open | Calls `PanelRegistry.disable()` — panel closes, drawer closes if it was the last open panel |
| Click Maps / Formulas / Stories dock item | Activates stub panel — PanelDrawer opens showing the "Coming soon" placeholder |
| Click dock item that activates a view (SuperGrid, Timeline, etc.) | No panel toggle — `onActivateItem` routes to view switch as before |

### PanelDrawer Open/Close Animation

Use existing CSS transition: `width var(--transition-normal) ease` (300ms). Transition from `width: 0` to `width: var(--panel-drawer-width, 300px)`. No JS animation — CSS only.

### Scroll-to-Section on Open

When a panel is enabled via dock click and the drawer is already open, scroll the activated section into view using `scrollIntoView({ behavior: 'smooth', block: 'start' })` on the CollapsibleSection header element.

### Focus Management

After panel activation via dock click, focus moves to the CollapsibleSection header of the newly opened panel (allows immediate keyboard interaction). Use `element.focus()` with `tabIndex: -1` if the header is not natively focusable.

## Accessibility Contract

- All stub panel placeholders must have `role="status"` on the "Coming soon" text for VoiceOver discoverability.
- Dock items for Maps, Formulas, Stories use the same ARIA pattern as existing items: `role="tab"`, `aria-selected`, `aria-label` matching the item label.
- PanelDrawer open/closed state announced via existing `aria-expanded` on toggle trigger (DockNav item already has this pattern from Phase 147).
- Focus-visible ring: 2px solid `--accent`, `outline-offset: -2px` — matches existing pattern on all interactive elements.
- New stub DOCK_DEFS entries must include a `label` and Lucide `icon` key consistent with existing DOCK_DEFS shape.

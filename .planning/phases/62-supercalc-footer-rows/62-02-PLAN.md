   - Define `CalcConfig` interface: `{ columns: Record<string, AggregationMode | 'off'> }`
   - Define `NUMERIC_FIELDS` as ReadonlySet<string> containing: `'priority'`, `'sort_order'`
   - All other AxisField values are text fields (name, folder, status, card_type, created_at, modified_at, due_at)
   - Constructor takes config: `{ bridge: WorkerBridge; pafv: PAFVProvider; container: HTMLElement; onConfigChange: (config: CalcConfig) => void }`
   - `mount()`:
     - Load persisted config from `bridge.send('ui:get', { key: 'calc:config' })` -- parse JSON or use empty defaults
     - Subscribe to PAFVProvider changes to rebuild dropdown list when axes change
     - Call `_render()` to build initial UI
   - `_render()`:
     - Clear container
     - Get current axes from `pafv.getStackedGroupBySQL()` -- extract unique fields from colAxes + rowAxes
     - For each axis field, create a row: `<div class="calc-row"><label>{displayName}</label><select>{options}</select></div>`
     - Numeric fields: options = SUM, AVG, COUNT, MIN, MAX, OFF (default: SUM)
     - Text fields: options = COUNT, OFF (default: COUNT)
     - Set select value from persisted config (or default if no config for that field)
     - On change: update internal config, call `_persist()`, call `onConfigChange` callback
   - `_persist()`: debounced (300ms) `bridge.send('ui:set', { key: 'calc:config', value: JSON.stringify(config) })`
   - `getConfig()`: returns current CalcConfig (for SuperGrid to read)
   - `destroy()`: unsubscribe from PAFVProvider, clear container
   - Export `CalcConfig` type for consumers

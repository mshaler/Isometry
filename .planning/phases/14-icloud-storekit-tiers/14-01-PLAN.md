**1. Create Isometry.entitlements:**

Create a new entitlements plist file at `native/Isometry/Isometry/Isometry.entitlements` with:
- `com.apple.developer.icloud-services` array containing `CloudDocuments`
- `com.apple.developer.ubiquity-container-identifiers` array containing `iCloud.works.isometry.Isometry`

Per CONTEXT.md decision: database file is hidden from iOS Files app -- do NOT add `NSUbiquitousContainers` to Info.plist. The file is stored in the ubiquity container root (NOT the `Documents/` subfolder).

**2. Add resolveStorageDirectory() and async factory to DatabaseManager:**

Add a `static func resolveStorageDirectory() -> URL` method that:
- Calls `FileManager.default.url(forUbiquityContainerIdentifier: nil)` -- returns the container root
- If available: uses `containerURL.appendingPathComponent("Isometry", isDirectory: true)` (NOT "Documents/Isometry" -- per CONTEXT.md the file is hidden from Files app)
- Creates the directory with `createDirectory(at:withIntermediateDirectories:true)`
- If nil (iCloud unavailable): falls back to `Application Support/Isometry/` (existing behavior)
- This method MUST NOT be called from the main thread (per Pitfall 1 from RESEARCH.md)

Add `static func makeForProduction() async throws -> DatabaseManager`:
- Uses `Task.detached(priority: .userInitiated)` to call `resolveStorageDirectory()` on a background thread
- Calls `autoMigrateIfNeeded(to:)` (see below) before creating the manager
- Returns `try DatabaseManager(baseDirectory: resolvedDir)`

**3. Add auto-migration from local to iCloud:**

Add `private static func autoMigrateIfNeeded(to icloudDir: URL)`:
- Check if `icloudDir` is in a ubiquity container (path contains "Mobile Documents" or "CloudDocs")
- If so, check if a local `Application Support/Isometry/isometry.db` exists
- If local db exists AND iCloud `isometry.db` does NOT exist: copy local to iCloud via `FileManager.copyItem(at:to:)` wrapped in NSFileCoordinator
- Log the migration result
- Do NOT delete the local file after migration (safety net)
- Per CONTEXT.md: "Auto-migration from local to iCloud should be seamless -- user enables iCloud and their data appears on other devices without any manual step"

**4. Wrap saveCheckpoint in NSFileCoordinator when using iCloud path:**

Add a private property `private let useCoordinator: Bool` set in `init(baseDirectory:)`:
- Detect if the directory is in a ubiquity container (path contains "Mobile Documents" or similar iCloud indicator)
- When `useCoordinator` is true, wrap the entire saveCheckpoint atomic write sequence (write .tmp, rotate .db to .bak, rename .tmp to .db) inside a single `NSFileCoordinator(filePresenter: nil).coordinate(writingItemAt: dbURL, options: .forReplacing)` block
- Per RESEARCH.md Pitfall 8: all intermediate operations (.tmp, .bak) must be inside the same coordination block to prevent the sync daemon from observing an inconsistent state
- When `useCoordinator` is false (local storage), keep the existing direct write pattern unchanged (no NSFileCoordinator overhead)
- NSFileCoordinator coordination blocks run synchronously on the calling thread -- since DatabaseManager is an actor, this is fine (actor serializes access, never main thread)

Keep the existing synchronous `init()` and `init(baseDirectory:)` working for tests and backward compat. The synchronous `init()` should set `useCoordinator = false` (local Application Support path).

Do NOT add `import StoreKit` -- this plan is purely iCloud Documents.

**1. Update ContentView.setupWebView() to use async factory:**

Replace the synchronous DatabaseManager initialization:
```swift
// OLD:
bridgeManager.databaseManager = try DatabaseManager()
```

With an async call:
```swift
// NEW:
Task {
    do {
        bridgeManager.databaseManager = try await DatabaseManager.makeForProduction()
    } catch {
        print("[Isometry] Failed to initialize DatabaseManager: \(error)")
        // App still functions without persistence -- data will not survive relaunch
    }
}
```

IMPORTANT: The WebView load must NOT wait for DatabaseManager initialization. The web app loads immediately; when JS sends `native:ready`, `sendLaunchPayload()` already handles `databaseManager == nil` (sends null dbData). The DatabaseManager will be available before any checkpoint save is needed (async factory completes in <2s, first checkpoint is at 30s). This is the correct ordering -- do NOT block WebView loading on iCloud resolution.

Remove the existing synchronous `do { try DatabaseManager() } catch` block. The async Task replaces it.

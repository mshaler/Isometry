**Bug 1 fix: Same-dimension reorder bypass.**
In `_handlePointerDrop` (line ~4812), after the drop zone hit-test loop (lines 4836-4868), BEFORE the `if (!targetDimension)` early return at line 4869:

Add a fallback: if `targetDimension` is still null AND `_dragPayload` exists AND `_lastReorderTargetIndex >= 0`, then set `targetDimension = _dragPayload.sourceDimension`. This allows same-dimension reorder to proceed without requiring the pointer to be over a drop zone. The `_lastReorderTargetIndex` was already correctly calculated during pointermove midpoint calculations.

The modified logic should be:
```typescript
if (!targetDimension && _dragPayload && this._lastReorderTargetIndex >= 0) {
    // Same-dimension reorder: pointer is in header area, not over a drop zone.
    // _lastReorderTargetIndex was set during pointermove midpoint calculations.
    targetDimension = _dragPayload.sourceDimension;
}
```

Place this AFTER the hit-test loops but BEFORE the existing `if (!targetDimension)` early return.

Note: read `_dragPayload` from the local `payload` variable (which was already extracted at line 4828) — `payload` not `_dragPayload` since `_dragPayload` was set to null at line 4829. Actually re-check: `_dragPayload` is nulled at line 4829 and `payload` is the local copy. Use `payload` for the check since `_dragPayload` is already null by this point.

Corrected:
```typescript
if (!targetDimension && payload && this._lastReorderTargetIndex >= 0) {
    targetDimension = payload.sourceDimension;
}
```

specifies token names, not raw hex values, so all three themes (dark/light/NeXTSTEP/Material) inherit

| Role | Token | Usage |

## Interaction Contract

This phase is primarily an interaction migration (HTML5 DnD → pointer events). The following contracts
are binding for all four DnD surfaces.

### Canonical Pointer-DnD Pattern (from Phase 95 — PROJ-02/PROJ-03)

Every DnD surface in this phase MUST implement this pattern identically:

```
pointerdown  → setPointerCapture(e.pointerId)
               create ghost element (clone of source, fixed position, pointer-events:none, z-index:9999)
               add .dragging class to source (opacity: 0.4)
               store module-level drag state

pointermove  → update ghost position: left = clientX - ghostWidth/2, top = clientY - 12px
               hit-test drop zones via getBoundingClientRect()
               add .drag-over (SuperGrid/Kanban) or --dragover modifier (ProjectionExplorer) to valid target
               cursor reflects grab state (set cursor:grabbing on body during drag)

pointerup    → commit or cancel the mutation
               ghost.remove()
               source.classList.remove('dragging')
               clear drop zone highlights
               release pointer capture
```

### Surface-Specific Interaction Contracts

**DND-01: SuperGrid row axis grip reorder**
- Trigger element: existing `⋮` grip in `.row-header` cells (cursor: grab)
- Drag type: within row dimension only (reorder within rows)
- Drop target: the `.row-drop-zone` overlay on the row header strip
- Visual feedback: midpoint calculation determines insertion slot (existing `_lastReorderTargetIndex` logic must be preserved)
- Drop zone highlight: add `.drag-over` class to the row-header drop zone element
- On drop: `PAFVProvider.setRowAxes()` with reordered array — existing mutation path

**DND-02: SuperGrid cross-dimension transpose (row ↔ col)**
- Trigger element: same `⋮` grip (shared with DND-01); sourceDimension tag distinguishes
- Drag type: row grip dropped on col drop zone or vice versa
- Drop targets: col-header strip (for row→col) and row-header strip (for col→row)
- Visual feedback: destination strip adds `.drag-over`; source header dims to opacity 0.4
- On drop: `PAFVProvider.setRowAxes()` + `PAFVProvider.setColAxes()` with field moved across
- Guard: single-axis source dimension blocks transpose (preserve existing guard)

**DND-03: KanbanView card drag between columns**
- Trigger element: entire `.card` element in kanban column body (cursor: grab)
- Drop target: `.kanban-column__body` (column body container)
- Visual feedback: dragging card dims to 0.4 opacity; destination column body adds `.drag-over`
- On drop: existing `onMutation(cardId, targetColumnValue)` callback — fires undoable MutationManager mutation
- Ghost: card-shaped ghost (clone of card element) follows cursor

**DND-04: DataExplorerPanel file drop**
- Primary interaction: pointer events on `.dexp-drop-zone` (replace HTML5 drop events)
- WKWebView constraint: native file drag may not reach the drop zone; fallback is mandatory
- Fallback CTA: "Import File" button (`.dexp-import-btn`) — already exists, triggers `<input type="file">`
- Drop zone state: idle = `.dexp-drop-zone` default; active (file over zone) = `.dexp-drop-zone--active`
- Note: `.dexp-drop-zone--active` applies `var(--drag-over-bg)` background + `var(--accent)` border (existing CSS already handles this)
- If pointer-based file drop is not viable in WKWebView context, the click-to-browse fallback satisfies DND-04 and must be clearly labelled

**DND-05: Ghost element standard (applies to all four surfaces)**
- Ghost is a `position:fixed` clone of the drag source
- `pointer-events: none` on ghost
- `z-index: 9999` on ghost
- `opacity: 0.8`
- `transform: scale(1.05)` (subtle lift)
- `width` pinned to source element width at drag start
- Positioned at `clientX - width/2, clientY - 12px` and updated on every pointermove
- Ghost appended to `document.body`; removed on pointerup/pointercancel
- Cursor: set `document.body.style.cursor = 'grabbing'` on pointerdown; restore on pointerup

## Phase-Specific Notes

### Why no shadcn

This project is a pure TypeScript + D3 + CSS custom properties stack. No React, no component library,
no Tailwind. The design system is entirely in `src/styles/design-tokens.css` and scoped CSS files.
Adding shadcn would introduce incompatible dependencies. All design tokens are already fully specified.

### Pointer-DnD vs HTML5 DnD — visual parity rule

The pointer-event migration must preserve visual parity with the existing HTML5 DnD behavior:
- `.dragging` opacity (0.4) remains unchanged on source elements
- `.drag-over` outline (1px solid `var(--accent)`) remains unchanged on drop targets
- New ghost element is the ONLY addition to the existing visual vocabulary

### Module-level drag state pattern

Each surface keeps its drag state in module-level variables (not instance state), consistent with
the existing `_dragPayload` singleton in `SuperGrid.ts` and the `_ghostEl/_dragField` pattern in
`ProjectionExplorer.ts`. This is a locked constraint per STATE.md.

### Transition timing

Ghost position updates on `pointermove` with no transition (immediate follow). Drop zone highlight
uses the existing `transition: 0.15s` on `.dexp-drop-zone` from `data-explorer.css`.

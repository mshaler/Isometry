Complete the per-tab zoom persistence by restoring saved zoom when activating a tab.

Current behavior (handleTabSwitch lines 190-196):
- Saves current zoom before switching: tabZoomRef.current[activeTab] = zoom
- Switches tab: setActiveTab(tab)
- Does NOT restore saved zoom for the new tab

Fix approach (using usePreviewSettings from Task 2):
1. In handleTabSwitch, after calling setActiveTab(tab):
2. Read the saved zoom for the new tab: const savedZoom = getTabConfig(tab).zoomLevel
3. If savedZoom exists and differs from current zoom, call setZoom(savedZoom)
4. The setZoom comes from useWebPreview hook (already imported)

Edge cases:
- If savedZoom is undefined/null, keep current zoom (don't reset to 100)
- web-preview tab uses its own zoom from useWebPreview; only restore for d3 tabs (timeline, network, d3-visualization)
- Only restore if the tab has a numeric zoomLevel saved

Implementation:
```typescript
const handleTabSwitch = useCallback((tab: PreviewTab) => {
  // Save current zoom before switching
  if (activeTab !== 'web-preview') {
    setTabZoom(activeTab, zoom);
  }
  setActiveTab(tab);

  // Restore saved zoom for the new tab
  if (tab !== 'web-preview') {
    const savedZoom = getTabConfig(tab).zoomLevel;
    if (typeof savedZoom === 'number' && savedZoom !== zoom) {
      setZoom(savedZoom);
    }
  }
}, [activeTab, zoom, setActiveTab, setTabZoom, getTabConfig, setZoom]);
```

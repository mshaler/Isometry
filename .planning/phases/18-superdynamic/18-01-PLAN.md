    **Step 1: Extend SuperGridProviderLike** in src/views/types.ts — add `setColAxes(axes: AxisMapping[]): void` and `setRowAxes(axes: AxisMapping[]): void` to the interface. PAFVProvider already implements both methods; this just widens the narrow interface contract.

    **Step 2: Add AxisDragPayload type and _dragPayload singleton** at module level in SuperGrid.ts (OUTSIDE the class):
    ```typescript
    interface AxisDragPayload {
      field: string;
      sourceDimension: 'col' | 'row';
      sourceIndex: number;
    }
    let _dragPayload: AxisDragPayload | null = null;
    ```
    The singleton is a STATE.md locked constraint: "HTML5 DnD dragPayload MUST be a module-level singleton."

    **Step 3: Add grip handles to column headers** in `_createColHeaderCell()`. Create a `<span class="axis-grip" draggable="true">` element prepended to the header cell. The grip fires `dragstart` which:
    - Sets `_dragPayload = { field, sourceDimension: 'col', sourceIndex }`
    - Calls `e.dataTransfer?.setData('text/x-supergrid-axis', '1')` (value irrelevant; presence gates dragover)
    - Sets `e.dataTransfer!.effectAllowed = 'move'`
    - Calls `e.stopPropagation()` to prevent header click (collapse) from firing

    **Step 4: Add grip handles to row headers** in the row header rendering block of `_renderCells()`. Same dragstart pattern as column grips, but with `sourceDimension: 'row'`.

    **Step 5: Add drop zones for cross-dimension transpose.** Two approaches (choose one):
    - Option A: The column header area itself is the "col drop zone" (accepts row-origin drags); the row header column (grid-column: 1) is the "row drop zone" (accepts col-origin drags).
    - Option B: Add dedicated drop zone elements (thin strips) at the boundary between dimension areas.

    Use Option A (simpler — no extra DOM). Add `dragover` and `drop` listeners to the col header area and row header area. `dragover` gates with `e.dataTransfer?.types.includes('text/x-supergrid-axis')` and checks that the drag source is from the OPPOSITE dimension (cross-dimension only in this task). Add `dragleave` to remove visual feedback.

    **Step 6: Implement cross-dimension drop handler** that:
    1. Reads `_dragPayload` (clear it immediately after reading)
    2. Reads current axes from `this._provider.getStackedGroupBySQL()`
    3. Checks source dimension has >1 axis (min-1 guard — per locked user constraint)
    4. Checks target dimension does not already contain the field (no-duplicate guard — per locked user constraint "a field can only appear in one dimension at a time")
    5. If both guards pass: removes field from source array, appends to target array
    6. Calls `this._provider.setColAxes()` and `this._provider.setRowAxes()` — StateCoordinator subscription fires `_fetchAndRender()` automatically (do NOT call _fetchAndRender directly — per anti-pattern in RESEARCH.md)
    7. If either guard fails: no-op (silently blocked)

    **Step 7: Add drop zone visual feedback** via CSS class `.drag-over` on dragover/dragleave/drop. Use `classList.add('drag-over')` on dragover, `classList.remove('drag-over')` on dragleave and drop. Style: subtle background highlight (e.g., rgba(59,130,246,0.1) — light blue tint).

    **IMPORTANT anti-patterns to avoid:**
    - Do NOT read `dataTransfer.getData()` in `dragover` — it's security-blocked. Use `types.includes()` only.
    - Do NOT call `_fetchAndRender()` directly from the drop handler — let the StateCoordinator subscription handle it.
    - Do NOT apply d3.drag to the grip handles — d3.drag intercepts dragstart and breaks dataTransfer.
    - Do NOT store axis state on SuperGrid instance — always read from/write to provider.

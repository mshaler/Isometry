## What Was Built

### Task 1: Panel-Rail Deletion + Sidebar Consolidation

**WorkbenchShell.ts:**
- Removed `_panelRailEl` private field and all panel-rail DOM creation
- New sidebar sub-structure: `.workbench-sidebar__nav` (SidebarNav mount), `.workbench-sidebar__sections` (CollapsibleSections), `.workbench-sidebar__resize-handle` (drag to resize)
- All 6 CollapsibleSections now mount into `sectionsContainer` (inside sidebar, not main column)
- `getSidebarEl()` now returns the `__nav` container — SidebarNav API in main.ts unchanged
- `getSectionsEl()` new method returns `__sections` container, replacing `getPanelRailEl()`
- `getPanelRailEl()` removed entirely
- `WorkbenchShellConfig.bridge` new required field for width persistence
- Drag logic: mousedown on handle starts drag, mousemove updates sidebar width clamped to [200, 480]px, mouseup persists to `ui_state` key `sidebar:width`
- Boot restore: calls `bridge.send('ui:get', { key: 'sidebar:width' })` and applies the stored width

**workbench.css:**
- Removed `.workbench-panel-rail` block entirely
- `.workbench-sidebar` expanded: `width: 280px`, `min-width: 200px`, `max-width: 480px`, `display: flex; flex-direction: column; position: relative`
- Added `.workbench-sidebar__nav` (flex-shrink: 0, overflow-y: auto)
- Added `.workbench-sidebar__sections` (flex: 1 1 auto, overflow-y: auto, min-height: 0)
- Added `.workbench-sidebar__resize-handle` (position: absolute right 0, width 8px, cursor: col-resize)
- Added hover state for resize handle (accent + 30% opacity)

**main.ts:**
- `const panelRailEl = shell.getPanelRailEl()` replaced with `const sectionsEl = shell.getSectionsEl()`
- All 9 `panelRailEl` references replaced with `sectionsEl`
- `bridge` cast and passed to WorkbenchShell config

### Task 2: ProjectionExplorer Overflow Fix

**projection-explorer.css:**
- Added `max-height: 120px` and `overflow-y: auto` to `.projection-explorer__well-body`
- The reset/footer button is appended to `_root` (not to any well), so it remains visible below the scrollable wells

**2. [Rule 1 - Bug] WorkerBridge type cast required**
- **Found during:** Task 1 TypeScript verification
- **Issue:** `WorkerBridge.send` is generic over `WorkerRequestType`, but `WorkbenchShellConfig.bridge.send` expects `(type: string, ...)`. TypeScript correctly rejects the assignment.
- **Fix:** Bridge interface on WorkbenchShellConfig uses `Promise<unknown>` return. In main.ts, `bridge` is cast as the interface type. No runtime impact.
- **Commit:** f95e66fb

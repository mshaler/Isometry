  <behavior>
    ## parseDateString(s: string): Date | null

    Sequential fallback chain (per CONTEXT.md locked decision):
    1. Strip ISO time suffix: s.split('T')[0] (date-only precision)
    2. Try ISO 8601: '%Y-%m-%d' (e.g., '2025-03-15')
    3. Try US: '%m/%d/%Y' (e.g., '03/15/2025')
    4. Try EU: '%d/%m/%Y' (e.g., '15/03/2025')
    5. First non-null result wins. All null = return null.

    Test cases:
    - parseDateString('2025-03-15') -> Date(2025, 2, 15)
    - parseDateString('03/15/2025') -> Date(2025, 2, 15)
    - parseDateString('15/03/2025') -> Date(2025, 2, 15)
    - parseDateString('2025-03-05T14:30:00') -> Date(2025, 2, 5) (strips time)
    - parseDateString('2025-03-05T14:30:00Z') -> Date(2025, 2, 5) (strips time+tz)
    - parseDateString('TBD') -> null
    - parseDateString('ASAP') -> null
    - parseDateString('') -> null
    - parseDateString('  ') -> null (whitespace-only)
    - parseDateString('N/A') -> null
    - parseDateString('2025-01-01') -> Date(2025, 0, 1) (January edge)
    - parseDateString('12/31/2025') -> Date(2025, 11, 31) (December edge)

    ## smartHierarchy(minDate: Date, maxDate: Date): TimeGranularity

    Threshold breakpoints targeting ~10-20 columns (per CONTEXT.md):
    - days <= 20 -> 'day'
    - days <= 140 -> 'week'  (7 * 20 = 140)
    - days <= 610 -> 'month' (30.5 * 20 = 610)
    - days <= 1825 -> 'quarter' (91 * 20 ~ 1825)
    - days > 1825 -> 'year'

    Where `days = d3.timeDay.count(minDate, maxDate)`.

    Test cases:
    - 1 day span -> 'day'
    - 15 days -> 'day'
    - 20 days -> 'day' (boundary)
    - 21 days -> 'week' (one past boundary)
    - 100 days -> 'week'
    - 140 days -> 'week' (boundary)
    - 141 days -> 'month' (one past boundary)
    - 400 days -> 'month'
    - 610 days -> 'month' (boundary)
    - 611 days -> 'quarter' (one past boundary)
    - 1200 days -> 'quarter'
    - 1825 days -> 'quarter' (boundary)
    - 1826 days -> 'year' (one past boundary)
    - 3650 days (10 years) -> 'year'
    - 0 days (same date) -> 'day' (single-date dataset edge case)

    Create `src/views/supergrid/SuperTimeUtils.ts`:

    ```typescript
    import * as d3 from 'd3';
    import type { TimeGranularity } from '../../providers/types';

    // Three parsers in fallback order (CONTEXT.md locked decision)
    const _ISO_PARSER = d3.timeParse('%Y-%m-%d');
    const _US_PARSER = d3.timeParse('%m/%d/%Y');
    const _EU_PARSER = d3.timeParse('%d/%m/%Y');
    const _PARSERS = [_ISO_PARSER, _US_PARSER, _EU_PARSER];

    /**
     * Parse a date string using sequential format fallback.
     * Order: ISO 8601 -> US (MM/DD/YYYY) -> EU (DD/MM/YYYY).
     * Returns null for unparseable strings ('TBD', 'ASAP', '', etc.)
     * — these map to the 'No Date' group in SuperGrid time axes.
     *
     * Date-only precision per CONTEXT.md: time components are stripped.
     */
    export function parseDateString(s: string): Date | null {
      if (!s || s.trim() === '') return null;
      // Strip ISO datetime suffix (date-only precision per CONTEXT.md)
      const dateOnly = s.split('T')[0]!;
      for (const parser of _PARSERS) {
        const result = parser(dateOnly);
        if (result) return result;
      }
      return null;
    }

    /**
     * Select the appropriate time hierarchy level for ~10-20 columns.
     * Thresholds: day<=20, week<=140, month<=610, quarter<=1825, year>1825.
     *
     * Uses d3.timeDay.count for DST-safe day counting.
     * Single-date (minDate === maxDate) returns 'day' (0 days <= 20).
     */
    export function smartHierarchy(minDate: Date, maxDate: Date): TimeGranularity {
      const days = d3.timeDay.count(minDate, maxDate);
      if (days <= 20) return 'day';
      if (days <= 140) return 'week';
      if (days <= 610) return 'month';
      if (days <= 1825) return 'quarter';
      return 'year';
    }
    ```

    Tests use Vitest. Each test creates concrete Date objects and compares parsed output.
    For smartHierarchy boundary tests, create dates with exact day offsets from a fixed start date.

    IMPORTANT: d3.timeParse('%d/%m/%Y') does NOT validate overflow months — '13/01/2025' parsed as US (%m/%d/%Y) gives Jan 13 2025 (succeeds before EU parser tried). This is the expected 'first successful parse wins' behavior per CONTEXT.md.

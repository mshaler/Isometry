    **Step 5: Add _computeSmartHierarchy() private method.**

    New private method that:
    1. Finds first time field among all active axes (`colAxes` and `rowAxes`).
    2. Collects all values for that field from the cells array.
    3. Calls `parseDateString()` on each value.
    4. Filters out nulls (unparseable dates).
    5. If no valid dates, returns null (no auto-detection possible).
    6. Computes min/max Date objects.
    7. Returns `smartHierarchy(minDate, maxDate)`.

    Import `{ parseDateString, smartHierarchy }` from `./supergrid/SuperTimeUtils`.

    **CRITICAL ANTI-PATTERN: Infinite loop guard.** The `smartLevel !== currentLevel` check prevents the loop: `_fetchAndRender()` -> `setGranularity()` -> subscriber -> `_fetchAndRender()` -> same level computed -> no `setGranularity()` call -> render proceeds. Document this in a comment.

    **CRITICAL: Auto-detection only parses raw date values when granularity is null.** When granularity is already set (e.g., 'month'), cell values are strftime-formatted ('2025-01') which `parseDateString` will attempt to parse but may or may not succeed. This is acceptable: on first mount (null granularity), raw ISO dates flow through and auto-detection sets the level. On subsequent renders (granularity already set), the smart level computed from strftime values is still meaningful — `parseDateString('2025-01')` returns null, so only raw-format values contribute to detection. If ALL values are strftime-formatted (no raw dates), `_computeSmartHierarchy` returns null and skips the setGranularity call — correct behavior since the level is already set.

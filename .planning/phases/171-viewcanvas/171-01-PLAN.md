    **ViewCanvas class:**

    Private fields:
    - `_config: ViewCanvasConfig`
    - `_viewManager: ViewManager | null = null`
    - `_wrapperEl: HTMLElement | null = null`
    - `_statusEl: HTMLElement | null = null` (reference to the status slot element, set externally or found via DOM)
    - `_currentViewType: ViewType | null = null`

    `mount(container: HTMLElement)`:
    1. Create wrapper div: `const wrapper = document.createElement('div'); wrapper.className = 'view-canvas';`
    2. Store `this._wrapperEl = wrapper`
    3. Create ViewManager: `this._viewManager = new ViewManager({ container: wrapper, coordinator: config.coordinator, queryBuilder: config.queryBuilder, bridge: config.bridge, pafv: config.pafv, filter: config.filter, announcer: config.announcer, getDimension: config.getDimension })`
    4. Wire `onViewSwitch` callback on ViewManager — this fires after each switchTo completes:
       ```typescript
       this._viewManager.onViewSwitch = (viewType: ViewType) => {
         this._currentViewType = viewType;
         this._updateStatus(viewType);
         this._notifySidecar(viewType);
       };
       ```
    5. Append wrapper to container: `container.appendChild(wrapper)`

    `onProjectionChange(proj: Projection)`:
    1. Validate `proj.activeTabId` is a valid ViewType by checking if it exists in `VIEW_DISPLAY_NAMES`
    2. If invalid: `console.error(\`[ViewCanvas] Unknown tab ID: \${proj.activeTabId}\`); return;`
    3. Cast: `const viewType = proj.activeTabId as ViewType`
    4. If same as `_currentViewType`, return (no-op)
    5. Look up factory: `const factory = this._config.viewFactory[viewType]`
    6. If no factory: `console.error(\`[ViewCanvas] No factory for view: \${viewType}\`); return;`
    7. Call: `void this._viewManager!.switchTo(viewType, factory)`

    `_updateStatus(viewType: ViewType)` (private):
    1. Find status element: walk up from `_wrapperEl` to find `[data-slot="status"]` — or accept it via a `setStatusEl(el)` method called by the registry wiring layer
    2. Use `setStatusEl(el: HTMLElement)` pattern: ViewCanvas stores a reference, status updates target it
    3. Get display name from `VIEW_DISPLAY_NAMES[viewType]`
    4. Get card count from `this._viewManager!.getLastCards().length`
    5. Idempotent DOM setup (same pattern as `renderStatusSlot`):
       - If `_statusEl` has no `.sw-status-bar`, create it with exactly this structure:
         ```
         div.sw-status-bar
           span.sw-status-bar__item[data-stat="view-name"]
           span.sw-status-bar__sep  (textContent = '\u00B7')
           span.sw-status-bar__item[data-stat="card-count"]
         ```
    6. Update spans:
       - `[data-stat="view-name"]`.textContent = display name
       - `[data-stat="card-count"]`.textContent = formatted count (use `N.toLocaleString('en-US')` for comma formatting) + ` card` / ` cards`

    `_notifySidecar(viewType: ViewType)` (private):
    1. Look up registry entry: `const entry = getRegistryEntry(this._config.canvasId)` — but this returns the ViewCanvas's own entry. Actually, `defaultExplorerId` is on the view-1 entry. The sidecar for supergrid should be driven by a per-view lookup, not the canvas entry.
    2. CORRECTION: The `defaultExplorerId` on the registry entry for 'view-1' is a single value. Per D-04/D-05/D-06, the sidecar visibility should be per-view. The registry entry `defaultExplorerId` is canvas-level, not view-level. We need a per-ViewType lookup.
    3. Approach: Define a `VIEW_SIDECAR_MAP: Partial<Record<ViewType, string>>` constant in ViewCanvas.ts. Only `'supergrid'` maps to `'explorer-1'` (ProjectionExplorer sidecar). All other views have no sidecar (undefined -> null).
    4. Call: `this._config.onSidecarChange(VIEW_SIDECAR_MAP[viewType] ?? null)`

    `setStatusEl(el: HTMLElement)`:
    - Store `this._statusEl = el`
    - If `_currentViewType` is set, call `_updateStatus` immediately to populate

    `destroy()`:
    1. If `_viewManager`: call `_viewManager.destroy()`, set to null
    2. If `_wrapperEl`: call `_wrapperEl.remove()`, set to null
    3. Set `_statusEl = null`, `_currentViewType = null`

    Per D-09 (CANV-06): ViewCanvas.ts must NOT be imported by SuperWidget.ts. It will be imported only by registry.ts and main.ts.
    Per D-10: ViewManager's container is the wrapper div, never _canvasEl directly.
    Per D-11: ViewManager already handles destroy-before-mount in switchTo().

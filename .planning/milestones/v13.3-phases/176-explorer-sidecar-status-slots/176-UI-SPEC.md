## Color

All values from `src/styles/design-tokens.css`. No new color tokens for this phase.

| Role | Token | Dark value | Usage |
|------|-------|------------|-------|
| Dominant (60%) | `--bg-primary` | `#1a1a2e` | SuperWidget root background, status slot background |
| Secondary (30%) | `--bg-surface` | `#252540` | Sidecar slot background (matches sidebar), separator borders |
| Accent (10%) | `--accent` | `#4a9eff` | Sync indicator "syncing" active state dot only |
| Muted text | `--text-muted` | `#858596` | All status bar items, separators, sync indicator idle state |
| Destructive | `--danger` | `#ff4a4a` | Sync indicator "error" state dot only |

Accent reserved for: sync indicator active/syncing state dot. Not used for view name, card count, filter count, selection count, card title, or dataset info — all of those render in `--text-muted`.

Sync indicator states:
- Idle (no active sync): `--text-muted` dot, no label or label reads "Synced"
- Syncing: `--accent` pulsing dot, label reads "Syncing…"
- Error: `--danger` dot, label reads "Sync error"

Sidecar background uses `--bg-surface` (same as sidebar at `[data-slot="sidebar"]`) to frame the explorer panels as a structural chrome area, not canvas content. Source: `superwidget.css` line 58.

## Layout Contract

### Sidecar Grid Column (SIDE-01, SIDE-02, SIDE-03, D-01, D-02)

Current grid (2 columns):
```css
grid-template-columns: auto 1fr;
grid-template-areas:
  "sidebar header"
  "sidebar tabs"
  "sidebar canvas"
  "sidebar status";
```

New grid (3 columns):
```css
grid-template-columns: auto 1fr 0;          /* sidecar hidden: 0 */
grid-template-areas:
  "sidebar header  sidecar"
  "sidebar tabs    sidecar"
  "sidebar canvas  sidecar"
  "sidebar status  sidecar";
transition: grid-template-columns var(--transition-normal) ease;
```

When sidecar is visible:
```css
grid-template-columns: auto 1fr var(--sw-sidecar-width);
```

The `sidecar` grid-area spans all 4 content rows (same row-span as `sidebar`). The column width transitions between `0` (hidden) and `var(--sw-sidecar-width)` (visible). No JS animation. Source: CONTEXT.md D-01, D-02.

Transition timing: `var(--transition-normal)` = 300ms with `ease` easing. Consistent with existing tab hover transitions at `var(--transition-fast)` = 150ms — the sidecar is a larger layout shift and warrants the slower 300ms. Source: `--transition-normal` in `design-tokens.css` line 129.

Overflow during transition: `[data-slot="sidecar"]` must have `overflow: hidden` to prevent explorer panels from bleeding outside the column during the 0→280px expand.

### Sidecar Slot DOM

```html
<div data-slot="sidecar">
  <div data-sidecar-slot="top-slot"></div>
  <div data-sidecar-slot="bottom-slot"></div>
</div>
```

- `data-slot="sidecar"` — follows existing `data-slot` naming convention (sidebar, header, tabs, canvas, status)
- `data-sidecar-slot="top-slot"` / `data-sidecar-slot="bottom-slot"` — internal sub-slots for multiple explorer panels (SIDE-04)
- `top-slot` height: `flex: 1 1 auto` (fills available space)
- `bottom-slot` height: `flex: 0 0 auto` (collapses to content height, max 40% of sidecar)

### Status Slot DOM After Phase 176

After D-06 (clear + re-append sync indicator pattern):

```html
<div data-slot="status">
  <!-- SuperWidget owns this: always re-appended after clear -->
  <div class="sw-sync-indicator" data-stat="sync-status">
    <span class="sw-sync-indicator__dot" data-sync-state="idle"></span>
    <span class="sw-sync-indicator__label"></span>
  </div>
  <!-- Canvas-owned: appended by each canvas on mount -->
  <!-- ViewCanvas appends .sw-view-status-bar -->
  <!-- EditorCanvas appends .sw-editor-status-bar -->
  <!-- ExplorerCanvas appends .sw-explorer-status-bar -->
</div>
```

Sync indicator is a standalone persistent element. SuperWidget re-creates and re-appends it after each `_statusEl.innerHTML = ''` in `commitProjection()`. Canvas-owned status content renders to the right of (or after) the sync indicator. Source: CONTEXT.md D-05, D-06.

`data-sync-state` values: `"idle"` | `"syncing"` | `"error"` — drives CSS color without JS class toggling.

## Status Slot Visual Spec

### ViewCanvas Status Bar (STAT-01, STAT-05, STAT-07)

Layout: horizontal flex row, items separated by `·` (`\u00B7`), all `--text-muted`, `--text-sm`.

```
[view-name] · [N cards] · [N filters active] · [N selected]
```

- `data-stat="view-name"` — always present; e.g. "SuperGrid", "List"
- `data-stat="card-count"` — always present; e.g. "1,234 cards"
- `data-stat="filter-count"` — present only when `filterCount > 0`; e.g. "3 filters active". Hidden (display:none or empty text content) when count is 0.
- `data-stat="selection-count"` — present only when `selectionCount > 0`; e.g. "2 selected". Hidden when count is 0.

Rendering: ViewCanvas subscribes to filter provider and selection events to update counts. DOM setup is idempotent (existing `sw-view-status-bar` pattern, `ViewCanvas._updateStatus()`).

### EditorCanvas Status Bar (STAT-02)

Layout: single span.

```
[card-title]
```

- `data-stat="card-title"` — card name from bridge `card:get`, or "No card selected" when selection is empty.
- Text: `--text-muted`, `--text-sm`, weight 400.
- Existing implementation in `EditorCanvas._updateStatus()` — no visual change, only extended to handle empty selection copy.

### ExplorerCanvas Status Bar (STAT-03)

Layout: horizontal flex row, separated by `·`.

```
[dataset-name] · [last imported: relative-time]
```

- `data-stat="dataset-name"` — e.g. "Apple Notes" or "3 datasets"
- `data-stat="last-import"` — e.g. "Last import: 2 hours ago". Reads from `CatalogWriter` or bridge `datasets:stats` response field `last_import_at`. Relative time formatted as human-readable (no exact timestamp in status bar).
- Text: `--text-muted`, `--text-sm`, weight 400.

### Sync Indicator (STAT-06)

Position: leftmost item in status slot, always visible. SuperWidget owns and re-appends after `innerHTML = ''`.

```
[●] [label]
```

- Dot: 6px × 6px circle (`border-radius: 50%`), `margin-right: var(--space-xs)`
- `data-sync-state="idle"`: dot color `--text-muted`, no label text (empty string)
- `data-sync-state="syncing"`: dot color `--accent`, label "Syncing…", CSS animation `sw-sync-pulse` (opacity 0.5→1.0, 800ms ease-in-out, alternate infinite)
- `data-sync-state="error"`: dot color `--danger`, label "Sync error"
- Label: `--text-xs` (10px), `--text-muted` color, weight 400

Separator between sync indicator and canvas-owned status content: `1px solid var(--border-subtle)` right border on sync indicator wrapper, `margin-right: var(--space-sm)`, `padding-right: var(--space-sm)`.

## Interaction Contract

### Sidecar Show/Hide (SIDE-01, SIDE-02, SIDE-03)

Trigger: `onSidecarChange(explorerId | null)` callback fires from `ViewCanvas._notifySidecar()` on view switch.

SuperWidget handler:
- `explorerId !== null` → set `grid-template-columns` 3rd value to `var(--sw-sidecar-width)`. CSS transition handles animation.
- `explorerId === null` → set 3rd column to `0`. CSS transition collapses sidecar.

Toggle mechanism: SuperWidget sets `grid-template-columns` directly on the root element (inline style or toggling a data attribute that drives a CSS rule). Prefer `data-sidecar-visible="true"` attribute on the root for testability:

```css
[data-component="superwidget"] {
  grid-template-columns: auto 1fr 0;
  transition: grid-template-columns var(--transition-normal) ease;
}
[data-component="superwidget"][data-sidecar-visible="true"] {
  grid-template-columns: auto 1fr var(--sw-sidecar-width);
}
```

No JS animation. No `requestAnimationFrame`. No `setTimeout`. Pure CSS grid-template-columns transition. Source: CONTEXT.md D-02.

### Explorer Mounting in Sidecar (SIDE-04)

Explorer panels mount into `[data-sidecar-slot="top-slot"]` or `[data-sidecar-slot="bottom-slot"]` via `main.ts` wiring (rewired from current passthrough slot mounting at lines 646-684). Explorer components do not know they are inside a sidecar — they receive a plain HTMLElement container.

### Status Slot Clearing on Canvas Switch (STAT-04, D-06)

`commitProjection()` sequence:
1. `this._statusEl.innerHTML = ''` — clears all canvas-owned content
2. SuperWidget re-creates and appends `.sw-sync-indicator` (preserving current sync state)
3. New canvas's `mount()` runs and appends its own status bar DOM

Canvas status bars must use idempotent DOM setup (check for existing bar before creating) to survive any edge cases where mount is called more than once.

## Accessibility

- Sidecar region: `<aside data-slot="sidecar" aria-label="Explorer panel">` — landmark for screen readers
- Sync indicator: `role="status"` on `.sw-sync-indicator`, `aria-live="polite"` — announces sync state changes without interruption
- Status bar items: no role required — presentational text in a `[data-slot="status"]` region
- Sidecar show/hide: no `aria-hidden` toggling needed — grid column collapsing to 0 does not require ARIA changes because the sidecar content remains in the DOM and accessible (consistent with existing `content-visibility: auto` pattern)
- Filter/selection counts: update in-place on existing `data-stat` spans — no live region needed (status bar as a whole is already presentational)

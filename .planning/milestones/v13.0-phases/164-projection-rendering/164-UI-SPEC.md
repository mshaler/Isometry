## Component Inventory

Phase 164 is a pure TypeScript behavioural layer over Phase 162's DOM substrate. No new visual components are introduced. The design contract governs state representation only.

### data-render-count (debug/test attribute — not visual)

- All four slots receive `data-render-count` initialized to `"0"` at SuperWidget construction.
- Attribute is a string-encoded integer. Incremented by `commitProjection` when a slot is re-rendered.
- Canvas slot increments on every `commitProjection` that results in a render.
- Header slot increments when `projection.zoneRole` changes (zone label text changes).
- Status and tabs slots do not increment on tab switch or canvas type switch (RNDR-03).
- Canvas slot resets to `"1"` (not increments) when canvas type changes — destroy prior + mount new = fresh render (RNDR-04, CONTEXT.md D-07).
- `data-render-count` is invisible to the user. It exists for test assertions only.

Source: CONTEXT.md D-05, D-06, D-07.

### Zone Theme Label (header slot text)

- The header slot's `textContent` is set to the capitalized zone role string.
- Lookup map (module-level constant in SuperWidget.ts or co-located module):

```
const ZONE_LABELS: Record<ZoneRole, string> = {
  primary: 'Primary',
  secondary: 'Secondary',
  tertiary: 'Tertiary',
};
```

- Assignment: `headerSlot.textContent = ZONE_LABELS[projection.zoneRole]`
- No wrapping element. Text is the direct `textContent` of `[data-slot="header"]`.
- Inherits all existing header slot styles (font-size: 13px, font-weight: 600, line-height: 1.3, color: var(--text-primary)).

Source: CONTEXT.md D-04, REQUIREMENTS.md RNDR-05.

### Canvas Slot Content

- Phase 164 mounts a `CanvasComponent` into `[data-slot="canvas"]` via the injected `canvasFactory`.
- The canvas component's `mount(containerEl)` is called with the canvas slot element as target.
- Phase 164 does not define the visual appearance of mounted canvas content — that is Phase 165 (stubs) and later.
- The canvas slot container itself stays: `background: var(--sw-canvas-bg); min-height: 0; overflow: hidden` (Phase 162, unchanged).

## Interaction Contract

### commitProjection(proj: Projection) — nominal path

1. Call `validateProjection(proj)`.
2. If invalid: emit `console.warn('[SuperWidget] commitProjection rejected: ' + reason)`. Leave DOM unchanged. Return.
3. If new `proj` has reference equality with current projection (same object): return without re-rendering (bail-out optimization — leverages Phase 163 no-op contract). No slot increments.
4. Determine which slots need re-rendering by diffing new vs. current projection fields:
   - `zoneRole` changed → re-render header slot (set `textContent`, increment `data-render-count`).
   - `canvasType` changed → destroy prior canvas (call `destroy()` on current `CanvasComponent`), call `canvasFactory(proj.canvasId)`, call `mount()` on new canvas component, reset canvas slot `data-render-count` to `"1"`.
   - `activeTabId` changed (tab switch) and `canvasType` unchanged → increment canvas slot `data-render-count` only. Header, status, tabs slots are NOT incremented. No destroy/mount cycle.
   - First commit (no prior projection): mount canvas, set header label, initialize all `data-render-count` to `"1"` on rendered slots, leave status and tabs at `"0"`.

### console.warn message format

```
[SuperWidget] commitProjection rejected: {reason}
```

Where `{reason}` is the `reason` field from `validateProjection`'s return value.

Source: CONTEXT.md D-01 (console.warn, no throw), REQUIREMENTS.md RNDR-02. Message format is Claude's discretion (CONTEXT.md discretion list).

### Canvas lifecycle on type switch (RNDR-04)

- Step 1: Call `currentCanvas.destroy()`.
- Step 2: Clear canvas slot children (or let mount() overwrite — executor decides).
- Step 3: Call `canvasFactory(proj.canvasId)`. If factory returns `undefined`: emit `console.warn('[SuperWidget] canvasFactory returned undefined for canvasId: ' + proj.canvasId)`. Leave canvas slot empty. Do not increment render count.
- Step 4: Call `newCanvas.mount(canvasSlot)`.
- Step 5: Set canvas slot `data-render-count` to `"1"`.

# v6.8 CSS Primitives — Roadmap

## Overview

Implement the @isometry/primitives CSS token system with a three-tier architecture:
- **Tier 1**: Design tokens (colors, spacing, typography)
- **Tier 2**: Layout primitives (per-view dimensions)
- **Tier 3**: CSS Chrome (presentation state via CSS, not React)

## File Structure (Final)

```
src/styles/
  index.css                      ← Import aggregator
  tokens.css                     ← Tier 1: Design tokens (dark theme default)
  tokens-light.css               ← Tier 1: Light theme override
  variables.css                  ← Legacy --cb-* tokens (preserved)
  primitives-supergrid.css       ← Tier 2: SuperGrid layout
  primitives-kanban.css          ← Tier 2: Kanban layout
  primitives-timeline.css        ← Tier 2: Timeline layout
  primitives-gallery.css         ← Tier 2: Gallery layout
  chrome/
    sidebar.css                  ← Chrome: Sidebar collapse
    accordion.css                ← Chrome: Native <details> panels
    sticky-headers.css           ← Chrome: position: sticky
    scroll-shadows.css           ← Chrome: Scroll indicators
    tooltip.css                  ← Chrome: CSS-only tooltips
    dialog.css                   ← Chrome: Native <dialog>
    selection.css                ← Chrome: Selection highlighting
```

## Incremental Strategy

1. **Add, don't replace**: New `--iso-*` tokens coexist with existing `--cb-*` tokens
2. **Dark theme first**: Match current app appearance
3. **One file at a time**: Each plan produces one or two CSS files
4. **Test incrementally**: Verify tokens accessible via browser DevTools after each change

## Success Criteria

- [ ] All `--iso-*` tokens accessible via `getComputedStyle`
- [ ] SuperGridCSS can read primitives at render time
- [ ] Zero React re-renders for sidebar collapse, accordion open/close
- [ ] Theme switch (dark/light) via single `data-theme` attribute change
- [ ] 60fps animations on all chrome interactions

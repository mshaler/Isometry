<objective>
Establish accessible-by-default color tokens and reduced-motion infrastructure.

Purpose: Every text and UI element must pass WCAG 2.1 AA contrast ratios in both dark and light themes without a separate high-contrast mode. All animations must respect the user's prefers-reduced-motion OS setting. Audit overlay must use shape+color dual encoding so color is never the sole differentiator.

Output: Contrast-validated design tokens, MotionProvider class, accessibility.css with reduced-motion overrides, contrast regression test suite, and shape-annotated audit overlay.
</objective>

  <action>
    **Focus Management in ViewManager:**

    1. In `src/views/ViewManager.ts`, after both the morph and crossfade paths complete `_fetchAndRender()`:
       - Call `this.container.focus()` to move focus to the main content area
       - Ensure the container has `tabindex="-1"` set (allows programmatic focus but does not appear in Tab order -- the view container inside will have tabindex="0")
       - This prevents focus loss on view switch (Pitfall 3 from research)

    **Composite Widget Pattern for SVG Views (ListView, GridView, TimelineView, NetworkView, TreeView):**

    2. In each SVG view's `mount()` method:
       - Set `tabindex="0"` on the view container (the SVG element or its wrapping div)
       - Add a `keydown` event listener implementing the composite widget pattern:
         - ArrowUp/ArrowDown: move focus to previous/next card (index-based for list/grid/timeline)
         - ArrowLeft/ArrowRight: additional navigation (grid columns, tree expand/collapse)
         - Enter/Space: activate the focused card (e.g., select it via SelectionProvider)
         - Escape: move focus to the toolbar (`document.querySelector('[role="navigation"]')?.focus()` or parent container)
       - Track the currently focused card index as a private field (`_focusedIndex: number`)
       - Visually indicate the focused card:
         - Option A (preferred): Use a CSS class `.card--focused` on the focused SVG `<g>` element with an outline/ring style
         - Do NOT add `tabindex` to individual SVG cards (composite widget pattern uses one tabindex on container)
       - In `destroy()`, remove the keydown listener and reset `_focusedIndex`

    3. **ListView, TimelineView**: Simple index-based navigation. ArrowUp = previous, ArrowDown = next. Cards are in a linear vertical list.

    4. **GridView**: ArrowUp/ArrowDown move between rows, ArrowLeft/ArrowRight move between columns. Calculate grid dimensions from the card count and container width.

    5. **NetworkView** (spatial nearest-neighbor per locked decision):
       - On ArrowUp/Down/Left/Right, find the card closest to the currently focused card in the arrow direction:
         - ArrowRight: find nearest card with x > current.x (prefer small dx, small |dy|)
         - ArrowLeft: find nearest card with x < current.x
         - ArrowUp: find nearest card with y < current.y
         - ArrowDown: find nearest card with y > current.y
       - Use Euclidean distance with a directional filter (only consider cards in the arrow's quadrant)
       - If no card exists in that direction, do nothing (stay on current)

    6. **TreeView** (WAI-ARIA TreeView APG pattern per research discretion recommendation):
       - ArrowDown: move to next visible node
       - ArrowUp: move to previous visible node
       - ArrowRight on collapsed node: expand it (show children)
       - ArrowRight on expanded node: move to first child
       - ArrowLeft on expanded node: collapse it (hide children)
       - ArrowLeft on collapsed or childless node: move to parent
       - Enter/Space: activate (select) the node -- do NOT toggle expand
       - Home: move to first node
       - End: move to last visible node

    **Composite Widget Pattern for HTML Views (GalleryView, CalendarView, KanbanView):**

    7. **GalleryView**: Similar to GridView -- ArrowUp/Down/Left/Right navigate tiles. Set `tabindex="0"` on the gallery container. Track focused tile index. Add `.gallery-tile--focused` CSS class.

    8. **CalendarView**: ArrowLeft/Right move between days, ArrowUp/Down move between weeks. If no events on target day, skip to next day with events. Set `tabindex="0"` on calendar container.

    9. **KanbanView**: ArrowLeft/Right move between columns, ArrowUp/Down move between cards within a column. Set `tabindex="0"` on kanban container.

    **Focus Indicator CSS:**

    10. In `src/styles/views.css`, extend the existing `:focus-visible` rule set:
        - Add `.card--focused` selector with the same `outline: 2px solid var(--accent); outline-offset: 2px;` style
        - Add `.gallery-tile--focused` with the same focus ring
        - For SVG elements, the outline may not render properly on `<g>` elements. Alternative: add a `<rect>` overlay or use `filter: drop-shadow()` to create a focus ring effect on SVG groups. Or apply the outline to a wrapping `<foreignObject>`. The simplest approach: use a CSS outline on the `<g>` element -- modern browsers (Safari 16+, Chrome, Firefox) support outline on SVG elements.
        - Add `[tabindex="0"]:focus-visible` for view containers so the container itself shows a subtle focus ring when first tabbed to (before arrow keys navigate within)

    **Toolbar grouped navigation:**

    11. The toolbar should act as a single Tab stop with arrow key internal navigation per locked decision. If the toolbar buttons currently each receive Tab focus individually:
        - Set `tabindex="-1"` on all toolbar buttons except the first (or currently active) one
        - Set `tabindex="0"` on the first/active toolbar button
        - Add a keydown handler on the toolbar container for ArrowLeft/ArrowRight to move between buttons (update tabindex="0" to the newly focused button, set previous to tabindex="-1")
        - This is the "roving tabindex" pattern for toolbar navigation
        - CHECK: Examine how toolbar buttons are created. If they are part of views or a shared component, locate the code.

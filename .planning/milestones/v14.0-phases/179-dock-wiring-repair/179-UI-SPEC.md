## Interaction Contract

### Click Handlers (WIRE-01 through WIRE-05)

Each dock item maps to exactly one action. No new UI chrome is introduced.

| Dock Item | Composite Key | Action | Toggle or Switch |
|-----------|--------------|--------|-----------------|
| Data Explorer | `integrate:catalog` | Toggle Data Explorer panel in sidecar | Toggle (independent on/off) |
| Filter | `analyze:filter` | Toggle LATCH filters panel in sidecar | Toggle (independent on/off) |
| Formulas | `analyze:formula` | Toggle formulas panel in sidecar | Toggle (independent on/off) |
| SuperGrid | `visualize:supergrid` | Switch active view to SuperGrid | Switch (mutually exclusive) |
| Timeline | `visualize:timeline` | Switch active view to Timeline | Switch (mutually exclusive) |
| Maps | `visualize:maps` | Switch active view to Maps | Switch (mutually exclusive) |
| Charts | `visualize:charts` | Switch active view to Charts | Switch (mutually exclusive) |
| Graphs | `visualize:graphs` | Switch active view to Graphs | Switch (mutually exclusive) |
| Settings | `help:settings` | Call `commandPalette.open()` | One-shot (no toggle state on dock) |
| Help | `help:help-page` | Call `helpOverlay.toggle()` | One-shot (no toggle state on dock) |

Source: CONTEXT.md D-01, D-02.

### Active State Rules (WIRE-06)

Governed by CONTEXT.md D-03: single `.dock-nav__item--active` class, same accent background, no visual distinction between view-selected and toggle-open.

| Situation | Visual State |
|-----------|-------------|
| View item currently selected | `.dock-nav__item--active` on that item; all other view items inactive |
| Panel toggle item with panel open | `.dock-nav__item--active` on that item |
| Panel toggle item with panel closed | No `.dock-nav__item--active` |
| Settings / Help items | Never receive `.dock-nav__item--active` (one-shot actions) |

### Mount-Time State Sync (WIRE-06, CONTEXT.md D-04)

After `PanelManager` initializes, the dock must reflect pre-existing panel state from first paint:

1. For each toggle-type item (`integrate:catalog`, `analyze:filter`, `analyze:formula`): query `PanelManager.isVisible(panelId)` and call `DockNav.setItemPressed(compositeKey, isVisible)` to set or clear `.dock-nav__item--active`.
2. For the active view item: call `DockNav.setActiveItem(compositeKey)` based on the current view from `StateManager` or equivalent.

No loading spinner, no transition animation, no new DOM elements — this is a synchronous state read on mount.

### Keyboard Accessibility

The existing ARIA `role="tablist"` / `role="tab"` pattern on `DockNav` uses ArrowUp/ArrowDown for vertical navigation. Phase 179 does not change keyboard direction (that is Phase 180's responsibility per STATE.md). No keyboard nav changes in this phase.

Focus-visible style: `outline: 2px solid var(--accent); outline-offset: -2px` — already defined in `dock-nav.css`, no changes.

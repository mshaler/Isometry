## Spacing Scale

All values reference existing tokens from `src/styles/design-tokens.css`. The project uses a non-standard internal scale — do not substitute standard 8pt multiples where the project has established values.

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| --space-xs | 4px | Icon gap within item button; section-header bottom padding |
| --space-sm | 8px | Gap between icon and label within each ribbon item |
| --space-md | 12px | Horizontal padding inside each ribbon item (left + right) |
| --space-lg | 16px | Inter-section gap — the visual "wider gap" between groups (D-01) |
| --space-xl | 24px | Horizontal padding at ribbon start/end edges |

**Ribbon height:** 56px fixed. Two-row internal layout: section label row (16px) + item row (32px) + 4px top/bottom inset = 56px total. Single compact row — never taller.

**Section label row height:** 16px (font 10px + vertical padding).

**Item button height:** 32px (matches --size-toolbar-btn × 1.14 — closest clean value at this density).

**Inter-section gap:** 16px (`--space-lg`) — implemented as `gap: 16px` on the flex row separating section groups.

**Touch target exception:** Ribbon items are pointer-device only (desktop app). No 44px override required.

## Color

All values reference existing tokens from `design-tokens.css`. Both dark (default) and light themes inherit via CSS custom property indirection — no hardcoded hex in component styles.

| Role | Token | Dark value | Light value | Usage |
|------|-------|-----------|-------------|-------|
| Dominant (60%) — ribbon surface | --bg-surface | #252540 | #eeeeef | Ribbon background; matches tab strip surface |
| Secondary (30%) — item default state | --text-secondary | #a0a0b0 | #5a5a6e | Icon + label color for inactive items |
| Accent (10%) | --accent | #4a9eff | #2563eb | Active item background fill only |
| Muted | --text-muted | #858596 | #74748a | Section header label text |
| Destructive | --danger | #ff4a4a | #dc2626 | Not used in this phase — no destructive actions |

**Accent reserved for:** `.dock-nav__item--active` background fill exclusively (D-09 from CONTEXT.md). Hover state uses `--cell-hover` (rgba semi-transparent), not accent. Focus-visible uses `2px solid var(--accent)` outline offset -2px — this is structural accessibility, not decorative.

**Active item color:** icon + label text on active item use `var(--on-accent, #fff)` to ensure contrast against the accent fill.

**Ribbon border:** 1px solid `var(--border-subtle)` on the bottom edge of the ribbon (separates ribbon from canvas). No top border (tabs row provides the visual separation above).

**Fade masks for overflow scroll:** CSS `mask-image` linear gradient — transparent 0% to black 32px from each edge — using `var(--bg-surface)` as the fade color. Applied only when overflow is present (JS adds `.dock-nav--overflow-left` / `.dock-nav--overflow-right` modifier classes).

## Layout Contract

This phase modifies the SuperWidget CSS Grid. The layout change is a load-bearing design decision — executors must not deviate.

### Before (current grid)

```
grid-template-columns: auto 1fr 0
grid-template-areas:
  "sidebar  header   sidecar"
  "sidebar  tabs     sidecar"
  "sidebar  canvas   sidecar"
  "sidebar  status   sidecar"
```

### After (Phase 180 target)

```
grid-template-columns: 1fr 0
grid-template-rows: auto auto auto 1fr auto
grid-template-areas:
  "header   sidecar"
  "tabs     sidecar"
  "ribbon   sidecar"
  "canvas   sidecar"
  "status   sidecar"
```

- `sidebar` column removed entirely (D-04 / HRIB-04 / HRIB-05).
- New `ribbon` row inserted between `tabs` and `canvas` rows.
- `[data-slot="sidebar"]` element replaced with `[data-slot="ribbon"]`.
- Canvas area spans full viewport width via `1fr` column with no sidebar offset.

**Ribbon slot CSS:**

```css
[data-component="superwidget"] [data-slot="ribbon"] {
  grid-area: ribbon;
  height: 56px;
  background: var(--bg-surface);
  border-bottom: 1px solid var(--border-subtle);
  overflow: hidden;        /* clip — JS manages scroll + fade masks */
  position: relative;
  z-index: 2;
}
```

### Ribbon Internal Layout

```
.dock-nav (ribbon mode)
  display: flex
  flex-direction: row
  height: 56px
  overflow-x: auto
  scrollbar-width: none        /* hides native scrollbar on all browsers */

  .dock-nav__section (×5)
    display: flex
    flex-direction: column
    gap: 0
    flex-shrink: 0

    .dock-nav__section-header
      height: 16px
      padding: 2px 12px 0
      font-size: 10px, weight 700, letter-spacing 0.08em, uppercase

    .dock-nav__section-items
      display: flex
      flex-direction: row
      gap: 2px

      .dock-nav__item (×N)
        display: flex
        flex-direction: row
        align-items: center
        height: 32px
        padding: 0 12px
        gap: 8px

  .dock-nav__section + .dock-nav__section
    margin-left: 16px        /* inter-section gap (--space-lg) */
```

## Interaction Contract

### Keyboard Navigation

- `aria-orientation="horizontal"` on the `<nav>` element.
- ArrowLeft / ArrowRight move focus between items using roving tabindex (existing pattern reused, direction swapped — source: CONTEXT.md, REQUIREMENTS.md HRIB-06).
- Home key focuses first item; End key focuses last item.
- Tab exits the ribbon and moves focus to the next focusable element in document order.
- ArrowUp / ArrowDown do nothing within the ribbon (no vertical navigation).

### Overflow Scroll

Decision from CONTEXT.md (Claude's Discretion): use `overflow-x: auto` with CSS fade masks. No chevron buttons. Rationale: the 5-section, 12-item ribbon fits comfortably at 1024px+ viewport widths; chevrons add DOM complexity for a rare edge case. At very narrow viewports (<768px), horizontal scroll with fade edges is adequate.

- Ribbon container: `overflow-x: auto; scrollbar-width: none; -webkit-overflow-scrolling: touch`
- Fade mask: `mask-image: linear-gradient(to right, transparent 0, black 32px, black calc(100% - 32px), transparent 100%)`
- Mask applied conditionally via JS: inspect `scrollLeft` and `scrollWidth` vs `clientWidth` on scroll event; toggle `.dock-nav--has-overflow` modifier class to activate/deactivate mask.

### Active State

- `.dock-nav__item--active`: `background-color: var(--accent); color: var(--on-accent, #fff)` (D-09 from CONTEXT.md — unchanged from Phase 179 active state logic).
- Only one item is active at a time. `setActiveItem(key)` removes the class from the previous active item before adding to the new one.

### Hover State

- `.dock-nav__item:hover`: `background-color: var(--cell-hover); color: var(--text-primary)`
- Active item hover: `background-color: var(--accent-hover, var(--accent)); color: var(--on-accent, #fff)` — accent-hover brightens the fill.

### Click Wiring

Event delegation pattern unchanged from Phase 179: single click handler on `.dock-nav` root, reads `data-key` from closest `.dock-nav__item`. No per-item event listeners.

## States

| State | Visual | Implementation |
|-------|--------|----------------|
| Default item | icon + label in `--text-secondary` | base `.dock-nav__item` |
| Hover item | `--cell-hover` background, `--text-primary` text | `.dock-nav__item:hover` |
| Active item | `--accent` background fill, white text | `.dock-nav__item--active` |

| Overflow left | left fade mask | `.dock-nav--has-overflow` + CSS mask |
| Overflow right | right fade mask | `.dock-nav--has-overflow` + CSS mask |

<task type="auto" tdd="true">
  <name>Task 2: VPOL-02 — Scroll-aware header label centering</name>
  <files>src/views/pivot/PivotGrid.ts, src/styles/pivot.css, tests/views/pivot/SuperStackSpans.test.ts</files>
  <behavior>
    - Test 1: A column span wider than the viewport has its label centered within the visible intersection (not the full span width)
    - Test 2: A row span taller than the viewport has its label centered within the visible intersection
    - Test 3: A span fully within the viewport has its label centered normally (no offset)
    - Test 4: Label offset updates on scroll
  </behavior>
  <action>
    **Approach: Inner label wrapper with dynamic offset**

    The key insight: each `.pv-col-span` and `.pv-row-span` is absolutely positioned and translated on scroll. When a span extends beyond the viewport, the text (centered via `justify-content: center`) appears off-screen. We need to shift the text so it centers within the visible portion.

    **PivotGrid.ts `_handleScroll` changes:**
    Add a private method `_centerSpanLabels()` called from `_handleScroll` and after render:

    1. For each `.pv-col-span` in the overlay:
       - Read its `left` style (absolute position) and `width` style
       - Compute the span's visible left/right edges relative to the viewport:
         - `visibleLeft = max(totalRowHeaderWidth, spanLeft - scrollLeft)`
         - `visibleRight = min(viewportWidth, spanLeft - scrollLeft + spanWidth)`
       - If `visibleRight - visibleLeft < spanWidth` (partially off-screen):
         - Compute `visibleCenter = (visibleLeft + visibleRight) / 2`
         - Compute `spanCenter = (spanLeft - scrollLeft) + spanWidth / 2`
         - Set `text-indent` or `padding-left` to shift text by `visibleCenter - spanCenter` px
       - If fully visible: reset to `text-indent: 0`

    2. Same logic for `.pv-row-span` but vertical axis (top/height, scrollTop, totalColHeaderHeight).

    **CSS pivot.css changes:**
    Add `.pv-col-span, .pv-row-span { text-indent: 0; }` as baseline. The `text-indent` property shifts the inline content without affecting box model — good for centering offset.

    Actually, for `display: flex` elements, `text-indent` doesn't work. Instead: wrap the text content in a `<span class="pv-span-label">` and use `transform: translateX(Npx)` on it. This is cleaner.

    **Revised approach — label wrapper span:**
    1. In `SuperStackSpans.ts`, wrap text content in `<span class="pv-span-label">`. For parent headers, the existing `chevron + textNode + count` structure needs the textNode wrapped.
    2. In `PivotGrid._renderOverlay`, wrap `.text()` content in a `<span class="pv-span-label">` for the base headers.
    3. In `_centerSpanLabels()`, find each `.pv-span-label` and set `transform: translateX(offsetPx)`.

    **CSS:**
    ```css
    .pv-span-label {
      display: inline-block;
      transition: transform 0.05s ease-out;
    }
    ```

    Call `_centerSpanLabels()` at end of `render()` (after overlay built) and in `_handleScroll()`.

    Keep the method lightweight — it reads computed positions from inline styles (already set as px values) and writes a single transform. No layout thrashing.
  </action>

  <name>Task 1: Extend GridLayout + create SuperSizeRowHeaderResize plugin</name>
  <files>src/views/pivot/plugins/PluginTypes.ts, src/views/pivot/plugins/SuperSizeRowHeaderResize.ts, src/views/pivot/plugins/FeatureCatalog.ts, tests/views/pivot/SuperSizeRowHeaderResize.test.ts</files>
  <behavior>
    - Test 1: clampRowHeaderWidth clamps below MIN (60) to 60
    - Test 2: clampRowHeaderWidth clamps above MAX (400) to 400
    - Test 3: clampRowHeaderWidth passes through values within range
    - Test 4: transformLayout sets rowHeaderWidths entries on GridLayout
    - Test 5: afterRender injects `.pv-resize-handle--row-header-width` handles on row span elements at each level
    - Test 6: onPointerEvent pointerdown on a handle initiates drag, pointermove updates width, pointerup ends drag
    - Test 7: FEATURE_CATALOG includes 'supersize.row-header-resize' entry
  </behavior>
  <action>
    **PluginTypes.ts:**
    Add `rowHeaderWidths: Map<number, number>;` to `GridLayout` interface. This maps dimension level index to pixel width.

    **All existing GridLayout construction sites** (PivotGrid.ts render(), mount() handlers, _handleScroll):
    Add `rowHeaderWidths: new Map()` to every GridLayout literal. There are ~4 places in PivotGrid.ts where GridLayout is constructed inline. Add the field to each.

    **SuperSizeRowHeaderResize.ts** (new file, model after SuperSizeColResize.ts):
    - Export `MIN_ROW_HEADER_WIDTH = 60`, `MAX_ROW_HEADER_WIDTH = 400`
    - Export `clampRowHeaderWidth(width: number): number`
    - Factory `createSuperSizeRowHeaderResizePlugin(): PluginHook`:
      - Internal `_widths: Map<number, number>` (level -> width)
      - Internal `_dragging: { level: number, startX: number, startWidth: number, pointerId: number } | null`
      - `transformLayout`: merge `_widths` into `layout.rowHeaderWidths`
      - `afterRender`: inject resize handles on row header spans. For each unique `data-level` among `.pv-row-span` elements, find the rightmost span at that level and append a `div.pv-resize-handle.pv-resize-handle--row-header-width[data-level=N]` on its right edge.
      - `onPointerEvent`:
        - pointerdown on `.pv-resize-handle--row-header-width`: read `data-level`, start drag
        - pointermove: update `_widths.set(level, clamped(startWidth + deltaX))`
        - pointerup: end drag
      - `destroy`: clear `_widths`, reset drag state

    **FeatureCatalog.ts:**
    - Add meta entry: `{ id: 'supersize.row-header-resize', name: 'Row Header Resize', category: 'SuperSize', description: 'Per-level drag-to-resize row header columns', dependencies: ['base.grid'], defaultEnabled: false }`
    - Register factory: `registry.setFactory('supersize.row-header-resize', createSuperSizeRowHeaderResizePlugin);`

    **CSS in pivot.css:**
    Add `.pv-resize-handle--row-header-width` styles — similar to `--width` but for row headers:
    ```css
    .pv-resize-handle--row-header-width {
      right: 0;
      top: 0;
      bottom: 0;
      width: 4px;
      cursor: col-resize;
    }
    ```
  </action>

  <files>src/views/pivot/PivotGrid.ts, src/views/pivot/plugins/SuperStackSpans.ts</files>
  <action>
    **PivotGrid.ts `render()` changes:**
    After the transform pipeline runs and `transformedLayout` is available, compute per-level row header widths:

    ```typescript
    // Resolve per-level row header widths (fallback to layout.headerWidth)
    const getRowHeaderWidth = (level: number): number =>
      transformedLayout.rowHeaderWidths.get(level) ?? transformedLayout.headerWidth;
    ```

    Update `totalRowHeaderWidth` calculation:
    ```typescript
    let totalRowHeaderWidth = 0;
    for (let i = 0; i < rowDimensions.length; i++) {
      totalRowHeaderWidth += getRowHeaderWidth(i);
    }
    ```

    Pass `getRowHeaderWidth` (or the resolved widths) to `_renderTable` and `_renderOverlay`.

    **PivotGrid.ts `_renderTable` changes:**
    Row header spacer `<th>` elements: use per-level width instead of uniform `layout.headerWidth`.
    Corner spacer `<th>` elements in thead: use per-level width.

    **PivotGrid.ts `_renderOverlay` changes:**
    Row span positioning: `left = sum of widths for levels 0..(dimIdx-1)` instead of `dimIdx * layout.headerWidth`.
    Row span width: `getRowHeaderWidth(dimIdx)` instead of `layout.headerWidth`.
    Corner cell width: `totalRowHeaderWidth` (already used, just ensure it uses the new calculation).

    **SuperStackSpans.ts `afterRender` changes:**
    Read `rowHeaderWidths` from the layout (via `ctxWithLayout.layout.rowHeaderWidths`).
    For row headers, compute per-level left offset as cumulative sum of resolved widths.
    For row headers, use resolved width instead of uniform `headerWidth`.

    **PivotGrid.ts `_handleScroll` / pointer handlers:**
    No changes needed — the overlay positions are set during render, scroll just translates.

    Ensure all existing tests still pass by adding `rowHeaderWidths: new Map()` to any test GridLayout fixtures that are missing it.

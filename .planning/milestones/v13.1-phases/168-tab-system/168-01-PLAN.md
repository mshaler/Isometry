Note: The last rule is a fallback in case `display: block` from JS conflicts with the flex layout. The executor should use `style.display = ''` (empty string) to show and `style.display = 'none'` to hide — letting CSS control the visible layout. Update the ExplorerCanvas onProjectionChange accordingly: use `container.style.display = 'none'` to hide, and `container.style.display = ''` (empty) to show (the CSS `[data-tab-container]` base rule can be `display: flex` for the default/visible state, with JS only adding `display: none` for hidden).

Actually, simpler approach: skip the `style*` selectors. Instead:
- Base CSS: `[data-tab-container] { display: none; flex-direction: column; ... }` — hidden by default
- Active CSS: `[data-tab-container].active { display: flex; }` — visible when `.active` class present
- JS toggles `.active` class instead of `style.display`

Use this class-toggle approach. Add class `active` to the active container, remove from others. CSS rules become:

```css
[data-component="superwidget"] .explorer-canvas [data-tab-container] {
  display: none;
  flex-direction: column;
  width: 100%;
  overflow: auto;
  flex: 1 1 auto;
  min-height: 0;
}

[data-component="superwidget"] .explorer-canvas [data-tab-container].active {
  display: flex;
}
```

Update the JS in ExplorerCanvas accordingly: `container.classList.add('active')` / `container.classList.remove('active')`.

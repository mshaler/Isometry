## DOM Assertion Contract

This section replaces the Copywriting Contract for a testing-only phase. It documents the exact selectors, attributes, and copy strings that EINT-01..04 tests must assert.

### EINT-01: ExplorerCanvas Mount Produces Real DataExplorerPanel Content

The test asserts the following DOM is present after registry-based mount:

| Assert | Selector / Attribute | Expected Value |
|--------|---------------------|----------------|
| Canvas wrapper present | `.explorer-canvas` | exists in DOM |
| Tab bar present | `.explorer-canvas [data-slot="tab-bar"]` | exists in DOM |
| Import/Export tab present | `[data-tab-id="import-export"]` | exists, label text "Import / Export" |
| Catalog tab present | `[data-tab-id="catalog"]` | exists, label text "Catalog" |
| DB Utilities tab present | `[data-tab-id="db-utilities"]` | exists, label text "DB Utilities" |
| Default active tab | `[data-tab-id="import-export"][data-tab-active="true"]` | present on mount |
| Import button present | `.data-explorer__import-btn` | exists inside import-export container |
| Panel not stub | `.explorer-canvas-stub` | must NOT be present |

### EINT-02: Tab Switching via commitProjection

The test asserts correct tab visibility after each `commitProjection()` call:

| Step | Action | Assert selector | Assert attribute |
|------|--------|----------------|-----------------|
| Switch to Catalog | `commitProjection({ activeTabId: "catalog" })` | `[data-tab-id="catalog"]` | `data-tab-active="true"` |
| Import tab inactive | — | `[data-tab-id="import-export"]` | `data-tab-active` absent or `"false"` |
| Catalog container visible | — | `[data-tab-container="catalog"].active` | class list includes `active` |
| Import container hidden | — | `[data-tab-container="import-export"]` | class list does NOT include `active` |
| Switch to DB Utilities | `commitProjection({ activeTabId: "db-utilities" })` | `[data-tab-id="db-utilities"]` | `data-tab-active="true"` |
| DB container visible | — | `[data-tab-container="db-utilities"].active` | class list includes `active` |

CSS mechanism: `display: none` on `[data-tab-container]`, `display: flex` on `.active` (D-03 from Phase 168). Tests use `.classList.contains('active')` — not computed display.

### EINT-03: Status Slot Updates After Simulated Import

The test asserts count values in `.sw-status-bar` update without canvas re-render:

| Assert | Selector | Expected content pattern |
|--------|----------|------------------------|
| Status bar present | `.sw-status-bar` | exists in `[data-slot="status"]` |
| Card count updates | `.sw-status-bar` | text includes updated card count integer |
| Connection count updates | `.sw-status-bar` | text includes updated connection count integer |
| Timestamp updates | `.sw-status-bar` | text includes relative time string (e.g. "just now") |
| Canvas slot unchanged | `.explorer-canvas` | same DOM node reference after update (no destroy/remount) |

Status bar copy format (from Phase 169 D-02, relative time D-02):
- `"{N} cards · {M} connections · {relative-time}"` where separators use `.sw-status-bar__sep` elements
- Zero-count variant: `"0 cards · 0 connections · Never imported"` (D-03 zero counts)

### EINT-04: Playwright WebKit Smoke Test

Playwright asserts against live DOM in superwidget-harness.html:

| Assert | Locator | Expected |
|--------|---------|---------|
| ExplorerCanvas renders | `.explorer-canvas` | visible |
| Tab bar renders | `.explorer-canvas [data-slot="tab-bar"]` | visible |
| Default tab active | `[data-tab-id="import-export"][data-tab-active="true"]` | visible |
| Tab switch works | after `__sw.commitProjection({ activeTabId: "catalog" })` | `[data-tab-id="catalog"][data-tab-active="true"]` visible |
| Previous tab inactive | — | `[data-tab-id="import-export"]` does not have `data-tab-active="true"` |
| Status slot present | `[data-slot="status"] .sw-status-bar` | visible or attached |

Playwright browser: WebKit only (CI gate constraint, from Phase 166 smoke pattern).
Page evaluate pattern: `page.evaluate(() => window.__sw.commitProjection({ activeTabId: "catalog" }))`.

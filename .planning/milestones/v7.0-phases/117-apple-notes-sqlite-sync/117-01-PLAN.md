# Phase 117-01 Plan: ETL Module Integration

## Goal

Integrate the existing `isometry-etl` package into the Isometry codebase, enabling direct Apple Notes SQLite access.

## Prerequisites

- [x] isometry-etl package verified working (`npm run apple-notes` succeeds)
- [x] NoteStore.sqlite accessible at `~/Library/Group Containers/group.com.apple.notes/NoteStore.sqlite`
- [x] Folder hierarchy verified correct (`Family/Stacey` for test note)

## Tasks

### Task 1: Copy ETL Module

Copy `isometry-etl/src/` contents to `src/etl/apple-notes-direct/`:

```
isometry-etl/src/
├── adapters/
│   └── apple-notes/
│       ├── adapter.ts       → AppleNotesAdapter class
│       ├── schema.ts        → SQL queries, types, utilities
│       └── content-extractor.ts → Protobuf decompression
└── core/
    └── types.ts            → CanonicalNode, CanonicalEdge, LATCH types
```

**Modifications needed:**
- Update import paths for ESM compatibility
- Align with Isometry's TypeScript configuration

### Task 2: Add Dependencies

Add to `package.json`:
```json
{
  "dependencies": {
    "better-sqlite3": "^11.0.0"
  },
  "devDependencies": {
    "@types/better-sqlite3": "^7.6.11"
  }
}
```

### Task 3: Type Alignment

Create `src/etl/apple-notes-direct/type-mapping.ts` to bridge:

| CanonicalNode (ETL) | Isometry nodes | Transformation |
|---------------------|----------------|----------------|
| `id` | `source_id` | "apple-notes:138083" → "138083" |
| `source` | `source` | "apple-notes" |
| `name` | `name` | Direct copy |
| `content` | `content` | Markdown |
| `time.created` | `created_at` | Date → ISO string |
| `time.modified` | `modified_at` | Date → ISO string |
| `category.hierarchy` | `folder` | ["Family", "Stacey"] → "Family/Stacey" |
| `category.tags` | `tags` | JSON array |
| `nodeType` | `node_type` | "note" |

### Task 4: Verify Compilation

Run `npm run gsd:build` to verify TypeScript compiles without errors.

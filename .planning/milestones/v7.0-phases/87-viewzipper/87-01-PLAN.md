Create `src/ui/ViewZipper.ts` exporting a `ViewZipper` class. Follow the exact pattern of ViewTabBar but with these changes:

**Config interface:**
```typescript
export interface ViewZipperConfig {
  container: HTMLElement;          // Mount target element
  onSwitch: (viewType: ViewType) => void;  // Called on tab click or cycle step
  announcer: Announcer;           // For screen reader announcements
}
```

**VIEW_TABS constant** — use UAT spec labels (NOT ViewTabBar labels). The order matches the UI-SPEC cycle sequence:
```typescript
const VIEW_TABS: { type: ViewType; label: string }[] = [
  { type: 'list', label: 'List' },
  { type: 'gallery', label: 'Gallery' },
  { type: 'kanban', label: 'Kanban' },
  { type: 'grid', label: 'Grid' },
  { type: 'supergrid', label: 'SuperGrid' },
  { type: 'calendar', label: 'Map' },
  { type: 'timeline', label: 'Timeline' },
  { type: 'network', label: 'Charts' },
  { type: 'tree', label: 'Graphs' },
];
```

Note: `calendar` type uses label "Map", `network` type uses label "Charts", `tree` type uses label "Graphs" per UAT spec.

**Constructor builds:**
1. A `<nav>` element with `role="tablist"` and `aria-label="View type"`, class `vzip-strip`
2. 9 `<button>` elements with `role="tab"`, class `vzip-tab`, `aria-selected="false"`, roving tabindex (first tab `tabindex="0"`, others `tabindex="-1"`)
3. A Play/Stop `<button>` with class `vzip-play-btn`, text content `"\u25B6 Play"`, `aria-label="Play auto-cycle"`, `aria-pressed="false"`
4. Appends the nav to `config.container`

**Import the CSS:**
```typescript
import '../styles/view-zipper.css';
```

**Tab click handler:** Call `config.onSwitch(type)`. If cycling is active, stop cycling first.

**Keyboard handler on the nav element:**
- `ArrowRight`: focus and activate next tab (wrapping)
- `ArrowLeft`: focus and activate previous tab (wrapping)
- `Home`: focus and activate first tab
- `End`: focus and activate last tab
- On each keyboard-triggered activation: call `config.onSwitch(type)` and `btn.focus()`

**setActive(viewType: ViewType) method:**
- Deactivate previous tab: remove `vzip-tab--active`, set `aria-selected="false"`, `tabindex="-1"`
- Activate new tab: add `vzip-tab--active`, set `aria-selected="true"`, `tabindex="0"`
- Update `_activeType` state
- Call `config.announcer.announce(\`Switched to \${label} view\`)` where `label` is the UAT label for the viewType

**Auto-cycle methods (Play/Stop):**
- `startCycle()`: Sets `_cycling = true`, swaps button to Stop state (class `vzip-stop-btn`, text `"\u25A0 Stop"`, `aria-label="Stop auto-cycle"`, `aria-pressed="true"`). Starts a `setInterval` at 2000ms that advances to next tab in VIEW_TABS order (wrapping). Calls `config.announcer.announce('Auto-cycle started')`.
- `stopCycle()`: Clears interval, sets `_cycling = false`, swaps button back to Play state (class `vzip-play-btn`, text `"\u25B6 Play"`, `aria-label="Play auto-cycle"`, `aria-pressed="false"`). Calls `config.announcer.announce(\`Auto-cycle stopped on \${label} view\`)`.
- Play/Stop button click handler: toggles between `startCycle()` and `stopCycle()`.

**Private state:**
- `_activeType: ViewType` (starts as `'list'`)
- `_cycling: boolean` (starts as `false`)
- `_cycleTimer: ReturnType<typeof setInterval> | null` (starts as `null`)
- `_buttons: Map<ViewType, HTMLButtonElement>`
- `_playStopBtn: HTMLButtonElement`
- `_el: HTMLElement` (the nav element)

**destroy() method:**
- Stop cycling if active
- Remove the nav element from DOM

**getActiveType(): ViewType** — returns `_activeType`

**isCycling(): boolean** — returns `_cycling`

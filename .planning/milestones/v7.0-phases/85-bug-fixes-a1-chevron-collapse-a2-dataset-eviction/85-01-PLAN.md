  <action>
The root cause is a CSS specificity conflict. The current rules are:

1. `.collapsible-section__body` — `max-height: 500px` (specificity 0,1,0)
2. `.collapsible-section--collapsed .collapsible-section__body` — `max-height: 0` (specificity 0,2,0)
3. `.collapsible-section__body--has-explorer` — `max-height: 2000px` (specificity 0,1,0) — OK, lower than #2
4. `.collapsible-section__body:has(> .properties-explorer)`, etc. — `max-height: 2000px` (specificity 0,2,0) — PROBLEM: same specificity as #2, but appears LATER in source order, so it wins

Fix: Add collapsed-state variants to both the `--has-explorer` class rule and the `:has()` progressive enhancement rules so they explicitly yield to the collapsed state.

Replace the existing collapsed + explorer override block (lines ~118-138) with:

```css
.collapsible-section--collapsed .collapsible-section__body {
  max-height: 0;
}

/* Override max-height for sections with real explorer content.
 * The collapse animation uses max-height transition, so 2000px
 * still works but is slightly less smooth -- acceptable tradeoff.
 * IMPORTANT: These rules MUST NOT apply when the section is collapsed.
 * The :not(.collapsible-section--collapsed) guard prevents the :has()
 * rules (specificity 0,2,0) from overriding the collapsed max-height: 0. */

/* Primary rule — class-based, always works: */
.collapsible-section__body--has-explorer {
  max-height: 2000px;
}

/* Collapsed state wins over --has-explorer (higher specificity): */
.collapsible-section--collapsed .collapsible-section__body--has-explorer {
  max-height: 0;
}

/* Progressive enhancement via :has() for browsers that support it: */
:not(.collapsible-section--collapsed) > .collapsible-section__body:has(> .properties-explorer),
:not(.collapsible-section--collapsed) > .collapsible-section__body:has(> .projection-explorer),
:not(.collapsible-section--collapsed) > .collapsible-section__body:has(> .latch-explorers),
:not(.collapsible-section--collapsed) > .collapsible-section__body:has(> .notebook-explorer),
:not(.collapsible-section--collapsed) > .collapsible-section__body:has(> .calc-explorer) {
  max-height: 2000px;
}
```

This ensures:
- When section is NOT collapsed: explorer sections get `max-height: 2000px` (from `:has()` or `--has-explorer`)
- When section IS collapsed: `max-height: 0` wins because the `:has()` rules are gated by `:not(.collapsible-section--collapsed)` and the `--has-explorer` class has a specific collapsed override
- The existing `transition: max-height 200ms ease-out` on `.collapsible-section__body` continues to animate smoothly

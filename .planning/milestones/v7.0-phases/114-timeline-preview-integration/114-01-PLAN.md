# Plan 114-01: Timeline View SQL Integration

## Objective

<objective>
Integrate TimelineView with LATCH filters via useSQLiteQuery, matching the pattern established by NetworkView in Phase 113-02. Add adaptive tick labels for zoom levels and ensure 60 FPS performance at 500 events.
</objective>

## Context

The `useTimeline` hook already uses `useSQLiteQuery` for data fetching. However, unlike `NetworkView`, it does not integrate with `useFilters` + `compileFilters` for LATCH filter support. This plan adds that integration, plus adaptive tick formatting and performance verification.

**Reference implementation:** `src/components/views/NetworkView.tsx` (Phase 113-02)

### Task 3: Add Adaptive Tick Labels to TimelineRenderer

Update `src/d3/visualizations/timeline/TimelineRenderer.ts` to compute tick format based on zoom level:

**Zoom level detection:**
- Very zoomed out (extent > 365 days): Year labels (`%Y`)
- Zoomed out (extent > 90 days): Month labels (`%b %Y`)
- Normal (extent > 7 days): Day labels (`%b %d`)
- Zoomed in (extent ≤ 7 days): Day+Time labels (`%b %d %H:%M`)

**Implementation:**
```typescript
const getAdaptiveTickFormat = (domain: [Date, Date]): string => {
  const extentMs = domain[1].getTime() - domain[0].getTime();
  const extentDays = extentMs / (1000 * 60 * 60 * 24);

  if (extentDays > 365) return '%Y';
  if (extentDays > 90) return '%b %Y';
  if (extentDays > 7) return '%b %d';
  return '%b %d %H:%M';
};
```

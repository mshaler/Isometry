   - After building `where`, `params`, `colAxes`, `rowAxes`, `granularity`, `searchTerm`:
   - Read calc config: `const calcConfig = this._calcExplorer?.getConfig() ?? { columns: {} }`
   - Build aggregates record: for each field in colAxes + rowAxes, look up calcConfig.columns[field] or use default (SUM for numeric, COUNT for text)
   - Fire both queries in parallel:
     ```typescript
     const calcPayload = {
       rowAxes,
       where,
       params,
       granularity: densityState.axisGranularity,
       ...(this._searchTerm ? { searchTerm: this._searchTerm } : {}),
       aggregates,
     };
     const [cells, calcResult] = await Promise.all([
       this._bridge.superGridQuery({ colAxes, rowAxes, where, params, ... }),
       this._bridge.send('supergrid:calc', calcPayload),
     ]);
     ```
   - CRITICAL: Both queries MUST use identical where/params/granularity/searchTerm (built from same variables in same invocation -- per Pitfall 3 in research)
   - Store calcResult for _renderCells: `this._lastCalcResult = calcResult`
   - Pass calcResult to _renderCells: `this._renderCells(cells, colAxes, rowAxes, calcResult)`

     - Actually, it's simpler to add ONE footer row after ALL data rows when there's a single group, or per-group footers when there are multiple groups
     - For a single group (no row axis stacking), add footer at `colHeaderLevels + visibleLeafRowCells.length + 1`
     - For multi-group: need to determine group boundaries from row axis values. Each group gets its own footer row.
   - SIMPLIFICATION per CONTEXT.md "start with leaf-level-only footers": render one footer per distinct innermost row group.
   - For SINGLE GROUP (only 1 unique row value or no row axis): one footer at the bottom
     - Footer row index = colHeaderLevels + visibleLeafRowCells.length + 1
   - For MULTIPLE GROUPS: requires inserting footer rows between groups in the grid
     - This affects gridTemplateRows count and all data cell gridRow assignments
     - RECOMMENDED APPROACH: Since footer rows bypass the virtualizer, render them as absolutely positioned overlays OR as extra grid rows appended at group boundaries
     - SIMPLEST APPROACH: Render ALL footer rows at the bottom of the grid (one row per group), not inline between groups. This avoids renumbering data cell gridRow values. Per CONTEXT.md "inline at group bottom" -- but implementing inline requires significant refactoring of the grid row numbering system. Start with bottom-appended grouped footers (one row per group, stacked at the bottom) and iterate to inline in a future polish pass.
     - ACTUALLY, the CONTEXT.md is explicit: "Inline at group bottom — footer renders as the last row of each group, scrolls with data." Honor this.
     - INLINE APPROACH:
       a. After computing visibleLeafRowCells and before assigning gridRow to data cells, insert "footer slot" rows
       b. Group visibleLeafRowCells by their row group (the compound key of all row axis values except the leaf level, or just the first N-1 row axis levels)
       c. For a single row axis: each unique value is a group. For stacked axes: group by all but the innermost.
       d. Actually, with a single row axis (most common), each unique row value is its own group. With stacked axes, groups are defined by the outermost axis values.
       e. SIMPLIFICATION: Group by the outermost row axis value. visibleLeafRowCells are already sorted by axis values. Walk the list, detect group boundaries (when outermost axis value changes), insert footer row at each boundary.
       f. Track `rowOffsets`: for each visible leaf row, compute its gridRow including footer row offsets. E.g., row 0-4 = group A (gridRow 1-5), footer A at gridRow 6, row 5-8 = group B (gridRow 7-10), footer B at gridRow 11.
       g. This means: each data cell's gridRow = colHeaderLevels + originalRowIndex + 1 + footerRowsBefore
       h. totalGridRows = colHeaderLevels + visibleLeafRowCells.length + numberOfGroups

   - Implementation:

        // Walk visibleLeafRowCells, split on group boundaries

     b. Compute `footerOffsetForRow(rowIdx)`: number of footer rows inserted before this data row
     c. Update `gridTemplateRows` to include extra rows for footers
     d. Update ALL data cell gridRow assignments to add footer offset: `gridRow = colHeaderLevels + rowIdx + 1 + footerOffsetForRow(rowIdx)`
     e. Update gutter cell gridRow assignments similarly
     f. After D3 data join, render footer cells for each group:

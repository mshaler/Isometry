  <name>Task 1: Undo-safe insertion engine + formatting methods</name>
  <files>src/ui/NotebookExplorer.ts, tests/ui/NotebookExplorer.test.ts</files>
  <behavior>
    - _undoSafeInsert: replaces selected text with given string via execCommand (fallback: direct value assignment in jsdom)
    - _formatInline('**', '**') with "hello" selected (0-5): textarea.value === "**hello**", cursor selects "hello" inside wrappers
    - _formatInline('**', '**') with empty selection at pos 0: textarea.value === "****", cursor at position 2
    - _formatInline('[', '](url)') with "link" selected: textarea.value === "[link](url)"
    - _formatLinePrefix('- ') with cursor mid-line: prefix added at line start, not cursor position
    - _formatLinePrefix('- ') with 3 lines selected: each line gets "- " prefix
    - _formatLinePrefix('> ') on "hello": textarea.value === "> hello"
    - _cycleHeading on plain line "hello": becomes "# hello"
    - _cycleHeading on "# hello": becomes "## hello"
    - _cycleHeading on "## hello": becomes "### hello"
    - _cycleHeading on "### hello": becomes "hello" (strips prefix)
    - _cycleHeading on "#### hello": treated as plain text, becomes "# #### hello" (no special handling for H4+)
    - After any formatting operation, _content field is synced with textarea.value
    - Existing Cmd+B/I/K shortcuts produce same results as before but use new _formatInline method
  </behavior>
  <action>
1. Add a private `_toolbarEl: HTMLElement | null = null` field to NotebookExplorer.

2. Add private `_undoSafeInsert(text: string)` method:
   - Call `textarea.focus()` FIRST (execCommand requires focus)
   - Save `selectionStart`/`selectionEnd` before toggle
   - Set `textarea.contentEditable = 'true'`
   - Try `document.execCommand('insertText', false, text)`
   - Catch: fallback to direct value assignment (slice before + text + slice after) + dispatch `new Event('input', { bubbles: true })`
   - Always: set `textarea.contentEditable = 'false'`
   - Always: explicitly sync `this._content = textarea.value` (do NOT rely on input event â€” execCommand may not fire it in all WebKit versions)

3. Add private `_formatInline(before: string, after: string)` method:
   - Get `selectionStart`, `selectionEnd`, `selected` text from textarea
   - Build `replacement = before + selected + after`
   - Set `textarea.selectionStart = start`, `textarea.selectionEnd = end` (ensure selection covers text to replace)
   - Call `this._undoSafeInsert(replacement)`
   - Reposition cursor: if `selected.length > 0`, select just the original text inside wrappers (`start + before.length` to `start + before.length + selected.length`). If no selection, place cursor at `start + before.length`.

4. Add private `_formatLinePrefix(prefix: string)` method:
   - Get `selectionStart`, `selectionEnd` from textarea
   - Find `lineStart = text.lastIndexOf('\n', start - 1) + 1`
   - Get `selectedText = text.substring(lineStart, end)`
   - Split on `'\n'`, map each line to `prefix + line`, join with `'\n'`
   - BUT: skip prefixing empty trailing lines (if last element after split is empty string, don't prefix it)
   - Set `textarea.selectionStart = lineStart`, `textarea.selectionEnd = end`
   - Call `this._undoSafeInsert(prefixed)`

5. Add private `_cycleHeading()` method:
   - Find current line boundaries: `lineStart = text.lastIndexOf('\n', start - 1) + 1`, `lineEnd = text.indexOf('\n', start)` (use text.length if -1)
   - Extract current line: `line = text.substring(lineStart, lineEnd)`
   - Match `line.match(/^(#{1,3})\s/)`:
     - No match: prepend `# ` (plain -> H1)
     - `#` match: replace `# ` with `## ` (H1 -> H2)
     - `##` match: replace `## ` with `### ` (H2 -> H3)
     - `###` match: strip `### ` prefix (H3 -> plain)
   - Note: `####` or more hashes do NOT match `/^(#{1,3})\s/` so they're treated as plain text -> H1
   - Set selection to cover full line, call `_undoSafeInsert(replacement)`

6. **Replace `_wrapSelection` entirely:** Delete the old method. Update the `_keydownHandler` in `mount()` to call `_formatInline` instead:
   - `e.key === 'b'` -> `this._formatInline('**', '**')`
   - `e.key === 'i'` -> `this._formatInline('_', '_')`
   - `e.key === 'k'` -> `this._formatInline('[', '](url)')`

7. Write tests in `tests/ui/NotebookExplorer.test.ts` â€” add a new describe block `'NotebookExplorer â€” formatting engine'` with tests for all behaviors listed above. Note: jsdom does NOT implement execCommand, so the fallback path runs. Tests verify formatting RESULT (textarea.value), not insertion mechanism.

   p. Verify card count still equals `baselineCardCount`

<task type="auto">
  <name>Task 2: E2E spec for Flow 4 — Card Selection Drives Notebook Binding</name>
  <files>e2e/notebook-binding.spec.ts</files>
  <action>
Create `e2e/notebook-binding.spec.ts` following the established Tier 2 spec pattern (reference `e2e/notebook-chart.spec.ts` for notebook interaction patterns):

1. Import `{ test, expect }` from `'./fixtures'`
2. Create a `test.describe('Flow 4: Card selection drives notebook binding')` block
3. Write a test `'selecting a card loads notebook, switching cards round-trips content via ui_state'`:
   a. Assert `baselineCardCount > 0`

   **Step 1: Get two card IDs from database**
   b. Query: `page.evaluate(async () => { const { bridge } = (window as any).__isometry; const r = await bridge.send('db:query', { sql: 'SELECT id FROM cards WHERE deleted_at IS NULL LIMIT 2', params: [] }); return r.rows.map((row: any) => row.id); })` — capture as `cardIds`
   c. Assert `cardIds.length >= 2`

   **Step 2: Select card A → notebook panel appears**
   d. Select: `page.evaluate((id) => { const { selection } = (window as any).__isometry; selection.select(id); }, cardIds[0])`
   e. Wait for notebook explorer to appear: `page.waitForFunction(() => { const nb = document.querySelector('.notebook-explorer'); return nb !== null; }, { timeout: 5_000 })`

   **Step 3: Ensure Write tab is active**
   f. `page.evaluate(() => { const tabs = document.querySelectorAll('.notebook-segmented-control .notebook-tab'); if (tabs.length >= 2 && !tabs[0]!.classList.contains('notebook-tab--active')) { (tabs[0] as HTMLElement).click(); } })`
   g. Wait 200ms

   **Step 4: Type content for card A**
   h. Type: `page.evaluate(() => { const textarea = document.querySelector('.notebook-textarea') as HTMLTextAreaElement; if (textarea) { textarea.value = '# Notes on Card A — E2E test marker'; textarea.dispatchEvent(new Event('input', { bubbles: true })); } })`
   i. Wait 800ms for debounced auto-save to ui_state

   **Step 5: Verify content saved to ui_state**
   j. Read: `page.evaluate(async (id) => { const { bridge } = (window as any).__isometry; const r = await bridge.send('ui:get', { key: 'notebook:' + id }); return r; }, cardIds[0])` — assert result contains 'Card A'

   **Step 6: Switch to card B**
   k. Select: `page.evaluate((id) => { const { selection } = (window as any).__isometry; selection.select(id); }, cardIds[1])`
   l. Wait 500ms for flush/load cycle

   **Step 7: Verify textarea changed (card B content, likely empty)**
   m. Read textarea: `page.evaluate(() => { const ta = document.querySelector('.notebook-textarea') as HTMLTextAreaElement; return ta?.value ?? ''; })` — assert does NOT contain 'Card A' marker

   **Step 8: Type content for card B**
   n. Type: `page.evaluate(() => { const textarea = document.querySelector('.notebook-textarea') as HTMLTextAreaElement; if (textarea) { textarea.value = '# Notes on Card B — E2E test marker'; textarea.dispatchEvent(new Event('input', { bubbles: true })); } })`
   o. Wait 800ms

   **Step 9: Switch back to card A → content restored**
   p. Select card A: `page.evaluate((id) => { const { selection } = (window as any).__isometry; selection.select(id); }, cardIds[0])`
   q. Wait for content restoration: `page.waitForFunction(() => { const ta = document.querySelector('.notebook-textarea') as HTMLTextAreaElement; return ta?.value?.includes('Card A'); }, { timeout: 5_000 })`
   r. Assert textarea value includes the marker string

   **Step 10: Switch back to card B → content restored**
   s. Select card B again
   t. Wait for content restoration with 'Card B' marker
   u. Assert textarea value includes Card B marker

**Pattern notes from existing specs:**
- Use `page.waitForFunction()` for content restoration (not `waitForTimeout`)
- The notebook flush/load cycle is: flush old → save to ui_state → load new → update textarea
- Auto-save debounce is typically 500-600ms; wait 800ms to be safe

**DO NOT:**
- Use `page.fill()` — the notebook textarea is managed by NotebookExplorer, use `textarea.value = ...` + `dispatchEvent(new Event('input'))` to match the existing spec pattern in `notebook-chart.spec.ts`
- Assert exact content equality — use `.includes()` for the marker string, as the content may have been modified by the save/load cycle

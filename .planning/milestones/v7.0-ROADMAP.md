# v7.0 ETL Enrichment — Roadmap

**Milestone:** v7.0 ETL Enrichment
**Status:** PLANNING

## Overview

Build a post-import enrichment pipeline that extracts structured data from markdown bodies, enabling dataset-specific properties in PAFV navigators and supporting rich data extraction (tasks, links, callouts).

## Phase 114: Contact Body Parser

**Goal:** Parse contact markdown bodies to extract job title, email, phone, and other structured fields into `node_properties`.

**Requirements:** REQ-ETL-001, REQ-ETL-002, REQ-ETL-004

### Plan 114-01: Contact Parser Implementation

**Deliverables:**
- `src/etl/parsers/contact-body-parser.ts` — Parse `**Key:** Value` patterns
- `src/etl/parsers/contact-body-parser.test.ts` — Unit tests for all field types
- Update `KNOWN_KEYS` to handle contact-specific fields

**Key Patterns to Parse:**
```markdown
## Professional
**Organization:** Company Name
**Title:** Job Title

## Email Addresses
- **Work:** email@work.com
- email@personal.com

## Phone Numbers
- **Mobile:** +1-555-1234
- **Work:** +1-555-5678

## Addresses
**Home:**
123 Main St
City, State 12345
```

**Success Criteria:**
- [ ] Parse `**Key:** Value` inline patterns
- [ ] Parse section headers (`## Professional`, `## Email Addresses`)
- [ ] Handle labeled items (`**Work:** value`) and unlabeled items
- [ ] Extract arrays (multiple emails, multiple phones)
- [ ] Unit tests for all contact field types

### Plan 114-02: Property Classifier Integration

**Deliverables:**
- Update `discoverDynamicProperties()` to return contact-specific properties
- Verify LatchNavigator shows new properties
- Integration test with real contact data

**Success Criteria:**
- [ ] Contact properties appear in Alphanumeric bucket (Job Title, Email)
- [ ] Category bucket shows Company/Organization
- [ ] Navigator reflects 5+ properties for Contacts dataset
- [ ] Integration test passes

## Phase 115: Enrichment Infrastructure

**Goal:** Build reusable enrichment service that can process nodes by type with progress tracking.

**Requirements:** REQ-ETL-003

### Plan 115-01: Enrichment Service + CLI

**Deliverables:**
- `src/etl/enrichment/EnrichmentService.ts` — Orchestrator for body parsing
- `src/etl/enrichment/types.ts` — BodyParser interface, ExtractedProperties
- Parser registry pattern for pluggable per-type parsers
- CLI command: `npm run enrich` / `npm run enrich:contacts`

**Success Criteria:**
- [ ] EnrichmentService accepts nodeType filter
- [ ] Parser registry loads contact-body-parser
- [ ] CLI can trigger enrichment from terminal
- [ ] Stores results in node_properties with proper typing

### Plan 115-02: Batch Processing + Progress

**Deliverables:**
- Batch processing (1000 nodes at a time) for memory efficiency
- Progress callback for UI integration
- Idempotent operation (re-run produces same results)
- Performance: 500+ nodes/second

**Success Criteria:**
- [ ] Process 5000 contacts in <10 seconds
- [ ] Memory stays under 100MB
- [ ] Progress callback fires at 10% intervals
- [ ] Second run produces identical node_properties

## Phase 116: Notes Enrichment

**Goal:** Extract tasks, callouts, and links from Notes markdown bodies.

**Requirements:** REQ-ETL-010, REQ-ETL-011, REQ-ETL-013

### Plan 116-01: Task Extraction

**Deliverables:**
- `src/etl/parsers/task-extractor.ts` — Parse `- [ ]` and `- [x]` checkboxes
- Store as `tasks` array in node_properties
- Link tasks to parent node for aggregation

**Patterns:**
```markdown
- [ ] Incomplete task
- [x] Completed task
- [ ] Task with **bold** and [[wikilink]]
```

**Success Criteria:**
- [ ] Extract task text, completion status
- [ ] Handle nested tasks
- [ ] Preserve inline formatting (strip for search)
- [ ] Aggregate task counts per node

### Plan 116-02: Callout Extraction

**Deliverables:**
- `src/etl/parsers/callout-extractor.ts` — Parse Obsidian/GFM callouts
- Extract callout type (NOTE, WARNING, TIP, etc.)
- Store callout content and type

**Patterns:**
```markdown
> [!NOTE]
> This is a note callout

> [!WARNING] Title here
> Warning content
```

**Success Criteria:**
- [ ] Parse callout type and optional title
- [ ] Extract multi-line callout content
- [ ] Store as structured array
- [ ] Support collapsible callouts (`[!NOTE]-`)

### Plan 116-03: Link Extraction with Context

**Deliverables:**
- `src/etl/parsers/link-extractor.ts` — Extract URLs with surrounding context
- Capture link text, URL, and ±50 chars context
- Classify link type (external, internal, image, etc.)

**Success Criteria:**
- [ ] Extract markdown links `[text](url)`
- [ ] Extract bare URLs `https://...`
- [ ] Extract wikilinks `[[page]]` and `[[page|alias]]`
- [ ] Store context snippet for preview

## Phase 117: Rich Data Features

**Goal:** Safari Reader preview, heading outlines, and embedded content metadata.

**Requirements:** REQ-ETL-020, REQ-ETL-022, REQ-ETL-023

### Plan 117-01: Safari Reader Preview

**Deliverables:**
- `src/etl/enrichment/reader-preview.ts` — Fetch and cache Reader-style content
- Integrate with existing link extraction
- Store preview in node_properties as `reader_preview`
- Respect rate limits and caching

**Success Criteria:**
- [ ] Fetch URL content via fetch API
- [ ] Extract article title, author, content
- [ ] Cache results (don't re-fetch)
- [ ] Handle errors gracefully (store error state)

### Plan 117-02: Heading Outline Extraction

**Deliverables:**
- `src/etl/parsers/heading-extractor.ts` — Parse H1-H6 hierarchy
- Build nested outline structure
- Store as `outline` in node_properties

**Success Criteria:**
- [ ] Extract heading level and text
- [ ] Build parent-child hierarchy
- [ ] Handle ATX (`#`) and Setext (`===`) headings
- [ ] Provide flat and nested output formats

### Plan 117-03: Code Block + Table Metadata

**Deliverables:**
- `src/etl/parsers/code-block-extractor.ts` — Extract fenced code blocks
- `src/etl/parsers/table-extractor.ts` — Parse markdown tables
- Store language, line count, table dimensions

**Success Criteria:**
- [ ] Extract code language identifier
- [ ] Count lines of code
- [ ] Parse table headers and row count
- [ ] Store as structured metadata

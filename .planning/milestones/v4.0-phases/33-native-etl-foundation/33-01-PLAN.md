Create `NativeImportCoordinator.swift`:

1. **`@MainActor class NativeImportCoordinator: ObservableObject`** with:
   - `weak var webView: WKWebView?`
   - `@Published var isImporting: Bool = false` â€” prevents concurrent imports (Pitfall 6)
   - `private var pendingChunkContinuation: CheckedContinuation<Bool, Never>?`
   - `func runImport(adapter: any NativeImportAdapter) async throws` â€” the main import pipeline:
     - Guard `!isImporting` else throw `.importInProgress`
     - Set `isImporting = true`, defer `isImporting = false`
     - Accumulate cards from `adapter.fetchCards()` AsyncStream
     - Slice into 200-card chunks
     - Send each chunk via `sendChunk()`, await ack, abort on failure
     - Send final partial chunk with `isLast: true`
   - `private func sendChunk(_ cards: [CanonicalCard], index: Int, isLast: Bool) async -> Bool`:
     - JSONEncoder encode cards
     - Base64 encode the JSON string (CRITICAL: avoids escaping issues in evaluateJavaScript â€” Pitfall 2)
     - Build JS call: `window.__isometry.receive({type:'native:import-chunk',payload:{chunkIndex:N,isLast:bool,cardsBase64:'...'}})`
     - Use `withCheckedContinuation` to await JS ack
     - Store continuation in `pendingChunkContinuation`
     - Call `webView?.evaluateJavaScript(js)` with do-catch (NOT `try?` â€” Pitfall from research)
   - `func receiveChunkAck(success: Bool)` â€” called by BridgeManager when `native:import-chunk-ack` arrives; resumes the pending continuation

CRITICAL: NativeImportCoordinator MUST be `@MainActor` because it calls `evaluateJavaScript` which requires main thread (Pitfall 7 from research).
CRITICAL: Use base64 encoding for the cards JSON payload, not raw JSON string embedding in JS (Pitfall 2 from research).

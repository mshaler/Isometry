**1. Modify the main SQL query to JOIN ZICNOTEDATA for ZDATA.**

The current query (line ~262) only reads from ZICCLOUDSYNCINGOBJECT. Add a LEFT JOIN:

```sql
SELECT
    n.ZIDENTIFIER,
    n.{titleColumn},
    n.ZSNIPPET,
    n.{creationDateColumn},
    n.{modificationDateColumn},
    n.{folderColumn},
    {encryptedFilter},
    n.Z_PK,
    nd.ZDATA                    -- NEW: gzipped protobuf blob
FROM ZICCLOUDSYNCINGOBJECT n
LEFT JOIN ZICNOTEDATA nd ON n.ZNOTEDATA = nd.Z_PK
WHERE n.ZIDENTIFIER IS NOT NULL
  AND n.{titleColumn} IS NOT NULL
  AND n.ZMARKEDFORDELETION != 1
```

**Critical:** ZDATA lives in ZICNOTEDATA, not ZICCLOUDSYNCINGOBJECT. The join key is `ZICCLOUDSYNCINGOBJECT.ZNOTEDATA -> ZICNOTEDATA.Z_PK` (Pitfall 6 from research). Also check if the ZNOTEDATA column exists via schema detection — add it to `NoteStoreSchema`.

Column 8 is now ZDATA (a BLOB). Read it with `sqlite3_column_blob` and `sqlite3_column_bytes` to get a `Data` value.

**1. Add note-link: prefix handling to etl-import-native.handler.ts.**

In the auto-connection creation loop (after the existing `attendee-of:` block at line ~73), add handling for the `note-link:` prefix. Per locked decisions: create bidirectional connections (forward `links_to` at weight 0.5, backlink `linked_from` at weight 0.3).

The link card's fields use this contract (set by NotesAdapter.swift in Task 1):
- `source_url`: `"note-link:{targetZIDENTIFIER}"` — used for prefix matching
- `source_id`: `"notelink:{sourceZIDENTIFIER}:{targetZIDENTIFIER}"` — colon-delimited, used to extract both note identifiers

```typescript
// Phase 36 (BODY-04): Auto-create note-to-note link connections.
// Link cards have source_url "note-link:{targetZID}" and
// source_id "notelink:{sourceZID}:{targetZID}" (colon-delimited).
if (card.source_url?.startsWith('note-link:')) {
    const targetSourceId = card.source_url.replace('note-link:', '');
    const targetUUID = dedupResult.sourceIdMap.get(targetSourceId);

    // Extract source note ZIDENTIFIER from colon-delimited source_id
    // Format: "notelink:{sourceZID}:{targetZID}"
    const parts = card.source_id.split(':');
    // parts[0] = 'notelink', parts[1] = sourceZID, parts[2] = targetZID
    if (parts.length === 3 && parts[0] === 'notelink') {
        const sourceZID = parts[1];
        const sourceNoteUUID = dedupResult.sourceIdMap.get(sourceZID);

        if (sourceNoteUUID && targetUUID) {
            // Forward: source note -> target note (links_to, weight 0.5)
            autoConnections.push({
                id: crypto.randomUUID(),
                source_id: sourceNoteUUID,
                target_id: targetUUID,
                via_card_id: null,
                label: 'links_to',
                weight: 0.5,
                created_at: new Date().toISOString(),
            });
            // Backlink: target note -> source note (linked_from, weight 0.3)
            autoConnections.push({
                id: crypto.randomUUID(),
                source_id: targetUUID,
                target_id: sourceNoteUUID,
                via_card_id: null,
                label: 'linked_from',
                weight: 0.3,
                created_at: new Date().toISOString(),
            });
        }
    }
}
```

This approach works because:
- ZIDENTIFIERs are UUIDs (contain only hex chars and dashes, never colons)
- `split(':')` cleanly separates the prefix, source, and target
- Both sourceZID and targetZID resolve via `sourceIdMap.get()` to their card UUIDs
- Connections are between the actual source and target NOTE cards, not the intermediary link card

# Plan 34-02: Calendar Adapter (EventKit + Attendees)

<objective>
Create CalendarAdapter.swift that uses EventKit to fetch calendar events (past year + future year), maps each EKEvent to a CanonicalCard with card_type "event", creates person cards for attendees with links_to convention, and handles recurring events as expanded individual occurrences.
</objective>

<tasks>

## Task 1: Create CalendarAdapter.swift

Create `native/Isometry/Isometry/CalendarAdapter.swift`:

- Conforms to `NativeImportAdapter` protocol
- `sourceType = "native_calendar"`
- Permission: `EKEventStore.authorizationStatus(for: .event)` and `requestFullAccessToEvents()`
- `fetchCards()`: uses `enumerateEvents(matching:)` with 1yr past + 1yr future date range
  - CALR-03: recurring events automatically expanded by EventKit's `enumerateEvents`
  - CALR-04: all-day events use `startDate` midnight → `endDate` midnight boundaries
- Maps each `EKEvent` to `CanonicalCard`:
  - `card_type: "event"` (CALR-06)
  - `source_id: eventIdentifier` (dedup key)
  - `folder: calendar.title` (CALR-01)
  - `event_start / event_end`: from `startDate` / `endDate`
  - `location_name`: from `location` property
  - `is_collective: true` when `attendees.count > 1` (CALR-06)
  - CALR-05: synthesize content from date range + location + attendee names when `notes` is nil
- CALR-02: For multi-attendee events, yield additional person cards:
  - `card_type: "person"`, `source_id: "person-{email}"`, `name: attendee.name`
  - `source_url: "attendee-of:{eventSourceId}"` — convention for auto-connection creation on TypeScript side
  - Person cards are deduped naturally (same email → same source_id across events)

**Commit**: `feat(34-02): add CalendarAdapter with attendee person cards`

</tasks>

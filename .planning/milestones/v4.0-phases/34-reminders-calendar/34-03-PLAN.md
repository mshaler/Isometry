# Plan 34-03: UI Wiring + Attendee Connections + WebBundle

<objective>
Wire all three adapters (Reminders, Calendar, Notes) into the app: enable sources in ImportSourcePickerView, add adapter routing with permission checks in ContentView, extend the TypeScript native import handler to auto-create attendee connections, and rebuild WebBundle.
</objective>

<tasks>

## Task 1: Enable sources in ImportSourcePickerView + add adapter routing in ContentView

Update `native/Isometry/Isometry/ImportSourcePickerView.swift`:
- Set `isAvailable: true` for Reminders, Calendar, Notes

Update `native/Isometry/Isometry/ContentView.swift`:
- Add cases in `runNativeImport(sourceType:)` for all 3 adapters
- Check permission before running import
- Show PermissionSheetView when permission not yet granted
- Open System Settings when permission denied
- Production builds use real adapters; DEBUG builds retain MockAdapter

**Commit**: `feat(34-03): wire adapters into import UI with permission flow`

## Task 2: Extend TypeScript handler for attendee auto-connections

Update `src/worker/handlers/etl-import-native.handler.ts`:
- After dedup, scan for person cards with `source_url` starting with `"attendee-of:"`
- For each, create a `CanonicalConnection` linking person → event via sourceIdMap
- Pass auto-created connections to `writer.writeConnections()`

**Commit**: `feat(34-03): auto-create attendee connections in native import handler`

## Task 3: Rebuild WebBundle

- Run `npm run build:native` to rebuild dist-native/
- Copy updated files to `native/Isometry/Isometry/WebBundle/`
- Remove old hashed JS files
- Force-add to git (WebBundle is in .gitignore)

**Commit**: `build(34-03): rebuild WebBundle with attendee connection support`

</tasks>

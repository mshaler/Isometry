# v6.8 CSS Primitives — Requirements

## Summary

Implement the @isometry/primitives CSS token system to provide composable building blocks for all views. This embraces CSS for what it does well (layout, transitions, presentation state) rather than fighting D3.js complexity.

## Guiding Principle

```
Presentation state  →  CSS    (visible/hidden, expanded/collapsed, highlighted/normal)
Application state   →  React  (selected facets, data mutations, undo stack)
Data state          →  D3     (enter/update/exit, force simulations)
```

**Litmus test:** "If this state were lost on page reload, would the user lose work?" If no → CSS. If yes → React.

## Architecture

```
SQL decides WHAT  →  LATCH separation (WHERE, GROUP BY, ORDER BY)
CSS decides WHERE →  PAFV spatial projection (grid templates, positioning)
D3  decides HOW   →  Data binding (enter/update/exit) + GRAPH topology
```

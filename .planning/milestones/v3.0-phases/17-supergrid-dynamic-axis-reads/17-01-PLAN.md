<objective>
Rewrite SuperGrid from zero-arg constructor with in-memory card filtering to a dependency-injected lifecycle with PAFVProvider axis reads, StateCoordinator subscription, and WorkerBridge query pipeline.

Purpose: SuperGrid must read live axis configuration from PAFVProvider and subscribe to state changes via StateCoordinator. This plan establishes the constructor injection pattern, lifecycle management (subscribe on mount, unsubscribe on destroy), render no-op, and main.ts wiring. The actual render pipeline (_fetchAndRender + _renderCells) is implemented here as a skeleton that Plan 02 will flesh out with D3 data join and CSS Grid positioning.

Output: SuperGrid.ts with constructor injection and lifecycle, types.ts with narrow interfaces, main.ts wiring, tests proving subscription/lifecycle/no-op behavior.
</objective>

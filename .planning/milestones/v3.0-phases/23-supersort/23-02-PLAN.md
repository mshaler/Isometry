<objective>
Extend SuperGridQueryConfig with sortOverrides and inject sort fields into buildSuperGridQuery's ORDER BY clause after axis fields.

Purpose: This implements SORT-04 (sort within groups) at the SQL level. By appending sortOverrides AFTER axis ORDER BY parts, the GROUP_CONCAT accumulates card IDs in the user's requested sort order within each (colKey, rowKey) group. No special boundary logic needed — SQL GROUP BY naturally defines group membership; ORDER BY secondary fields control card order within each group's result set.

Output: SuperGridQueryConfig with optional sortOverrides field, buildSuperGridQuery producing correct compound ORDER BY, validated against allowlist.
</objective>

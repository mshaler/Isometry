**Create `src/views/supergrid/SuperGridSizer.ts`:**

Following the SuperZoom lifecycle pattern (class with attach/detach), create a SuperGridSizer class:

```typescript
// Constants
export const MIN_COL_WIDTH = 48;    // Minimum column width in base px
export const AUTO_FIT_PADDING = 24; // Breathing room for auto-fit
export const AUTO_FIT_MAX = 400;    // Maximum auto-fit base width
```

**Class structure:**
- `_colWidths: Map<string, number>` — stores base px widths per colKey
- `_leafColKeys: string[]` — current ordered leaf column keys (set by SuperGrid after each render)
- `_gridEl: HTMLElement | null` — grid element reference for style updates
- `_getZoomLevel: () => number` — callback to read current zoom from positionProvider
- `_onWidthsChange: ((widths: Map<string, number>) => void) | null` — callback fired on pointerup
- `_dragging: { colKey: string; startX: number; startWidth: number } | null` — active drag state

**Constructor:** `constructor(getZoomLevel: () => number, onWidthsChange?: (widths: Map<string, number>) => void)`

**Public methods:**
- `attach(gridEl: HTMLElement): void` — stores gridEl reference
- `detach(): void` — nulls gridEl reference
- `getColWidths(): Map<string, number>` — defensive copy
- `setColWidths(widths: Map<string, number>): void` — stores copy
- `resetColWidths(): void` — clears map
- `getLeafColKeys(): string[]` — returns current leaf keys
- `setLeafColKeys(keys: string[]): void` — stores keys (called by SuperGrid after each render)
- `addHandleToHeader(headerEl: HTMLElement, colKey: string): void` — creates 8px resize handle and wires pointer events
- `applyWidths(leafColKeys: string[], zoomLevel: number, gridEl: HTMLElement, rowHeaderWidth?: number): void` — calls buildGridTemplateColumns and sets gridEl.style.gridTemplateColumns

**`addHandleToHeader` implementation:**
1. Create a `div.col-resize-handle` with: `position:absolute, top:0, right:0, width:8px, height:100%, cursor:col-resize, zIndex:4, userSelect:none, backgroundColor:transparent`
2. Ensure headerEl has `position:relative` (sticky already works — see RESEARCH Pitfall 1)
3. Wire `pointerdown` handler:
   - Check `e.button === 0` (left click only)
   - `e.preventDefault()` (prevent text selection)
   - `e.stopPropagation()` (prevent header collapse click)
   - `handle.setPointerCapture(e.pointerId)`
   - Set `_dragging = { colKey, startX: e.clientX, startWidth: this._colWidths.get(colKey) ?? BASE_COL_WIDTH }`
   - Add `pointermove`, `pointerup`, `pointercancel` listeners to the handle
4. Wire `pointermove` handler:
   - If no `_dragging`, return
   - `dx = e.clientX - _dragging.startX`
   - `zoomLevel = _getZoomLevel()`
   - `newBase = Math.max(MIN_COL_WIDTH, _dragging.startWidth + dx / zoomLevel)`
   - If `e.shiftKey`: set ALL leafColKeys to newBase (populate missing keys from BASE_COL_WIDTH first — per RESEARCH Open Question 2, normalize only currently visible leaf columns)
   - Else: set only `_dragging.colKey` to newBase
   - Call `_rebuildGridTemplate()` to update gridEl.style.gridTemplateColumns
5. Wire `pointerup` handler:
   - Release pointer capture: `handle.releasePointerCapture(e.pointerId)`
   - Remove pointermove/pointerup/pointercancel listeners from handle
   - Clear `_dragging = null`
   - Call `_onWidthsChange(new Map(_colWidths))` to persist
6. Wire `pointercancel` handler:
   - Revert: `_colWidths.set(_dragging.colKey, _dragging.startWidth)` (don't persist partial drag — per RESEARCH Open Question 3)
   - Release pointer capture, remove listeners, clear _dragging
   - Call `_rebuildGridTemplate()` to restore visual state
   - Do NOT call _onWidthsChange
7. Wire `dblclick` handler:
   - `e.stopPropagation()` (prevent collapse click)
   - Measure header label: `headerEl.querySelector('.col-header-label')?.scrollWidth ?? 0`
   - Measure all data cells: `gridEl.querySelectorAll(\`.data-cell[data-col-key="${CSS.escape(colKey)}"]\`)` — iterate, take max `scrollWidth`
   - `zoomLevel = _getZoomLevel()`
   - `fittedWidth = Math.min(AUTO_FIT_MAX, Math.max(MIN_COL_WIDTH, maxContentWidth + AUTO_FIT_PADDING))`
   - `baseWidth = fittedWidth / zoomLevel`
   - Set colWidth and call `_rebuildGridTemplate()` + `_onWidthsChange`
   - NOTE: `data-col-key` attribute must be set on data cells in SuperGrid._renderCells — add this in Task 2

**Private `_rebuildGridTemplate()`:** Calls `buildGridTemplateColumns(_leafColKeys, _colWidths, _getZoomLevel(), ROW_HEADER_WIDTH)` and sets `_gridEl.style.gridTemplateColumns`. Import `buildGridTemplateColumns` from `./SuperStackHeader` and `BASE_COL_WIDTH` from `./SuperZoom`.

   The existing zoom change handler in SuperGrid (inside `mount()`, where SuperZoom is created with `onZoomChange`) should trigger sizer width reapplication:
   ```typescript
   // After existing onZoomChange logic (toast update):
   if (this._gridEl && this._sizer.getLeafColKeys().length > 0) {
     this._sizer.applyWidths(

   ```
   Wait — the onZoomChange already calls `zoom.applyZoom()` which sets `--sg-col-width`. But with per-column widths, `--sg-col-width` is no longer used in grid-template-columns (Plan 01 replaced it with literal px values). `SuperZoom.applyZoom()` still sets `--sg-row-height` and `--sg-zoom` which ARE still needed. So after `applyZoom()`, also call `_sizer.applyWidths()` to rebuild grid-template-columns with the new zoom level.

7. **Update _renderCells():**
   After computing `colLeafCells` (the leaf-level header cells):

   d. After creating leaf column header elements: for each leaf header cell element, call `this._sizer.addHandleToHeader(el, colKey)` where `el` is the header DOM element and `colKey` is the HeaderCell.value

   The leaf header cell creation happens in the existing D3-style loop that creates `.col-header` elements. Find where header elements for the deepest level are created and add the handle call there. The header cell creation already sets `.col-header` class. After creating each leaf-level header element, check if it is at the leaf level (level === colHeaders.length - 1) and if so, call addHandleToHeader.

8. **Critical: Resize handle must not conflict with axis grip and collapse click.**

   - The collapse click handler is on the header element. `stopPropagation` on the handle prevents it from reaching the header's click handler.
   - On `dblclick`, the handle also calls `stopPropagation()` to prevent double-collapse.

**3. Add highlight rendering in D3 .each() callback:**

Inside the `.each(function(d) { ... })` callback, AFTER the existing view-mode branching (matrix/spreadsheet/empty — ends around line 1196) and BEFORE the click handler (line ~1198), add the search highlight logic:

```typescript
// -----------------------------------------------------------------
// Phase 25 SRCH-03 — Search highlight rendering
// Applied AFTER view-mode content rendering so highlights overlay content.
// -----------------------------------------------------------------
const isSearchActive = self._searchTerm.trim().length > 0;
const isMatch = isSearchActive && d.matchedCardIds.length > 0;

// Opacity: dim non-matches, restore matches, clear when search inactive
// CRITICAL (Pitfall 4): Always set opacity — empty string removes inline style
el.style.opacity = isSearchActive ? (isMatch ? '1' : '0.4') : '';

if (isSearchActive && isMatch && densityStateForView.viewMode === 'matrix') {
  // Matrix mode: amber outline on matching cells
  el.classList.add('sg-search-match');
} else {
  el.classList.remove('sg-search-match');
}

// Spreadsheet mode: wrap matching text in <mark> tags via DOM manipulation (SRCH-03)
if (isSearchActive && isMatch && densityStateForView.viewMode === 'spreadsheet') {
  const searchTerms = self._searchTerm.trim().split(/\s+/).filter(Boolean);
  if (searchTerms.length > 0) {
    // Build case-insensitive regex from search terms
    const escapedTerms = searchTerms.map(t => t.replace(/[.*+?^${}()|[\]\\]/g, '\\$&'));
    const regex = new RegExp(`(${escapedTerms.join('|')})`, 'gi');

    el.querySelectorAll('.card-pill').forEach(pill => {
      const text = pill.textContent ?? '';
      if (regex.test(text)) {
        // Clear pill and rebuild with <mark> wrapped matches via DOM nodes (not innerHTML)
        // Reset regex lastIndex before split
        regex.lastIndex = 0;
        const parts = text.split(regex);
        pill.textContent = ''; // Clear existing text
        for (const part of parts) {
          if (regex.test(part)) {
            regex.lastIndex = 0;
            const mark = document.createElement('mark');
            mark.style.cssText = 'background:rgba(245,158,11,0.4);color:inherit;padding:0 1px;border-radius:2px;';
            mark.textContent = part;
            pill.appendChild(mark);
          } else {
            pill.appendChild(document.createTextNode(part));
          }
        }
      }
    });
  }
}
```

**5. Handle the regex test pattern carefully:**

The `regex.test()` advances `lastIndex` for a global regex. The pattern above uses `split(regex)` to get alternating text/match parts, then tests each part. The key insight: `String.split(regex_with_capture_group)` includes captured groups in the result array. So parts alternate between non-match and match segments. Use index parity or re-test each part:

Actually, a cleaner approach — use a simpler split pattern:

```typescript
// Split text into alternating non-match / match segments
// String.split() with a capturing group includes the captured text in the array
const parts = text.split(regex);
pill.textContent = '';
for (let i = 0; i < parts.length; i++) {
  const part = parts[i]!;
  if (part.length === 0) continue;
  // Odd-indexed parts are the captured matches (from split with capture group)
  // But simpler: just test each part against the regex
  const isTermMatch = escapedTerms.some(t =>
    new RegExp(`^${t}$`, 'i').test(part)
  );
  if (isTermMatch) {
    const mark = document.createElement('mark');
    mark.style.cssText = 'background:rgba(245,158,11,0.4);color:inherit;padding:0 1px;border-radius:2px;';
    mark.textContent = part;
    pill.appendChild(mark);
  } else {
    pill.appendChild(document.createTextNode(part));
  }
}
```

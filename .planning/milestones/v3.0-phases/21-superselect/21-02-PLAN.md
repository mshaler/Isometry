**Step 2 — TDD GREEN: Implement SuperGridSelect**

Create `src/views/supergrid/SuperGridSelect.ts`:

```typescript
export type ClickZone = 'header' | 'data-cell' | 'supergrid-card' | 'grid';

export function classifyClickZone(target: EventTarget | null): ClickZone {
  const el = target instanceof Element ? target : null;
  if (!el) return 'grid';
  if (el.closest('.col-header') || el.closest('.row-header')) return 'header';
  if (el.closest('.supergrid-card')) return 'supergrid-card';
  if (el.closest('.data-cell')) return 'data-cell';
  return 'grid';
}
```

The class follows the attach/detach lifecycle pattern (same as SuperZoom, SuperGridSizer):

- `attach(rootEl, gridEl, bboxCache, selection)`:
  1. Create SVG via `document.createElementNS('http://www.w3.org/2000/svg', 'svg')`
  2. Style: position absolute, inset 0, width 100%, height 100%, pointerEvents none, zIndex 5
  3. Create rect via `document.createElementNS('http://www.w3.org/2000/svg', 'rect')`
  4. Set rect attributes: fill, stroke, stroke-dasharray, stroke-width per locked decisions
  5. rect.style.display = 'none'
  6. Append rect to svg, append svg to rootEl
  7. Wire pointerdown on rootEl:
     - Check `classifyClickZone(e.target)` — only start lasso for 'data-cell' or 'grid'
     - Store anchor: `{ x: e.clientX, y: e.clientY }`
     - Set `_isDragging = false` initially (becomes true on first pointermove > 4px threshold)
     - `e.preventDefault()` to suppress scroll during drag
     - `rootEl.setPointerCapture(e.pointerId)` for reliable tracking
  8. Wire pointermove on rootEl:
     - If no anchor stored, return
     - Calculate distance from anchor — if < 4px and not yet dragging, return (not yet a lasso)
     - If first time exceeding threshold: set `_isDragging = true`, set `svg.style.pointerEvents = 'all'`, show rect
     - Compute SVG-relative coordinates: `clientX - rootEl.getBoundingClientRect().left` (account for scroll)
     - Set rect attributes (x, y, width, height) from min/max of anchor and current position
     - Call `bboxCache.hitTest()` with client-coordinate lasso rect (NOT SVG-relative — cache stores client coords)
     - Store hit cell keys for live highlight callbacks
  9. Wire pointerup on rootEl:
     - `rootEl.releasePointerCapture(e.pointerId)`
     - `svg.style.pointerEvents = 'none'`
     - Hide rect
     - If was dragging and has hits: finalize selection
     - If `e.metaKey || e.ctrlKey`: `selection.addToSelection(cardIds)`, else `selection.select(cardIds)`
     - Clear drag state
  10. Wire pointercancel on rootEl: same cleanup as pointerup but NO selection call

- `detach()`:
  1. Remove SVG from rootEl
  2. Remove all pointer event listeners from rootEl
  3. Null all references
  4. Clear drag state

**The lasso needs to pass cell card_ids to the selection, not just cell keys.** The SuperGridSelect constructor should accept a `getCellCardIds: (cellKey: string) => string[]` callback that SuperGrid provides from its `_lastCells` cache. This keeps SuperGridSelect decoupled from CellDatum internals.

Add a `cellCardIdsFn` parameter to `attach()`:
```typescript
attach(
  rootEl: HTMLElement,
  gridEl: HTMLElement,
  bboxCache: SuperGridBBoxCache,
  selection: SuperGridSelectionLike,
  getCellCardIds: (cellKey: string) => string[]
): void
```

On pointerup, collect card IDs:
```typescript
const allCardIds: string[] = [];
for (const cellKey of hitCellKeys) {
  allCardIds.push(...getCellCardIds(cellKey));
}
```

**CRITICAL: Use `document.createElementNS('http://www.w3.org/2000/svg', 'svg')` — NOT `document.createElement('svg')`. The latter creates an HTML element, not an SVG namespace element.**

    **Step 4: Add rAF-throttled scroll handler** to SuperGrid:
    ```typescript
    private _scrollRafId: number | null = null;
    private _boundScrollHandler = (): void => {
      if (this._scrollRafId !== null) return;
      this._scrollRafId = requestAnimationFrame(() => {
        this._scrollRafId = null;
        if (this._rootEl && this._positionProvider) {
          this._positionProvider.savePosition(this._rootEl);
        }
      });
    };
    ```
    In `mount()`: `root.addEventListener('scroll', this._boundScrollHandler, { passive: true })`.
    In `destroy()`: `this._rootEl?.removeEventListener('scroll', this._boundScrollHandler)`. Also: `if (this._scrollRafId !== null) { cancelAnimationFrame(this._scrollRafId); this._scrollRafId = null; }`.

    ```typescript
    // In mount(), after the initial _fetchAndRender:

      }
    });
    ```
    This replaces the current bare `void this._fetchAndRender();` call. Position restore MUST happen after render because the scroll container needs content to have a scrollable range (Pitfall 4 from RESEARCH.md).

    **Step 8: Add scroll reset on filter/transpose changes** in `_fetchAndRender()`:
    After the bridge query succeeds and before rendering, detect whether this is a re-render that should reset scroll position. The approach: compare current axes to _lastColAxes/_lastRowAxes. If axes changed (cross-dimension transpose or filter-triggered re-render), reset scroll:
    ```typescript
    // After successful bridge response, before _renderCells:
    const axesChanged = this._axesChanged(colAxes, rowAxes);
    // ... _renderCells() ...
    // After render, reset if axes changed:
    if (axesChanged && this._rootEl) {
      this._rootEl.scrollTop = 0;
      this._rootEl.scrollLeft = 0;
      this._positionProvider.savePosition(this._rootEl);
    }
    ```
    The `_axesChanged()` helper compares field arrays. Per user decisions:
    - Filter change → reset scroll to (0,0) (different data)
    - Cross-dimension transpose → reset scroll to (0,0) (deliberate reflow)
    - Same-dimension reorder → preserve scroll (same data, reshuffled layout)
    - View switch → restore from provider (handled in mount())

    Note: Filter changes trigger _fetchAndRender via StateCoordinator; the axes may or may not change. To differentiate filter-only changes, check if axes are identical to last and data changed. Simplest approach: always reset scroll when _fetchAndRender is called from coordinator (not from mount). Add a `_isInitialMount: boolean` flag that is true during mount, false afterward.

    **Step 9: Add zoom toast overlay:**
    ```typescript
    private _toastEl: HTMLDivElement | null = null;
    private _toastTimeout: ReturnType<typeof setTimeout> | null = null;

    private _showZoomToast(zoomLevel: number): void {
      if (!this._rootEl) return;
      // Create or reuse toast element
      if (!this._toastEl) {
        this._toastEl = document.createElement('div');
        this._toastEl.className = 'supergrid-zoom-toast';
        this._toastEl.style.position = 'absolute';
        this._toastEl.style.top = '50%';
        this._toastEl.style.left = '50%';
        this._toastEl.style.transform = 'translate(-50%, -50%)';
        this._toastEl.style.backgroundColor = 'rgba(0, 0, 0, 0.7)';
        this._toastEl.style.color = 'white';
        this._toastEl.style.padding = '8px 16px';
        this._toastEl.style.borderRadius = '8px';
        this._toastEl.style.fontSize = '14px';
        this._toastEl.style.fontWeight = 'bold';
        this._toastEl.style.pointerEvents = 'none';
        this._toastEl.style.zIndex = '100';
        this._toastEl.style.transition = 'opacity 0.3s ease';
        this._rootEl.appendChild(this._toastEl);
      }
      // Update text and show
      this._toastEl.textContent = `${Math.round(zoomLevel * 100)}%`;
      this._toastEl.style.opacity = '1';
      // Clear previous timeout and set fade
      if (this._toastTimeout) clearTimeout(this._toastTimeout);
      this._toastTimeout = setTimeout(() => {
        if (this._toastEl) this._toastEl.style.opacity = '0';
      }, 1000);
    }
    ```
    Per user decisions: dark semi-transparent rounded pill, centered in visible grid area, shows on every zoom change including Cmd+0 reset. Fades after ~1 second.

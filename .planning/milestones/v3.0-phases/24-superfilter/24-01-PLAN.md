  <behavior>
    - setAxisFilter('card_type', ['note', 'task']) stores values; compile() includes `card_type IN (?, ?)` with params ['note', 'task']
    - setAxisFilter('folder', []) removes the axis filter entirely (FILT-05: empty = unfiltered); compile() does NOT include that field
    - setAxisFilter replaces previous values for same field (not append); two calls with different values for same field = last wins
    - Multiple axis filters on different fields: compile() includes both as AND clauses
    - hasAxisFilter('card_type') returns true when filter is set with non-empty values, false otherwise
    - getAxisFilter('card_type') returns defensive copy of values; getAxisFilter on unset field returns []
    - clearAxis('card_type') removes that field's filter; other axis filters unchanged
    - clearAllAxisFilters() removes all axis filters; regular filters unchanged
    - clearFilters() clears both _filters array AND _axisFilters map AND searchQuery
    - resetToDefaults() clears _axisFilters along with existing state
    - setAxisFilter validates field via validateFilterField — invalid field throws "SQL safety violation"
    - toJSON() serializes axisFilters as Record<string, string[]>; setState() restores them
    - setState() with missing axisFilters field defaults to {} (backward compat — Phase 15 pattern)
    - setState() with invalid axisFilters shape (non-object, non-array values) throws
    - Axis filter compile order: regular filters first, then axis filters — deterministic
    - _scheduleNotify fires after setAxisFilter, clearAxis, clearAllAxisFilters
  </behavior>
  <action>
Add `private _axisFilters: Map<string, string[]> = new Map()` field to FilterProvider.

Add public methods:
- `setAxisFilter(field: string, values: string[]): void` — validates field via validateFilterField, if values.length === 0 then delete from map, else set map entry with [...values] copy. Call _scheduleNotify().
- `clearAxis(field: string): void` — validates field, delete from map, _scheduleNotify().
- `hasAxisFilter(field: string): boolean` — return _axisFilters.has(field) && (_axisFilters.get(field)?.length ?? 0) > 0.
- `getAxisFilter(field: string): string[]` — return [...(_axisFilters.get(field) ?? [])].
- `clearAllAxisFilters(): void` — _axisFilters.clear(), _scheduleNotify().

Extend `compile()`: After the existing `_filters` loop, iterate `_axisFilters` entries. For each entry with values.length > 0, validateFilterField(field), build `${field} IN (${placeholders})`, push values to params.

Update `clearFilters()`: Also call `this._axisFilters.clear()`.
Update `resetToDefaults()`: Also call `this._axisFilters.clear()`.

Update `FilterState` interface: Add optional `axisFilters?: Record<string, string[]>`.
Update `toJSON()`: Include `axisFilters: Object.fromEntries(this._axisFilters)`.
Update `setState()`: If `state.axisFilters` exists and is a valid Record<string, string[]>, populate _axisFilters. If missing, default to empty Map (backward compat). If present but invalid shape, throw.
Update `isFilterState()`: Accept optional `axisFilters` field — if present, must be object with string keys and string[] values.
Update `fromJSON()`: No changes needed (delegates to setState).

**Step 1: Generate placeholder app icon PNG**

Use a CLI command to create a simple solid-color 1024x1024 PNG icon. The icon should be a dark blue/indigo square (#1a1a2e or similar) — matching the web runtime's dark theme.

Use `sips` (macOS built-in) or Python to generate the image:
```bash

python3 -c "
from PIL import Image
img = Image.new('RGB', (1024, 1024), (26, 26, 46))
img.save('native/Isometry/Isometry/Assets.xcassets/AppIcon.appiconset/AppIcon.png')

```

If PIL is not available, use an alternative approach:
```bash
# Use sips to create from a temporary file, or use ImageMagick if available
# Fallback: create a minimal valid PNG programmatically with Python's built-in modules
python3 << 'PYEOF'
import struct, zlib

def create_png(width, height, r, g, b, path):
    """Create a minimal solid-color PNG using only stdlib."""
    # PNG signature
    sig = b'\x89PNG\r\n\x1a\n'
    # IHDR chunk
    ihdr_data = struct.pack('>IIBBBBB', width, height, 8, 2, 0, 0, 0)
    ihdr = make_chunk(b'IHDR', ihdr_data)
    # IDAT chunk (raw pixel data)
    raw = b''
    row = bytes([r, g, b] * width)
    for _ in range(height):
        raw += b'\x00' + row  # filter byte + pixel data
    compressed = zlib.compress(raw)
    idat = make_chunk(b'IDAT', compressed)
    # IEND chunk
    iend = make_chunk(b'IEND', b'')
    with open(path, 'wb') as f:
        f.write(sig + ihdr + idat + iend)

def make_chunk(chunk_type, data):
    chunk = chunk_type + data
    return struct.pack('>I', len(data)) + chunk + struct.pack('>I', zlib.crc32(chunk) & 0xffffffff)

create_png(1024, 1024, 26, 26, 46,
    'native/Isometry/Isometry/Assets.xcassets/AppIcon.appiconset/AppIcon.png')
print('Created AppIcon.png (1024x1024, #1a1a2e)')
PYEOF
```

**Step 4: Configure UILaunchScreen via pbxproj build settings**

The project uses `GENERATE_INFOPLIST_FILE = YES` with no explicit Info.plist file. Launch screen configuration is already handled via `INFOPLIST_KEY_UILaunchScreen_Generation[sdk=iphoneos*] = YES` and the iphonesimulator variant in `project.pbxproj`.

To wire the `LaunchBackground` color asset, replace the auto-generation keys with explicit UILaunchScreen keys in `project.pbxproj`. In both the Debug and Release build settings sections for the main target, make these changes:

Replace:
```
"INFOPLIST_KEY_UILaunchScreen_Generation[sdk=iphoneos*]" = YES;
"INFOPLIST_KEY_UILaunchScreen_Generation[sdk=iphonesimulator*]" = YES;
```

With:
```
"INFOPLIST_KEY_UILaunchScreen_Generation[sdk=iphoneos*]" = YES;
"INFOPLIST_KEY_UILaunchScreen_Generation[sdk=iphonesimulator*]" = YES;
"INFOPLIST_KEY_UILaunchScreen[sdk=iphoneos*]" = "{UIColorName=LaunchBackground;}";
"INFOPLIST_KEY_UILaunchScreen[sdk=iphonesimulator*]" = "{UIColorName=LaunchBackground;}";
```

This adds the `UILaunchScreen` dictionary with the `UIColorName` key pointing to the `LaunchBackground` color asset. The `GENERATE_INFOPLIST_FILE` mechanism merges these INFOPLIST_KEY entries into the generated Info.plist at build time. Both Debug and Release configurations must be updated (there are two pairs of these lines in the pbxproj, around lines 447-448 and 491-492).

**Note:** The `_Generation` keys can remain alongside the explicit keys. The explicit `UILaunchScreen` key takes precedence when both are present.

**Setup helper (at top of describe block):**

Before each test or in a shared setup:
1. `page.goto('/')`
2. Wait for `__isometry` bootstrap (bridge + viewManager + sampleManager)
3. Load sample data via `page.evaluate(() => (window as any).__isometry.sampleManager.load('sample'))`
4. Trigger coordinator update: `page.evaluate(() => (window as any).__isometry.coordinator.scheduleUpdate())`
5. Switch to Network view: `page.evaluate(() => (window as any).__isometry.viewManager.setViewType('network'))`
6. Wait for NetworkView SVG to have at least one `g.node` element: `await page.locator('g.node').first().waitFor({ timeout: 15_000 })`
7. Expand the Algorithm sidebar section if collapsed: click the CollapsibleSection header for "Algorithm" — use `page.evaluate(() => (window as any).__isometry.shell.setSectionState('algorithm', 'ready'))` or find and click the section toggle.

**Test 1: "PageRank compute writes graph_metrics and encodes nodes"**
1. Select PageRank radio: `page.locator('input[type=radio][name=algorithm][value=pagerank]').check()`
2. Click Run: `page.locator('[data-testid="algorithm-run"]').click()`
3. Wait for status text to show results (not "Running..."): `await page.locator('.algorithm-explorer__status').waitFor({ state: 'visible' })` then assert text contains "pagerank" and a card count
4. Assert graph_metrics rows via queryAll:
   ```typescript
   const metrics = await page.evaluate(async () => {
     return (window as any).__isometry.queryAll('SELECT COUNT(*) as cnt FROM graph_metrics WHERE pagerank IS NOT NULL');
   });
   expect(metrics.rows[0].cnt).toBeGreaterThan(0);
   ```
5. Assert at least one node circle has a non-default radius (encoding active). Get the `r` attribute of the first `g.node circle`:
   ```typescript
   const firstR = await page.locator('g.node circle').first().getAttribute('r');
   expect(Number(firstR)).toBeGreaterThan(0);
   ```
6. Assert legend is visible: `await expect(page.locator('.nv-legend--visible')).toBeVisible()`

**Test 2: "Multi-algorithm overlay — community color then centrality size (PAFV-04)"**
1. Select Community radio, click Run, wait for completion (status text changes from "Running...")
2. Assert at least one node circle fill is a d3.schemeCategory10 color (not the default source-provenance color):
   ```typescript
   const fills = await page.locator('g.node circle').evaluateAll((els) =>
     els.map((el) => el.getAttribute('fill')).filter(Boolean)
   );
   // schemeCategory10 colors start with # and are hex RGB
   const hasCommunityColor = fills.some((f) => f && f.startsWith('#') && f.length === 7);
   expect(hasCommunityColor).toBe(true);
   ```
3. Select Centrality radio, click Run, wait for completion
4. Assert community colors are STILL present (cumulative — color persists):
   ```typescript
   const fillsAfter = await page.locator('g.node circle').evaluateAll((els) =>
     els.map((el) => el.getAttribute('fill')).filter(Boolean)
   );
   const stillHasCommunityColor = fillsAfter.some((f) => f && f.startsWith('#') && f.length === 7);
   expect(stillHasCommunityColor).toBe(true);
   ```
5. Assert graph_metrics has both community_id and centrality populated:
   ```typescript
   const both = await page.evaluate(async () => {
     return (window as any).__isometry.queryAll(
       'SELECT COUNT(*) as cnt FROM graph_metrics WHERE community_id IS NOT NULL AND centrality IS NOT NULL'
     );
   });
   expect(both.rows[0].cnt).toBeGreaterThan(0);
   ```

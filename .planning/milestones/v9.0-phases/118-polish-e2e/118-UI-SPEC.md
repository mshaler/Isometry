## Spacing Scale

Declared values from `src/styles/design-tokens.css` (--space-* tokens):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (--space-xs) | Icon gaps, inline padding, radio group item gap |
| sm | 8px (--space-sm) | Compact element spacing, legend swatch gap, param field gap |
| md | 12px (--space-md) | Explorer section padding, legend panel padding |
| lg | 16px (--space-lg) | Default element spacing, button horizontal padding |
| xl | 24px (--space-xl) | Legend panel bottom/right offset from SVG edge |

> Note on `--space-md: 12px`: This token is a project legacy value that predates this phase. It is a multiple of 4, it exists in `design-tokens.css`, and it is already in use across the codebase. It is accepted as-is within the standard 4-point scale for this project. All Phase 118 usages of `--space-md` reference this token directly rather than hardcoding 12px.

Exceptions:
- Tooltip internal padding: `padding: var(--space-sm) var(--space-md)` — 8px vertical, 12px horizontal (compact floating surface; both values are existing --space-* tokens)
- Stale badge dot size: 8px diameter (purely decorative indicator, not a touch target)

## Color

All values reference existing tokens from `src/styles/design-tokens.css`. Dark theme shown; light theme values are in the token file.

| Role | Token | Dark Value | Usage |
|------|-------|------------|-------|
| Dominant (60%) | --bg-primary | #1a1a2e | App background, SVG canvas area |
| Secondary (30%) | --bg-surface / --bg-card | #252540 / #1e1e2e | Legend panel, tooltip surface, AlgorithmExplorer panel |
| Accent (10%) | --accent | #4a9eff | Run button focus ring, active algorithm indicator |
| Destructive | --danger | #ff4a4a | Not applicable in this phase |
| Warning (stale state) | --audit-modified | #fb923c | Stale badge dot + inline warning text tint |

Accent reserved for: Run button focus ring only. All other interactive states use `--bg-surface` / `--border-muted` hover patterns already established in algorithm-explorer.css.

Warning color (`--audit-modified`, #fb923c dark / #ea580c light) is the designated stale indicator color. It is already used for audit "modified" overlays — reusing it establishes consistent "data-changed" semantics. The stale badge must be visually subtle: 8px dot, no animation (no pulse or flash). Inline status text uses `color: var(--audit-modified)` at `--text-xs` size.

Tooltip surface: `var(--bg-surface)` background, `var(--border-muted)` border, `var(--overlay-shadow-heavy)` box-shadow. Matches the established `.nv-legend` panel pattern exactly.

Combined legend panel: extends existing `.nv-legend` / `.nv-legend--visible` classes. No new surface color needed.

## Interaction Contract

### Stale Indicator (GFND-04)

- Trigger: any `MutationManager` operation that inserts, updates, or deletes cards or connections.
- Display: 8px warning dot badge appears on the AlgorithmExplorer CollapsibleSection header. When section is expanded, `_statusEl` also shows "Results may be outdated" in `var(--audit-modified)` color.
- Clearing: any successful algorithm Run clears the flag; Reset button also clears the flag.
- Persistence: session-only (in-memory flag). No ui_state write. No badge on page reload.
- The badge dot sits inline after the section title text, with 8px left margin. It uses `background-color: var(--audit-modified)`, `border-radius: 50%`, `width: 8px`, `height: 8px`, `display: inline-block`.
- CSS class for badge dot: `.algorithm-explorer__stale-dot`. Hidden by default (`display: none`), shown when stale (`display: inline-block`).
- CSS class for stale status text state: `.algorithm-explorer__status--stale` adds `color: var(--audit-modified)`.

### Hover Tooltip (CTRL-03)

- Primary focal point: the bold card name at the top of the tooltip panel is the visual anchor. It is the first element the eye lands on, rendered at Label weight (11px, semibold) against the `--bg-surface` panel. Metric rows below it are subordinate (10px, regular).
- Library: d3-tip (or equivalent positional tooltip). Positioned above the hovered node using `d3-tip` offset pattern.
- Activation: `mouseenter` on node `<g>` shows tooltip; `mouseleave` hides. Nodes with no graph_metrics data (all columns null) do not trigger tooltip.
- Structure: card name in bold at top (Label, 11px, semibold), then one row per non-null metric (Micro, 10px, regular).
- CSS class: `.nv-tooltip`. Matches `.nv-legend` surface: `background: var(--bg-surface)`, `border: 1px solid var(--border-muted)`, `border-radius: var(--radius-md)`, `box-shadow: var(--overlay-shadow-heavy)`, `padding: var(--space-sm) var(--space-md)`.
- Metric display order: PageRank, Centrality, Community, Clustering (consistent, regardless of compute order).
- Numeric format: 3 decimal places for PageRank/Centrality/Clustering (0.042, not 0.0421857). Community shows integer only.
- data-testid: `data-testid="nv-tooltip"` on the tooltip container for E2E assertions.

### Multi-Algorithm Overlay Legend (PAFV-04)

- Primary focal point: the color swatch row (community section) is the dominant visual anchor of the combined legend panel because community color appears on every visible node. The size scale gradient (lower section) is secondary. Section headings ("Color — Community", "Size — PageRank") are the typographic anchors that distinguish the two encodings.
- When only color encoding active: legend shows community swatches section with heading "Color — Community".
- When only size encoding active: legend shows size scale gradient section with heading "Size — {algorithm}".
- When both active: legend shows both sections stacked. Color section first, size section second. Divider: 1px `var(--border-subtle)` horizontal rule between sections.
- No user interaction on the legend — display only.
- The combined legend replaces the existing single-section legend: `_updateLegend()` must clear and rebuild both sections on each `applyAlgorithmEncoding` call.

### Reset (CTRL-04)

- Clicking "Reset Encoding" calls `NetworkView.resetEncoding()` + clears stale indicator flag + hides legend.
- No confirmation required.
- Status text returns to empty string after reset.
- Pick mode (source/target nodes) is also cleared on reset.

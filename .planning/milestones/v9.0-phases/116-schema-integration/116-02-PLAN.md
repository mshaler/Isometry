    **Internal state:**
    - `_selectedAlgorithm: AlgorithmId = 'pagerank'` (default)
    - `_running = false`
    - `_louvainResolution = 1.0`
    - `_pagerankAlpha = 0.85`
    - `_centralitySamplingThreshold = 2000`

    - `_runButton: HTMLButtonElement | null = null`
    - `_paramsContainer: HTMLElement | null = null`

    **mount() method:**
    1. Create `div.algorithm-explorer` container inside config.container.
    2. Create radio group: `fieldset.algorithm-explorer__radios` with `legend` "Algorithm".
       - For each ALGORITHMS entry: `label.algorithm-explorer__radio-label` containing `input[type=radio][name=algorithm]` + text span.
       - Wire `change` event on each radio to update `_selectedAlgorithm` and call `_renderParams()`.
    3. Create params container: `div.algorithm-explorer__params`.
    4. Create Run button: `button.algorithm-explorer__run` with text "Run", `data-testid="algorithm-run"`.
       - Wire `click` to `_onRun()`.
    5. Create status line: `div.algorithm-explorer__status` for showing "Running..." or result summary.
    6. Call `_renderParams()` to show initial parameter controls.

    **_renderParams() method:**
    Clear `_paramsContainer`, then based on `_selectedAlgorithm`:
    - `'community'`: Render `label` "Resolution" + `input[type=range]` min=0.1, max=10.0, step=0.1, value=`_louvainResolution`, `data-testid="louvain-resolution"`. Wire `input` event to update `_louvainResolution`. Display current value in a `span.algorithm-explorer__param-value`.
    - `'pagerank'`: Render `label` "Damping Factor" + `input[type=number]` min=0.1, max=0.99, step=0.01, value=`_pagerankAlpha`, `data-testid="pagerank-alpha"`. Wire `input` event to update `_pagerankAlpha`.
    - `'centrality'`: Render `label` "Sampling Threshold" + `input[type=number]` min=100, max=100000, step=100, value=`_centralitySamplingThreshold`, `data-testid="centrality-threshold"`. Wire `input` event to update `_centralitySamplingThreshold`. Add helper text: "Nodes > this value use sqrt(n) sampling".
    - Other algorithms: no parameter controls (params container empty).

    **_onRun() async method:**
    1. If `_running`, return (debounce).
    2. Set `_running = true`, disable Run button, update status to "Running...".
    3. Get filtered card IDs:
       - Compile current filter: `const compiled = this._filter.compile();`
       - If `compiled.where !== 'deleted_at IS NULL'` (i.e., there are active filters beyond the base):
         - Query card IDs: `const result = await this._bridge.send('db:query', { sql: 'SELECT id FROM cards WHERE ' + compiled.where, params: compiled.params });`
         - Wait — actually, we should use the bridge to get filtered IDs. Use `bridge.queryCards()` or `bridge.send('card:list', ...)`.
         - Simpler approach: `bridge.send('card:list-ids', ...)` — but this message type may not exist.
         - Best approach: use `bridge.listCards({ where: compiled.where, params: compiled.params })` — but listCards returns full Card objects.
         - Most pragmatic: pass the `where` and `params` directly to graph:compute. But CONTEXT.md says "graph:compute message includes current filtered card IDs from FilterProvider."
         - Since Plan 01 adds `cardIds?: string[]` to the payload, we need actual IDs. Query them: `await bridge.send('db:query', { sql: 'SELECT id FROM cards WHERE ' + compiled.where, params: compiled.params })`. Then extract IDs from the result rows.
         - Alternatively, check if there's a simpler path — FilterProvider.compile() always starts with `deleted_at IS NULL`. If the compiled.where is just `deleted_at IS NULL` (no additional filters), skip cardIds (let handler use all cards). If there are extra filters, query the filtered IDs.
       - Set `cardIds` to the filtered ID array (or undefined if no active filters).
    4. Build params object based on `_selectedAlgorithm`:
       - `'community'`: `{ community: { resolution: _louvainResolution } }`
       - `'pagerank'`: `{ pagerank: { alpha: _pagerankAlpha } }`
       - No special params for other algorithms.
    5. Call `await bridge.computeGraph({ algorithms: [_selectedAlgorithm], params, cardIds })`.
    6. On success:
       - Call `schema.addGraphMetricColumns()` to make columns available.
       - Call `coordinator.scheduleUpdate()` to trigger SuperGrid re-render.
       - Update status: "Computed {algorithmsComputed} — {cardCount} cards, {edgeCount} edges, {durationMs}ms".
    7. On error: update status with error message.
    8. Set `_running = false`, re-enable Run button.

    **destroy() method:** Remove the container element, null out references.

    **getSelectedAlgorithm():** Returns `_selectedAlgorithm`.

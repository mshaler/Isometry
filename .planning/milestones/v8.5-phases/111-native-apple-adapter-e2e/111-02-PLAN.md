Key insight for schema branching tests (NATV-06):
The NoteStore schema branching is a Swift-side concern (NotesAdapter.swift). The TypeScript handler
only receives pre-parsed CanonicalCard[] — it does not know about ZTITLE1 vs ZTITLE2.
The seam test for NATV-06 verifies that cards with different `name` values (representing what the
Swift adapter would extract from ZTITLE1 vs ZTITLE2) pass through the handler correctly and appear
in the cards table with the correct name. The test validates the contract boundary: if Swift gives
correct names, the handler preserves them.

Key insight for protobuf tests (NATV-07):
Similarly, protobuf extraction is Swift-side. The seam tests verify the handler preserves the content
field as-is for each tier: full body string (tier 1), snippet fallback string (tier 2), and empty
string (tier 3). No crash, no mutation.

it('NATV-04a: attendee person card creates connection row linking person to event', async () => {
  const eventCard = makeCard({
    card_type: 'event' as any,
    name: 'Team Standup',
    source: 'native_calendar',
    source_id: 'EVT-001',
    source_url: null,
    event_start: '2026-03-25T09:00:00Z',
    event_end: '2026-03-25T09:30:00Z',
  });
  const attendeeCard = makeCard({
    card_type: 'person' as any,
    name: 'Alice Smith',
    source: 'native_calendar',
    source_id: 'ATT-001',
    source_url: 'attendee-of:EVT-001',
  });

  const result = await handleETLImportNative(db, {
    sourceType: 'native_calendar',
    cards: [eventCard, attendeeCard],
  });

  expect(result.connections_created).toBeGreaterThanOrEqual(1);

  // Verify connection row exists with correct topology
  const connections = db.exec(`

/**
 * Isometry v5 -- Phase 109 ETL Test Helpers
 *
 * Extends the E2E helper library for ETL-specific operations:
 *   - importNativeCards: inject CanonicalCard[] through bridge (same path as native adapters)

 *   - resetDatabase: clear cards/connections for test isolation
 */

**importNativeCards(page, cards, sourceType)**
- Signature: `(page: Page, cards: CanonicalCard[], sourceType: string) => Promise<{ inserted: number; updated: number; errors: number }>`
- Where `CanonicalCard` is a locally-defined interface with the minimum fields needed (id, card_type, name, content, source, source_id -- plus all other fields from src/etl/types.ts CanonicalCard)
- Implementation: `page.evaluate()` calling `window.__isometry.bridge.send('etl:import-native', { cards, sourceType })` then `window.__isometry.coordinator.scheduleUpdate()`
- Wait 500ms after for render settle (matching existing pattern in isometry.ts)
- Return the ImportResult shape `{ inserted, updated, errors }`

**assertCatalogRow(page, sourceType, expectedMinCards)**
- Signature: `(page: Page, sourceType: string, expectedMinCards: number) => Promise<void>`
- Uses `window.__isometry.queryAll()` (the new API from Task 1) to:
  1. Query `SELECT * FROM import_sources WHERE source_type = ?` with params `[sourceType]` -- assert at least 1 row
  2. Query `SELECT * FROM import_runs WHERE source_type = ? ORDER BY started_at DESC LIMIT 1` with params `[sourceType]` -- assert 1 row, assert `card_count >= expectedMinCards`
  3. Query `SELECT COUNT(*) as cnt FROM cards WHERE source = ? AND deleted_at IS NULL` with params `[sourceType]` -- assert `cnt >= expectedMinCards`
- Uses Playwright `expect()` for assertions so failures produce clear test output

**resetDatabase(page)**
- Signature: `(page: Page) => Promise<void>`
- Uses `window.__isometry.exec()` to run:
  1. `DELETE FROM connections`
  2. `DELETE FROM cards`
  3. `DELETE FROM datasets` (wrapped in try/catch for table-may-not-exist)
  4. `DELETE FROM import_runs` (wrapped in try/catch)
  5. `DELETE FROM import_sources` (wrapped in try/catch)
- Then calls `window.__isometry.coordinator.scheduleUpdate()` and waits 300ms

All functions follow the existing `page.evaluate()` pattern from isometry.ts. Export a `CanonicalCard` type alias for use in downstream test files.

RESEARCH.md Pattern 4 (PAFVProvider View Family Suspension):
```typescript
setViewType(viewType: ViewType): void {
  const currentFamily = getViewFamily(this.state.viewType);
  const newFamily = getViewFamily(viewType);
  if (currentFamily !== newFamily) {
    this.suspendedStates.set(currentFamily, structuredClone(this.state));
    const restored = this.suspendedStates.get(newFamily);
    this.state = restored ? { ...structuredClone(restored), viewType } : { ...VIEW_DEFAULTS[viewType], viewType };
  } else {
    this.state.viewType = viewType;
  }
  this.scheduleNotify();
}
```

const STRFTIME_PATTERNS: Record<TimeGranularity, (field: string) => string> = {
  day:     field => `strftime('%Y-%m-%d', ${field})`,
  week:    field => `strftime('%Y-W%W', ${field})`,
  month:   field => `strftime('%Y-%m', ${field})`,
  quarter: field => `strftime('%Y', ${field}) || '-Q' || ((CAST(strftime('%m', ${field}) AS INT) - 1) / 3 + 1)`,
  year:    field => `strftime('%Y', ${field})`,
};

Internal state shape:

interface PAFVState {
  viewType: ViewType;
  xAxis: AxisMapping | null;
  yAxis: AxisMapping | null;
  groupBy: AxisMapping | null;
}

Key implementation details:
- `suspendedStates: Map<ViewFamily, PAFVState>` — stores suspended family states
- `getViewFamily(viewType: ViewType): ViewFamily` — helper: 'network'|'tree' → 'graph', everything else → 'latch'
- `setViewType(viewType)`: Check family boundary. If crossing, suspend current via structuredClone, restore or default for new family. Always call scheduleNotify().
- `compile(): CompiledAxis`: Validate all axis fields via validateAxisField(). Build orderBy from xAxis and yAxis (comma-separated if both). Build groupBy from groupBy axis. Return { orderBy, groupBy }.
- `VIEW_DEFAULTS`: Map of ViewType → default PAFVState. At minimum: list defaults to sort by modified_at desc, kanban defaults to groupBy status.
- Subscribe pattern: identical to FilterProvider (Set + queueMicrotask + pendingNotify guard)
- Implements PersistableProvider (toJSON, setState, resetToDefaults)

IMPORTANT: When compiling orderBy, format as `field ASC` or `field DESC` (no ORDER BY keyword — QueryBuilder adds that). Same for groupBy — just the field name, no GROUP BY keyword.

  <action>
Create `src/providers/QueryBuilder.ts`:

```typescript
interface CompiledQuery {
  sql: string;
  params: unknown[];
}

class QueryBuilder {
  constructor(
    private filter: FilterProvider,
    private axis: PAFVProvider,
    private density: DensityProvider
  ) {}

  buildCardQuery(options?: { limit?: number; offset?: number; includeDeleted?: boolean }): CompiledQuery;
  buildCountQuery(): CompiledQuery;
  buildGroupedQuery(): CompiledQuery;
}
```

Implementation notes:
- `buildCardQuery()`:
  1. Start with `SELECT * FROM cards`
  2. Call filter.compile() → get {where, params}. If includeDeleted, strip 'deleted_at IS NULL' from where (or compile without it — FilterProvider always includes it, so QueryBuilder may need to handle this).
  3. Append `WHERE ${where}` (filter.compile() always returns at least 'deleted_at IS NULL')
  4. Call axis.compile() → get {orderBy}. If non-empty, append `ORDER BY ${orderBy}`
  5. If limit, append `LIMIT ?` and add to params. If offset, append `OFFSET ?`.
  6. Return {sql, params}

- For `includeDeleted`: The simplest approach is to replace 'deleted_at IS NULL' with '1=1' or omit the WHERE entirely if no other filters. Since FilterProvider always includes 'deleted_at IS NULL', QueryBuilder should handle this by either: (a) asking FilterProvider to compile without the base clause, or (b) stripping it. Prefer option (b): strip 'deleted_at IS NULL AND ' or 'deleted_at IS NULL' from the where string.

- `buildCountQuery()`: Same WHERE as buildCardQuery but `SELECT COUNT(*) as count FROM cards WHERE ...`. No ORDER BY. No LIMIT/OFFSET.

- `buildGroupedQuery()`: Uses axis.compile().groupBy for GROUP BY. If density.compile().groupExpr is available and the query involves time grouping, include it. Return query with GROUP BY.

CRITICAL: QueryBuilder does NOT accept raw SQL strings. No `rawSql` parameter, no `customWhere` escape hatch. Every SQL fragment comes from a provider compile().

Tests use mock providers (objects with compile() returning specific values) — no real Database needed.

  <action>
Create `src/providers/StateManager.ts`:

```typescript
class StateManager {
  private providers = new Map<string, PersistableProvider>();
  private debounceTimers = new Map<string, ReturnType<typeof setTimeout>>();
  private autoPersistUnsubs = new Map<string, () => void>();

  constructor(
    private bridge: WorkerBridge,
    private debounceMs: number = 500
  ) {}

  registerProvider(key: string, provider: PersistableProvider): void;
  markDirty(key: string): void;
  async persist(key: string): Promise<void>;
  async persistAll(): Promise<void>;
  async restore(): Promise<void>;
  enableAutoPersist(): void;
  disableAutoPersist(): void;
  destroy(): void;
}
```

Key implementation:
- `markDirty(key)`: Clear existing timer for key, set new setTimeout at debounceMs. On fire: `this.persist(key)`.
- `persist(key)`: Get provider, call toJSON(), send to bridge: `this.bridge.send('ui:set', { key, value: provider.toJSON() })`.
- `restore()`: Call `this.bridge.send('ui:getAll', {})`. For each returned row, find registered provider by key. Try JSON.parse(value) and provider.setState(parsed). Catch: console.warn, provider.resetToDefaults().
- `enableAutoPersist()`: For each registered provider, subscribe and markDirty on change. Store unsubscribe functions.
- `destroy()`: Clear all debounce timers. Call all autoPersist unsubscribe functions.

IMPORTANT: StateManager requires WorkerBridge. However, WorkerBridge.send() is currently private. Plan 02 extended the protocol types but the public API methods for ui:* are not on WorkerBridge yet. StateManager should use a `send` method. Two approaches:
1. Make WorkerBridge.send() public (or add a public `sendRaw` method)
2. Add convenience methods to WorkerBridge for ui operations

Recommendation: Make the `send` method on WorkerBridge public so that StateManager and MutationManager can use `bridge.send('ui:set', {...})` directly. This is the simplest approach and follows the existing pattern. Add this change to this task.

For tests: Mock the WorkerBridge. StateManager tests do NOT need a real Worker — mock bridge.send() to return test data. Use vi.useFakeTimers() for debounce testing.

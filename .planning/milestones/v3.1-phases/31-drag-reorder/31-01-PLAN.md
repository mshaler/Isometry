  - level = axis level index (0-based)
  - parentPath = ancestor values joined by \x1f (empty string for level 0)
  - value = the specific header value at that level

    1. Add `reorderColAxes(fromIndex: number, toIndex: number): void` and `reorderRowAxes(fromIndex: number, toIndex: number): void` to SuperGridProviderLike interface in `src/views/types.ts`.

    2. Implement both methods in PAFVProvider:
       - Guard: fromIndex === toIndex → return (no-op, no notify)
       - Guard: fromIndex or toIndex out of bounds → return (no-op, no notify)
       - Splice: copy axes array, splice(fromIndex, 1), splice(toIndex, 0, moved)

       - For axisCount >= 3: clear all collapse state (return []). Document: "Pragmatic simplification — parentPath encoding makes surgical remap error-prone at 3+ levels. User's collapsed headers reset on 3+ level reorder."

    4. Update ALL inline SuperGridProviderLike objects in tests/views/SuperGrid.test.ts to include the two new methods (reorderColAxes, reorderRowAxes as no-ops).

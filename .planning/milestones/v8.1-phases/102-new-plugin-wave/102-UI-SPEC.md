## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap in toolbar controls; chevron margin |
| sm | 8px | Plugin row padding gap; toolbar item gap; search input inner padding |
| md | 16px | Toolbar area vertical padding; density segmented button gap; toolbar flex gap |
| lg | 24px | Harness sidebar section padding; config panel padding |
| xl | 32px | Header bar padding (matches existing .pv-header padding: 16px 24px) |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Density cell heights deviate from 8-pt scale by design: compact=24px, normal=32px, comfortable=48px, spacious=64px — these are content-driven row heights, not spacing gaps.
- Lasso overlay div: tracks pointer position at sub-pixel level; border is 1px (below scale minimum).
- Audit left-border stripe: 3px width (matches existing .pv-agg-cell `border-left: 3px solid var(--pv-accent)` pattern).
- Source provenance left-border stripe: 3px width (consistent with audit stripe pattern).

### Accent Reserved For (explicit list)

The `--pv-accent` color is reserved exclusively for:
1. Active/pressed state of density mode-switch segmented buttons
2. Count badge background in active state (`.pv-count-badge--active`)
3. Active plugin indicator in harness sidebar (existing `.hns-badge-active-bg`)
4. Search input focus ring border

The `--pv-accent` is NOT used for: hover states on cells, lasso drag overlay, audit tints, or source stripes.

Destructive: `--danger` (`#ff4a4a` dark / `#dc2626` light). No destructive actions in this phase — no confirmations needed.

## Component Inventory

All new UI components in this phase are DOM fragments injected via `afterRender()`. No new HTML files or top-level view scaffolding.

### SuperDensity: Toolbar Segmented Button Group

```
.pv-density-toolbar            — flex row, injected into toolbar area above grid
  .pv-density-label            — "Density" uppercase label, 11px/600
  .pv-density-segmented        — flex row with 1px gap, border: 1px solid --pv-btn-border
    .pv-density-btn            — each level button (Compact / Normal / Comfortable / Spacious)
    .pv-density-btn--active    — active level, background: --pv-btn-active-bg, color: --pv-btn-active-fg
```

Anatomy:
- Button height: 28px
- Button horizontal padding: 16px (--space-md)
- Font: 11px / weight 400 (label weight reserved for uppercase section heads; buttons use body weight)
- Active button: background `--pv-btn-active-bg`, color `--pv-btn-active-fg`, no border radius on inner edges (flush segmented)
- First button: `border-radius: var(--pv-chip-radius) 0 0 var(--pv-chip-radius)`
- Last button: `border-radius: 0 var(--pv-chip-radius) var(--pv-chip-radius) 0`
- Transition: background-color 120ms ease

### SuperDensity: Mini Card Cell Layout

CSS classes applied to `.pv-data-cell` by superdensity.mini-cards at compact density:

```
.pv-cell--mini-card            — triggers compact icon+title layout
```

Layout at compact density:
- Cell uses `overflow: hidden`, `white-space: nowrap`, `text-overflow: ellipsis`
- First line: card title at 11px / weight 400
- No multi-line wrapping at compact level

### SuperDensity: Count Badge

```
.pv-count-badge                — inline in each data cell at lowest density
```

- Font: 11px monospace (`--pv-number-font`) / weight 600
- Color: `--pv-fg`
- Text-align: right
- No background, no border — numeric count replaces full card list rendering

### SuperSearch: Search Input Toolbar

```
.pv-search-toolbar             — flex row, injected into toolbar area above grid (alongside density and zoom)
  .pv-search-input             — <input type="search">, 13px, height 28px, debounced 300ms
  .pv-search-clear             — "✕" clear button, aria-label="Clear search", appears when input.value.length > 0
```

- Input width: flex-grow 1, max-width 200px
- Border: 1px solid `--pv-btn-border`
- Border-radius: `var(--pv-chip-radius)` (6px)
- Background: `--pv-cell-bg`
- Focus ring: 1px solid `--pv-accent`
- Placeholder: "Search cells...", italic, `--pv-muted-fg`

### SuperSearch: Match Highlight Class

```
.search-match                  — applied to matching .pv-data-cell elements
```

- Background: `var(--search-highlight)` (amber 40% opacity)
- Outline: `1px solid var(--search-match-outline)`
- No inline `<mark>` tags; class-only highlighting

### SuperSelect: Selection Classes

```
.selected                      — applied to selected .pv-data-cell elements
.pv-lasso-overlay              — positioned <div> created by superselect.lasso
```

`.selected` styles:
- Outline: `2px solid var(--selection-outline)` (matches existing `--sg-selection-border` pattern)
- Background: `var(--selection-bg)` (replaces cell's normal background)
- Applied via data-attribute selector: `[data-selected="true"]` with class `.selected`

`.pv-lasso-overlay` styles:
- `position: absolute` on parent grid container (grid container gets `position: relative`)
- `border: 1px solid var(--selection-outline)`
- `background: var(--selection-bg)`
- `pointer-events: none`
- `z-index: 20`
- Created/destroyed per drag gesture (not persistent in DOM)

### SuperAudit: Change Tracking Classes

```
.audit-new                     — applied to .pv-data-cell for inserted cards
.audit-modified                — applied to .pv-data-cell for updated cards
.audit-deleted                 — applied to .pv-data-cell for removed cards
```

Each class:
- `border-left: 3px solid var(--audit-{state})` (stripe, not background)
- `background: rgba` tint at 8% opacity derived from the same audit color

Dark theme tint values (pre-computed for CSS):
- `.audit-new`: `background: rgba(74, 222, 128, 0.08)`
- `.audit-modified`: `background: rgba(251, 146, 60, 0.08)`
- `.audit-deleted`: `background: rgba(248, 113, 113, 0.08)`

### SuperAudit: Source Provenance Class

```
.audit-source                  — applied to .pv-data-cell; reads source type from data attribute
```

- `border-left: 3px solid var(--source-{type})` — source color stripe
- Source type stored as `data-source="{type}"` on the cell element
- No background tint for source (avoids conflict with audit-new/modified/deleted tints)
- Source type key matches SOURCE_COLORS keys in audit-colors.ts (e.g., "apple_notes", "csv", "json")

### CSS Composability Contract

All 4 plugin categories add classes to the same `.pv-data-cell` element. The following combinations must render without visual conflict:

| Combination | Visual outcome |
|-------------|---------------|
| `.selected` + `.search-match` | Blue selection outline + amber background tint |
| `.selected` + `.audit-new` | Blue selection outline + green left stripe (stripe shows through selection fill) |
| `.audit-new` + `.search-match` | Amber background from search-match (dominant); green left stripe visible |
| `.selected` + `.audit-new` + `.search-match` | Blue outline + green stripe + amber tint |
| `.pv-cell--mini-card` + `.selected` | Compact layout + selection outline; layout unchanged |

Resolution rule: when multiple background tints conflict, `--selection-bg` wins (declared last in class application order by superselect plugin afterRender, which runs after audit/search afterRender).

## Toolbar Area Layout Contract

The toolbar area above the pivot grid hosts 3 controls injected by separate plugins. They must coexist in a single flex row:

```
.pv-toolbar                    — flex row, align-items: center, gap: 16px, padding: 8px 16px
  .pv-zoom-control             — SuperZoomSlider (existing, from Phase 100)
  .pv-density-toolbar          — SuperDensity segmented button (new, Phase 102)
  .pv-search-toolbar           — SuperSearch input (new, Phase 102)
```

Layout rules:
- Zoom control: fixed width per existing implementation
- Density toolbar: fixed width (fits 4 buttons at ~80px each + label = ~380px max)
- Search toolbar: flex-grow 1, max-width 200px — fills remaining space
- When density plugin is disabled: `.pv-density-toolbar` is removed from DOM by plugin destroy()
- When search plugin is disabled: `.pv-search-toolbar` is removed from DOM by plugin destroy()
- Overflow: toolbar wraps to second row at `< 600px` grid width (flex-wrap: wrap)

## Density Level Contract

| Level name | CSS class | Cell height | Content rendered |
|------------|-----------|-------------|-----------------|
| Compact | `.pv-density--compact` | 24px | Count badge only (superdensity.count-badge) |
| Normal | `.pv-density--normal` | 32px | Default cell rendering |
| Comfortable | `.pv-density--comfortable` | 48px | Default cell rendering, increased padding |
| Spacious | `.pv-density--spacious` | 64px | Default cell rendering, maximum padding |

Applied to the grid container element (`.pv-root` or `.pv-grid-wrapper`) so all cells inherit via CSS:

```css
.pv-density--compact  .pv-data-cell { height: 24px; padding: 0 8px; }
.pv-density--normal   .pv-data-cell { height: 32px; padding: 0 8px; }
.pv-density--comfortable .pv-data-cell { height: 48px; padding: 0 16px; }
.pv-density--spacious .pv-data-cell { height: 64px; padding: 0 16px; }
```

Default level: Normal. No density class on the container renders Normal sizing (no CSS change needed).

## Interaction States

### Density Mode Switch
| State | Visual |
|-------|--------|
| Default | All buttons unselected style (`--pv-btn-bg`, `--pv-btn-border`) |
| Active button | `--pv-btn-active-bg` fill, `--pv-btn-active-fg` text |
| Hover (non-active) | `--pv-btn-hover-bg` background |
| Focus | 1px `--pv-accent` focus ring via `:focus-visible` |

### Search Input
| State | Visual |
|-------|--------|
| Empty | Placeholder "Search cells...", no clear button |
| Has value | Clear button (✕) appears at right |
| Focused | 1px `--pv-accent` border replaces `--pv-btn-border` |
| Active filter | Matching cells get `.search-match`; non-matching cells get `opacity: 0.35` |

### Cell Selection
| State | Visual |
|-------|--------|
| Unselected | No decoration |
| Selected (click) | `.selected` class — blue outline + fill |
| Multi-selected (Cmd+click) | Same `.selected` class; multiple cells |
| Lasso dragging | `.pv-lasso-overlay` div tracks cursor; cells within bounds get `.selected` on mouseup |
| Keyboard range | Shift+arrow extends selection; new cells added to `.selected` set |
| Deselect | Click on non-cell or Escape removes all `.selected` classes |

### Audit Overlay
| State | Visual |
|-------|--------|
| Overlay disabled | No audit classes applied |
| Overlay enabled | `.audit-new`, `.audit-modified`, `.audit-deleted` applied to respective cells |
| Source disabled | No `.audit-source` or `data-source` applied |
| Source enabled | `data-source` attribute set, CSS reads `--source-*` token for border color |

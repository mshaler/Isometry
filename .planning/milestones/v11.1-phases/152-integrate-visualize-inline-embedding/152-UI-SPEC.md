## Layout Contract

This section captures the structural decisions from CONTEXT.md (D-01..D-04) as visual
requirements the executor must not deviate from.

### Top-Slot Child Structure

`.workbench-slot-top` contains three child `<div>` elements in this stacking order:

```
.workbench-slot-top
  └── .slot-top__data-explorer       ← Data Explorer (DataExplorerPanel)
  └── .slot-top__properties-explorer ← Properties Explorer (PropertiesExplorer)
  └── .slot-top__projection-explorer ← Projections Explorer (ProjectionExplorer)
```

- Child divs are `display: block; width: 100%` by default.
- Visibility is controlled by `element.style.display = 'block'` / `element.style.display = 'none'`
  (matching the existing DataExplorer toggle pattern — no CSS class toggling).
- `.workbench-slot-top` itself becomes `display: none` via inline JS when all three
  children are hidden (parent collapse collapses layout space so the view fills correctly).

### Data + Properties Toggle (INTG-01, INTG-02)

**Trigger:** Single dock item `integrate:catalog` (database icon, label "Data").

**Visible state:** Both `.slot-top__data-explorer` and `.slot-top__properties-explorer`
are `display: block`. `.workbench-slot-top` is `display: block`.

**Hidden state:** Both are `display: none`. If `.slot-top__projection-explorer` is also
`display: none`, `.workbench-slot-top` is `display: none`.

**Dock item active indicator:** When Data+Properties are shown, the `integrate:catalog`
dock item renders with `aria-pressed="true"` and the active visual indicator (left-border
accent or background highlight per existing DockNav active-item pattern).

### Projections Auto-Visibility (VIZ-01, VIZ-02, VIZ-03)

**Trigger:** View switch in `onActivateItem` visualize branch.

**SuperGrid active:** `.slot-top__projection-explorer` is `display: block`.
`.workbench-slot-top` is `display: block`.

**Any other view active:** `.slot-top__projection-explorer` is `display: none`.
`.workbench-slot-top` visibility recalculated (hide if all children hidden).

**No dock item, no manual toggle.** Projections has no toggle affordance in the dock.
The user cannot dismiss Projections while SuperGrid is active.

## Interaction States

### Dock Item: "Data" (`integrate:catalog`)

| State | Visual |
|-------|--------|
| Default (explorers hidden) | Icon + label at rest; no active indicator |
| Active (explorers shown) | `aria-pressed="true"`; left-border `3px solid var(--accent)` or background `var(--accent-bg)` per existing DockNav active pattern |
| Hover | `background: var(--cell-hover)` (existing DockNav hover pattern) |
| Focus-visible | `outline: 2px solid var(--accent); outline-offset: -2px` |

### Top-Slot Container

| State | Visual |
|-------|--------|
| One or more explorers visible | `display: block`; `border-bottom: 1px solid var(--border-subtle)` already in workbench.css |
| All explorers hidden | `display: none` — collapses entirely, view content fills available space |
| Overflow | `overflow-y: auto` (pre-existing in `.workbench-slot-top`) |

### Explorer Child Divs

No additional CSS states. Each child div inherits its explorer's existing styles
(`.data-explorer`, `.properties-explorer`, `.projection-explorer` scoped classes).
No new wrappers or visual chrome are introduced around them.

## Accessibility Contract

- `integrate:catalog` dock button must have `aria-pressed="true"` when Data+Properties
  are visible, `aria-pressed="false"` when hidden. Source: existing DockNav ARIA tablist
  pattern established in Phase 147.
- `aria-label` on the slot-top container: not required (decorative grouping, not
  a landmark).
- No new focus order changes — explorers appear above the active view in DOM source
  order, matching visual reading order top-to-bottom.
- Projections auto-visibility change must not steal focus — view switch focus management
  is unchanged.

## Component Anatomy

### 3 Collapse States

| State | CSS Modifier Class | Sidebar Width | Content |
|-------|--------------------|---------------|---------|
| Hidden | `dock-nav--hidden` | 0px | Toggle button only (floats outside or pins to edge) |
| Icon-only | `dock-nav--icon-only` | 48px | Toggle + icon + label (default layout, same as Phase 146) |
| Icon+Thumbnail | `dock-nav--icon-thumbnail` | 160px | Toggle + icon + label + 96×48 placeholder box |

State cycle on each toggle click: Hidden → Icon-only → Icon+Thumbnail → Hidden (CONTEXT.md D-03).

### Toggle Button

- Pinned to top of dock, always visible including when dock is Hidden (CONTEXT.md D-01)
- Icon: `iconSvg('panel-left', 20)` — Lucide `PanelLeft` glyph (rectangle-with-sidebar)
- Static icon — does not change between states (CONTEXT.md D-02)
- Size: 48×48px touch target
- CSS class: `dock-nav__toggle`
- Position behavior: toggle is a sibling of the main `.dock-nav__list` wrapper inside `.dock-nav`, OR a sibling of `.dock-nav` entirely. Claude's Discretion — see note in CONTEXT.md. Recommended: sibling outside `.dock-nav` so it remains visible when the nav is `display: none` in Hidden state.

### Animation

Only one animated transition (CONTEXT.md D-04, D-05):

| Transition | Property | From | To | Duration | Easing |
|-----------|----------|------|----|----------|--------|
| Hidden ↔ visible content | `grid-template-rows` | `0fr` | `1fr` | 200ms | `ease` (matches `--transition-fast`) |
| Width change (state swap) | none | — | — | instant | CSS class swap only — no `transition: width` |

Animation implementation: wrap `.dock-nav__list` in a grid container. The wrapper uses `display: grid; grid-template-rows: 0fr` (Hidden) or `grid-template-rows: 1fr` (visible). The inner div has `min-height: 0` to allow collapse. `transition: grid-template-rows 200ms ease` on the wrapper.

### Minimap Placeholder Box (Icon+Thumbnail state only)

- Dimensions: 96px wide × 48px tall
- Border: `1px dashed var(--border-subtle)`
- Background: `var(--bg-primary)`
- Positioned below icon + label inside each dock item in the 160px layout
- No content — intentionally empty until Phase 148
- CSS class: `dock-nav__item-thumb`

### 160px Item Layout (Icon+Thumbnail state)

Dock item in `dock-nav--icon-thumbnail` state uses column flex:

```
[icon 20px] [label text] — row 1 (flex row, align center, gap --space-xs)
[96×48 placeholder box] — row 2 (full width block)
```

Overall dock item: `flex-direction: column; align-items: flex-start; padding: var(--space-xs) var(--space-sm); width: 160px; height: auto;`

## Interaction Contract

### State Persistence

- Key: `dock:collapse-state`
- Values: `"hidden"` | `"icon-only"` | `"icon-thumbnail"`
- API: `StateManager.saveUiState('dock:collapse-state', value)` / `StateManager.loadUiState('dock:collapse-state')`
- Read on mount: restore persisted state before first render (CONTEXT.md D-08, REQUIREMENTS.md CLPS-04)
- Default (no persisted value): `"icon-only"`

### Keyboard Navigation — Roving Tabindex

Matches established pattern from `ListView.ts` line 125, `CommandBar.ts` line 331 (CONTEXT.md D-11):

| State | Focusable Elements | Tab stop behavior |
|-------|--------------------|-------------------|
| Hidden | Toggle button only | Toggle has `tabindex="0"`. All `.dock-nav__item` buttons have `tabindex="-1"`. |
| Icon-only | Toggle + all dock items | One dock item has `tabindex="0"` at a time (roving). |
| Icon+Thumbnail | Toggle + all dock items | Same roving pattern as Icon-only. |

**Arrow key navigation (CONTEXT.md D-10):**
- `ArrowDown`: Move focus to next actionable dock item, skipping section headers
- `ArrowUp`: Move focus to previous actionable dock item, skipping section headers
- Wrap: ArrowDown on last item moves focus to first item; ArrowUp on first moves to last
- Section headers (`.dock-nav__section-header`) receive no focus — they are `aria-hidden="true"` or `role="presentation"`

**Keyboard handler** on `.dock-nav` element (event delegation), listening for `keydown`. Filter for `.dock-nav__item` as the focused element.

### VoiceOver Announcements (CONTEXT.md D-12, D-13)

| Trigger | Announcement text | Implementation |
|---------|-------------------|----------------|
| Toggle click → Hidden | `"Hidden"` | `config.announcer.announce("Hidden")` |
| Toggle click → Icon-only | `"Icon only"` | `config.announcer.announce("Icon only")` |
| Toggle click → Icon+Thumbnail | `"Icon and thumbnail"` | `config.announcer.announce("Icon and thumbnail")` |
| Dock item activated | (already handled) | `aria-current="page"` on active button — pre-existing from Phase 146 |

The `announcer` interface is already defined in `DockNavConfig` and connected in `DockNav.ts` — Phase 147 activates it.

## ARIA Contract

| Element | ARIA attribute | Value | Notes |
|---------|----------------|-------|-------|
| `<nav>` | `role` | `navigation` | Pre-existing from Phase 146 |
| `<nav>` | `aria-label` | `"Main navigation"` | Pre-existing from Phase 146 |
| Dock item list `<ul>` | `role` | `tablist` | REQUIREMENTS.md A11Y-01 — add in Phase 147 |
| Each dock item `<button>` | `role` | `tab` | Paired with `role="tablist"` |
| Active item | `aria-current` | `"page"` | Pre-existing from Phase 146 `_setActive()` |
| Active item | `aria-selected` | `"true"` | Add alongside `aria-current` for tablist semantics |
| Inactive item | `aria-selected` | `"false"` | All non-active items |
| Section headers | `aria-hidden` | `"true"` | Skip in AT reading order and roving tabindex |
| Toggle button | `aria-label` | `"Toggle navigation"` | Static; does not change with state |
| Toggle button | `aria-expanded` | `"true"` / `"false"` | `false` when Hidden, `true` for both visible states |
| Thumbnail placeholder | `aria-hidden` | `"true"` | Decorative; no AT value |

## Component Inventory

New components introduced in this phase:

### MinimapRenderer module (`src/ui/MinimapRenderer.ts`)
- Exported function: `renderMinimap(thumbEl: HTMLElement, viewKey: string, cards: CardDatum[], pafv: PAFVState): void`
- Inserts an inline `<svg width="96" height="48">` into `thumbEl`, replacing any previous content
- Dispatches to per-view sketch functions based on `viewKey`
- Source: CONTEXT.md D-03

### Per-view sketch functions (Claude's Discretion — centralized vs per-file)
- Recommended: centralized in `MinimapRenderer.ts` as named functions (`sketchGrid`, `sketchNetwork`, `sketchTimeline`, `sketchTree`, `sketchChart`)
- Each function signature: `(svg: SVGSVGElement, cards: CardDatum[], colors: ColorMap): void`
- Draws simplified schematic shapes using actual data colors

### Loupe overlay (inline SVG element within the 96×48 thumbnail SVG)
- Two SVG `<rect>` elements implementing the inverted-dimming pattern (D-05):
  - Dimming rects: four rectangles covering the area outside the viewport, fill `--overlay-bg`
  - Viewport outline rect: clear fill, 2px stroke `--accent`, `rx: 1`
- Pointer events: `pointerdown` / `pointermove` / `pointerup` on the SVG element with `setPointerCapture` for drag-to-pan (D-04)
- Click-to-jump: `click` event on the SVG maps thumbnail coordinates to main view scroll position (D-04)
- Event delegation via DockNav's existing `_navEl` listener is preferred, but loupe pointer capture requires direct attachment to the SVG — attach to the `<svg>` element directly, not delegated

### PAFV Caption Bar (inline SVG `<g>` inside the 96×48 SVG)
- Rendered as a `<rect>` background + `<text>` pairs for each mapped axis
- Position: bottom 14px of SVG (`y: 34` to `y: 48`)
- Layout: P/A/F/V glyphs in `--accent`, field names in `--text-muted`, truncated with SVG `textLength` if overflow
- Source: CONTEXT.md D-06, D-07

## Interaction States

### Thumbnail — states across dock collapse modes

| Dock State | Thumbnail Visibility | Render Status |
|---|---|---|
| Hidden | `display: none` (existing `dock-nav__item-thumb` rule) | Not rendered, no idle callback queued |
| Icon-only | `display: none` (existing rule) | Not rendered, no idle callback queued |
| Icon+Thumbnail | `display: block` (existing rule) | Rendered lazily via `requestIdleCallback` stagger on state transition |

### Thumbnail — render lifecycle

| Trigger | Action |
|---|---|
| Dock transitions to `icon-thumbnail` state | Queue all visible dock items via `requestIdleCallback` (batch of 3 per idle callback, per D-08) |
| Dock transitions away from `icon-thumbnail` | Cancel pending idle callbacks; thumbnails remain stale in DOM |
| Filter change / PAFV change / data import / view switch while `icon-thumbnail` active | Debounced re-render of affected thumbnail, 300ms debounce (`--transition-normal` value, D-10) |
| Dock item not a visualization view | No thumbnail rendered; `dock-nav__item-thumb` remains hidden or shows a static placeholder |

### Loupe — interaction states

| State | Visual |
|---|---|
| Default | Dimming overlay rects + viewport outline visible; no cursor change |
| Hover over thumbnail | `cursor: crosshair` on the SVG |
| Pointer down (drag start) | `setPointerCapture` on SVG; `cursor: grabbing` |
| Drag in progress | Loupe viewport rect moves in real time; main view scrolls to match |
| Click (no drag) | Main view jumps to clicked position; loupe repositions instantly |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | No CTA in this phase — thumbnail renders automatically on dock expansion |
| Empty state (no data) | Thumbnail shows no sketch shapes; PAFV caption bar still renders with axis field names; no explicit text label needed |
| Empty state (view not renderable) | `dock-nav__item-thumb` stays hidden — no placeholder text shown inside thumbnail |
| Error state (render failure) | Silent fail — thumbnail remains as dashed-border placeholder from Phase 147; no error copy shown to user |
| Destructive confirmation | None — this phase has no destructive actions |
| PAFV axis glyph labels | P, A, F, V (single uppercase letter per axis; no full words) |
| Axis field name (unmapped) | `—` (em dash; do not show "None" or "null") |

## Accessibility Contract

| Concern | Contract |
|---|---|
| Thumbnail SVG accessible label | `<svg aria-hidden="true">` — thumbnail is decorative; the dock item's existing `aria-label` provides the accessible name |
| Loupe interactivity | Loupe click/drag affordances are enhancement only — keyboard nav (roving tabindex from Phase 147) already navigates views; no keyboard contract for loupe drag |
| Reduced motion | Thumbnail rendering is not animated; loupe position updates are instant (no CSS transitions on loupe rect position) |
| PAFV caption contrast | `--text-muted` on `--bg-surface` overlay must meet 4.5:1 — existing token adjustments in design-tokens.css already satisfy this for all 5 themes |

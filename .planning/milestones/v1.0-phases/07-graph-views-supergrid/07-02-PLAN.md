  <action>
Create `src/views/NetworkView.ts` implementing the IView interface for force-directed graph rendering.

**Architecture:**
- Constructor receives ViewConfig (container, coordinator, queryBuilder, bridge)
- Also accepts optional SelectionProvider for interaction (can be null for test simplicity)
- Internal state: `positionMap: Map<string, NodePosition>` for warm start
- Internal state: `svg`, `graphLayer` (inner g with zoom transform), `zoom` behavior

**mount(container):**
1. Create SVG element (class: 'network-view', viewBox matches container size)
2. Create inner `g.graph-layer` — all rendering happens inside this group
3. Create sub-groups: `g.links` (rendered first, behind), `g.nodes` (rendered second, in front)
4. Set up d3-zoom on SVG:
   ```typescript
   const zoom = d3.zoom<SVGSVGElement, unknown>()
     .scaleExtent([0.1, 8])
     .on('zoom', (event) => {
       this.graphLayer.attr('transform', event.transform.toString());
     });
   this.svg.call(zoom);
   ```
5. Store zoom reference for later programmatic reset

**render(cards):**
1. If cards is empty, clear SVG content and return
2. Build node list from cards (id, name, card_type + previous position from positionMap for warm start)
3. Fetch all connections for visible cards via `bridge.send('connection:get', ...)` — OR use a dedicated query. Since we need connections between visible cards, use `bridge.send('db:exec', { sql: 'SELECT * FROM connections WHERE source_id IN (...) OR target_id IN (...) AND deleted_at IS NULL', params })` — but this violates SQL safety. Instead, fetch connections for each card and deduplicate. For performance, use a single db:exec with parameterized IN clause (this is acceptable because the field 'source_id'/'target_id' is known safe).

   IMPORTANT: The simplest correct approach is to send a db:exec query:
   ```sql
   SELECT DISTINCT c.* FROM connections c
   WHERE (c.source_id IN (?, ?, ...) OR c.target_id IN (?, ?, ...))
   ```
   with card IDs as params. This gets all connections between visible cards.

4. Compute degree for each node (count of connections where node is source or target)
5. Build SimulatePayload with warm-start positions from positionMap
6. Send `graph:simulate` to Worker, await NodePosition[] response
7. Update positionMap with new positions
8. Render nodes (circles + labels) with D3 data join, key: `d => d.id`
9. Render edges (lines) with D3 data join, key: `d => d.source + '-' + d.target` or a link ID

**Node rendering details (from CONTEXT.md):**
- Circle: `cx`, `cy` from NodePosition, `r` from degreeScale
- Fill color: by card_type (use a simple ordinal scale with d3.schemeCategory10)
- Label: `<text>` element below circle, text = card name, font-size 10px, text-anchor middle

**Edge rendering details:**
- Line: `x1`, `y1` from source position, `x2`, `y2` from target position
- Stroke: semi-transparent gray (#666, opacity 0.4)
- No labels by default (shown on hover — added in Task 2)

**destroy():**
- Remove SVG from container
- Clear positionMap
- Set all references to null

- Render with cards calls bridge.send('graph:simulate')
- D3 data join uses key function (verify circles have correct data binding)
- Zoom behavior is attached to SVG (verify d3.zoom is called)

1. Add interaction handlers to NetworkView.ts:

**Click-to-select:**
- On node circle click event:
  - If shift key held: `selectionProvider.addToSelection(d.id)`
  - Otherwise: `selectionProvider.toggle(d.id)`
- Subscribe to SelectionProvider changes to update visual highlights:
  - Selected nodes get a thicker stroke (e.g., 3px white stroke)
  - Connected edges of selected nodes get brighter opacity

**Drag-to-pin (d3-drag):**
- Attach d3.drag() to node circles:
  ```typescript
  const drag = d3.drag<SVGCircleElement, NodeDatum>()
    .on('start', (event, d) => {
      d.fx = event.x;
      d.fy = event.y;
    })
    .on('drag', (event, d) => {
      d.fx = event.x;
      d.fy = event.y;
      // Update circle position immediately for responsiveness
      d3.select(event.sourceEvent.target as SVGCircleElement)
        .attr('cx', d.fx)
        .attr('cy', d.fy);
      // Update connected edges
      this._updateEdgesForNode(d.id, d.fx, d.fy);
    })
    .on('end', (event, d) => {
      // Node stays pinned (fx/fy remain set)
      this.positionMap.set(d.id, { id: d.id, x: d.fx!, y: d.fy!, fx: d.fx, fy: d.fy });
    });
  ```
- IMPORTANT: Drag must work WITH zoom — d3-drag and d3-zoom share pointer events. The drag handler should call `event.sourceEvent.stopPropagation()` in the 'start' handler to prevent zoom from activating during drag. Actually, d3-drag and d3-zoom coordinate automatically when both are applied to nested elements (zoom on SVG, drag on circles inside g).

**Hover dimming (immediate, no transition delay per CONTEXT.md):**
- On node circle mouseenter:
  - Get this node's ID
  - Find all connected node IDs (from the links data)
  - Set all non-connected nodes to opacity 0.2
  - Set all non-connected edges to opacity 0.1
  - Keep hovered node and its connections at full opacity
  - No transition duration — immediate (per CONTEXT.md "Hover dimming should be immediate")
- On node circle mouseleave:
  - Restore all nodes/edges to default opacity (1.0 for nodes, 0.4 for edges)

**Edge hover label:**
- On edge line mouseenter: show connection label as a `<title>` element (SVG tooltip) or a temporary `<text>` element near the edge midpoint
- On edge line mouseleave: hide label

# Milestone v4.2: Polish + QoL

**Status:** SHIPPED 2026-03-07
**Phases:** 42-47
**Total Plans:** 15

## Overview

Clean up build pipeline, fill UX gaps (empty states, keyboard shortcuts, visual polish), harden stability, and validate end-to-end ETL across all sources and views -- dev-ready foundation for next feature milestone.

## Phases

- [x] 42-02: Fix Xcode Run Script input path, provisioning profile checkpoint

### Phase 46: Stability + Error Handling

## Milestone Summary

**Key Decisions:**
- Biome 2.x uses `includes` key (not `include`) in files config
- 8 Biome lint rules disabled for tsconfig strictness compatibility
- ShortcutRegistry uses single keydown listener (eliminates duplicated guard logic)
- AuditOverlay Shift+A shortcut not migrated to registry (owns its own lifecycle)
- ViewSwitchReceiver extracted as ViewModifier for SwiftUI type-checker performance
- 12px font-size mapped to --text-sm (11px) as closest token value
- rgba(0,0,0,0.2) in import-toast kept as-is (no exact token match)
- Error categories ordered: parse > database > network > import > unknown
- 20-card subset per source for rendering matrix (keeps 81-combo test fast)
- DedupEngine connection dedup: pre-check existing connections (SQLite UNIQUE ignores NULL)

**Issues Resolved:**
- Fixed Xcode Run Script inputPaths ($(SRCROOT)/../../package.json)
- Fixed provisioning profile with CloudKit + iCloud Documents entitlements
- Eliminated all hardcoded inline colors/font-sizes across all view JS files
- Fixed DedupEngine connection dedup for NULL via_card_id

**Issues Deferred:**
- CMD-01: Command palette (Cmd+K) with fuzzy search
- A11Y-01: Full WCAG 2.1 AA accessibility audit
- THEME-01: Light mode / system theme preference
- SMART-01: Smart per-view empty state CTAs

**Technical Debt Incurred:**
- audit-colors.ts retains hardcoded hex values (documented mapping to CSS tokens)
- 9px sort badge and 8px chevron kept as literal px (below --text-xs 10px token scale)
- Teal drag-over accent rgba(0,150,136,0.18) kept as-is (distinct from --selection-bg)
- Test assertions check var(--token) strings directly (jsdom cannot resolve CSS custom properties)

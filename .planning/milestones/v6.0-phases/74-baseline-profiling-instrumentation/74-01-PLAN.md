  <files>src/profiling/PerfTrace.ts, src/vite-env.d.ts, vite.config.ts, vite.config.native.ts, tests/profiling/PerfTrace.test.ts</files>
  <behavior>
    - startTrace('test') creates a performance.mark('test:start')
    - endTrace('test') creates a performance.mark('test:end') and performance.measure('test', 'test:start', 'test:end')
    - getTraces('test') returns PerformanceEntry array with duration > 0
    - When __PERF_INSTRUMENTATION__ is false, startTrace/endTrace/getTraces are no-ops (return [] for getTraces)
    - Multiple traces with different names do not interfere
  </behavior>
  <action>
    1. Create `src/profiling/PerfTrace.ts`:
       - `declare const __PERF_INSTRUMENTATION__: boolean;` at top
       - `startTrace(name: string): void` — guards on `__PERF_INSTRUMENTATION__`, calls `performance.mark(\`${name}:start\`)`
       - `endTrace(name: string): void` — guards on `__PERF_INSTRUMENTATION__`, calls `performance.mark(\`${name}:end\`)` then `performance.measure(name, \`${name}:start\`, \`${name}:end\`)`
       - `getTraces(name: string): PerformanceEntry[]` — guards on `__PERF_INSTRUMENTATION__`, returns `performance.getEntriesByName(name, 'measure')` or `[]`
       - `clearTraces(): void` — guards on `__PERF_INSTRUMENTATION__`, calls `performance.clearMarks()` and `performance.clearMeasures()`

    2. Add to `src/vite-env.d.ts`:
       - `declare const __PERF_INSTRUMENTATION__: boolean;`

    3. Add `define` to both `vite.config.ts` and `vite.config.native.ts`:
       ```
       define: {
         __PERF_INSTRUMENTATION__: process.env.NODE_ENV !== 'production',
       }
       ```
       In vite.config.ts: add inside the defineConfig object at top level.
       In vite.config.native.ts: add inside the defineConfig object at top level.

    4. Create `tests/profiling/PerfTrace.test.ts`:
       - Test startTrace + endTrace + getTraces returns entry with duration >= 0
       - Test getTraces for non-existent name returns empty array
       - Test clearTraces removes all marks and measures
       - Note: In test environment, `__PERF_INSTRUMENTATION__` is true (vitest runs with NODE_ENV=test, not production). Add `globalThis.__PERF_INSTRUMENTATION__ = true` in a vi.stubGlobal or define in vitest.config if needed.
  </action>

  </verify>

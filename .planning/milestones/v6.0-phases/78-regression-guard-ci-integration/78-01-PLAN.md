## Goal

Add a 4th GitHub Actions job (`bench`) to `.github/workflows/ci.yml` that runs `npm run bench:budgets` on every push, with `continue-on-error: true` as a soft gate and `timeout-minutes: 5` for safety.

## Context

- Existing CI has 3 parallel jobs: `typecheck`, `lint`, `test` — all triggered on `push` to any branch
- `bench:budgets` npm script already exists and runs exactly 2 budget test files in isolation
- Budget tests use `PerfBudget.ts` constants as CI-relative thresholds (jsdom overhead baked in)
- No `needs:` dependency — bench runs in parallel with all other jobs for maximum signal per PR

## Tasks

<task id="78-01-01">
Add a `bench` job to `.github/workflows/ci.yml` after the `test` job. The job must:
- Use `runs-on: ubuntu-latest`
- Set `timeout-minutes: 5` at the job level (NOT step level)
- Set `continue-on-error: true` (soft gate — see promotion procedure in 78-02)
- Add a YAML comment above `continue-on-error` explaining the promotion procedure: "SOFT GATE: flip to `false` after 3 consecutive green runs on main. See PROJECT.md Performance Contracts for promotion procedure."
- Steps: `actions/checkout@v5` → `actions/setup-node@v6` (node 22, npm cache) → `npm ci` → `npm run bench:budgets`
- No `needs:` dependency — runs in parallel with typecheck, lint, test
- No artifact upload — pass/fail is sufficient
</task>

<task id="78-01-02">
Run `npm run bench:budgets` locally to confirm budget tests pass before the CI wiring is considered complete. All assertions must be green.
</task>

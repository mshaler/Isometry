- Sorts: "Accepts column references with direction (ASC/DESC); compiles to the SQL ORDER BY clause, ordered by chip position."
Include compilation targets, composition rules (AND for Filters per FE-RG-03, lexicographic by chip order for Sorts per FE-RG-04, DAG for Calculations per FE-RG-05 — reference WA-2 for algorithm, do NOT include pseudocode per D-04/Pitfall 3).
Out-of-scope list: "Formulas Explorer never owns GROUP BY" (FE-RG-01), plus any other prohibitions from handoff.

**§ Marks Explorer (view-layer):** One chip well category:
- Conditional encoding: "Accepts boolean predicates over row columns; produces a Map&lt;rowId, string[]&gt; of CSS class assignments via post-query annotation; never modifies row membership."
Compilation target: post-query CSS class annotation. Composition rules.
Out-of-scope list: "Marks never alters row membership" (FE-RG-07), "Theme palette/typography/banding live in theme config, NOT in Marks" (FE-RG-09).

**§ Audits Explorer (semantic-flag layer):** Two chip well categories:
- Anomaly rules: "Accepts boolean predicates that identify statistically or semantically anomalous rows; produces flag annotations per row via post-query annotation; never excludes rows."
- Validation rules: "Accepts boolean predicates that identify data-quality violations; produces warning annotations per row via post-query annotation; never excludes rows."
Compilation target: post-query flag annotation. Composition rules.
Out-of-scope list: "Audits never excludes rows" (FE-RG-08).

**§ Cross-Explorer Composition:** Fixed SQL clause order table (Calculations → SELECT, Filters → WHERE, Sorts → ORDER BY, GROUP BY from view explorer per FE-RG-01). **§ Anti-Features (SPEC-05, per D-03 format):** Bullet + one-sentence rationale + FE-RG cross-reference for each:
1. OR-composition across filter chips — "Composition is commutative and idempotent at chip level; OR semantics require a single chip's DSL" — FE-RG-03
2. Marks altering result set row membership — "Marks are view-layer; any chip that excludes rows is a Filter" — FE-RG-07
3. Audits excluding rows from result set — "Audits surface attention; excluding rows is Filter behavior" — FE-RG-08
4. FormulasProvider owning GROUP BY — "Grouping comes from view explorers; same calc chip produces different SQL per view context" — FE-RG-01
5. Inline cell formula editing — "Cards have rich content with DSL/SQL/notes; formulas live in chip wells, not cell-edit affordances" — FE-RG-06
6. Automatic chip-to-card promotion — "Prevents accidental library accumulation; promotion is always explicit ('Save as Formula')" — FE-RG-10
7. Theme palette/typography in Marks — "Predicates must work across themes without rewriting; Marks decide class, theme decides appearance" — FE-RG-09
8. Formulas Explorer as the A in LATCH — "LATCH letters are organizational axes; Formulas is an orthogonal operator surface" — FE-RG-11

**§ Discussion.md Example Placement (SPEC-06):** Table with columns: Item in discussion.md | Explorer / Category | Rationale.

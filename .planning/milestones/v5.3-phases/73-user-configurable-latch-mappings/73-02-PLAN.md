## Goal

Add LATCH family reassignment dropdown, field disable/enable UI, SchemaProvider subscriber, and footer buttons to PropertiesExplorer. Disabled fields remain visible in their LATCH column with greyed-out styling (using `getAllAxisColumns()` from Plan 01). This plan modifies PropertiesExplorer only; boot wiring, persistence restoration, and tests are in Plan 03.

## must_haves

1. Each property row in PropertiesExplorer has a chip badge showing the current LATCH family letter (e.g., `[C]`)
2. Chip badge opens a native `<select>` dropdown listing all 5 LATCH families; the heuristic-assigned family shows "(default)" suffix
3. Selecting a different family calls `schemaProvider.setOverrides()` with the updated map and persists via `bridge.send('ui:set', { key: 'latch:overrides', ... })`
4. Selecting the "(default)" family clears the override for that field (per-field reset)
5. Overridden fields show a dot indicator on the chip badge (`[C*]` or CSS `::after` dot)
6. Disabling a field (unchecking its checkbox) calls `schemaProvider.setDisabled()`, removes active filters on that field via `FilterProvider`, and persists via `bridge.send('ui:set', { key: 'latch:disabled', ... })`
7. Disabled fields shown greyed out in place within their LATCH column (reduced opacity, `--disabled` class) — uses `getAllAxisColumns()` NOT `getAxisColumns()`
8. PropertiesExplorer footer has "Reset all LATCH mappings" button (visible only when overrides exist) with `window.confirm()` dialog
9. PropertiesExplorer footer has "Enable all" button (visible only when disabled fields exist)
10. PropertiesExplorer subscribes to SchemaProvider; on notification, rebuilds column field membership (`_rebuildColumnFields()` using `getAllAxisColumns()`) so fields move between LATCH columns while disabled fields remain visible
11. `PropertiesExplorerConfig` gains `bridge?: WorkerBridgeLike` and `filter?: FilterProvider` properties

```typescript
private async _persistOverrides(): Promise<void> {
  if (!this._config.bridge) return;
  const record: Record<string, string> = {};
  for (const [field, family] of this._config.schema!.getOverrides()) {
    record[field] = family;

    value: JSON.stringify(record),

private async _persistDisabled(): Promise<void> {
  if (!this._config.bridge) return;
  const arr = [...this._config.schema!.getDisabledFields()];
  await this._config.bridge.send('ui:set', {
    key: 'latch:disabled',
    value: JSON.stringify(arr),
  });
}
```

   ```typescript
   private _handleToggle(field: AxisField): void {
     if (this._enabledFields.has(field)) {
       this._enabledFields.delete(field);
       // Sync to SchemaProvider disabled set
       if (this._config.schema) {
         const disabled = new Set(this._config.schema.getDisabledFields());
         disabled.add(field);
         this._config.schema.setDisabled(disabled);

       // Clear active filters for this field
       if (this._config.filter) {

           if (filters[i]!.field === field) this._config.filter.removeFilter(i);
         }
         this._config.filter.clearRangeFilter(field);
         if (this._config.filter.hasAxisFilter(field)) {
           this._config.filter.setAxisFilter(field, []);

       if (this._config.schema) {
         const disabled = new Set(this._config.schema.getDisabledFields());
         disabled.delete(field);
         this._config.schema.setDisabled(disabled);
       }
       void this._persistDisabled();
     }

     this._renderColumns();
     for (const cb of this._subscribers) {
       cb();
     }
   }
   ```

<details>
Add a private `_rebuildColumnFields()` method that re-derives which fields belong to each LATCH column.

**CRITICAL:** Use `getAllAxisColumns()` (NOT `getAxisColumns()`) so that disabled fields remain visible in their LATCH column (greyed out). `getAxisColumns()` excludes disabled fields, which would cause them to vanish entirely — violating the locked decision: "Disabled fields shown greyed out in place within their LATCH column."

```typescript
private _rebuildColumnFields(): void {
  for (const col of this._columns) {
    // getAllAxisColumns() includes disabled fields with override-applied latchFamily
    const allFields: AxisField[] = this._config.schema?.initialized
      ? this._config.schema.getAllAxisColumns().map(c => c.name as AxisField)
      : [...ALLOWED_AXIS_FIELDS];
    col.fields = allFields.filter(f => getLatchFamily(f) === col.family);

    // Update empty state
    const emptyEl = col.bodyEl.querySelector('.properties-explorer__empty');
    if (col.fields.length === 0 && !emptyEl) {
      const e = document.createElement('div');
      e.className = 'properties-explorer__empty';
      e.textContent = 'No properties';
      col.bodyEl.appendChild(e);
    } else if (col.fields.length > 0 && emptyEl) {
      emptyEl.remove();
    }
  }

  // Sync _enabledFields set from SchemaProvider disabled state
  if (this._config.schema?.initialized) {
    const disabled = this._config.schema.getDisabledFields();
    // Use getAllAxisColumns() to include disabled fields in the sync
    const allCols = this._config.schema.getAllAxisColumns();
    for (const c of allCols) {
      if (!disabled.has(c.name)) {
        this._enabledFields.add(c.name as AxisField);
      }
    }
    for (const f of disabled) {
      this._enabledFields.delete(f as AxisField);
    }
  }
}
```

## Goal

Wire boot-time restoration of LATCH overrides and disabled fields from `ui_state` in `main.ts`, connect `bridge` and `filter` to PropertiesExplorer config, wire LatchExplorers remount and ProjectionExplorer update on SchemaProvider change notifications, and write comprehensive tests for the Plan 02 UI. Ensures overrides persist across sessions (UCFG-03) and LatchExplorers reflects changes immediately (UCFG-04).

## must_haves

1. `main.ts` boot sequence restores `latch:overrides` and `latch:disabled` from `ui:get` after `await bridge.isReady`, calling `schemaProvider.setOverrides()` and `schemaProvider.setDisabled()`
2. `main.ts` wires `bridge` and `filter` to PropertiesExplorer config
3. LatchExplorers reflects overrides automatically via `destroy()` + `mount()` remount triggered by SchemaProvider subscriber in `main.ts`
4. ProjectionExplorer reflects disabled fields automatically via `update()` call triggered by SchemaProvider subscriber in `main.ts`
5. Tests cover: chip badge rendering, dropdown options with "(default)", family change handler, disable field handler with FilterProvider cleanup, footer button visibility, reset all with confirmation, enable all, column field rebuild after override, disabled field greyed-out in place

**1. Restore overrides at boot** — after `await bridge.isReady` and after SchemaProvider wiring (after `setLatchSchemaProvider(schemaProvider)` around line 122), add override restoration:

```typescript
// Phase 73: Restore LATCH overrides and disabled fields from ui_state
{
  const overridesRow = await bridge.send('ui:get', { key: 'latch:overrides' }) as { value?: string };
  if (overridesRow?.value) {
    try {
      const parsed = JSON.parse(overridesRow.value) as Record<string, string>;
      schemaProvider.setOverrides(new Map(Object.entries(parsed)) as Map<string, import('./worker/protocol').LatchFamily>);
    } catch { /* ignore corrupt data */ }
  }
  const disabledRow = await bridge.send('ui:get', { key: 'latch:disabled' }) as { value?: string };
  if (disabledRow?.value) {
    try {
      const parsed = JSON.parse(disabledRow.value) as string[];
      schemaProvider.setDisabled(new Set(parsed));
    } catch { /* ignore corrupt data */ }
  }
}
```

**2. Wire bridge and filter to PropertiesExplorer config** — modify the PropertiesExplorer instantiation (around line 647):
```typescript
const propertiesExplorer = new PropertiesExplorer({
  alias,
  schema: schemaProvider,
  container: propertiesBody!,
  bridge,                    // NEW: for ui:set/ui:get persistence
  filter,                    // NEW: for clearing filters on disable
  onCountChange: (_count) => {},
});
```

**3. Wire LatchExplorers remount on SchemaProvider change** — after `latchExplorers.mount(latchBody!)` (around line 684), add:
```typescript
// Phase 73: Remount LatchExplorers when LATCH overrides change
// (fields move between family sections, requiring full DOM rebuild)
schemaProvider.subscribe(() => {
  latchExplorers.destroy();
  latchExplorers.mount(latchBody!);
});
```

Note: This is a full destroy+remount which is acceptable because LATCH override changes are rare user-initiated events. CollapsibleSection collapse state persists to localStorage so it survives remount.

**4. Wire ProjectionExplorer update on SchemaProvider change** — after `projectionExplorer.mount()`:
```typescript
// Phase 73: Update ProjectionExplorer when disabled fields change
schemaProvider.subscribe(() => projectionExplorer.update());
```

**Tests to write:**

1. **Chip badge renders with current LATCH family letter**
   - Mount PropertiesExplorer with schema
   - Assert: each property row contains a `select.prop-latch-chip__select` element
   - Assert: the select's value matches the field's LATCH family

2. **Dropdown options include all 5 families with "(default)" suffix on heuristic**
   - Mount, find the select for field "priority" (Hierarchy)
   - Assert: 5 options present
   - Assert: the Hierarchy option text contains "(default)"
   - Assert: other options do NOT contain "(default)"

3. **Selecting new family triggers override and persistence**
   - Mount with bridge mock
   - Change select value for "priority" to "Category"
   - Dispatch change event
   - Assert: `schemaProvider.getLatchOverride('priority') === 'Category'`
   - Assert: bridge mock received `ui:set` call with key `latch:overrides`

4. **Selecting "(default)" family clears override**
   - Set initial override for "priority" to Category
   - Change select back to "Hierarchy" (the default)
   - Assert: `schemaProvider.getLatchOverride('priority') === undefined`

5. **Override indicator: chip has data-overridden="true"**
   - Override priority to Category
   - Re-render
   - Assert: chip wrapper for priority has `data-overridden="true"`

6. **Disable field: triggers SchemaProvider.setDisabled + FilterProvider cleanup**
   - Mount with filter mock
   - Uncheck "priority" checkbox
   - Assert: `schemaProvider.getDisabledFields().has('priority')`
   - Assert: filter mock's removeFilter or clearRangeFilter was called for "priority"

7. **Disabled field remains visible greyed-out in LATCH column**
   - Mount with schema, disable "priority"
   - Assert: "priority" row still exists in the Hierarchy column DOM
   - Assert: row has class `properties-explorer__row--disabled`
   - Assert: checkbox is unchecked

8. **Column field rebuild: field moves between columns after override**
   - Mount with "priority" in Hierarchy column
   - Override to Category
   - Allow SchemaProvider subscriber to fire
   - Assert: "priority" no longer in Hierarchy column fields, now in Category column fields

9. **"Reset all LATCH mappings" button: visible only with overrides, confirms, resets**
   - Mount, assert reset button hidden
   - Set override, trigger re-render
   - Assert reset button visible
   - (Can't easily test window.confirm in vitest, so test the underlying handler with schema state assertion)

10. **"Enable all" button: visible only with disabled fields, re-enables all**
    - Disable a field, assert enable button visible
    - Click enable button
    - Assert: `schemaProvider.hasAnyDisabled() === false`

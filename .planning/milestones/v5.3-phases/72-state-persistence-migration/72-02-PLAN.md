## Goal

AliasProvider preserves aliases for fields not in the current schema. When a column disappears from the database and returns in a future import, its alias re-activates automatically. The `isValidAxisField()` gate in `setState()` is removed so any string key is accepted.

## must_haves

1. `AliasProvider.setState()` accepts any string key (removes `isValidAxisField()` gate)
2. Aliases for fields not in the current schema are preserved in the internal `_aliases` Map
3. `getAlias()` returns stored alias regardless of current schema validity
4. The existing test `'setState ignores invalid fields'` is updated to reflect orphan-preservation behavior
5. New test verifies orphan preservation across schema changes
6. The `isValidAxisField` import can be removed from AliasProvider if no other usage remains

<title>Remove isValidAxisField gate from AliasProvider.setState()</title>
<type>code</type>
<files>src/providers/AliasProvider.ts</files>
<details>
In `AliasProvider.setState()` (line 89-98), change the inner loop condition:

**Before:**
```typescript
if (isValidAxisField(key) && typeof value === 'string') {
    this._aliases.set(key, value);
}
```

**After:**
```typescript
if (typeof value === 'string') {
    this._aliases.set(key as AxisField, value);
}
```

The cast `key as AxisField` is safe because AxisField is `KnownAxisField | (string & {})` (widened in Phase 71-01), so any string is structurally valid.

Also remove the `isValidAxisField` import from line 12 if it is the only usage in the file. Check that `setAlias()` and `clearAlias()` do NOT use `isValidAxisField` -- they accept typed `AxisField` parameters which already accept any string.
</details>
</task>

<title>Update and extend AliasProvider tests for orphan preservation</title>
<type>test</type>
<files>tests/providers/AliasProvider.test.ts</files>
<details>
1. **Update existing test** `'setState ignores invalid fields'` (line 110-114):

   **Before:** Asserts `bogus_field` is dropped (only `folder` alias present).
   **After:** Asserts `bogus_field` is preserved alongside `folder`:
   ```typescript
   it('setState preserves aliases for unknown fields (orphan preservation)', () => {
       const provider = new AliasProvider();
       provider.setState({ bogus_field: 'Bad', folder: 'Good' });
       expect(provider.getAlias('folder')).toBe('Good');
       expect(provider.getAlias('bogus_field' as any)).toBe('Bad');
   });
   ```

2. **Add orphan round-trip test:**
   ```typescript
   it('orphan aliases survive toJSON/setState round-trip', () => {
       const provider = new AliasProvider();
       provider.setAlias('folder', 'Project');
       // Simulate orphan: set state with a field that exists + one that does not
       provider.setState({ folder: 'Project', vanished_column: 'Old Name' });

       const json = provider.toJSON();
       const restored = new AliasProvider();
       restored.setState(JSON.parse(json));

       expect(restored.getAlias('folder')).toBe('Project');
       expect(restored.getAlias('vanished_column' as any)).toBe('Old Name');
   });
   ```

3. **Add test confirming getAlias returns orphan alias:**
   ```typescript
   it('getAlias returns stored alias for fields not in schema', () => {
       const provider = new AliasProvider();
       provider.setState({ unknown_field: 'Custom Name' });
       expect(provider.getAlias('unknown_field' as any)).toBe('Custom Name');
   });
   ```

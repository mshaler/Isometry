## Copywriting Contract

Phase 130 introduces no new UI text surfaces. The copywriting contract is carried from the existing system with no additions.

| Element | Copy |
|---------|------|
| Primary CTA | not applicable — no new CTAs in this phase |
| Empty state heading | not applicable — no new empty states |
| Empty state body | not applicable |
| Error state | not applicable — migration failures are silent-log only (no user-visible error for key rename) |
| Destructive confirmation | not applicable — no destructive user actions |

**Migration transparency note (FNDX-01):** The boot-time key migration (flat keys renamed to namespaced keys) is invisible to the user. No toast, no progress indicator, no confirmation. It completes synchronously before the first view renders.

**isSwitching guard note (FNDX-02):** Silently dropped notifications produce no user-visible change — the newly mounted view fetches fresh data on mount. No loading state or indicator is required.

**Preset key reservation note (FNDX-03):** `preset:` prefix rejection in StateManager.registerProvider() is a developer-facing guard (throws or logs), not a user-facing message. No UI copy required.

Create `src/presets/LayoutPresetManager.ts`:

Constructor accepts:
- `getSectionStates: () => Map<string, boolean>` — from WorkbenchShell
- `restoreSectionStates: (states: Map<string, boolean>) => void` — from WorkbenchShell
- `bridge: { send(cmd: string, args: Record<string, unknown>): Promise<unknown> }` — for ui_state persistence

Methods:
- `async loadCustomPresets(): Promise<void>` — reads all `preset:name:*` keys from ui_state via `ui:getAll`, populates internal Map. Called at boot.
- `listAll(): Array<{ name: string; isBuiltIn: boolean }>` — returns built-in + custom presets sorted (built-in first, then custom alphabetically)
- `getPreset(name: string): Record<string, boolean> | null` — returns panel dict for built-in or custom preset
- `isBuiltIn(name: string): boolean` — checks against BUILT_IN_PRESETS names
- `applyPreset(name: string): Map<string, boolean> | null` — gets preset dict, converts to Map, calls restoreSectionStates, returns the previous states (from getSectionStates) for undo. Returns null if preset not found.
- `async saveCustom(name: string, states?: Record<string, boolean>): Promise<void>` — if states not provided, reads current via getSectionStates and converts Map to Record. Persists to ui_state as `preset:name:{name}` with JSON.stringify value.
- `async deleteCustom(name: string): Promise<boolean>` — no-op returning false if built-in. Deletes `preset:name:{name}` from ui_state. Returns true on success.
- `async setAssociation(datasetId: string, presetName: string): Promise<void>` — persists `preset:association:{datasetId}` = presetName to ui_state
- `async getAssociation(datasetId: string): Promise<string | null>` — reads `preset:association:{datasetId}` from ui_state, returns null if not found
- `captureCurrentState(): Record<string, boolean>` — calls getSectionStates, converts Map to Record for serialization

Key serialization format (per D-10, PRST-04): `Record<storageKey, boolean>` where keys are the 6 section storageKeys and values are collapsed booleans. Stored as JSON string in ui_state.

Storage key conventions:
- Custom preset data: `preset:name:{presetName}`
- Dataset association: `preset:association:{datasetId}`

## Interaction Contracts

### Sidebar Resize (D-01, D-02)

- Drag handle: 8px-wide `div.workbench-sidebar__resize-handle` positioned at the right edge of `.workbench-sidebar`
- Cursor: `col-resize` on handle hover
- Min width: 200px; max width: 480px
- Default width before first user resize: 280px
- Persistence: store as integer pixels in `ui_state` key `sidebar:width`
- Resize is live (no ghost/snap): update `--sidebar-width` CSS custom property on `mousemove`, commit to `ui_state` on `mouseup`
- Transition: `none` during active drag; restore `transition` on `mouseup`

### Sidebar Section Collapse (D-03)

- Multiple sections can be open simultaneously — no accordion constraint
- Collapse/expand via click on `.sidebar-section__header`
- Chevron rotates 90deg when open (existing CSS `data-state="visible"`)
- Per-section open/closed state persisted via `CollapsibleSection` `storageKey` in `ui_state` (already implemented)

### ProjectionExplorer Overflow Fix (D-08)

- Drop zone chip container: `overflow-y: auto` with explicit `max-height` so chips scroll within the well
- Action buttons below drop zones must not be obscured
- Chip container max-height: `120px` per well (4 wells stack vertically inside sidebar)

### Lucide Icon Rendering (D-12)

- Inline SVG pattern: each icon is a `<svg>` element injected via `innerHTML` with `aria-hidden="true"` and `focusable="false"`
- Size: `width: 14px; height: 14px` for sidebar section icons; `width: 12px; height: 12px` for leaf item icons
- Stroke: `currentColor` — inherits from parent text color token
- Stroke-width: `1.5` (Lucide default — monoline appearance)
- No sprite sheet: each icon is a standalone SVG string constant in a shared `icons.ts` module

### App Icon Swap (D-11)

- Element: `.workbench-command-bar__app-icon` button (28x28px)
- Replace `textContent = '◆'` with `<img>` or inline `<svg>` of the isometric cube app icon
- Static asset path: `assets/app-icon.png` or `assets/app-icon.svg` (PNG preferred for initial implementation per CONTEXT.md)
- Size: `width: 20px; height: 20px` inside the 28px button container (4px padding on each side)
- Alt attribute: `""` (decorative — button already has accessible label via aria-label or title)

### SuperGrid Zoom (D-06)

- `--sg-zoom` CSS custom property set on `.pv-root` element by `SuperPositionProvider`
- All zoom-dependent sizes use `calc(base * var(--sg-zoom, 1))` — do not hardcode pixel values
- Zoom range: 0.5 to 2.0 (existing provider contract)

### SuperGrid Density (D-07)

- `SuperDensityProvider` Extent/View level classes applied to `.pv-root` or `.sg-root`
- 1x / 2x / 5x size levels must apply row height, cell padding, and font size scaling
- Implementation references existing `--sg-cell-padding-spreadsheet` and `--sg-cell-padding-matrix` tokens

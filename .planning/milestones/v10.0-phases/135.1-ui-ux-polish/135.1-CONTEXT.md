# Phase 135.1: UI/UX Polish - Context

**Gathered:** 2026-03-31
**Status:** Ready for planning

<domain>
## Phase Boundary

Consolidate the dual navigation surfaces (SidebarNav + panel-rail) into a single resizable sidebar, fix 7 SuperGrid styling regressions from the plugin architecture migration, replace the diamond icon with the app icon asset, and migrate all emoji icons to a consistent SVG icon set.

</domain>

<decisions>
## Implementation Decisions

### Navigation Consolidation
- **D-01:** Sidebar absorbs panel-rail. The `.workbench-panel-rail` and its 6 CollapsibleSections move into `.workbench-sidebar`. The main area (`.workbench-main`) becomes 100% visualization — panel-rail is deleted.
- **D-02:** Sidebar is resizable via drag handle. Width persisted in ui_state. No fixed width — user controls it.
- **D-03:** All explorer sections stack vertically and scroll inside the sidebar. Multiple sections can be open simultaneously (no accordion constraint). Same behavior as current panel-rail, just relocated.

### SuperGrid Styling Regressions
- **D-04:** Style unification — SuperGrid CSS diverged during plugin migration. Colors, typography, sizing, themed appearance must be reconnected to the Designer Workbench design token system (`--text-*`, `--color-*` from `design-tokens.css`). Not new tokens — reconnect existing ones.
- **D-05:** Data cell and header cell styling — same token/theme mismatch as D-04. Cells must match the DW main window appearance.
- **D-06:** Zoom broken — SuperPositionProvider / CSS custom property zoom (`--sg-zoom`) not functioning. Must be restored.
- **D-07:** Size broken — 1x/2x/5x density size levels not applying (SuperDensityProvider Extent/View levels). Must be restored.
- **D-08:** Projection Explorer drop zone overflow — chips spill over action buttons below the drop zones. Scroll containment broken. Will be fixed as part of sidebar migration but the containment logic itself needs repair.
- **D-09:** Existing `supergrid.css` and `design-tokens.css` already define `--sg-*` custom properties — the fix is reconnecting SuperGrid's plugin-era CSS classes to the existing token system, not creating new tokens.

### Explorer Panel Positioning
- **D-10:** Fully covered by the sidebar migration (D-01 through D-03). No additional positioning changes needed. Explorers live in the sidebar; view gets 100% of main area.

### Icon Refresh
- **D-11:** Replace `◆` (U+25C6 diamond) in CommandBar with the app icon as a PNG or SVG static asset. The app icon is a 3D isometric grid cube. Simple image swap in `CommandBar.ts` line 69.
- **D-12:** Replace all emoji icons across SidebarNav section definitions and WorkbenchShell section configs with a consistent SVG icon set (Lucide or similar monoline style). Covers: section headers, leaf items, action buttons.

### Claude's Discretion
- Default sidebar width before user resizes (reasonable starting point, e.g., 280-320px)
- Specific Lucide icon mappings for each section/action (pick icons that best match current emoji semantics)
- Order of operations for the 7 SuperGrid fixes (may batch related token fixes together)
- Whether to inline SVG or use an icon sprite/component pattern

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `CollapsibleSection` — Already handles expand/collapse/loading states with `storageKey` persistence. Can be reused directly inside the sidebar without modification.
- `design-tokens.css` — Full token system with `--sg-*` SuperGrid properties already defined. Regression fix is reconnection, not creation.
- `SidebarNav` — Has the section definition structure and keyboard accessibility (roving tabindex). Will be extended to host explorer content.

### Established Patterns
- CSS custom property theming (`data-theme` attribute + token palettes) — SuperGrid must adopt the same pattern
- `ui_state` table for persisting UI preferences — sidebar width will use this same mechanism
- CollapsibleSection `storageKey` convention — explorer sections already persist open/collapsed state

### Integration Points
- `WorkbenchShell.constructor()` — Panel-rail creation logic (lines 98-123) needs to move into sidebar
- `main.ts` — Wires SidebarNav and WorkbenchShell together; will need updated wiring
- `src/main.ts` — Explorer mounting currently targets panel-rail sections; must retarget to sidebar sections
- ViewManager mounts into `.workbench-view-content` — unchanged, but gains full width once panel-rail is removed

</code_context>

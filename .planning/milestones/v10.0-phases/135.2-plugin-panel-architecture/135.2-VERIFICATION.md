### Human Verification Required

The following behaviors require visual inspection in a running app. All automated checks pass.

#### 1. Icon Strip and Drawer Open/Close

**Test:** Run `npm run dev`, observe the 40px icon strip between sidebar and visualization. Click a panel icon.
**Expected:** Drawer slides open with a CSS width transition. Panel content (e.g. Properties Explorer) appears inside a CollapsibleSection. Icon button becomes highlighted (accent color).
**Why human:** CSS transition timing and visual layout cannot be verified programmatically.

#### 2. Multiple Panels Simultaneously Open

**Test:** Click Properties icon, then click Projection icon.
**Expected:** Both panels appear stacked in the drawer with independent CollapsibleSection headers. Neither displaces the other.
**Why human:** DOM order and visual stacking requires visual confirmation.

#### 3. Chip Well Overflow in Drawer Context

**Test:** Open Projection panel in drawer. Drag many fields into a single chip well.
**Expected:** Chip well scrolls at 120px height. Action buttons below the well remain visible.
**Why human:** Overflow scroll behavior requires actual DOM interaction with content.

#### 4. Drag-to-Reorder on Icon Strip

**Test:** Press and hold a panel icon for >150ms, then drag to a different position.
**Expected:** Icon moves to new position. Release commits the order. Reload page and verify order persists.
**Why human:** Pointer timing, drag feedback, and persistence round-trip require manual interaction.

#### 5. Resize Handle

**Test:** Drag the 8px resize handle between drawer and view content.
**Expected:** Drawer width changes smoothly between 200px and 480px. Resize persists after page reload.
**Why human:** Pixel-level resize clamping and persistence require manual interaction.

#### 6. Sidebar Has No Explorer Sections

**Test:** Open the app and inspect the sidebar.
**Expected:** Sidebar shows only: Data Explorer, Visualization Explorer, GRAPH Explorers, Formula Explorer, Interface Builder. No Properties, Projection, or LATCH toggle sections.
**Why human:** Sidebar rendering requires visual confirmation.

#### 7. DataExplorer Sidebar Toggle

**Test:** Click "Data Explorer" section in sidebar.
**Expected:** DataExplorer catalog appears in its dedicated container (not inside the panel drawer). Closing that section hides it.
**Why human:** Requires confirming the DataExplorer is NOT rendered in the drawer (separate DOM container).

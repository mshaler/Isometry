## Spacing Scale

All spacing uses existing project tokens from `src/styles/design-tokens.css`. These are project-locked — do not introduce new values.

| Token | Value | Usage |
|-------|-------|-------|
| --space-xs | 4px | Icon strip icon padding, chip well label padding |
| --space-sm | 8px | Icon strip gap, drawer section header padding |
| --space-md | 12px | Drawer section body padding |
| --space-lg | 16px | Drawer outer padding |
| --space-xl | 24px | Panel stack gap |

Exceptions:
- `--space-md: 12px` is a pre-existing project token from `src/styles/design-tokens.css` — value pre-dates this phase, confirmed as intentional project deviation from 4/8/16/24/32/48/64 standard set.
- Icon strip total width: 40px (D-04, D-05 from CONTEXT.md) — this is a fixed layout dimension, not a spacing token
- Chip well max-height: 120px (D-10 from CONTEXT.md) — fixed overflow constraint
- Drag handle hit target: 8px wide, 100% height — meets minimum interactive target requirement
- Default drawer width before user resize: 300px (midpoint of 280-320px range from Claude's Discretion)
- Drawer min-width: 200px — prevents unusable collapsed state
- Drawer max-width: 480px — prevents drawer dominating layout

Source: CONTEXT.md D-04, D-06, D-10; `src/styles/design-tokens.css`

## Typography

All type uses existing project tokens from `src/styles/design-tokens.css`. Do not introduce new sizes.

| Role | Size | Weight | Line Height | Token |
|------|------|--------|-------------|-------|
| Panel section heading | 14px | 600 (semibold) | 1.4 | var(--text-md) |
| Panel body / label | 11px | 400 (regular) | 1.4 | var(--text-sm) |
| Panel metadata / badge | 10px | 400 (regular) | 1.2 | var(--text-xs) |

Notes:
- Only 2 weights used: 400 (regular) and 600 (semibold) — matches project convention
- Icon strip icons render at 16px (iconSvg name, 16) — size matches current sidebar section icon usage at 14px, bumped 2px for standalone strip context
- Panel section headings follow existing `.collapsible-section` heading pattern (var(--text-md), weight 600)
- Panel body (11px) and panel metadata/badge (10px) are 1px apart in size. Roles are distinguishable by both size AND color: panel body uses `var(--text-primary)` while metadata/badge uses `var(--text-secondary)`, ensuring the two roles never rely on size difference alone.

## Color

All color uses existing project tokens from `src/styles/design-tokens.css`. Multi-theme compatible — never use raw hex in new CSS; always use var(--token-name).

**Primary visual anchor:** the visualization view (flex: 1, fills remaining space). Secondary anchor: active panel icon in the icon strip (accent color).

| Role | Dark Token | Light Token | Usage |
|------|-----------|-------------|-------|
| Dominant (60%) | var(--bg-primary) #1a1a2e | #ffffff | Visualization view background |
| Secondary (30%) | var(--bg-surface) #252540 | #eeeeef | Drawer surface, icon strip background |
| Accent (10%) | var(--accent) #4a9eff | #2563eb | Active panel icon, drag handle hover, focus outlines |
| Destructive | var(--danger) #ff4a4a | #dc2626 | Not used in this phase |

Accent reserved for:
1. Active panel icon in the icon strip (icon whose panel is currently open)
2. Focus-visible outline on icon strip buttons (2px solid var(--accent))
3. Drag handle hover state highlight
4. CollapsibleSection focus ring within the drawer (pre-existing pattern, unchanged)

Icon strip states:
- Inactive icon: color var(--text-secondary), background transparent
- Active icon: color var(--accent), background var(--accent-bg)
- Hover (inactive): background var(--cell-hover)
- Hover (active): background var(--accent-bg), color var(--accent-hover)
- Focus-visible: outline 2px solid var(--accent), outline-offset -2px

Drawer drag handle:
- Default: background var(--border-subtle)
- Hover/drag: background var(--accent-border)

Chip well overflow (D-10):
- Well body: max-height 120px, overflow-y auto
- Scrollbar uses browser default — no custom scrollbar styling

Source: CONTEXT.md D-04, D-05, D-08, D-10; `src/styles/design-tokens.css`

## Layout Contract

This section is specific to Phase 135.2's structural changes.

### Workbench Body Layout (replacing `.workbench-panel-rail`)

Current layout (being replaced):
```
.workbench-shell
  .workbench-body
    .workbench-sidebar (200px fixed)
    .workbench-main
      .workbench-panel-rail (max-height 40vh, scrollable)
      .workbench-view (flex: 1)
```

New layout:
```
.workbench-shell
  .workbench-body
    .workbench-sidebar (200px fixed — navigation only, simplified)
    .panel-icon-strip (40px fixed — always visible)
    .panel-drawer (var(--panel-drawer-width, 300px) — collapsible, resizable)
    .panel-drawer__resize-handle (8px — drag target)
    .workbench-view (flex: 1 — visualization)
```

### Icon Strip

- Width: 40px fixed, full height of `.workbench-body`
- Background: var(--bg-surface)
- Right border: 1px solid var(--border-subtle)
- Each icon button: 40px × 40px, centered icon at 16px
- Icons stacked vertically, ordered by `panel-drawer:order` (persisted in ui_state)
- Drag-to-reorder: pointer events (not HTML5 DnD — avoids DnD ghost image limitations in constrained layout)

### Panel Drawer

- Width: CSS custom property `--panel-drawer-width` initialized to 300px, updated on drag
- Min-width: 200px, max-width: 480px
- Background: var(--bg-surface)
- Right border: 1px solid var(--border-subtle)
- When closed (no active panels): width collapses to 0, icon strip remains visible
- Open/close animation: CSS `width` transition, var(--transition-normal) 300ms ease
- Overflow: hidden on drawer root, auto on inner scroll container

### Drawer Panel Stack

- Each active panel rendered as a `CollapsibleSection` (reusing existing component)
- Panels stack vertically in order
- Inner container: overflow-y auto, height 100% of drawer
- CollapsibleSection storageKey convention: `panel-section:{panelId}`

### Resize Handle

- Element: `.panel-drawer__resize-handle`
- Width: 8px, full height
- Background: transparent default, var(--border-subtle) on hover after 150ms delay
- Cursor: col-resize
- Pointer events: captures pointerdown for resize drag initiation

Source: CONTEXT.md D-04, D-05, D-06, D-07, D-08, D-09; `src/styles/workbench.css`

## Interaction Contracts

### Panel Toggle (Icon Strip Click)

1. User clicks icon strip button for panel P
2. If panel P is closed: open drawer (if closed), mount panel P, add CollapsibleSection to drawer, highlight icon with accent
3. If panel P is open: unmount panel P, remove CollapsibleSection, deactivate icon. If no panels remain open, close drawer (width → 0)
4. PanelRegistry broadcasts `update()` to all currently mounted panels after toggle completes

### Drawer Resize (Drag Handle)

1. User presses pointer on `.panel-drawer__resize-handle`
2. Pointer capture set on handle element
3. On pointermove: compute new width = handleX - drawerLeft, clamp to [200, 480]
4. Set `--panel-drawer-width` on `.panel-drawer` element
5. On pointerup: persist new width to ui_state key `panel-drawer:width`

### Icon Strip Reorder (Drag-to-Reorder)

1. User presses pointer on icon strip button
2. After 150ms threshold without release: begin drag mode — show drag indicator
3. On pointermove: update visual order of icons (CSS `order` or DOM reorder)
4. On pointerup: commit new order, persist to ui_state key `panel-drawer:order`
5. Short press (< 150ms): treated as toggle (see Panel Toggle above)

### Provider Update Broadcast

- When SchemaProvider reloads or PAFVProvider axis changes: `PanelRegistry.broadcastUpdate()` calls `update()` on all mounted panels
- Each panel's `update()` is responsible for re-querying its data — no central coordinator subscription required

### CollapsibleSection Within Drawer

- Existing `CollapsibleSection` collapse/expand behavior unchanged
- storageKey: `panel-section:{panelId}` (e.g., `panel-section:properties`)
- Each section independently collapsible within the drawer

## Accessibility Contract

- All icon strip buttons: `<button>` elements with `aria-label` and `aria-pressed` (true when panel is open)
- Icon strip buttons use `role="toolbar"` container with `aria-label="Panel controls"` and roving tabindex
- Drawer resize handle: `role="separator"` with `aria-orientation="vertical"`, `aria-valuenow` (current width px), `aria-valuemin="200"`, `aria-valuemax="480"`
- Resize handle keyboard: ArrowLeft/ArrowRight adjust width by 16px steps; Home/End snap to min/max
- CollapsibleSection within drawer: pre-existing roving tabindex and focus management unchanged
- Focus remains on icon strip button after toggle (do not move focus into drawer on panel open)
- Drag-to-reorder: keyboard not required in this phase (power-user feature)

Source: WCAG 2.1 AA; existing project accessibility patterns from `src/styles/accessibility.css`

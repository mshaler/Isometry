# Phase 135.2 Plan 02: Integration — WorkbenchShell + SidebarNav + Explorer Registration Summary

**One-liner:** Rewired WorkbenchShell to use PanelDrawer (no panel-rail), registered all 6 explorer panels via PanelRegistry with lazy factories, simplified SidebarNav to navigation-only, and fixed ProjectionExplorer chip well overflow with 120px max-height.

## What Was Built

### WorkbenchShell.ts (rewritten)

Old architecture: SECTION_CONFIGS array, 6 CollapsibleSection instances, panel-rail DOM, getSectionBody/setSectionState/collapseAll/getPanelRailEl methods.

New architecture:
- `_panelDrawer: PanelDrawer` — mounts icon strip + collapsible drawer + resize handle
- `_dataExplorerEl: HTMLElement` — dedicated container for DataExplorer (not a panel plugin)
- `getDataExplorerEl()` — new accessor for main.ts DataExplorer mounting
- `getPanelDrawer()` — new accessor for init() call and future control
- `getSectionStates()` / `restoreSectionStates()` — retained as stubs for LayoutPresetManager backward compat
- DOM layout: sidebar | icon-strip | drawer | resize-handle | data-explorer | view-content

### workbench.css

Removed `.workbench-main` and `.workbench-panel-rail` rules. Added `.workbench-data-explorer { flex-shrink: 0; overflow-y: auto; }` (display managed by JS).

### main.ts

- `PanelRegistry` created before WorkbenchShell, passed via config
- `await shell.getPanelDrawer().init()` called after shell construction
- 6 `panelRegistry.register()` calls with lazy factories:
  1. `properties` — PropertiesExplorer (defaultEnabled: true)
  2. `projection` — ProjectionExplorer (defaultEnabled: true)
  3. `latch` — LatchExplorers (defaultEnabled: true)
  4. `notebook` — NotebookExplorer (defaultEnabled: false)
  5. `calc` — CalcExplorer (defaultEnabled: false)
  6. `algorithm` — AlgorithmExplorer (defaultEnabled: false)
- `coordinator.subscribe(() => panelRegistry.broadcastUpdate())` — D-03 wired
- `schemaProvider.subscribe()` — broadcastUpdate + projectionExplorer.update()
- DataExplorer: mounts into `shell.getDataExplorerEl()`, shown/hidden by `dataExplorerVisible` flag
- Focus mode shortcut (Cmd+\\) kept as no-op (panel drawer owns panel visibility)
- `algorithmExplorer` and `calcExplorer` typed as `| null` (lazy-initialized)

### SidebarNav.ts

Removed `properties`, `projection`, `latch` from SECTION_DEFS — these are now toggled via the icon strip. Removed `LEAF_SECTION_KEYS` constant and the leaf section branch in `_toggleSection`. Sidebar is now navigation-only with 5 sections: data-explorer, visualization, graph, formula, interface-builder.

### projection-explorer.css

Added `max-height: 120px; overflow-y: auto;` to `.projection-explorer__well-body`. Action buttons below each chip well now stay visible even with many chips assigned (D-10).

### Auto-fixed Issues

**1. [Rule 1 - Bug] Guarded propertiesExplorer.getDepth() in SuperGrid factory**
- **Found during:** Task 1
- **Issue:** SuperGrid factory at boot referenced `propertiesExplorer.getDepth()` but explorer is now lazy-initialized (created when Properties panel first opens, not at boot)
- **Fix:** Changed to `propertiesExplorer?.getDepth() ?? 1` (optional chain + default)
- **Files modified:** src/main.ts
- **Commit:** bca7cd80

**2. [Rule 1 - Bug] Changed calcExplorer/algorithmExplorer to nullable types**
- **Found during:** Task 1
- **Issue:** Both forward-declared without initializer (`let calcExplorer: CalcExplorer`) but referenced in window.__isometry before factory runs — TypeScript TS2454 error
- **Fix:** Changed to `let calcExplorer: CalcExplorer | null = null` and `algorithmExplorer: AlgorithmExplorer | null = null`
- **Files modified:** src/main.ts
- **Commit:** bca7cd80

**3. [Rule 2 - Missing functionality] Null-safe algorithmExplorer.nodeClicked() in NetworkView factory**

- **Files modified:** src/main.ts
- **Commit:** bca7cd80

**4. [Rule 2 - Missing functionality] Track data explorer visibility with local flag**

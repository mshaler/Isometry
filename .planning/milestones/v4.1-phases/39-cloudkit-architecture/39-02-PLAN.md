  <name>Task 2: Create SyncManager actor and wire into app lifecycle</name>
  <files>native/Isometry/Isometry/SyncManager.swift, native/Isometry/Isometry/IsometryApp.swift</files>
  <action>
Create SyncManager.swift as a dedicated actor conforming to CKSyncEngineDelegate:

1. **Properties:**
   - `private var syncEngine: CKSyncEngine?`
   - `private let stateURL: URL` -- Application Support/Isometry/sync-state.data
   - `private let queueURL: URL` -- Application Support/Isometry/sync-queue.json
   - `private var pendingChanges: [PendingChange]` -- in-memory copy of offline queue
   - `private let logger = Logger(subsystem: "works.isometry.app", category: "Sync")`
   - `weak var bridgeManager: BridgeManager?` -- for sending incoming records to JS (used in Plan 39-03)

2. **Initialization:**
   - `init(appSupportDir: URL)` -- stores stateURL and queueURL paths, loads pendingChanges from queueURL
   - `func initialize()` -- creates CKSyncEngine with:
     - `CKContainer.default().privateCloudDatabase`
     - Loaded state serialization (nil on first launch)
     - `self` as delegate
   - Log initialization status

3. **CKSyncEngineDelegate -- handleEvent:**
   Handle these event cases:
   - `.stateUpdate(let update)` -- persist `update.stateSerialization` immediately via `persistStateSerialization()` (SYNC-03: this is how change tokens survive restart)
   - `.accountChange(let change)` -- handle `.signIn` (ensure zone exists), `.signOut` (clear state), `.switchAccounts` (clear and re-initialize). Log each case.
   - `.fetchedRecordZoneChanges(let changes)` -- collect fetched records and deletions. For now, log them (Plan 39-03 will wire these to BridgeManager.sendSyncNotification). Store this as a TODO comment: `// TODO(39-03): Forward fetched records to JS via BridgeManager`
   - `.sentRecordZoneChanges(let sent)` -- handle successful sends: remove from pendingChanges, persist queue. Handle failures: log errors, leave in queue for retry. Archive system fields of successfully sent records for future conflict resolution via `encodeSystemFields(with:)`.
   - `.fetchedDatabaseChanges(let changes)` -- handle zone deletions: `.purged` (clear local state), `.encryptedDataReset` (re-upload all), `.deleted` (normal cleanup). Log each case.
   - Default: break (ignore unknown events)

4. **CKSyncEngineDelegate -- nextRecordZoneChangeBatch:**
   - Filter pendingChanges for the requested zone
   - Create CKSyncEngine.RecordZoneChangeBatch with:
     - For "save" operations: create CKRecord with appropriate record type and fields
     - For "delete" operations: create CKRecord.ID for deletion
   - Return the batch (or nil if no pending changes)

5. **State persistence (SYNC-03):**
   - `persistStateSerialization(_ serialization: CKSyncEngine.State.Serialization)` -- use NSKeyedArchiver.archivedData(withRootObject:requiringSecureCoding:true), write atomically to stateURL
   - `loadStateSerialization() -> CKSyncEngine.State.Serialization?` -- read from stateURL, unarchive with NSKeyedUnarchiver

6. **Offline queue persistence (SYNC-10):**
   - `func addPendingChange(_ change: PendingChange)` -- append to pendingChanges array, persist to queueURL, add to CKSyncEngine via `syncEngine?.state.add(pendingRecordZoneChanges:)`
   - `persistQueue()` -- JSONEncoder encode pendingChanges, write atomically to queueURL
   - `loadQueue() -> [PendingChange]` -- JSONDecoder from queueURL, return empty array if file missing
   - On initialization, load queue from disk and add any pending items to CKSyncEngine state

7. **System fields archival (Pitfall 2 prevention):**
   - `private var archivedSystemFields: [String: Data]` -- maps recordName to archived system fields
   - `private let systemFieldsURL: URL` -- Application Support/Isometry/record-metadata.json
   - After successful fetch or send, archive the CKRecord's system fields via `record.encodeSystemFields(with: NSKeyedArchiver)` and store in the dictionary
   - Before creating a CKRecord for upload, check for archived system fields and restore via `CKRecord(coder: NSKeyedUnarchiver)` to preserve server metadata

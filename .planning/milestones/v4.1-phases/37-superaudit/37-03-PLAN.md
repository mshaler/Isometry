    4. Modify `src/main.ts`:
       - Import { auditState, AuditOverlay } from './audit'
       - After ViewManager creation (step 6), before step 7:
         - Create AuditOverlay instance and mount it on the container
       - After ImportToast setup (step 9), modify the import result handling:
         - The current bridge.onnotification handles import_progress. But import RESULTS come back as return values from bridge.importFile() and bridge.importNative().
         - Since these methods are called from external code (native bridge or dev tools via window.__isometry.bridge), we need to wrap them.
         - Create wrapper functions that intercept the ImportResult and feed it to auditState:
           ```typescript
           const originalImportFile = bridge.importFile.bind(bridge);
           bridge.importFile = async (source, data, options) => {
             const result = await originalImportFile(source, data, options);
             auditState.addImportResult(result, source);
             return result;
           };
           const originalImportNative = bridge.importNative.bind(bridge);
           bridge.importNative = async (sourceType, cards) => {
             const result = await originalImportNative(sourceType, cards);
             auditState.addImportResult(result, sourceType);
             return result;
           };
           ```
         - This approach avoids modifying WorkerBridge.ts — the wiring is in the app entry point.
       - Add auditState to the window.__isometry exposure (step 10)

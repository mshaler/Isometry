# Plan 84-01: Wire aggregation and displayField into superGridQuery

## Objective

ProjectionExplorer exposes Aggregation and Display Field controls that write to PAFVProvider and SuperDensityProvider respectively. SuperGridQuery already has typed aggregation and displayField fields. But _fetchAndRender() never reads or passes them — the query always runs with default 'count'/'name'. Wire the existing state through to the query.

<tasks>

<task id="1">
**Extend SuperGridProviderLike interface**

In `src/views/types.ts`, add `getAggregation(): AggregationMode` to the `SuperGridProviderLike` interface. `AggregationMode` is already imported at line 124 — no new imports needed. PAFVProvider already implements this method at line 592, so it will satisfy the contract without changes.
</task>

<task id="2">
**Update _noOpDensityProvider stub**

In `src/views/SuperGrid.ts`, add `displayField: undefined` to the `_noOpDensityProvider.getState()` return object (around line 84). This ensures the no-op stub satisfies the full `SuperDensityState` shape when displayField is accessed.
</task>

<task id="3">
**Read and spread projectionOpt in _fetchAndRender()**

After the existing `searchTermOpt` pattern (around line 1368), add a `projectionOpt` block that reads `this._provider.getAggregation()` and `densityState.displayField`. Only spread when aggregation !== 'count' to preserve backward compatibility:

```typescript
const projectionOpt: { aggregation?: AggregationMode; displayField?: AxisField } = {};
const aggregation = this._provider.getAggregation();
if (aggregation !== 'count') {
  projectionOpt.aggregation = aggregation;
  const displayField = densityState.displayField;
  if (displayField) projectionOpt.displayField = displayField;
}
```

Add `...projectionOpt` to the superGridQuery() call args.
</task>

<task id="4">
**Verify PAFVProvider.setAggregation() notifies**

Check PAFVProvider.ts around line 603. Confirm setAggregation() calls this._scheduleNotify() after mutating state. If absent, add it — without this, _fetchAndRender() won't fire when aggregation changes.
</task>

<task id="5">
**Add three behavioral tests**

```typescript
it('passes aggregation=sum from provider into superGridQuery')
it('passes displayField from densityState when aggregation is non-count')
it('omits aggregation/displayField spread when aggregation is count (default)')
```

Use the existing SuperGrid test harness with mock provider/bridge. Assert on bridge.superGridQuery call args.
</task>

</tasks>

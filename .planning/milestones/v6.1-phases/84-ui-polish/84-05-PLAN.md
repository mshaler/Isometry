# Plan 84-05: HistogramScrubber inline error state

## Objective

_fetchAndRender() catches errors, logs to console, and falls through to _render([]). A failed query renders an empty histogram — visually identical to a legitimately empty dataset. Replace the silent fallback with an inline error element and Retry button.

In `src/ui/HistogramScrubber.ts`, add:

```typescript

private _showError(message: string): void {
  this._render([]);  // clear stale chart
  if (!this._errorEl) {
    // Lazy-create error element with message span + Retry button
    // Retry button calls _clearError() then void _fetchAndRender()
    // Append to _wrapperEl
  }
  // Update message text, show element
}

private _clearError(): void {
  if (this._errorEl) this._errorEl.style.display = 'none';
}
```

CSS classes: `.histogram-scrubber__error`, `.histogram-scrubber__error-msg`, `.histogram-scrubber__retry`
</task>

<task id="2">
**Update _fetchAndRender() error path**

```typescript
try {
  // ... existing fetch logic ...
```

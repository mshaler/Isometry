**Bridge spy pattern** (from CONTEXT.md locked decision):
```typescript
const bridgeSpy = vi.fn();
coordinator.subscribe(() => {
  const compiled = filter.compile();
  const densityState = density.getState();
  bridgeSpy({ ...compiled, densityState });
});
```

**describe('CORD-01: filter change propagates to bridge params'):**
1. Single filter change → bridge spy called once with correct filter params
   - `filter.setAxisFilter('folder', ['Work'])` → flush → spy called 1x
   - Assert `call.where` contains 'folder' and `call.params` contains 'Work'
2. Different filter type also propagates
   - `filter.setAxisFilter('status', ['active'])` → flush → spy called 1x
   - Assert `call.where` contains 'status'

**describe('CORD-02: rapid filter changes batch into one re-query'):**
3. 3 distinct synchronous filter mutations → exactly 1 spy call with final state
   - `filter.setAxisFilter('folder', ['Film'])` then `['Film', 'Award']` then `['Film', 'Award', 'Doc']`
   - Flush → assert `toHaveBeenCalledTimes(1)`
   - Assert final call params contain all 3 values (Film, Award, Doc)

**describe('CORD-03: destroy prevents stale re-queries'):**
4. After destroy, filter mutation does not trigger spy
   - Subscribe spy → `coordinator.destroy()` → `filter.setAxisFilter(...)` → `density.setHideEmpty(true)`
   - `vi.advanceTimersByTime(100)` + `await Promise.resolve()` x2
   - Assert `bridgeSpy.not.toHaveBeenCalled()`

5. Destroy mid-batch cancels pending callback
   - Subscribe spy → trigger filter change → flush microtasks only (2x Promise.resolve) → `coordinator.destroy()` → advance timers → assert spy NOT called

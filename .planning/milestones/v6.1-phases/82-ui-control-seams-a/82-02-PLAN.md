Helper to dispatch keyboard events:
```typescript
function fireKey(key: string, opts?: Partial<KeyboardEvent>): void {
  document.dispatchEvent(new KeyboardEvent('keydown', {
    key,
    bubbles: true,
    cancelable: true,
    ...opts,
  }));
}
```

**describe('CMDB-01: keyboard shortcuts invoke correct callbacks'):**

1. ShortcutRegistry Cmd+K fires registered handler
   - Create ShortcutRegistry, register `Cmd+K` with a vi.fn() spy

   - Assert spy called once
   - Clean up: registry.destroy()

2. ShortcutRegistry Cmd+F fires registered handler
   - Register `Cmd+F` with spy
   - `fireKey('f', { metaKey: true })`
   - Assert spy called once

3. ShortcutRegistry Escape fires registered handler
   - Register `Escape` with spy (plain key, no modifier)
   - `fireKey('Escape')`
   - Assert spy called once

4. ShortcutRegistry input field guard skips INPUT elements
   - Create an INPUT element, append to body, focus it
   - Register `Cmd+K` with spy
   - Fire keydown ON the input element (not document)
   - Trick: dispatch event from input element — the guard checks `event.target`
   - Actually: the ShortcutRegistry listener is on document, so dispatch from document but with target being input — use `input.dispatchEvent(new KeyboardEvent(...))`
   - NOTE: In jsdom, `event.target` is the element that dispatches if it bubbles. So dispatch from the input element with `bubbles: true`.
   - Assert spy NOT called

1. setRangeFilter with numeric min/max produces WHERE clause
   - `providers.filter.setRangeFilter('priority', 20, 40)`
   - `const compiled = providers.filter.compile()`
   - Assert `compiled.where` contains 'priority'
   - Query db → returns Beta, Gamma, Delta (priority 20, 30, 40)

2. setRangeFilter with tight range (single value)
   - `providers.filter.setRangeFilter('priority', 30, 30)`
   - Query db → returns only Gamma

3. setRangeFilter with full range returns all cards
   - `providers.filter.setRangeFilter('priority', 10, 50)`
   - Query db → returns all 5 cards

4. setRangeFilter with date field (due_at) produces WHERE clause
   - `providers.filter.setRangeFilter('due_at', '2026-02-15T00:00:00Z', '2026-04-25T00:00:00Z')`
   - Query db → returns Beta, Gamma, Delta (due_at between Feb and Apr inclusive)

5. setRangeFilter with tight date range (single month)
   - `providers.filter.setRangeFilter('due_at', '2026-03-01T00:00:00Z', '2026-03-31T00:00:00Z')`
   - Query db → returns only Gamma (due_at 2026-03-20)

   - `providers.filter.clearRangeFilter('priority')`
   - Query → returns all 5 cards

8. clearRangeFilter is idempotent (no error when no range exists)

   - Query → returns all 5 cards (no crash, no filter applied)

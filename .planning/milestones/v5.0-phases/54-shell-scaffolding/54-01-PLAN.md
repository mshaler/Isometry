  <files>src/ui/CollapsibleSection.ts, tests/ui/CollapsibleSection.test.ts</files>
  <behavior>
    - mount(container) creates header button + body div, appends to container
    - Header button has aria-expanded="true" by default (all sections expanded on first load per CONTEXT.md)
    - Header button has aria-controls pointing to body element id
    - Body element has role="region" and aria-labelledby pointing to header button id
    - Clicking header toggles collapsed state (body hidden, aria-expanded updated, chevron rotated)
    - Enter key on header toggles collapsed state
    - Space key on header toggles collapsed state (preventDefault to avoid scroll)
    - Chevron shows ▾ when expanded, ▸ when collapsed
    - Count badge displayed in header when provided (e.g., "Properties (12)")
    - Collapse state persists to localStorage keyed by `workbench:${storageKey}`
    - On mount, reads localStorage — if stored value exists, uses it; otherwise defaults to expanded
    - setCollapsed(true/false) programmatically sets state and persists
    - getCollapsed() returns current collapsed state
    - destroy() removes DOM element and cleans up event listeners
    - Stub content (icon + "coming soon" text) rendered inside body when provided
  </behavior>
  <action>
Write tests first in `tests/ui/CollapsibleSection.test.ts` with `// @vitest-environment jsdom` directive. Test mount/destroy lifecycle, aria-expanded toggle, keyboard operation (Enter + Space), chevron indicator, localStorage persistence (mock localStorage), count badge rendering, programmatic setCollapsed/getCollapsed, and stub content rendering.

Then implement `src/ui/CollapsibleSection.ts`:

```typescript
export interface CollapsibleSectionConfig {
  title: string;
  icon: string;           // Unicode icon for stub content
  storageKey: string;      // localStorage key suffix
  defaultCollapsed?: boolean; // Default: false (expanded)
  stubContent?: string;    // "coming soon" description text
  count?: number;          // Badge count for collapsed header
}

export class CollapsibleSection {
  mount(container: HTMLElement): void;
  setCollapsed(v: boolean): void;
  getCollapsed(): boolean;
  setCount(n: number): void;  // Update count badge dynamically
  getElement(): HTMLElement | null; // Access the root element
  destroy(): void;
}
```

DOM structure created by mount():
```html
<div class="collapsible-section" data-section="{storageKey}">
  <button class="collapsible-section__header"
          id="section-{storageKey}-header"
          aria-expanded="true"
          aria-controls="section-{storageKey}-body">
    <span class="collapsible-section__chevron">▾</span>
    <span class="collapsible-section__title">{title}</span>
    <span class="collapsible-section__count" style="display:none">(0)</span>
  </button>
  <div class="collapsible-section__body"
       id="section-{storageKey}-body"
       role="region"
       aria-labelledby="section-{storageKey}-header">
    <div class="collapsible-section__stub">
      <span class="collapsible-section__stub-icon">{icon}</span>
      <span class="collapsible-section__stub-text">{stubContent}</span>
    </div>
  </div>
</div>
```

When collapsed: add `.collapsible-section--collapsed` class to root, set aria-expanded="false", chevron shows ▸. Body uses CSS max-height transition for smooth animation.

localStorage key format: `workbench:${storageKey}` storing "true"/"false".

Import workbench.css at top of file: `import '../styles/workbench.css';`

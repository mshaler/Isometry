Build the ProjectionExplorer component with 4 horizontal projection wells (Available, X, Y, Z) where users drag property chips between wells to configure SuperGrid axes. HTML5 native DnD with duplicate rejection and minimum enforcement.

Purpose: Users configure what SuperGrid displays by dragging properties into X (columns), Y (rows), and Z (values) wells. Every change flows through PAFVProvider to trigger live re-render.
Output: ProjectionExplorer.ts, projection-explorer.css, ProjectionExplorer.test.ts

    Module-level DnD state (not on dataTransfer due to async limitations):
    - _dragSourceWell: WellId | null
    - _dragField: AxisField | null
    - _dragIndex: number | null (for within-well reorder)

    MIME type constant: MIME_PROJECTION = 'text/x-projection-field'

    mount():
    - Create .projection-explorer root with .projection-explorer__wells container
    - For each well: create well div with label and body [role="listbox"]
    - Call _renderChips()
    - Subscribe to pafv and alias providers for re-render
    - Append root to container

    _renderChips():
    - Read current state: colAxes from pafv.getState().colAxes, rowAxes from pafv.getState().rowAxes
    - Compute Available: all enabled fields (from enabledFieldsGetter()) MINUS fields in colAxes/rowAxes/zAxes
    - For each well, use D3 select(wellBody).selectAll('.projection-explorer__chip').data(fields, d => d).join(enter, update, exit):
      - enter: create chip div with draggable="true", role="option", LATCH colored border span + label span
      - Set up dragstart/dragend on each chip
      - update: sync chip label to alias, sync disabled state for toggled-OFF fields
      - exit: remove
    - Set up dragover/dragleave/drop on each well body

    DnD handlers (following KanbanView pattern with custom MIME text/x-projection-field):
    - _handleDragStart(e, field, wellId, index):
      - e.dataTransfer.setData(MIME_PROJECTION, field)
      - e.dataTransfer.effectAllowed = 'move'
      - Store _dragSourceWell, _dragField, _dragIndex in module-level vars
      - Add .dragging class to chip

    - _handleDragOver(e, targetWellId):
      - Check e.dataTransfer.types.includes(MIME_PROJECTION)
      - If _dragField already in target well (duplicate) -> do not preventDefault (invalid drop)
      - If removing last from X or Y (_dragSourceWell is 'x'/'y' and that well would have 0 after move) -> do not preventDefault if target is different well
        - Actually, minimum enforcement happens at drop time. During dragover, just validate duplicates.
      - If valid: e.preventDefault(), add .projection-explorer__well--dragover class
      - If invalid: do not preventDefault

    - _handleDragLeave(e, targetWellId):
      - Remove .projection-explorer__well--dragover class

    - _handleDrop(e, targetWellId):
      - e.preventDefault(), e.stopPropagation() (prevent bubbling to SuperGrid)
      - Read field from e.dataTransfer.getData(MIME_PROJECTION) or _dragField
      - Remove dragover class

      WITHIN-WELL REORDER (source == target):
      - Compute drop index from e.clientY/clientX relative to chip positions (find closest chip boundary)
      - If X well: call pafv.reorderColAxes(_dragIndex, dropIndex)
      - If Y well: call pafv.reorderRowAxes(_dragIndex, dropIndex)
      - Available and Z wells: reorder is visual only (no provider call needed)

      BETWEEN-WELL MOVE:
      - Validate: duplicate check (field already in target)
      - Validate: minimum check -- if removing from X/Y and that well would have 0 remaining:
        - actionToast.show('X axis requires at least one property')
        - Return without mutating
      - Compute new axes arrays:
        - Remove field from source well's axis array
        - Add field to target well's axis array (with direction 'asc' default)
      - Call pafv.setColAxes() for X well changes, pafv.setRowAxes() for Y well changes
      - If moving to/from Available: just recalculate (Available is derived, not stored)
      - If moving to/from Z: Z is not currently stored on PAFVProvider (deferred to Plan 04 Z-controls)

    - _handleDragEnd():
      - Remove .dragging class
      - Clear module-level DnD state

    setEnabledFieldsGetter(getter: () => ReadonlySet<AxisField>):
    - Update the getter and re-render chips (for when PropertiesExplorer toggle changes affect Available well)

    update(): re-render all chips (called externally)

    destroy():
    - Remove root element
    - Unsubscribe from pafv and alias providers
    - Null out references

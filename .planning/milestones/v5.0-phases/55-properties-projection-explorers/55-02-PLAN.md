  <behavior>
    - mount() creates 5 LATCH columns (L, A, T, C, H) arranged horizontally with correct property assignments
    - L column shows 0 properties with "(0/0)" badge and empty state text
    - A column shows 1 property (name) with "(1/1)" badge
    - T column shows 3 properties (created_at, modified_at, due_at)
    - C column shows 3 properties (folder, status, card_type)
    - H column shows 2 properties (priority, sort_order)
    - Each column header is clickable to collapse/expand independently
    - Toggle checkbox: clicking checkbox toggles property enabled state
    - Toggle OFF: property row gets dimmed opacity (0.5) + strikethrough text
    - Count badge updates when toggle changes (e.g., T goes from "(3/3)" to "(2/3)")
    - Inline rename: clicking property name text swaps span to input, auto-focuses, selects all
    - Enter key or blur on input: calls aliasProvider.setAlias(), swaps back to span showing new alias
    - Escape key on input: cancels edit, swaps back to span with original value
    - Clear/reset button visible during edit: calls aliasProvider.clearAlias(), swaps back to span with original field name
    - destroy() removes DOM and unsubscribes from providers
    - Uses D3 selection.join for property rows within each column body (INTG-03)
    - update() re-renders all columns reflecting current alias and toggle state
    - getEnabledFields() returns Set of currently enabled AxisField values
    - subscribe(callback) fires when toggle state changes (so ProjectionExplorer can react)
  </behavior>

    Constructor config interface PropertiesExplorerConfig:
    - alias: AliasProvider
    - container: HTMLElement (the section body element from WorkbenchShell.getSectionBody('properties'))
    - onCountChange?: (count: number) => void -- callback to update CollapsibleSection badge

    Internal state:
    - _enabledFields: Set<AxisField> -- starts with all 9 fields enabled
    - _columnCollapsed: Map<LatchFamily, boolean> -- per-column collapse state (localStorage keyed by `workbench:prop-col-${family}`)
    - _editingField: AxisField | null -- which field is currently being inline-edited
    - _subscribers: Set<() => void> -- fires when toggle state changes
    - _unsubAlias: (() => void) | null -- alias provider subscription cleanup

    mount():
    - Create .properties-explorer root
    - Create .properties-explorer__columns flex container
    - For each family in LATCH_ORDER: create column with header (chevron + label + badge) and body
    - Call _renderColumns() to populate property rows via D3 selection.join
    - Subscribe to aliasProvider to call _renderColumns() on alias changes
    - Append root to container (which was obtained via WorkbenchShell.getSectionBody('properties'))

    _renderColumns():
    - For each family in LATCH_ORDER:
      - Get AxisField values belonging to this family (filter LATCH_FAMILIES entries)
      - Use D3 select(columnBody).selectAll('.properties-explorer__property').data(fields, d => d).join(enter, update, exit) for INTG-03
      - enter: create .properties-explorer__property div with checkbox + name span
      - update: sync checkbox checked state, toggle disabled class, update name text to alias
      - exit: remove
    - Update each column badge text: "(enabled/total)"
    - Fire onCountChange with total enabled count

    _handleToggle(field: AxisField):
    - Toggle field in _enabledFields set
    - Call _renderColumns() to update UI
    - Fire _subscribers (so ProjectionExplorer reacts to toggle changes)

    _handleNameClick(field: AxisField, spanEl: HTMLElement):
    - If already editing another field, commit that edit first
    - Set _editingField = field
    - Create input element, set value to current alias, select all text
    - Create clear/reset button (x icon or "reset" text)
    - Replace span with input + clear button
    - Add keydown listener: Enter -> commit, Escape -> cancel
    - Add blur listener -> commit
    - Clear button click: aliasProvider.clearAlias(field), exit edit mode

    _commitEdit(field, value) / _cancelEdit(field):
    - Commit: aliasProvider.setAlias(field, value.trim() || original field name), re-render
    - Cancel: just re-render (alias unchanged)

    getEnabledFields(): ReadonlySet<AxisField>
    subscribe(cb: () => void): () => void

    update(): re-render all columns (called externally after provider state changes)

    destroy():
    - Remove root element from DOM
    - Unsubscribe from aliasProvider
    - Clear subscribers
    - Null out references

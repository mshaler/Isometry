## Spacing Scale

Declared values from existing `design-tokens.css` — phase must use these tokens only:

| Token | Value | Usage |
|-------|-------|-------|
| --space-xs | 4px | Icon gaps, inline padding, tag chip gaps |
| --space-sm | 8px | Card internal padding, row gaps |
| --space-md | 12px | Default element spacing |
| --space-lg | 16px | Section padding, card content padding |
| --space-xl | 24px | Panel gaps, 10x detail page content padding |

Source: `design-tokens.css` `:root` block (existing tokens — do not introduce new spacing values)

Exceptions:
- 1x card height: 28px (compact row — matches existing sidebar-item min-height pattern of 32px, capped at 28px for density)

- 10x detail page: full view area takeover, no height constraint
- Dimension switcher segmented buttons: 24px height touch target minimum

## Typography

Source: detected from `design-tokens.css` `:root` block.

4-size scale — no two sizes within 2px of each other:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Card content preview (2x/5x), card property labels (5x), dimension switcher button label | var(--text-xs) = 10px | 400 (regular) for preview; 600 (semibold) for labels and buttons | 1.5 for preview; 1.4 for labels; 1 for button |
| Card title (1x/2x/5x), card type icon badge | var(--text-base) = 13px | 600 (semibold) | 1.4 |
| 10x detail content | var(--text-md) = 14px | 400 (regular) | 1.6 |
| 10x detail title | var(--text-xl) = 18px | 600 (semibold) | 1.2 |

Notes:
- `--text-sm` (11px) is NOT used in Phase 94. Card titles at 1x/2x/5x use `--text-base` (13px) for legibility.
- The 4-size scale produces clear hierarchical jumps: 10px → 13px → 14px → 18px.
- Body line-height 1.5 for content preview (truncated text, 2-line max at 2x).
- Heading line-height 1.2 for 10x detail title.

## Component Inventory

### New: `[data-dimension]` CSS attribute system

The `data-dimension` attribute is placed on the **view container root element** (not per-card). This matches the existing `data-view-mode` pattern from SuperGrid (Phase 58).

```
[data-dimension="1x"] .card { ... }   /* compact row */
[data-dimension="2x"] .card { ... }   /* preview — default */
[data-dimension="5x"] .card { ... }   /* full card */
```

10x is NOT a `data-dimension` value. It is a per-card triggered state (class `.card--hero-open` on the card element, or a dedicated overlay/panel).

### Card CSS Classes (new file: `src/styles/card-dimensions.css`)

| Class | Purpose |
|-------|---------|
| `.card` | Base card element — D3 data join target (div.card) |
| `.card__icon` | Card type icon badge — always visible |
| `.card__title` | Card name — always visible, truncated at 1x |
| `.card__preview` | Content/summary text — hidden at 1x, 2-line truncation at 2x, full display at 5x |
| `.card__tags` | Tag chips — hidden at 1x/2x, shown at 5x |
| `.card__props` | Collapsible properties section — only at 5x |
| `.card__props-toggle` | Chevron button to collapse/expand properties in 5x |
| `.card--hero` | Applied to a single card to trigger 10x detail overlay |

### New: DimensionExplorer (Workbench sidebar panel)

Implemented as a section within the **Visual Explorer** panel OR as a new collapsible sidebar section. Per CONTEXT.md Claude's Discretion, this spec prescribes: **add as a new section within VisualExplorer** (avoids a new sidebar entry for a 3-button control, keeps the panel rail from growing).

The section header: "Size" with a grid/resize icon glyph.

Segmented control: three `<button>` elements inside a `.dim-switcher` container.

```html
<div class="dim-switcher" role="group" aria-label="Card size">
  <button class="dim-btn" data-size="1x" aria-pressed="false">1×</button>
  <button class="dim-btn dim-btn--active" data-size="2x" aria-pressed="true">2×</button>
  <button class="dim-btn" data-size="5x" aria-pressed="false">5×</button>
</div>
```

| Class | Purpose |
|-------|---------|
| `.dim-switcher` | Flex row container, border grouped |
| `.dim-btn` | Individual toggle button, 32px wide, 24px tall |
| `.dim-btn--active` | Selected state — accent background |

### 10x Detail Overlay

Triggered by double-click on card OR Enter on selected card. Rendered as a full-area overlay inside `.workbench-view-content`. Implemented as a `<div class="card-detail-overlay">` absolutely positioned over the view area (not a `<dialog>` since it's view-scoped, not app-scoped).

| Class | Purpose |
|-------|---------|
| `.card-detail-overlay` | Position absolute, inset 0, z-index 20, bg var(--bg-card) |
| `.card-detail-overlay__header` | Title + card_type badge + close button row |
| `.card-detail-overlay__content` | Markdown-rendered content (existing marked+DOMPurify pattern) |
| `.card-detail-overlay__props` | All property fields displayed below content (read-only display at Phase 94, editable via Phase 93 CardPropertyFields in a future integration) |
| `.card-detail-overlay__close` | "×" close button, top-right corner, 32×32px touch target |

## Dimension Level Visual Specifications

Prescriptive sizing for each dimension level. Claude's Discretion per CONTEXT.md.

### 1x — Compact Row

```
Height: 28px
Layout: single horizontal flex row
  [icon --text-base 13px] [gap 4px] [title truncated, flex-grow, --text-base 13px weight-600]
Background: var(--bg-primary)
Border-bottom: 1px solid var(--border-subtle)
Padding: 0 var(--space-sm)   /* 0 8px */
```

Hidden at 1x: `.card__preview`, `.card__tags`, `.card__props`

SuperGrid at 1x: cells show count badge only (`N` label, --text-xs 10px, centered), no card elements rendered.

### 2x — Preview (default)

```
Height: 56px (fixed)
Layout: two-row flex column
  Row 1: [icon --text-base 13px] [gap 4px] [title, --text-base 13px weight-600, single-line truncate]
  Row 2: [content preview, --text-xs 10px weight-400, 1-line truncate with ellipsis]
Background: var(--bg-card)
Border: 1px solid var(--border-subtle)
Border-radius: var(--radius-sm)   /* 4px */
Padding: var(--space-xs) var(--space-sm)   /* 4px 8px */
Margin-bottom: var(--space-xs)   /* 4px — between cards in list */
```

Hidden at 2x: `.card__tags`, `.card__props`

SuperGrid at 2x: cells show card name list, one name per line, --text-xs 10px, up to N names fitting in cell height.

### 5x — Full Card

```
Min-height: 120px
Max-height: 320px (overflow: hidden, expand via properties toggle)
Layout: flex column
  Header band: [icon 16px] [title, --text-base 13px weight-600] — bg var(--bg-surface), padding var(--space-xs) var(--space-sm)
  Content area: [preview text, --text-xs 10px, up to 4 lines, line-height 1.5] — padding var(--space-sm)
  Tags row: [tag chips] — padding 0 var(--space-sm) var(--space-xs)
  Properties section: [collapsible, CLOSED by default] — cpf-group pattern
Background: var(--bg-card)
Border: 1px solid var(--border-muted)
Border-radius: var(--radius-md)   /* 8px */
Margin-bottom: var(--space-sm)   /* 8px — between cards */
```

Properties toggle: `.card__props-toggle` button, full-width, "Properties ▾" label, --text-xs 10px, weight-600, var(--bg-surface) background. On expand: reveals `.card__props` with key-value property rows reusing `.cpf-row` / `.cpf-row__label` pattern (read-only display, not editable inputs at Phase 94).

SuperGrid at 5x: cells show mini-card divs at reduced scale (--sg-zoom applied). Each cell renders up to 3 mini-cards with icon + title + 1-line preview.

### 10x — Hero Detail

```
Position: absolute, inset 0, z-index 20
Background: var(--bg-card)
Overflow-y: auto
Layout: flex column
  Header: [card_type icon 24px] [title --text-xl 18px weight-600] [close button 32×32]
          — padding var(--space-lg) var(--space-xl)   /* 16px 24px */
  Content: [markdown rendered, --text-md 14px, line-height 1.6]
           — padding 0 var(--space-xl) var(--space-xl)
  Properties: [all 26 fields in labeled rows, read-only, --text-xs 10px]
              — padding 0 var(--space-xl) var(--space-xl)
              — section label "Properties" --text-md 14px weight-600, border-top 1px solid var(--border-subtle)
```

Scroll behavior: the overlay itself scrolls; the view behind is frozen (pointer-events: none applied to view container when overlay is open).

## Interaction Contract

### Dimension Switcher

| Interaction | Behavior |
|-------------|----------|
| Click `1x` button | Sets `data-dimension="1x"` on view container; saves `dimension:{viewType}` to ui_state; button gets `dim-btn--active`, aria-pressed="true" |
| Click `2x` button | Same for 2x |
| Click `5x` button | Same for 5x |
| No persisted value | Default to `2x` |
| View switch | Load dimension from ui_state for new view on mount |

No transition animation on dimension switch — INSTANT CSS recalculation. Source: CONTEXT.md decision.

Keyboard: segmented buttons support arrow-key navigation (roving tabindex — left/right arrows move focus, Space/Enter activates). Matches existing roving tabindex pattern from v6.1 (Phase 84).

### 10x Trigger and Exit

| Interaction | Behavior |
|-------------|----------|
| Double-click `.card` | Opens `.card-detail-overlay` for that card |
| Enter key on focused `.card` | Opens `.card-detail-overlay` |
| Click `.card-detail-overlay__close` | Closes overlay, returns to previous dimension level |
| Press Escape | Closes overlay |
| Click outside overlay content area | NOT a close trigger — avoids accidental dismissal |

10x does not modify `data-dimension` on the view container. The overlay is layered above it. When closed, the view re-emerges at its current 1x/2x/5x dimension unchanged.

### 5x Properties Toggle

| Interaction | Behavior |
|-------------|----------|
| Click `.card__props-toggle` | Toggles `.card__props` visibility; rotates chevron -90deg when collapsed (matching cpf-group__chevron pattern) |
| Default state | CLOSED (per CONTEXT.md decision) |

Per-card toggle state is NOT persisted — it resets on re-render. This is intentional: 5x cards collapse their properties by default to avoid height overflow in dense lists.

## CSS Architecture

### New file: `src/styles/card-dimensions.css`

All new CSS for Phase 94 goes in this single file. Scoped under:
- `.card` prefix (base card element)
- `.dim-*` prefix (dimension switcher control)
- `.card-detail-overlay` prefix (10x overlay)
- `[data-dimension="1x"]`, `[data-dimension="2x"]`, `[data-dimension="5x"]` attribute selectors

Do NOT add dimension styles to existing `views.css`, `supergrid.css`, or `visual-explorer.css`.

### Import order in main CSS entry point

```css
/* Existing */
@import './design-tokens.css';
@import './views.css';
@import './supergrid.css';
/* ... other existing imports ... */

/* New in Phase 94 */
@import './card-dimensions.css';
```

### data-dimension selector pattern

```css
/* Base card — visible at all dimension levels */
.card {
  background: var(--bg-card);
  border: 1px solid var(--border-subtle);
  border-radius: var(--radius-sm);
  overflow: hidden;
}

/* 1x — compact row */
[data-dimension="1x"] .card {
  height: 28px;
  display: flex;
  align-items: center;
  padding: 0 var(--space-sm);
  border-radius: 0;
  border-bottom: 1px solid var(--border-subtle);
  border-top: none;
  border-left: none;
  border-right: none;
}

/* 2x — preview (default styling, also the fallback) */
[data-dimension="2x"] .card {
  height: 56px;
  display: flex;
  flex-direction: column;
  justify-content: center;
  padding: var(--space-xs) var(--space-sm);
  margin-bottom: var(--space-xs);
}

/* 5x — full card */
[data-dimension="5x"] .card {
  min-height: 120px;
  max-height: 320px;
  display: flex;
  flex-direction: column;
  border: 1px solid var(--border-muted);
  border-radius: var(--radius-md);
  margin-bottom: var(--space-sm);
}
```

## Accessibility Requirements

All patterns must match existing WCAG 2.1 AA compliance standard established in v4.4 (Phase 50).

| Requirement | Implementation |
|-------------|----------------|
| Dimension switcher keyboard | roving tabindex, arrow key navigation, Space/Enter activation |
| `aria-pressed` on active button | true on selected, false on others |
| 10x overlay focus management | focus moves to overlay on open; returns to triggering card on close |
| 10x overlay Escape close | keydown Escape closes overlay; must call event.stopPropagation() to prevent other handlers |
| Card focus ring | `outline: 2px solid var(--accent); outline-offset: 2px` on `.card:focus-visible` |
| 5x properties toggle | `aria-expanded` attribute toggles true/false |
| Minimum touch target | 28px height at 1x (acceptable for desktop-first app; iOS users on 2x/5x) |

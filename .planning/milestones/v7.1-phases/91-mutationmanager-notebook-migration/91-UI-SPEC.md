## Spacing Scale

Source: `src/styles/design-tokens.css` — theme-independent tokens block. These are the project's declared values.

| Token | Value | Usage |
|-------|-------|-------|
| --space-xs | 4px | Icon gaps, toolbar button gaps, inline padding |
| --space-sm | 8px | Textarea/input padding, toolbar padding, compact element spacing |
| --space-md | 12px | Section padding, notebook body dividers |
| --space-lg | 16px | Default element spacing, list padding |
| --space-xl | 24px | Major section breaks |

All spacing values are multiples of 4. No exceptions — use `var(--space-sm)` (8px) for title input padding on all sides.

## Color

Source: `src/styles/design-tokens.css` — all values are token references, not raw hex, to respect the 3-theme system (dark/light/nextstep/material).

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | var(--bg-primary) | Notebook panel background, title input background |
| Secondary (30%) | var(--bg-surface) | Segmented control bar, toolbar strip, idle state background |
| Accent (10%) | var(--accent) | Active tab indicator on segmented control, title input focus ring, textarea top border on focus |
| Destructive | var(--danger) | Not used in Phase 91 — no destructive actions |

Accent reserved for:
- `.notebook-tab--active` background (Write/Preview active tab)
- Title input focus ring: `outline: 2px solid var(--accent)` on `:focus-visible`
- Textarea top border on focus: `border-top-color: var(--accent)` (existing pattern from `.notebook-textarea:focus`)
- Nothing else — do not apply accent to placeholder text, idle state, or migration status

## Component Inventory

New CSS classes introduced in Phase 91. All scoped under `.notebook-*` prefix (established convention per SHEL-06).

### Title Input — `.notebook-title-input`

```
.notebook-title-input {
  width: 100%;
  padding: var(--space-sm) var(--space-sm);
  font-family: var(--font-sans);
  font-size: var(--text-lg);       /* 16px — document title scale */
  font-weight: 600;
  line-height: 1.3;
  color: var(--text-primary);
  background: var(--bg-primary);
  border: none;
  border-bottom: 1px solid var(--border-subtle);
  outline: none;
  box-sizing: border-box;
}

.notebook-title-input::placeholder {
  color: var(--text-muted);
  font-weight: 400;                /* placeholder lighter than typed text */
}

.notebook-title-input:focus-visible {
  outline: 2px solid var(--accent);
  outline-offset: -2px;
}
```

- Placeholder text: "Untitled" (source: CONTEXT.md decisions — matches Notion/Apple Notes convention)
- No border-radius — consistent with existing borderless inputs in the workbench
- Positioned above the segmented control (Write | Preview), above the toolbar strip

### Idle State — `.notebook-idle`

Shown when no card is selected (zero-selection state, or after active card is deleted by undo).

```
.notebook-idle {
  display: flex;
  align-items: center;
  justify-content: center;
  min-height: 120px;
  padding: var(--space-xl);
  font-size: var(--text-sm);
  color: var(--text-muted);
  text-align: center;
  background: var(--bg-primary);
}
```

- No icon, no button — text only. Copy defined in Copywriting Contract below.
- This replaces the current empty panel state. Existing `_setVisible(false)` hides the explorer entirely; Phase 91 should show `.notebook-idle` instead when selection is null or card was deleted.

### Layout Order (top-to-bottom within `.notebook-explorer`)

1. `.notebook-title-input` — always visible when a card is active, hidden in idle state
2. `.notebook-segmented-control` — existing Write | Preview tab bar
3. `.notebook-toolbar` — existing formatting toolbar (Write mode only)
4. `.notebook-body` — existing textarea or preview pane
5. `.notebook-idle` — replaces all above when no card is selected

## Interaction Contract

### Shadow-Buffer Lifecycle

| Trigger | Action | Visual feedback |
|---------|--------|-----------------|
| Card selected (SelectionProvider change) | Load full card row into `_snapshot`. Populate title input value and textarea content. | Input populates instantly — no loading indicator |
| User types in title input | Update `_bufferName` | No visual indicator — browser native cursor/caret |
| Title input blur | Commit: `updateCardMutation(_snapshot.id, _snapshot, { name: _bufferName })` via MutationManager.execute() | No visual indicator — commit is silent (CONTEXT.md: no dirty indicator) |
| User types in textarea | Update `_bufferContent` | No visual indicator |
| Textarea blur | Commit: `updateCardMutation(_snapshot.id, _snapshot, { content: _bufferContent })` via MutationManager.execute() | No visual indicator — commit is silent |
| Cmd+S anywhere in notebook panel | Force-blur active field → triggers blur commit | No visual indicator beyond standard blur |
| Card switch (new SelectionProvider event while buffer has unsaved changes) | Force-commit current buffer before loading next card | No visual indicator |
| Active card deleted by undo (MutationManager subscriber fires, card no longer exists) | Reset to idle state: show `.notebook-idle`, hide title input + segmented control + body | Panel transitions to idle — no animation |

### Undo Layering

| Context | Cmd+Z behavior |
|---------|---------------|
| Focus inside `.notebook-title-input` | Browser-native undo (keystroke-level, per HTML input element) |
| Focus inside `.notebook-textarea` | Browser-native undo (keystroke-level, via `_undoSafeInsert()` execCommand pattern) |
| Focus outside notebook panel (anywhere else in app) | MutationManager.undo() — undoes last committed edit session as a single step |

ShortcutRegistry already skips TEXTAREA and INPUT elements — no change required. This two-layer behavior is transparent to the user.

### Focus Ring Standard

All interactive elements in the notebook panel follow the existing focus standard:
- `outline: 2px solid var(--accent)` on `:focus-visible`
- `outline-offset: -2px` (inset, consistent with `.notebook-tab:focus-visible` and `.notebook-toolbar-btn:focus-visible`)

## Migration — No Visual Contract

EDIT-05 (notebook:migration:v1) runs silently on boot. No progress bar, no toast, no banner. The sentinel guard means it only fires once. If migration fails for a specific card, that card is silently skipped (cards.content already non-null wins). No user-visible state change during or after migration.

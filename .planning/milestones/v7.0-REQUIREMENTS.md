# v7.0 ETL Enrichment — Requirements

**Milestone:** v7.0 ETL Enrichment
**Status:** PLANNING
**Created:** 2026-02-17
**Goal:** Extend ETL pipeline with post-import enrichment to extract structured data from markdown bodies, enabling dataset-specific properties in PAFV navigators.

## Problem Statement

The current ETL pipeline only extracts frontmatter keys into `node_properties`. Many data sources (especially Alto-Index Contacts) store structured information in **markdown body** format that never reaches the property classifier:

```markdown
---
title: "Andrew Quinlan"
organization: "Conga"
---

## Professional
**Title:** Vice President, Global OEM/ISV Program

## Email Addresses
- drew.quinlan@gmail.com
```

Result: Navigator shows only universal properties (Name, Folder, Tags) for Contacts instead of dataset-specific properties (Job Title, Email, Company).

## Technical Constraints

1. **Non-destructive enrichment** — Must not modify original node data, only add to `node_properties`
2. **Incremental processing** — Support enriching single nodes or filtered subsets
3. **Idempotent operations** — Re-running enrichment produces same results
4. **Performance** — Process 10K nodes in <30 seconds
5. **Memory-bounded** — Stream processing, don't load all nodes into memory

## Architecture

### Enrichment Pipeline

```
┌─────────────┐     ┌──────────────┐     ┌─────────────────┐
│ node record │ ──► │ Body Parser  │ ──► │ node_properties │
│ (summary)   │     │ (per type)   │     │ (extracted)     │
└─────────────┘     └──────────────┘     └─────────────────┘
                           │
                    ┌──────┴──────┐
                    │             │
              ┌─────▼─────┐ ┌─────▼─────┐
              │ Contacts  │ │ Notes     │
              │ Parser    │ │ Parser    │
              └───────────┘ └───────────┘
```

### Parser Interface

```typescript
interface BodyParser {
  /** Which node_type this parser handles */
  nodeType: string;

  /** Extract structured data from markdown body */
  parse(summary: string): ExtractedProperties;
}

interface ExtractedProperties {
  /** Key-value pairs for node_properties table */
  properties: Record<string, unknown>;

  /** Extracted tasks (for task-extraction feature) */
  tasks?: ExtractedTask[];

  /** Extracted links (for link-preview feature) */
  links?: ExtractedLink[];
}
```
